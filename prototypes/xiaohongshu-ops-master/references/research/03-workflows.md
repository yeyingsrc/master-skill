# Track 03 — Workflows (小红书运营)

> Mini-viable for prototype.

### 1. 品牌新号冷启动

- **One-liner**: 从 0 到第一篇爆文 (≥ 1k 互动) 的标准动作
- **Trigger**: 新建品牌专业号 / 新建矩阵账号
- **Output**: 账号站稳, 至少 1 篇爆文 + 数据基准建立
- **入门 SOP**:
  1. 完善账号资料 (头像 / 名称 / 简介 / 第一篇置顶笔记)
  2. 找 3-5 个对标账号, 拆其爆款笔记选题 / 封面 / 结构
  3. 第一周发 5-7 篇笔记, 选题分散测试品类边界
  4. 看 24 小时数据 — 互动率 + 收藏率 + 点击率
  5. 数据异常 (爆 / 死) 都立刻复盘, 复刻爆款 / 砍掉死款
- **资深路径**:
  - skip 非必要资料完善, 资深直接进入选题阶段
  - optimize 用千瓜批量拆对标爆款, 不一篇篇看
  - add 第一周直接做矩阵 (3 个账号同步发), 不押单点
- **常见失败模式**:
  - 资料没完善就发笔记 → 第一篇被限流, 后续起量难
  - 选题太集中 → 测不出账号最适合的内容方向
  - 数据差还硬撑 → 浪费账号权重, 后面爆款都救不回来
- **Last_updated**: 2025-12-01
- **Decay risk**: high

### 2. 商单笔记生产

- **One-liner**: 接到品牌商单后, 从 brief 到爆款内容的标准产出流程
- **Trigger**: 接到品牌商单 (蒲公英报备 / 私单)
- **Output**: 报备笔记 + 数据达标 (品牌投后 KPI)
- **入门 SOP**:
  1. 拆品牌 brief — 必填卖点 / 禁忌词 / 投放预期
  2. 选题反推 — 用户痛点角度切入, 不是品牌角度
  3. 内容创作 — 真实场景 + 软植入 + 真人感
  4. 蒲公英平台报备 (商单笔记必须报)
  5. 发布后跟数据, 24 小时内决定是否投流加热
- **资深路径**:
  - skip 大段品牌信息植入, 资深做法是只在最后 1-2 句话提
  - optimize 选题用品牌目标用户的搜索词反推, 不是从品牌卖点出发
  - add 报备同时做内容备份方案 (限流时可以快速换号发)
- **常见失败模式**:
  - 品牌信息过密 → 用户跳出 → 数据死
  - 不报备 → 平台一查就限流, 商单也黄了
  - 选题从品牌卖点出发 → 没人愿意看
- **Last_updated**: 2025-11-15
- **Decay risk**: medium
