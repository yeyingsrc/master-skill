### 1. 品牌新号冷启动 (KFS 阶段 1)

- **One-liner**: 从 0 到第一篇爆文 (≥ 1k 互动) 的标准动作 — 30 天内站稳基线 + 找到内容方向.
- **Trigger**: 新建品牌专业号 / 新建矩阵账号 / 沉睡号重启动
- **Output**: 账号站稳, 至少 1 篇爆文 + 数据基准建立 + 选题方向锁定 (3-5 个内容主题)
- **入门 SOP**:
  1. **完善账号资料** (Day 1-2): 头像 / 名称 / 简介 / 第一篇置顶笔记. 品牌专业号必须做企业认证 (¥ 600/年).
  2. **对标账号研究** (Day 2-3): 用千瓜找 3-5 个对标账号, 拆其爆款笔记选题 / 封面 / 标题 / 结构. 把它们的选题做成 sheet.
  3. **第一周内容 5-7 篇** (Day 3-10): 选题分散, 测试品类边界. 不押单点, 至少覆盖 3-4 个不同选题方向.
  4. **24 小时数据看** (Day 10): 互动率 / 收藏率 / 点击率 / 完读率 — 4 个核心指标. 互动率 ≥ 类目均值 1.5x 的算「潜力爆款」.
  5. **复盘 + 复刻** (Day 10-15): 数据异常 (爆 / 死) 都立刻复盘. 复刻爆款 (48 小时内 2-3 条同主题). 砍掉死款方向.
  6. **第二三周深耕优胜方向** (Day 15-30): 围绕复盘出的优胜方向, 持续输出 5-10 篇. 同步开始小额薯条测试 (¥ 75-300/篇).
- **资深路径**:
  - **skip 非必要资料完善**: 资深直接进入选题阶段, 资料用模板批量完成 (¥ 200 找代运营 1 天搞定)
  - **optimize 用千瓜批量拆对标爆款**: 不一篇篇看, 用工具导数据后 sheet 排序
  - **add 第一周直接做矩阵**: 3 个账号同步发, 不押单点 (合规边界注意 — 矩阵号必须不同手机号 / 不同设备)
  - **add 上线前 AIPS 人群预设定**: 头部品牌跟代理预先做人群分层, 笔记上线就有预设流量池
- **常见失败模式**:
  - 资料没完善就发笔记 → 第一篇被限流, 后续起量难
  - 选题太集中 → 测不出账号最适合的内容方向
  - 数据差还硬撑 → 浪费账号权重, 后面爆款都救不回来
  - 矩阵号同手机号管理 → 平台识别为「营销号」, 整组限流
- **Last_updated**: 2026-04-15
- **Decay risk**: high (平台规则 6 个月一变)

### 2. 商单笔记生产 (蒲公英报备流程)

- **One-liner**: 接到品牌商单后, 从 brief 到爆款内容的标准产出流程 — 报备 + 内容 + 数据三件套.
- **Trigger**: 接到品牌商单 (蒲公英报备 / 私单)
- **Output**: 报备笔记 + 数据达标 (品牌投后 KPI) + 后链路转化数据回归
- **入门 SOP**:
  1. **拆品牌 brief** (Day 1): 必填卖点 / 禁忌词 / 投放预期. 不要全盘照搬, 提炼出 1-2 个真正能种草的点.
  2. **选题反推** (Day 1-2): 用户痛点角度切入, 不是品牌角度. 用千瓜 / 新红 看竞品同类商单选题, 选差异化角度.
  3. **内容创作** (Day 2-4): 真实场景 + 软植入 + 真人感. 品牌信息植入比例 ≤ 30% (软植入是 5-10%). 至少 80% 内容是真实使用 / 体验 / 测评.
  4. **蒲公英平台报备** (Day 4): 商单笔记必须报. 报备后才能合规接单. 商单笔记会带「赞助」标识 (但不影响算法).
  5. **发布后 24 小时跟数据** (Day 5): 看互动率 / 收藏率 / 完读率. 数据达品牌 KPI 80% 以上算合格.
  6. **决定是否投流加热** (Day 5-7): 自然流好但还没到 KPI → 上薯条加热. 自然流差 → 不要硬投流, 节省预算.
- **资深路径**:
  - **skip 大段品牌信息植入**: 资深做法是只在最后 1-2 句话提品牌, 前面全是真实体验
  - **optimize 选题用品牌目标用户的搜索词反推**: 不是从品牌卖点出发, 是从用户「在小红书会搜什么」出发
  - **add 报备同时做内容备份方案**: 限流时可以快速换号发 (但不能换号发同一条, 平台会识别)
  - **add 后链路 AIPS 数据归因**: 头部博主跟品牌方建议接 AIPS, 看自己内容真正贡献的人群资产
- **常见失败模式**:
  - 品牌信息过密 → 用户跳出 → 数据死
  - 不报备 → 平台一查就限流, 商单也黄了
  - 选题从品牌卖点出发 → 没人愿意看
  - 一次接 ≥ 5 条同品类商单 → 蒲公英信用等级降级 (Lv2 → Lv1)
- **Last_updated**: 2026-03-20
- **Decay risk**: high (蒲公英规则每年改 1-2 次)

### 3. KFS 组合营销 (中大品牌完整投放)

- **One-liner**: KOL × Feeds × Search 三件套组合投放 — 内容摸索 → 内容验证 → 规模化的标准 90 天打法.
- **Trigger**: 品牌方投放预算 ≥ ¥ 30k/月, 准备做规模化推广
- **Output**: 累积 30+ 篇爆款笔记 + AIPS 人群资产沉淀 + 跨平台 ROI 归因
- **入门 SOP**:
  1. **阶段 1 内容摸索** (Day 1-30, K:F:S = 7:3:0):
     - K (KOL): 用蒲公英投 8-15 个中腰部博主 (10 万-50 万粉) + 5-10 个 KOC (千粉-10 万粉)
     - F (Feeds): 薯条小额加热 (¥ 75-300/篇, 测试爆款潜力)
     - S (Search): 暂不做
     - 目标: 10-20 篇潜力爆款笔记 (互动率 ≥ 类目均值 1.5x)
  2. **阶段 2 内容验证** (Day 30-60, K:F:S = 5:3:2):
     - K: 复用阶段 1 跑出来的优胜博主 + 头部 KOL (50 万-100 万粉) ×2-3
     - F: 上聚光跑量 (起投 ¥ 5000+), 重点跑爆款笔记
     - S (Search): 接入品牌词 + 类目词 + 痛点词. 占大盘 20%
     - 目标: 单品爆款验证 + 搜索词渗透
  3. **阶段 3 规模化** (Day 60-90, K:F:S = 4:4:2):
     - K: 头部 KOL ×5+ 中腰部 ×30+ KOC 矩阵
     - F: 聚光大额跑量 + 跨人群投放
     - S: 搜索词全覆盖 (品牌词 / 类目词 / 长尾痛点词)
     - 目标: 30+ 爆款笔记 + AIPS 人群资产沉淀
- **资深路径**:
  - **skip 阶段 1 内容摸索**: 资深品牌跨大单品时直接复用既有人群 + 内容 IP, 跳到阶段 2
  - **optimize K 的人群匹配**: 不只看博主粉丝量, 用蒲公英智能选博主功能 (人群匹配 + 种草力)
  - **add 第一方数据反哺**: 头部品牌接小红书种草联盟, 把自家用户数据反哺人群定向
  - **add 跨平台联动**: 小红书种草 + 天猫直播 / 抖音引流 — 跨平台 ROI 归因
- **常见失败模式**:
  - 阶段 1 跳过直接上聚光 → 内容没验证, 投流跑不动 → 投放预算浪费
  - 头部 KOL ROI 不如中腰部 → 多数类目下中腰部 ROI 更高, 不要押头部
  - Search 不做品牌词 → 别人 SEO 抢走你的搜索词
  - K:F:S 比例死板, 不按品牌阶段动态调
- **Last_updated**: 2026-04-01
- **Decay risk**: medium (KFS 框架稳定, 具体比例随平台政策动)

### 4. 矩阵号 + 数据迭代 (中大品牌长期路径)

- **One-liner**: 5-30 个矩阵号同步运营, 数据驱动持续迭代, 抗算法波动 + 抗单点风险.
- **Trigger**: 中大品牌 / MCN 方 / 单 IP 风险大, 需要长期路径
- **Output**: 矩阵号 5-30 个, 月度爆款 ≥ 5 篇, 数据迭代闭环建立
- **入门 SOP**:
  1. **矩阵号规划** (Week 1): 按人群 (年龄 / 性别 / 价位 / 风格) 分层. 每个矩阵号锁定 1-2 个细分人群.
  2. **账号矩阵搭建** (Week 1-3): 5-10 个账号同步搭, 不同手机号 / 不同设备 / 不同 IP. 每个号有不同昵称 / 头像 / 简介定位.
  3. **内容生产分工** (Week 3+): 头部矩阵号做精品 (周 3-5 篇) + 中腰部铺量 (周 5-10 篇) + KOC 长尾 (周 1-3 篇).
  4. **数据迭代闭环** (持续): 周复盘 — 哪些选题爆了 / 哪些死了. 月复盘 — 矩阵号方向是否要调整.
  5. **合规边界守住** (持续): 不报备的私单一概不接 / 不刷量 / 不用同设备同 IP / 不发同一条内容.
- **资深路径**:
  - **skip 头部矩阵号自营**: 资深 MCN 把头部号交给签约博主, 平台只做选题 + 数据
  - **optimize 用千瓜批量监控**: 5-30 个号一次性数据看完, 不一个个手动
  - **add 跨平台矩阵**: 小红书 + 抖音 + 视频号同步矩阵, 跨平台分散风险
  - **add 品牌专业号 + 个人 IP 双线**: 品牌号沉淀长期资产, 个人 IP 矩阵跑量
- **常见失败模式**:
  - 矩阵号同设备 / 同 IP 管理 → 平台识别 → 整组限流
  - 矩阵号内容同质化 → 平台判定为「营销号」 → 推荐量降权
  - 不做数据迭代 → 矩阵号沦为僵尸号 → 浪费维护成本
  - 矩阵规模过快 → 平台合规边界一动就团灭
- **Last_updated**: 2026-04-10
- **Decay risk**: high (矩阵合规边界平台一直在收紧)

### 5. 跨平台联动 (小红书种草 + 站外转化)

- **One-liner**: 小红书种草 + 天猫 / 抖音 / 京东 / 美团等站外平台直播 / 闭环转化, 跨平台 ROI 归因.
- **Trigger**: 头部品牌 / 跨平台投放预算 ≥ ¥ 100k/月
- **Output**: 跨平台 GMV 闭环 + AIPS 人群资产 + 后链路 ROI 数据
- **入门 SOP**:
  1. **种草联盟接入** (Week 1): 跟小红书 BD / 代理对接, 开通小红星 (X 阿里) / 小红盟 (X 京东) / 小红链 (X 唯品会等)
  2. **种草直达配置** (Week 2): 笔记里挂跳转链接 (天猫 / 京东 / 美团等), 用户点击直接跳第三方电商
  3. **跨平台投放节奏** (持续):
     - 小红书做种草 (KFS 阶段 1-2): 内容沉淀 + 人群积累
     - 天猫 / 抖音直播做转化 (大促 / 周期性): 把小红书人群导到直播间
     - 京东 / 美团做长尾承接 (持续): 搜索流量承接
  4. **AIPS 人群资产沉淀** (持续): 每月看 A / I / TI / P / S 五个层级人群, 重点扩 TI → P (深度兴趣 → 购买)
  5. **ROI 归因闭环** (季度): 跟代理 / 平台跑后链路数据 — 哪些种草内容真正贡献了站外 GMV
- **资深路径**:
  - **skip 种草联盟手动配置**: 头部品牌直接跟代理打包, 配置由代理负责
  - **optimize 用第一方数据深度合作**: 头部 2100+ 品牌跟小红书有第一方数据合作通道
  - **add 跨平台节奏复用**: 小红书爆款选题复用到抖音 / 视频号, 跨平台流量协同
  - **add 大促期 KFS + 直播双轨**: 双 11 / 618 期间小红书 KFS 重金 + 天猫直播闭环
- **常见失败模式**:
  - 种草联盟接通了但没配 ROI 归因 → 数据看不到, 老板不批预算
  - 小红书内容做得好但站外承接不力 → 用户跳过去转化不下来
  - 跨平台节奏脱节 → 小红书种草 ≠ 直播间需求, 浪费种草人群
  - AIPS 人群资产看了不动 → 5 层人群每月不扩, 沉淀价值随时间衰减
- **Last_updated**: 2026-04-20
- **Decay risk**: medium (跨平台机制 2024-2026 大方向稳定, 工具迭代但策略稳定)

## 工作流之间的关系

```
品牌新号冷启动 (workflow 1)
   │
   │  站稳基线 + 找到内容方向后
   ↓
矩阵号 + 数据迭代 (workflow 4)  ←→  KFS 组合营销 (workflow 3)
   │                                    │
   │  矩阵号承接                        │  规模化投放
   ↓                                    ↓
商单笔记生产 (workflow 2)            跨平台联动 (workflow 5)
                                          │
                                          ↓
                                      头部品牌闭环
```

**典型路径**:
- 个人博主 / 中小品牌: workflow 1 → workflow 2 (商单变现)
- 中大品牌: workflow 1 → workflow 3 → workflow 4 (规模化 + 矩阵)
- 头部品牌: workflow 3 + workflow 4 + workflow 5 同时跑

## Phase 2 提炼提示

每个工作流抽: **一句话抽象 + 决策规则 + 失败模式**.

- workflow 1 → 决策规则: 「冷启动期不押单点, 矩阵分散测试 + 数据反馈优化」
- workflow 2 → 决策规则: 「商单内容先 80% 真实体验 + 后 20% 软植入, 不报备一概不接」
- workflow 3 → 决策规则: 「KFS 比例随阶段动: 7:3:0 → 5:3:2 → 4:4:2」
- workflow 4 → 决策规则: 「矩阵号合规第一 (不同手机号 / 设备 / IP), 数据迭代第二」
- workflow 5 → 决策规则: 「种草联盟接入 + AIPS 人群资产 + 跨平台 ROI 闭环」
