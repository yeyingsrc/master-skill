## A. 必备工具栈 (品牌方上小红书的最小集)

### 1. 蒲公英 (官方报备 + 商单交易)

- **类型**: 官方
- **核心定位**: 商单报备 + 创作者匹配 + 后链路结算
- **价格**: 平台抽佣 10% (品牌方付给博主的金额, 平台抽 10%)
- **代表功能**:
  - 智能选博主 (基于「人群匹配 + 内容种草力」, 不只看粉丝量) — 2025 升级
  - 商单笔记 72 小时审核 + 7 天确认订单
  - 信用等级 Lv0~Lv3+ (Lv0 无接单 / Lv3+ 「人间种草机」无上限)
- **典型用法**:
  - 品牌方搜索 + 筛选博主 (粉丝段 / 类目 / 数据)
  - 询单 + 报价 + 简介 brief
  - 收报备笔记 + 确认结算
- **适用对象**: 所有品牌 (不论预算大小)
- **常见错误**:
  1. 不报备走私单 → 平台一查就限流博主 + 品牌
  2. brief 写得过细把博主框死 → 内容失真 → 数据死
  3. 只看粉丝量不看「人群匹配 + 种草力」 → 投错人
- **来源**: [Primary] pgy.xiaohongshu.com/help/docs

### 2. 千瓜数据 (第三方数据头部)

- **类型**: 第三方数据
- **核心定位**: 小红书行业数据 + 博主拆解 + 品牌投放数据
- **价格**: 个人 ¥ 199/月 + 品牌 ¥ 999~9999/月 + 企业版定制
- **代表功能**:
  - 笔记数据趋势图 + 互动率 / 收藏率 / 完读率分析
  - 博主拆解 (粉丝画像 / 商单数据 / 互动健康度)
  - 品牌竞品监控 + 类目榜单
  - 千瓜月度行业报告 (美妆 / 母婴 / 食品 / 服饰)
- **典型用法**:
  - 选题前: 看竞品爆款笔记数据反推选题
  - 投放前: 拆博主商单数据避坑
  - 投放后: 数据复盘 + 类目对比
- **适用对象**: 中型以上品牌 / 头部博主 / 代理商
- **常见错误**:
  1. 工具数据是反推, 不等于平台真实数据 — 关键决策要交叉
  2. 互动率高但「收藏率 < 点赞率」 ← 内容没用, 别复刻
- **来源**: [Primary] qian-gua.com 官网 + 月度报告

### 3. 薯条 (官方轻量化投放)

- **类型**: 官方
- **核心定位**: 笔记 / 直播自助加热 (低门槛投放)
- **价格**: 75 元起投, 按 CPM 计费
- **代表功能**:
  - 自然流量 + 信息流双轨
  - 自助选投放目标 (曝光 / 互动 / 粉丝 / 私信)
  - 24-48 小时数据反馈
- **典型用法**:
  - 内容摸索期 (KFS 阶段 1, K:F:S = 7:3:0): 用薯条做小额测试 (¥ 75-300/篇)
  - 笔记爆款验证后再上聚光跑量
- **适用对象**: 个人博主 / 中小商家 / 内容测试
- **常见错误**:
  1. 数据差还硬投流救 → 浪费钱 + 标记账号低权重
  2. 投流的笔记和自然流的笔记内容用同一篇 → 两边都打不出量
  3. 一上来就投大额 → 没数据基线, 后续没法优化
- **来源**: [Primary] 小红书广告平台 xiao-ad.com + 平台官方

### 4. 聚光 (官方专业级投流)

- **类型**: 官方
- **核心定位**: 大账户 / 大品牌的精细化投放 (跟巨量引擎对标)
- **价格**: 起投 ¥ 5000+ (品牌投放标配)
- **代表功能**:
  - 多目标多创意 A/B 测试
  - 精细人群定向 (类目 / 兴趣 / 行为 / 地域 / 设备)
  - 转化目标 (互动 / 收藏 / 加购 / 进店)
  - 跨平台归因 (种草联盟接入)
- **典型用法**:
  - 品牌专业号矩阵投放 (多账号 + 多 SKU)
  - KFS 阶段 2-3 (K:F:S = 5:3:2 或 4:4:2): F 主力工具
- **适用对象**: 中大品牌 / 投放预算 ≥ ¥ 5000/月
- **常见错误**:
  1. 没做 KFS 阶段 1 就直接上聚光 → 内容没验证, 投流跑不动
  2. 创意不分层 (头部 KOL + 中腰部 + KOC 内容混着投) → ROI 差
- **来源**: [Primary] 小红书广告平台官方 + 朴朴超市 KFS 案例

### 5. 千帆 (官方品牌专业号工具)

- **类型**: 官方
- **核心定位**: 品牌专业号管理 + 后台数据 + 私信运营
- **价格**: 免费 (品牌专业号标配)
- **代表功能**:
  - 多账号统一管理 (矩阵号一键切换)
  - 商家后台数据 (笔记 / 客资 / 销售)
  - 群聊功能 + 私信批量回复
  - 直播 + 商品发布
- **典型用法**: 品牌专业号日常运营 + 矩阵号统一管理
- **适用对象**: 所有品牌专业号
- **常见错误**:
  1. 不开品牌专业号 (只用个人号) → 没法用千帆 + 没后台数据
  2. 矩阵号用同一手机号管理 → 平台识别风险
- **来源**: [Primary] qf.xiaohongshu.com 商家后台

## 决策树 (按场景选工具)

### 场景 1: 个人博主 / 千粉以下

```
必备: 千帆 (品牌专业号免费) / 蒲公英 (Lv0 待升级)
内容: AI 笔记辅助 (脑暴 + 标题) + 真人编辑
不必要: 千瓜 / 蝉小红 / 聚光 (预算太大)
```

### 场景 2: 中小品牌 (月预算 ¥ 5k-¥ 30k)

```
必备: 蒲公英 (报备) + 千帆 (后台) + 千瓜个人版 (¥ 199) + 薯条 (轻投)
按需: 新红数据 (选题方法论) + AI 工具 (内容辅助)
不必要: 聚光 / 蝉小红 / 灰豚 (预算还不够)
```

### 场景 3: 中大品牌 (月预算 ¥ 30k-¥ 200k)

必备: 蒲公英 + 千帆 + 千瓜品牌版 (¥ 999+) + 薯条 + 聚光 (轻 KFS)

## 工具栈坑点 (诚实边界)

1. **第三方数据工具不等于平台真实数据** — 千瓜 / 新红 / 蝉小红 都是反推, 大方向准但精度有限. 关键决策必须官方后台数据交叉.

2. **官方工具组合「KFS 三件套」是平台导向** — 实际效果跟品牌内容质量强相关, 工具不是万能.

3. **AI 内容必须人工编辑** — 直接 AI 生成发布会被平台 Helpful Content 类机制限流. 至少要加真人场景 + 真人感.

4. **价格跳档明显** — 个人版 ¥ 199 → 品牌版 ¥ 999 → 企业版 ¥ 9999, 多数中型品牌选品牌版即可, 不必跳企业版.

5. **跨平台归因仍在演化** — AIPS / 种草联盟 / 第一方数据是 2024-2026 大方向, 但具体 ROI 归因精度还在迭代.

6. **平台政策 6 个月一变** — 蒲公英 / 薯条 / 聚光 规则更新频繁, 工具方法论 6 个月一改.
