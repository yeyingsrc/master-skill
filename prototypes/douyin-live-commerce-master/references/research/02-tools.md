> 调研日期 2026-05-14，时间盒 ~12 min。

>
> **行业开篇诚实声明**：抖音直播带货的工具栈是一个**强平台依赖、单一供应商主导**的体系——核心必备工具 80% 是字节跳动自家产品（千川 / 抖店 / 巨量百应 / 罗盘 / DOU+ / 巨量算数 / 巨量学），第三方工具（蝉妈妈 / 飞瓜 / 新抖）只做「外部视角的估算数据」，且**估算数据与抖音真实数据有 10-30% 偏差**（intake 警示，本轮多源证实）。这意味着：①「新兴工具」≈「字节自家产品的年度升级」（如 2025-12 的千川·乘方），不是独立创业公司的新产品；②工具栈衰减点高度集中在「千川产品形态」——2025 年标准推广下线、全域推广上位、千川·乘方 AI 化，一年内工作流被改写一次；③选型决策树的核心分支不是「选哪个工具」（必备工具没有替代品），而是「同一个千川，用标准推广还是全域推广、用控成本还是放量、自己投还是托管」。本文件按 02-tools.md「主流工具背后是闭源 → license: proprietary + 第三方 case study」「工具栈进入稳态期 → 新兴层数量少不算调研失败」两条失败处理路径执行。

## 必备层（≥ 80% 从业者用）

### 1. 巨量千川（Qianchuan）

- **One-liner**: 抖音直播间付费投流的主战场 —— 整合直播/短视频/商品卡全场景的电商广告平台；行业铁律「千川是撬自然流的杠杆，不是买观众」。
- **Tier**: 必备
- **Maintainer / Owner**: 巨量引擎（字节跳动）；2024 起千川团队并入抖音电商，与电商组织共担 GMV/ROI 目标（T02-S017）→ 反映「投放」与「经营」在平台侧已不分家
- **License**: proprietary（平台广告产品，按消耗计费 + 技术服务费）
- **Maturity signals**:
  - 行业采用率：投放派从业者 100% 使用；任何有付费投放需求的直播间绕不开（intake + Track 04/06 共识）
  - 关键产品节点：2020-12 巨量千川发布；2024 商品全域推广 + 商品卡投放；2025-05-22【推商品-标准推广】下线；2025-12-11「千川·乘方」3.0 AI 升级（last checked: 2026-05-14）
- **典型使用工作流**（非 marketing）:
  1. **建计划**：选投放场景（推直播间 / 推短视频 / 推商品）→ 选模式（标准推广 / 全域推广）→ 设定向（全域推广弱化定向，强调「素材即定向」）→ 上素材（每计划通常 3 条，3+3+3 原则）
  2. **设出价/预算**：控成本投放（出价=可接受最高价，成本曲线平稳）or 放量投放（前期消耗快、成本由低拉高，适合快速起量）；全域推广只需设整店 ROI 目标
  3. **过冷启动/学习期**：新计划积累约 20-50 个转化前模型不稳定，期间忌频繁改计划/调价
  4. **盯盘**：实时看消耗速度、ECPM、pCTR/pCVR、直播间 GPM/UV 价值；素材衰减（投 1-2 天成本上升）就换素材
  5. **复盘**：拆「计划维度」（pCVR/ECPM）和「直播间维度」（GPM/承接），决定加预算 / 关计划 / 换素材
- **CLI 化暗示（v0.6+）**：核心决策节点「标准推广 vs 全域推广」「控成本 vs 放量」「冷启动期改不改计划」可直接抽成 `cli/decision/qianchuan-mode.sh` 的分支（见 §选型决策树 决策树 B）。
- **不适合 / 替代方案**: ① 直播间承接（UV 价值/话术/品组）不行时**不该加投千川** —— 等于加速亏钱（Track 04/06 强共识）；② 纯内容涨粉/非电商场景用 DOU+ 不用千川；③ 极小商家/达人轻量投放用「小店随心推」（千川移动端轻量版）。**没有第三方替代品** —— 抖音生态内付费投放只有巨量引擎一家。
- **生产案例**: 行业「ROI 70 深度分析抖音投放逻辑」类操盘手长文（数英多篇）；千川大会案例「GMV 破亿品牌」增长路径（T02-S016）；追觅高客单年销破 6 亿的千川开年攻略（数英 890259）
- **维护者背景（sub_skill_link）**: 巨量引擎产品负责人 / 千川大会产品发声人（intake Track 01 strategy note 点名「巨量引擎产品负责人」；Track 05 已标「千川大会产品负责人姓名待 wave 2 落实」）

### 2. 抖音小店（抖店）+ 飞鸽客服

- **One-liner**: 抖店是交易闭环的底座（开店 / 商品管理 / 订单 / 资质），飞鸽是抖店官方客服工作台（IM / 售后 / 退货退款处理）—— 两者是「能不能做生意」的及格线，不是「做得好不好」的加分项。
- **Tier**: 必备
- **Maintainer / Owner**: 抖音电商（字节跳动）
- **License**: proprietary（开店免费，按交易抽佣 + 保证金）
- **Maturity signals**:
  - 行业采用率：所有自营商家 100%（不开抖店无法挂小黄车交易）；飞鸽是抖店强制客服入口（2022-08 起新店需通过「平台客服考试」才能进会话页，T02-S005 类）
  - last checked: 2026-05-14（官网持续运营）
- **典型使用工作流**（飞鸽侧，售后是本行业高频痛点）:
  1. 商品/订单管理在抖店后台（fxg.jinritemai.com）；客服会话/售后在飞鸽（im.jinritemai.com 或抖店后台右上角工作台进入）
  2. **服务体验考核硬指标**：IM 3 分钟平均回复率 ≥ 95%；仅退款自主完成 ≤ 1 小时；退货退款自主完成 ≤ 6 小时；服务体验占店铺体验分 35%（T02-S020 类来源）
  3. **降退货率操作**：开「代申售后」功能帮顾客填退货信息、通过飞鸽选「非品质退货原因」（区分「品退」=商家造成 vs 非品退，影响店铺权重）
- **不适合 / 替代方案**: 抖店是入驻抖音电商的唯一官方交易工具，**无替代**；飞鸽同理是抖店生态内官方客服工作台。第三方客服外包/SCRM 工具可补充，但 IM 主入口必须是飞鸽。

### 11. FACT 经营矩阵 ⚠️【框架，非工具软件 — intake 显式要求标注】

- **One-liner**: 抖音电商官方提出的「四阵地配比」经营诊断框架 —— **F**ield 商家自播 / **A**lliance 达人矩阵 / **C**ampaign 营销活动 / **T**op-KOL 头部大 V；2022 升级为 S-FACT（S=Source 流量供给）。**这是一套思维框架/坐标系，不是一个可登录的工具平台。**
- **Tier**: 场景特化（按「方法论工具」收录，但 type 明确标注为 framework）
- **Maintainer / Owner**: 抖音电商 x 贝恩咨询（《2021 抖音电商商家经营方法论白皮书》，T02-S008）

- **典型使用场景**（针对场景特化层）:

## 选型决策树

> 本行业的核心特征：**必备工具没有替代品**（千川/抖店/罗盘/百应都是字节独家）。所以决策树的真正分支不是「选 A 还是 B」，而是「同一个工具，怎么用 / 用哪个模式 / 自己做还是交给系统」。决策树节点数 = 7（在 5-10 区间）。

### 决策树 A：你的核心目标/瓶颈是什么？

#### Branch 1：还没开始 / 0→1 起号阶段
→ **必装**：抖音小店（抖店）+ 飞鸽（交易底座）、抖音电商罗盘（看真数据）、巨量学（学投放/运营，免费且最新）
→ **先用**：巨量算数（选品定方向）；DOU+（轻量加热短视频引流，门槛低）
→ **先别碰**：千川重投放（承接没跑通前投千川 = 加速亏钱，Track 04/06 强共识）；蝉妈妈/飞瓜年费（0→1 阶段免费工具够用，别先付费）；千川·乘方（首年产品，新手先用基础全域推广）
→ 真实依据：Track 06「起号有大量运气和试错成分」+「投流是撬自然流的杠杆，承接不行白搭」；intake「不适合 90% 想做主播的人」

#### Branch 2：已能稳定出单，要放量（1→10 阶段）
##### Branch 2a：瓶颈在「付费流量效率」（投放派场景）
→ **主战场**：巨量千川 —— 【推直播间】场景可在残存的标准推广里精细控，或用直播全域推广；【推商品】场景只能用商品全域推广（标准推广 2025-05-22 已下线）
→ **投放模式选择**（→ 见决策树 B）
→ **复盘**：抖音电商罗盘（真数据，四大指标）+ 巨量百应（达人合作真数据）
→ 不推荐：只靠 DOU+ 放量（弱、不可控，不是带货成交工具）
##### Branch 2b：瓶颈在「选品/货」（选品派场景 —— Track 04「货是天花板」）
→ **选品工具链**：巨量算数（行业趋势/垂类热度，定方向）→ 蝉妈妈或飞瓜（查具体商品销量/竞品/达人，但记住是估算，偏差 10-30%）→ 巨量百应（看自己商品被带的真数据）
→ 单抖音 + 直播电商深度 → 蝉妈妈；跨平台对标（抖音+快手+淘宝）→ 飞瓜；内容/账号/MCN/明星视角 → 新榜「新抖」
→ 避坑：不要拿蝉妈妈/飞瓜的估算数当真数据做达人结算或合规举证
##### Branch 2c：瓶颈在「经营结构」（不知道该往哪扩）
→ **用 FACT 框架（不是工具，是坐标系）**：诊断当前是不是「只有 F 没有 A」；按预算阶段排序 F→A→C/T，不要四件齐做
→ 配合：巨量百应（铺达人 = 落地 FACT 的 A 阵地）

#### Branch 3：规模化阶段 / 多直播间 / 想用 AI 提效（10→100 阶段）
→ **可评估**：千川·乘方（千策整店预算托管 + 千意 AIGC 素材 + 千寻需求预判）—— 平台用 0.6% 费率优惠强推
→ **权衡**：托管 = 交出投放控制权，资深操盘手对「黑箱化」有疑虑（Track 04 投放派 vs 运营派之争）；首年产品、无独立 case 验证 → 用 `stability: experimental` 心态，小预算试，别 all in
→ 真实案例：`[no public independent case found — 2025-12 发布距今 ~5 个月]`，仅平台官方发布信息，本 branch 可信度降级
→ 配套：矩阵号/切片场景下，蝉妈妈/飞瓜做矩阵账号监测

### 决策树 B：千川投放，用哪个模式？

#### Branch B1：【推商品】场景
→ **只能用「商品全域推广」**（标准推广已于 2025-05-22 下线该场景）—— 上素材 + 设 ROI，核心竞争力转移到「素材质量」（多上一条视频 ≈ 多花约 65 元/天，T02-S012）
→ 配套：AI 辅助产素材（控制创作成本）+ 系统化素材生命周期管理（测试/跑量/衰减）+ 快速监控竞品素材（30 分钟级）

#### Branch B2：【推直播间】场景 + 有专业操盘能力
→ 可用残存的精细投放（手动控计划/出价/定向）或直播全域推广
→ 出价方式：求成本稳 → 控成本投放；求快速起量 → 放量投放；低客单宽人群品 → 通投拉满
→ 注意：平台在主动收回「精细可控」选项，操盘手要做好「迁移到全域」的预案

#### Branch B3：中小商家 / 无专业操盘 / 求省事
→ 直接用全域推广（一键投放、设 ROI 即可，操作次数 -65%）；或评估千川·乘方千策（整店预算托管）
→ 不要硬学标准推广 —— 既学不会又正在被下线

## 避坑清单

- ❌ **不要在直播间承接没跑通时加投千川**：UV 价值/话术/品组不行时投千川 = 加速亏钱。「投流就能卖爆」是课程方神话，效果由素材、选品、承接共同决定，没人能「保 ROI」（Track 04/06 强共识，intake 厂商话术拒绝清单）。
- ❌ **不要把蝉妈妈/飞瓜的估算数据当真数据**：与抖音真实数据有 10-30% 偏差；直播场景下「一个商品多直播间在带 + 临时改价/防拍价」会让偏差放大（案例：同链接 30 天热推达人，飞瓜显示 34 人、蝉妈妈显示 547 人）。决策/合规/达人结算必须以巨量百应/罗盘的真数据为准（T02-S021）。
- ❌ **不要把 DOU+ 当千川用**：DOU+ 是弱内容加热工具（涨粉/曝光），千川才是直播间成交投放主战场，量级和逻辑完全不同。用 DOU+ 做精细成交 = 用错工具（Track 06 外行破绽 top 10）。
- ❌ **不要把 FACT 当「工具」或「四件都要满分的清单」**：FACT 是经营框架/坐标系不是软件，且是「平台官方话术」不是中立行业标准；多数中小商家只做得动 F+A。课程方大量把 FACT 二次包装成「秘籍」售卖，是割韭菜信号（Track 06）。
- ❌ **不要在 2025 年还硬学千川「标准推广」**：【推商品-标准推广】2025-05-22 已下线，全面切全域推广。新手既学不会标准推广的精细操作，又在学一个正在消失的东西 —— 直接学全域推广 + 素材能力（T02-S012/S014）。
- ❌ **不要 all in 千川·乘方（首年产品）**：「整店预算托管」=进一步交出投放控制权，AIGC 日产百万素材的实际转化质量待验证，无独立第三方 case。平台用 0.6% 费率强推不代表已成熟 —— 小预算试，别梭哈（T02-S015）。
- ❌ **不要忽视飞鸽的服务体验硬指标**：IM 3 分钟回复率 ≥ 95%、退货退款自主完成 ≤ 6 小时，服务体验占店铺体验分 35%，做不到直接砸店铺权重。退货率不只是「损失钱」，是「权重杀手」+ 监管红线（T02-S020 + Track 06）。
- ❌ **不要指望第三方工具能给「内部真相」**：本行业工具栈是单一供应商主导，蝉妈妈/飞瓜/新抖只能给「外部估算」。想要真数据，唯一来源是字节自家的罗盘/巨量百应（仅商家本人可见）。

## Phase 2 提炼提示

### 反复出现 ≥ 3 source 都强调的「工具选型原则」（候选 playbook 规则）

- **「承接 > 投流」/ 工具不能补能力的洞**：如果直播间「加了流量但 GMV 不涨」，先查承接（UV 价值/品组/话术），**不要继续加投千川**。（出现于：Track 04 canon / Track 06 glossary / T02-S012 / 千川操盘手长文）
- **「真数据 vs 估算数据」二分**：决策/合规/结算用字节自家的罗盘/巨量百应（真数据，仅本人可见）；选品/对标用蝉妈妈/飞瓜（估算，偏差 10-30%，不可作 ground truth）。（出现于：Track 05 / Track 06 / T02-S021 / T02-S019）
- **「选品定方向 → 查具体品」工具链分层**：巨量算数定行业趋势/垂类热度 → 蝉妈妈/飞瓜查具体商品/竞品 → 巨量百应看自己真数据。不要用一个工具干所有事。（出现于：T02-S006 / T02-S018 / Track 04）
- **「平台在收回控制权，操盘手要顺势」**：标准推广下线、全域推广上位、千川·乘方托管 —— 工具演进方向是「从管计划到管目标」「从操盘手依赖到算法+内容驱动」。选型要顺这个方向，不要逆势死守精细手动。（出现于：T02-S012 / T02-S014 / T02-S015 / T02-S016 / T02-S017）
- **「FACT 是坐标系不是清单」**：用 FACT 诊断经营结构，但按预算阶段排序 F→A→C/T，不四件齐做。（出现于：T02-S008 / Track 04 / Track 06）

### 显著的工具流派分裂（候选「智识谱系」条目 — 直接对应 Track 04/06 已识别的派系）

- **流派 A：精细操盘派（手动控）** —— 代表工具：千川标准推广、手动控成本/放量/定向；代表心智：「投放是手艺，操盘手价值在精细调控」；代表人物：一线投手类操盘手（Track 01 待补）。**这一派的工具正在被平台主动下线**（标准推广 2025-05-22 退场）。
- **流派 B：系统智能派（托管）** —— 代表工具：全域推广、千川·乘方（千策整店托管）、千意 AIGC 素材；代表心智：「投放交给算法，人的价值在素材和选品」；代表载体：巨量引擎产品团队（平台官方主推方向）。
- → 这组分裂的底层是 Track 04 已点的「投放派 vs 运营派」之争，但 2025 年有了新变量：**平台用产品下线 + 费率优惠，强行把行业从 A 推向 B**。Phase 2 心智模型可提炼：「这一行的工具自主权在收缩 —— 资深的标志从『会精细操盘』变成『会喂好素材 + 选对货 + 看懂真数据』」。
- 另一组（次要）：**官方真数据派 vs 第三方估算派** —— 罗盘/巨量百应（真、闭、仅本人）vs 蝉妈妈/飞瓜/新抖（估算、开放、可看竞品）。不是对立是互补，但「把估算当真数据」是高频反模式。

### 新兴工具信号

- **当前活跃/上升的新工具数：2**（千川·乘方、商品全域推广）—— 但都是字节自家产品的年度 AI 升级，**不是独立创业公司的新产品**。
- **出现 → 主流速度估计**：极快，**< 6 个月**。依据：标准推广 2025-05 下线、千川·乘方 2025-12 发布、平台用 0.6% 费率强推迁移 —— 平台主导的工具更替，没有市场自然扩散的缓冲期，「发布即主流」。
- **结构判断**：本行业工具栈**已进入「平台主导的稳态期」** —— 不会有独立第三方颠覆性新工具（生态闭合在字节内），但「字节自家产品的形态」高频变（年度 AI 升级）。这符合 02-tools.md「工具栈进入稳态期，新兴层数量少，不算调研失败」。

### 冷僻 / 信号薄弱

按 02-tools.md Step 6 自检逐条：

- **必备层 < 3 个？** ✅ 否（7 个）。
- **必备层证据 < 50% 采用率？** ✅ 否 —— 必备层 7 个均为「不用就无法运营」的字节独家产品，采用率实质 100%（无 survey 数字，但「无替代品 + 平台强制」是更硬的证据）。
- **新兴层 0 个？** ✅ 否（2 个）。
- **三层 floor**：必备 ≥ 3（实 7）✅ / 场景特化 ≥ 5（实 5）✅ / 新兴 ≥ 2（实 2）✅。
- **候选数**：retained 14 个（必备 7 + 场景特化 5 + 新兴 2）；探索约 18 个（额外探索但未单独立卡：小店随心推 = 千川移动端轻量版并入 #1 描述 / 抖查查 = 与新抖同类第三方未单独深挖 / 巨量引擎广告平台 = 千川的上层广告主后台并入 #1 / 考古加 = 第三方数据小众未深挖）。**未触发冷僻协议**（≥ 15）。
- **一手来源占比**：manifest 22 条中 verified_primary 0 条 + surrogate_primary 7 条（约 32%，按 manifest 规范 surrogate_primary 计入「一手」）。**低于 02-tools.md「≥ 50% 一手」门槛**，且**没有一条真 `verified_primary`** —— 这是本行业的结构性问题，不是调研缺漏：
  - 必备工具的「一手」只能是 vendor 官网（字节不开放产品文档给公开抓取，巨量学课程需登录），且 `source_verifier.py` 不认 zh-CN vendor 域名、机械口径下只能落 `surrogate_primary`，官网首页信息量也有限；
  - 真正能讲清「实际使用工作流」的是亿邦动力/数英/IT之家/B站操盘手的二手长稿（本文件大量依赖）；
  - 这与 Track 05 的发现一致：「本行业一手高度依赖字节官方单一渠道」。
- **触发诚实边界标注**：→ **Phase 2.8 应标注「工具栈维度：一手占比偏低（~32%），且全部是 `surrogate_primary`（vendor 官网），无一条独立 `verified_primary`；必备工具 100% 是单一供应商（字节跳动）的闭源产品；工具的『实际使用工作流』主要从行业媒体二手长稿和 B 站操盘手实操视频还原，非来自 vendor 一手文档。工具栈衰减极快且由平台单方主导 —— 2025 年内标准推广下线、全域推广上位、千川·乘方 AI 化，工作流被改写一次。用户使用本 skill 的工具栈节时，务必去巨量学/千川官网核对当前产品形态。」**
