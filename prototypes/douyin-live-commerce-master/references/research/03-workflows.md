## Workflow 详表

### 1. 跑一场完整直播（场前选品 → 场前预热 → 场中节奏 → 场后复盘）

- **One-liner**: 从「决定开一场带货直播」到「这场播完且复盘出下一场优化清单」的完整闭环 —— 这一行最高频、最核心的单位工作流。
- **Trigger**: 商家/团队已有抖音小店、已搭好直播间、有可播的货，决定开一场直播（日播商家则是每天一次）。 evidence: [T03-S001, T03-S002]
- **Output**: 一场已执行完毕的直播 + 产出的 GMV/UV 价值/GPM 数据 + 一份四象限复盘报告 + 下一场的排品/话术/投放优化清单。 evidence: [T03-S001, T03-S005]
- **入门 SOP（minimum viable steps）**:
  1. **场前选品 + 排品（提前 3-7 天）**: 操盘手按账号标签定选品方向，把货分成「五类款」—— 引流款/福利款（低价拉人留人）、主推款/爆款（人气峰值时主卖）、利润款（赚钱）、对比款（衬托主推款）、形象款（拉调性）；按「127 原则」(福利+主推 : 利润 : 对比+形象 = 1:2:7) 排出上架顺序，注意品之间有关联性、价格梯度不跳。**跳过会发生什么**：没引流款拉不起实时数据，全是利润款直播间起不来；选品与账号标签不符则流量进来不转化（UV 价值低）。 evidence: [T03-S001, T03-S002]
  2. **写脚本 + 定岗位分工（提前 1-3 天）**: 写整场脚本框架（主题/日期/时长/目标 + 前中后每个岗位做什么动作配什么福利）；话术按六板块准备（活动/互动/讲款/打单/逼单/气氛话术）；明确主播（讲款+互动）、中控（后台上下架/改价/库存/发福袋/递话）、场控（节奏把控：何时过款/憋单/拉互动）分工。**跳过会发生什么**：没脚本则节奏全靠主播即兴，数据节奏散；岗位不分则中控/场控/主播互相抢动作，后台操作出错（上错链接、改错价）。 evidence: [T03-S001, T03-S002]
  3. **场前预热（开播前 3 小时～提前 3-5 天）**: 提前 3-5 天发短视频作品（主打卖点/颜值/性价比）为直播间导初始精准流量；开播前 3 小时发带直播预告+福利的预热短视频。**跳过会发生什么**：开播时无初始流量，冷启动期数据更难看，算法没数据可判断；据二手测算预热视频可提升约 40% 初始流量。 evidence: [T03-S001, T03-S002]
  4. **场中执行 — 开场暖场（开播 0-10 分钟）**: 主播自我介绍 + 告知直播时间和福利预告 + 用利益点直给（「今天专属价仅 50 单」）和互动引导（「扣 1 抽免单」）快速建立观看理由、留住人。**跳过会发生什么**：开场留不住人，停留时长不达标（冷启动期第一权重指标，目标人均≥2 分钟），算法判定内容差不再推流。 evidence: [T03-S001, T03-S002]
  5. **场中执行 — 过款 + 憋单 + 节奏循环**: 按排品过款（单品讲解固定流程：卖点引出/手拿介绍/用户评价/促单，约 2+3+3+3 分钟），用憋单（低价钩子留人不上库存）蓄水实时数据，人气峰值时放主推款，每 15 分钟设福利节点（福袋/秒杀）防流失；场控盯实时数据调节奏。**跳过会发生什么**：没节奏设计则在线人数曲线平直、无法在峰值收割；不设福利节点观众流失快。 evidence: [T03-S001, T03-S002]
  6. **场后复盘（播后当天/次日）**: 四象限复盘 —— 流量数据（场观/峰值环比）、互动数据（停留率/点赞率/加粉率）、交易数据（GMV/客单价/转化率/UV 价值）、现场反馈（人员分工/技术问题）；梳理爆品与滞销品，定位异常时段，产出下一场优化项。**跳过会发生什么**：不复盘则同样的选品/话术/节奏错误每场重犯，直播间无法迭代。 evidence: [T03-S001, T03-S005]
- **资深路径（差异点）**:
  - **skip**: 资深操盘手对成熟账号会**跳过繁琐的「养号期模拟互动」动作**（新手教程强调的前 3 天刷同行+点赞养标签），因为账号标签已稳定，靠选品和承接维持标签即可。
  - **optimize**: ① **选品和排品从「套 127 公式」优化为「按账号实时标签 + 供应链能给什么」动态调」**—— 资深人知道公式是死的，垂类不同（服饰退货率 40-60% vs 食品低）配比要变。② **话术从「临摹同行」优化为「临摹→拆分归类→融合创新」形成自己的话术库**，新主播照搬，资深主播有沉淀。③ **场中节奏靠盯盘实时调**，不是脚本写死。 evidence: [T03-S001]
  - **add**: ① **倒推账号结构** —— 资深操盘手先设想理想账号形态，反向规划这场直播该产出什么数据，新手只想「这场卖多少」。② **短视频端 + 直播推荐双轮驱动** —— 资深人重视视频端流量培育，新手只盯直播推荐流量（这是「99% 失败模式」之一）。③ **付费流量「超长周期低烧」** —— 资深人慢投长投校准模型，新手快速消耗预算。④ **合规自查前置** —— 资深人开播前过一遍绝对化用语/资质，新手播完被罚才知道（见 workflow 6）。 evidence: [T03-S001, T03-S010]
  - 差异点统计：skip 1 + optimize 3 + add 4 = 共 8 处，远超 retained 合格线（≥2 处 + ≥2 类型）。
- **近期变化**（近 12 个月）:
  - **2025-05 起，由巨量千川全面切换「全域推广」模式触发**，原 step 3「场前预热只发短视频导直播间」扩展为「短视频 + 直播 + 商品卡三载体协同预热」；场中承接也要兼顾货架场（商城/搜索）流量。 evidence: [T03-S003, T03-S004]
  - **2025-12 起，由千川·乘方（千意 AIGC 模块）上线触发**，原 step 3 的「场前预热短视频靠人工拍剪」开始转向「AIGC 辅助批量产素材」。 evidence: [T03-S009]
  - **2026-02 起，由《直播电商监督管理办法》施行触发**，step 2 写脚本时新增「绝对化用语自查」前置环节，AI 数字人直播需加显著标识。 evidence: [T03-S011]
  - 触发事件类型：新工具（全域推广、千川·乘方）+ 法规变化（117 号令）。
  - 当前采用率：全域推广为千川强制默认（非可选），故场中兼顾货架场承接是 2025-05 后所有商家的既成事实；AIGC 素材采用率无公开数据，估计仍在早期。
- **典型耗时**:
  - 入门 SOP: 单场直播本身 2-6 小时 + 场前准备分散在 3-7 天 + 场后复盘 1-3 小时。
  - 资深路径: 单场直播时长相近，但场前准备更高效（话术库/选品池已沉淀），复盘更深（带可视化沉淀）。
- **关键工具（与 Track 02 关联）**: 抖音小店 + 飞鸽客服（交易闭环，必备）、抖音电商罗盘（场后真实数据复盘，必备）、巨量百应（商家工具，必备）、福袋/小黄车（场中互动，平台功能）、蝉妈妈/飞瓜（选品对标，场景特化）、千川（场中投放，必备 — 见 workflow 3）。
- **关键人物（与 Track 01 关联）**: 尹晨在《抖音直播带货实操指南》中详细谈过「算法→人货场→投放→复盘」全链路（Track 04 canon 已收录）；操盘手类自媒体（Track 01 待补）在数英/刀法访谈中谈过场中节奏与 GPM。
- **常见失败模式**:
  - **只盯直播推荐流量、忽视短视频端培育**: 触发原因 —— 新手以为直播间流量只来自直播广场。后果 —— 流量结构单一、抗风险弱、起不来（数英稿称「99% 失败模式」之一）。怎么避免：**场前预热必须发短视频导流，把短视频端当与直播同等重要的流量入口，不要只在直播开始后才想流量**。 evidence: [T03-S001]
  - **选品与账号标签不符**: 触发原因 —— 跟风大主播选品、不看自己账号的用户画像。后果 —— 流量进来不转化，UV 价值低，加投流等于加速亏钱。怎么避免：**选品前先拉账号实时标签和历史成交人群，按自己的用户画像选品，不要因为某品在别人直播间卖爆就上**。 evidence: [T03-S001, T03-S002]
  - **憋单时间过长 / 承诺福利不兑现**: 触发原因 —— 为冲实时数据过度憋单。后果 —— 引发投诉和限流，且 2026 新规下虚假承诺有合规风险。怎么避免：**憋单设明确时间上限，承诺的福利款一定按量兑现，把「真实节奏控制」和「虚假紧缺欺骗」分清**。 evidence: [T03-S001, T03-S011]
  - **场后不复盘 / 复盘只看 GMV 总量**: 触发原因 —— 嫌复盘麻烦，或只关心卖了多少钱。后果 —— 靠投流堆出的高 GMV 可能 GPM/UV 价值很低、不可持续，同样的错误每场重犯。怎么避免：**每场必复盘四象限，重点看 GPM/UV 价值/停留时长这组「算法评分代理」指标，而非只看 GMV**。 evidence: [T03-S001, T03-S005]
  - **CLI 化暗示 (v0.6+)**: 上述每个 failure 写法已 actionable，可直接抽进 `cli/workflow/run-one-live.sh` 的失败模式自检环节，每条变成 yes/no 检查项（如「你这场的场前预热短视频发了吗？」「你的选品是按自己账号标签选的还是跟风选的？」「这场复盘你看了 GPM 和 UV 价值吗？」）。

### 2. 商家从入驻到稳定输出（0→1 / 1→10 / 10→100 三阶段全生命周期）

- **One-liner**: 从「商家决定把抖音直播当经营阵地」到「拥有一个流量结构健康、能稳定月销的直播间」的全生命周期路径 —— 跨度数月到一年以上，是 intake 点名的两大主干工作流之一。
- **Trigger**: 商家（品牌方/工厂/个体）决定入驻抖音、把直播带货作为正式经营渠道。 evidence: [T03-S006, T03-S008]
- **Output**: 一个标签精准、能稳定月销（百万→千万级）、流量结构健康（自然流 + 短视频 + 付费三条腿）的直播间 + 一套可复用的运营 SOP。 evidence: [T03-S006, T03-S008]
- **入门 SOP（minimum viable steps）**:
  1. **入驻 + 基建（第 1 周）**: 注册抖音小店（个体工商户营业执照 + 缴店铺保证金），可绑定多个抖音账号实现 0 粉开播；完成主播实名认证（身份证+人脸，开播权限前提）；搭直播间（补光灯/麦克风/支架/功能分区）；组团队（最少 4 人：运营/场控/主播/助理）。**跳过会发生什么**：无营业执照/未实名无法开播；团队不齐则一人多岗、动作变形。 evidence: [T03-S002, T03-S006]
  2. **0→1 冷启动期（第 1 周～约 1 个月，目标月销百万级）**: 跑通「流量进来→留得住→转化掉」正循环。打法 = 货上亏钱留人（福利款占比高，约 80%）+ 抓停留和转化两个核心指标 + 投放加速。不追求销售额和利润率，只追账号标签精准化。**跳过会发生什么**：跳过冷启动直接追 GMV，则账号标签泛、后续流量进来都不转化，越投越亏。 evidence: [T03-S006, T03-S008]
  3. **1→10 增长/拉升期（约 1-3 个月，目标月销千万级）**: 爆品主推款占比上升（约 70%），同时关注人气/互动/停留/关注 + 转化率/销售额；流量上靠「短视频 + 自然流量」双驱动；用付费撬免费，免费流量提升转化后让标签更精准，再带来更多免费流量。**跳过会发生什么**：拉升期不上短视频端、只靠直播推荐，流量天花板很快到顶。 evidence: [T03-S006, T03-S008]
  4. **10→100 稳定/放大期（约 3-6 个月+，月销千万以上）**: 各分层产品数据均衡，重点转向「直播间整体数据 + 供应链 + 售前售后服务」；流量上引入「付费流量 + 短视频 + 自然流量」三条腿，付费占比上升。**跳过会发生什么**：放大期供应链跟不上则爆单即翻车（缺货/发货慢/退货率飙升）。 evidence: [T03-S006, T03-S008]
  5. **持续迭代（贯穿全程）**: 每场直播按 workflow 1 跑 + 复盘；把直播切片成短视频二次分发；坚持稳定开播频次（每周 3+ 场，二手测算第 8 周 GMV 可显著增长）。**跳过会发生什么**：开播不稳定则算法无法积累账号权重，每次开播都像重新冷启动。 evidence: [T03-S006]
- **资深路径（差异点）**:
  - **skip**: 资深商家**坚决跳过「盲目矩阵化」** —— 新手在 0→1 还没跑通就复制十几个账号「赛马」，结果账号在同一用户池内互相抢量、每个直播间流量稀薄、「花很多钱都干不成」。资深人先把一个号跑通到稳定，再考虑有分工的矩阵。 evidence: [T03-S008]
  - **optimize**: ① **三阶段的流量配比不是死的** —— 资深人按类目和供应链能力调（标品 0→1 可纯自然流，非标品可能 0→1 就要投流）。② **「倒推账号结构」** —— 资深操盘手在入驻时就设想好理想账号终态，反向规划每个阶段的目标，新手走一步看一步。 evidence: [T03-S001, T03-S006]
  - **add**: ① **供应链前置** —— 资深人入驻前先确认供应链能不能扛住放大期的量（「99% 死在供应链短板」，线下厂商尤其盲目自信），新手等爆单了才发现缺货。② **全域承接布局** —— 2025 后资深商家从入驻就规划「直播 + 短视频 + 商品卡」三载体，不只做直播场。③ **心流管理 / 预期管理** —— 资深人初期主动降低预期、长期布局，新手把直播当「弯道超车」、投产不佳就摇摆放弃（「执行力不足」失败模式）。 evidence: [T03-S001, T03-S003, T03-S008]
  - 差异点统计：skip 1 + optimize 2 + add 3 = 共 6 处，合格。
- **近期变化**（近 12 个月）:
  - **2025-05 起，由千川全域推广 + 货架场崛起触发**，原本「商家全生命周期 = 把直播间做大」升级为「把直播 + 短视频 + 商品卡三载体协同做透，形成拉新-转化-复购闭环」；10→100 阶段的「放大」不再只是直播间放大，而是全域放大。 evidence: [T03-S003, T03-S004]
  - **2022 起（薇娅案后强化），由税务全面转查账征收触发**，头部商家/工作室在 10→100 阶段的收入结构和开票流程改变，「核定征收 + 个独/个体户」的老避税工作流作废。 evidence: [T03-S011]（Wave 1 Track 06 seed）
  - 触发事件类型：新工具 + 行业事件（货架场战略）+ 法规变化（税务）。
  - 当前采用率：全域经营是平台强力推动的默认叙事，主流商家已在跟。
- **典型耗时**:
  - 入门 SOP: 0→1 约 1 个月，1→10 约 1-3 个月，10→100 约 3-6 个月+ —— 全程半年到一年以上（且大量商家卡在 0→1 上不去，见失败模式）。
  - 资深路径: 阶段划分相近，但资深团队 0→1 更快（标签校准经验 + 选品池），且不会在 0→1 反复打转。
- **关键工具（与 Track 02 关联）**: 抖音小店 + 飞鸽客服（必备）、抖音电商学习中心（官方成长路径课，必备）、千川（增长/放大期必备）、抖音电商罗盘 + 巨量百应（数据，必备）、蝉妈妈/飞瓜（选品对标 + 竞品监测，场景特化）。
- **关键人物（与 Track 01 关联）**: 尹晨在《抖音直播带货实操指南》中谈「90% 直播间起不来」「团队组建是规模化前提」（Track 04 canon）；抖音电商官方在「商家成长路径」课程体系中给出 0→1→进阶分层（Track 04/05）。
- **常见失败模式**:
  - **0→1 没跑通就盲目矩阵化**: 触发原因 —— 以为多开号能多拿量。后果 —— 同主体账号在同一用户池互相抢量，每个直播间流量都稀薄，钱花了号都没起来。怎么避免：**先把一个账号跑到稳定月销，验证 SOP 可复制，再做有明确分工的矩阵；不要在 0→1 阶段开第二个号**。 evidence: [T03-S008]
  - **供应链短板 —— 爆单即翻车**: 触发原因 —— 入驻时只看流量打法、不看供应链，线下厂商尤其「盲目自信」。后果 —— 10→100 放大期一爆单就缺货/发货慢/退货率飙升，号被限流。怎么避免：**入驻前就确认供应链能扛住放大期的量级和退换货，把「货是天花板」当铁律，供应链优势比流量技巧更核心**。 evidence: [T03-S001, T03-S008]
  - **心态失衡 —— 把直播当「弯道超车」**: 触发原因 —— 用「卖货要即时回报」的心态做需要长期布局的事。后果 —— 投产不佳就摇摆、延迟投入、错失早期红利。怎么避免：**入驻时就做预期管理（「心流管理」），初期主动降低预期、长期投入，认知到品牌和卖货是两件事**。 evidence: [T03-S001, T03-S008]
  - **运营人才短缺 —— 招到的都是「庸才」**: 触发原因 —— 优秀操盘手倾向自己创业而非受雇。后果 —— 品牌组不起高效团队，整个生命周期推进慢。怎么避免：**接受「招不到顶级操盘手」是常态，靠 SOP 化 + 官方课程培训现有团队补齐，不要把整盘生意压在「找到一个神操盘手」上**。 evidence: [T03-S008]
  - **CLI 化暗示 (v0.6+)**: 可抽进 `cli/workflow/merchant-lifecycle.sh`，每条 failure 变 yes/no 检查项（如「你当前账号 0→1 跑通了吗？跑通前不要开第二个号」「你的供应链能扛住月销翻 10 倍吗？」）。

### 3. 千川直播间冷启动投放

- **One-liner**: 从「新账号/新计划没有付费数据」到「跑出一个过了学习期、ROI 达标、人群标签精准的千川计划组」的投放工作流 —— 投放派的核心日常，也是行业最高频痛点。
- **Trigger**: 新账号要起量 / 新千川计划要跑通 / 直播间自然流不够要付费撬动。 evidence: [T03-S007]
- **Output**: 一组过了学习期（积累约 20-50 个转化、模型稳定）、ROI 达到目标（≥保本 ROI×1.2）、跑出精准人群标签的千川计划。 evidence: [T03-S007]（Wave 1 Track 06 seed）
- **入门 SOP（minimum viable steps）**:
  1. **确认前端「人货场」最优**: 投放前先保证直播间承接没问题（平均停留≥30 秒、进场留存≥30% 是流量承载力达标线）。**跳过会发生什么**：承接不行就投千川 = 加速亏钱，钱花了流量留不住。 evidence: [T03-S007]
  2. **选投放工具 + 定预算**: 冷启动期可用「小店随心推 + 相似达人定向」轻量起步，单场预算 300-500 元；或用千川标准推广。聚焦「进入直播间」「商品点击」等浅层目标。**跳过会发生什么**：一上来就投深层成交目标，模型没数据可学，跑不动。 evidence: [T03-S007]
  3. **搭计划 + 定向「先窄后宽」**: 定向先窄（精准人群优先跑出模型），跑稳后再放宽找增量。**跳过会发生什么**：一开始就宽定向，人群泛、标签校不准、ROI 难看。 evidence: [T03-S007]
  4. **跑量过学习期**: 让计划快速消耗预算跑出第一波转化数据，积累约 20-50 个转化前模型不稳定，期间**忌频繁改计划/调价**。**跳过会发生什么**：频繁改计划导致模型一直「学不会」，永远过不了学习期。 evidence: [T03-S007]
  5. **盯数据 + 优化 ROI**: 对有潜力的计划优化 ROI，衰减的素材及时换（素材投一两天后成本快速上升）。**跳过会发生什么**：不换衰减素材则成本越跑越高、ROI 持续下滑。 evidence: [T03-S007]
- **资深路径（差异点）**:
  - **skip**: 资深投手在学习期**坚决不碰计划**（不改定向、不调价、不暂停），新手忍不住天天调，把模型搞乱。
  - **optimize**: ① **「先窄后宽」是资深人的标准动作**，新手要么全程窄（跑不出量）要么一开始就宽（标签乱）。② **素材策略 3+3+3**（每计划 3 条素材、每 3 天换 3 条、淘汰 CTR<3% 的）—— 资深人有素材节奏，新手等素材彻底跑死才换。③ 资深人知道 **ROI 不是越高越好**（高 ROI = 不敢花钱 = 做不大），追的是「能放量的可接受 ROI」。 evidence: [T03-S007]（Wave 1 Track 06 seed）
  - **add**: ① 资深投手把 **GPM 纳入复盘**（不只看 ROI），因为 GPM 已进千川推荐排序。② 资深人理解「投流 = 撬自然流的杠杆」不是「买成交」，会算付费流量带来的自然流量增量。 evidence: [T03-S010]
  - 差异点统计：skip 1 + optimize 3 + add 2 = 共 6 处，合格。
- **近期变化**（近 12 个月）:
  - **2025-05 起，由千川全面切换「全域推广」模式触发**，原本「商家自己搭计划、设人群定向」变成「绝大多数普通商家不再支持手动人群定向，靠内容向量化（商品卡标题/直播间场景）让 AI 自动识别意图」；投放决策从「操纵参数」转向「优化内容标签 + 数据反馈」。 evidence: [T03-S003, T03-S004]
  - **2025-12 起，由千川·乘方上线触发**，「千策」模块自动完成预算分配/出价调控/选品匹配，「千意」AIGC 自动生成素材 —— 投手的「手动搭计划 + 人工产素材」工作量大幅压缩，转向「给系统喂目标 + 喂内容标签」。 evidence: [T03-S004, T03-S009]
  - 触发事件类型：新工具（全域推广、千川·乘方）。
  - 当前采用率：全域推广为强制默认（2025-05 起全量切换），故「不再手动定向」是既成事实；千川·乘方 2025-12 上线，2026 上半年逐步铺开。
- **典型耗时**: 入门 SOP 单个计划过学习期约 1-3 天，整个账号冷启动期约 7-30 天。资深路径相近，但资深人废计划率低、过学习期更稳。
- **关键工具（与 Track 02 关联）**: 巨量千川（必备主战场）、小店随心推（冷启动轻量工具，必备）、抖音电商罗盘/巨量百应（投放数据复盘，必备）、千川·乘方（千意/千寻/千策，新兴）。
- **关键人物（与 Track 01 关联）**: 操盘手在数英/刀法访谈中谈过千川推流机制与 GPM 进排序（T03-S010）；巨量学官方课程给出账户冷启动标准流程（T03-S014）。
- **常见失败模式**:
  - **承接不行就投千川**: 触发原因 —— 以为投流能解决一切。后果 —— 钱花了流量留不住，加速亏损。怎么避免：**投千川前先确认直播间停留≥30 秒、进场留存≥30%，承接不达标先修承接再投流**。 evidence: [T03-S007]
  - **学习期频繁改计划**: 触发原因 —— 看到数据难看忍不住调。后果 —— 模型一直学不会，永远过不了学习期，预算白烧。怎么避免：**学习期（积累约 20-50 个转化前）锁死计划不动，给算法学习时间**。 evidence: [T03-S007]
  - **追求 ROI 越高越好**: 触发原因 —— 直觉觉得 ROI 高是好事。后果 —— ROI 高意味着不敢花钱、跑不出量，规模做不大。怎么避免：**目标 ROI 设为「保本 ROI×1.2」左右、能放量即可，不要把 ROI 拉到不敢投钱的程度**。 evidence: [T03-S007]
  - **不换衰减素材**: 触发原因 —— 没有素材更换节奏。后果 —— 素材衰减后成本飙升、ROI 持续下滑。怎么避免：**按 3+3+3 节奏管理素材，CTR 跌破 3% 就淘汰**。 evidence: [T03-S007]
  - **CLI 化暗示 (v0.6+)**: 可抽进 `cli/workflow/qianchuan-coldstart.sh`，每条变 yes/no（如「你的直播间停留时长到 30 秒了吗？没到先别投千川」「这个计划还在学习期吗？在的话不要改」）。
- **来源** (≥ 3):
  - [Secondary] 蝉妈妈运营圈 随心推 vs 千川 ROI 对比 + 冷启动投放打法: https://www.chanmama.com/yunyingquan/article/1151.html （collected: 2026-05-14）
  - [Secondary] 品玩 2025 抖音电商千川大会「主战场变了」— 千川升级为经营平台: https://www.pingwest.com/a/309973 （collected: 2026-05-14）
  - [Secondary] 京报网 千川·乘方上线 — 千意/千寻/千策三模块: https://news.bjd.com.cn/2025/12/19/11473118.shtml （collected: 2026-05-14）
  - [Reference] Wave 1 Track 06 glossary — 学习期/3+3+3/素材衰减/ROI 认知（T06-S013, T06-S017）；Track 04 canon — 巨量学官方课程（T04-S008）

- **Decay risk**: high — 千川产品年度甚至季度迭代（全域推广 2025-05、千川·乘方 2025-12），投放工作流是本 skill 里衰减最快的部分之一，强烈建议每季度刷新。

### 4. 新账号起号（0 粉到稳定出单）

- **One-liner**: 从「新开一个直播间、0 粉无标签」到「跑通『流量-转化』正循环、能稳定出单」的冷启动工作流 —— 是 workflow 2「0→1 阶段」的更细粒度展开，也是行业黑话「起号」对应的标准动作。
- **Trigger**: 新开直播间 / 老号数据做烂要重新起 / 矩阵新增账号。 evidence: [T03-S006]
- **Output**: 一个有精准算法标签、能稳定出单的直播间（注意：0 粉也能起号，涨粉是副产物不是目标）。 evidence: [T03-S006]（Wave 1 Track 06 seed）
- **入门 SOP（minimum viable steps）**:
  1. **账号基建（前 3 天）**: 实名认证（身份证+人脸）；设备正常使用 3 天以上避免被判营销号；账号头像/昵称/简介体现赛道定位。**跳过会发生什么**：未实名无开播权限；新设备秒开播易被判营销号限流。 evidence: [T03-S006]
  2. **养标签（前 3 天，与上并行）**: 每天 30 分钟+看同类目直播，做点赞/评论/停留互动，让系统给账号打精准标签。**跳过会发生什么**：标签不准，开播后流量推给不对的人，转化差。 evidence: [T03-S006]
  3. **选起号路径**: 三选一 —— 短视频起号（靠优质短视频拿自然推荐导流）、千川单品起号（一个爆品+优质素材跑千川）、螺旋起号（让各项数据按 7 天螺旋上升骗系统给更高流量池）。**跳过会发生什么**：没有明确起号路径则动作散、数据不成体系。 evidence: [T03-S006]（Wave 1 Track 06 seed）
  4. **跑数据过测试**: 新直播间要过平台多次（约 3 次）流量测试，重点把停留时长（权重占比约 40%，目标人均≥2 分钟）和进场留存做达标。**跳过会发生什么**：测试不达标则流量池晋级失败、掉级，起号失败。 evidence: [T03-S006]
  5. **稳定开播 + 切片复用**: 每周稳定开播 3 场以上；把直播切成短视频二次分发引流。**跳过会发生什么**：开播不稳定算法不给积累权重；不切片则一次直播内容浪费。 evidence: [T03-S006]
- **资深路径（差异点）**:
  - **skip**: 资深人**跳过「先涨粉再带货」的误区** —— 知道直播流量和短视频是两套算法、0 粉能起号，新手非要先涨粉到几千才敢带货（白等）。
  - **optimize**: 资深人**按账号定位和供应链选起号路径**（标品/有爆品 → 千川单品起号；内容能力强 → 短视频起号），新手乱试或只会一种。
  - **add**: ① 资深人从起号期就**布局直播切片**，用切片矩阵放大；② 资深人知道**账号一旦数据做烂、再救比开新号还难**，所以起号期就严控数据质量、不为冲量做虚假动作。 evidence: [T03-S006]（Wave 1 Track 06 seed）
  - 差异点统计：skip 1 + optimize 1 + add 2 = 共 4 处，合格。
- **近期变化**（近 12 个月）:
  - **2026-02 起，由《直播电商监督管理办法》施行触发**，「螺旋起号」等靠刷数据骗流量池的灰色起号法合规风险加大（数据真实性条款 + 黑名单制度封堵换号）。 evidence: [T03-S011]
  - **2025 起，由货架场崛起触发**，起号不再只是「把直播间数据做起来」，也要兼顾商品卡/搜索的货架场冷启动。 evidence: [T03-S003]
  - 触发事件类型：法规变化 + 新工具（货架场）。
  - 当前采用率：螺旋起号在民间仍流传，但正规商家和 MCN 趋于规避。
- **典型耗时**: 入门 SOP 约 7-30 天（前 3 天养号 + 后续 1-4 周跑通正循环）。资深路径相近，但资深人废号率低。
- **关键工具（与 Track 02 关联）**: 千川/小店随心推（单品起号必备）、抖音电商罗盘（盯起号数据，必备）、剪辑工具+切片（场景特化）、蝉妈妈/飞瓜（选起号品对标，场景特化）。
- **关键人物（与 Track 01 关联）**: 尹晨书谈「为什么 90% 直播间起不来」（Track 04 canon）；操盘手类自媒体在 B 站长讲中拆解起号路径（Track 05 sources「B 站操盘手长视频圈」）。
- **常见失败模式**:
  - **「先涨粉再带货」**: 触发原因 —— 把直播间当短视频账号运营。后果 —— 白白等涨粉，错过起号窗口（直播和短视频是两套算法，涨粉≠能卖货）。怎么避免：**0 粉就开播带货，把「跑通流量-转化模型」当目标，涨粉只是副产物**。 evidence: [T03-S006]
  - **新设备秒开播被判营销号**: 触发原因 —— 不知道有养号期。后果 —— 账号被限流，起号还没开始就废。怎么避免：**新设备正常使用 3 天以上、账号资料填完整再开播**。 evidence: [T03-S006]
  - **靠螺旋起号刷数据骗流量池**: 触发原因 —— 想走捷径。后果 —— 2026 新规下数据真实性违规、黑名单封堵换号，号废了也救不回。怎么避免：**用真实承接（停留/转化）跑数据，不靠刷量；账号数据做烂比开新号还难救**。 evidence: [T03-S006, T03-S011]
  - **CLI 化暗示 (v0.6+)**: 可抽进 `cli/workflow/account-bootstrap.sh`（如「你的直播间停留时长到人均 2 分钟了吗？」「你是 0 粉就开播了还是在等涨粉？」）。
- **来源** (≥ 3):
  - [Secondary] 蝉妈妈运营圈〈抖音直播新手必看：5步数据起号法〉: https://www.chanmama.com/yunyingquan/article/1350.html （collected: 2026-05-14）
  - [Verified primary] 《直播电商监督管理办法》— 数据真实性 + 黑名单（影响螺旋起号合规性）: https://www.cac.gov.cn/2026-01/07/c_1769516655383334.htm （collected: 2026-05-14）
  - [Reference] Wave 1 Track 06 glossary — 起号/螺旋起号/单品起号/短视频起号 词条（T06-S011, T06-S017）；Track 04 canon — 尹晨书（T04-S010）
- **Last_updated**: 2026-05-14
- **Decay risk**: high — 起号是算法博弈的直接产物，算法规则一变起号路径就变；且灰色起号法（螺旋）的合规边界正在收紧。

### 5. 直播复盘与下一场优化

- **One-liner**: 从「一场直播刚结束」到「产出一份带异常值归因 + 下一场可执行优化项的复盘报告」的闭环工作流 —— 是 workflow 1 的 step 6 的独立展开，因为它是「直播间能不能迭代」的关键，单列。
- **Trigger**: 每场直播结束后（日播商家则每天一次）。 evidence: [T03-S005]
- **Output**: 一份四象限/五步复盘报告 + 定位到的异常时段 + 下一场的选品/话术/节奏/投放优化清单。 evidence: [T03-S005]
- **入门 SOP（minimum viable steps）**:
  1. **整体回顾关键数据**: 从人气维度（总场观/平均在线/停留时长/新增粉丝/互动率）和销售维度（销售额/转化率/UV 价值）拉数据。**跳过会发生什么**：不看数据则复盘变成「凭感觉」，无法定位真问题。 evidence: [T03-S005]
  2. **时段对比精细复盘**: 找直播过程中的异常值（某时段在线/转化突然掉或涨），对应到当时的直播流程动作；用 AB 测试（如福袋投放时机）找最佳路径。**跳过会发生什么**：只看整体均值，不知道问题出在哪个环节。 evidence: [T03-S005]
  3. **商品复盘优化选品**: 对比爆品和滞销品的销讲次数/时段/话术，判断选品是否符合用户标签。**跳过会发生什么**：滞销品下次还排进去，爆品没给够讲解时长。 evidence: [T03-S005]
  4. **用户分析匹配标签**: 检验引入的流量是不是目标用户，分析购买和价格偏好。**跳过会发生什么**：流量不精准的根因找不到，继续投错人。 evidence: [T03-S005]
  5. **流量分析优化渠道**: 区分自然流量（搜索/推荐/关注）和付费流量，分析引流视频为什么有效、复制或优化。**跳过会发生什么**：不知道哪个流量渠道有效，预算乱分配。 evidence: [T03-S005]
- **资深路径（差异点）**:
  - **optimize**: 资深运营**靠「时段异常值 → 对应流程动作」精准定位**，新手只会笼统「学会抓重点」，定位不到具体环节。 evidence: [T03-S005]
  - **add**: ① 资深运营做**数据可视化记录**（把每场数据沉淀成可对比的表/图），新手复盘完就忘；② 资深运营复盘**团队职责**（谁的动作出了问题），不只复盘数据；③ 资深运营把复盘结论**迭代回 SOP**，形成闭环，新手复盘和 SOP 是两张皮。 evidence: [T03-S005]
  - 差异点统计：optimize 1 + add 3 = 共 4 处，合格（≥2 类型 + ≥2 处）。
- **近期变化**（近 12 个月）:
  - **2025 起，由 GPM 进千川推荐排序触发**，复盘指标从「主要看 ROI」扩展到「ROI + GPM + 内容互动」多指标。 evidence: [T03-S010]（Wave 1 Track 06 seed）
  - **2025-05 起，由全域推广触发**，复盘范围从「只复盘直播场数据」扩展到「直播 + 短视频 + 商品卡」全域数据。 evidence: [T03-S003]
  - 触发事件类型：新工具（GPM 进排序、全域推广）。
- **典型耗时**: 入门 SOP 1-3 小时/场。资深路径相近（有可视化模板后更快），但复盘更深。
- **关键工具（与 Track 02 关联）**: 抖音电商罗盘（真实复盘数据，必备）、巨量百应（必备）、蝉妈妈/飞瓜（竞品对标复盘，场景特化）。
- **关键人物（与 Track 01 关联）**: 尹晨书强调「数据复盘要看 GPM/UV 价值/停留/转化率这组指标而非只看 GMV」（Track 04 canon）。
- **常见失败模式**:
  - **复盘只看 GMV 总量**: 触发原因 —— 只关心卖了多少钱。后果 —— 看不出 GPM/UV 价值低的不可持续问题。怎么避免：**复盘必看 GPM/UV 价值/停留时长这组「算法评分代理」指标**。 evidence: [T03-S005]（+ Track 04 seed）
  - **复盘不落到具体流程环节**: 触发原因 —— 只看整体均值。后果 —— 知道「这场不行」但不知道哪步不行，下场重犯。怎么避免：**用时段异常值对应到当时的直播动作，定位到具体环节再优化**。 evidence: [T03-S005]
  - **复盘和 SOP 两张皮**: 触发原因 —— 复盘完不更新 SOP。后果 —— 同样的错误每场重犯，复盘白做。怎么避免：**每次复盘结论必须迭代回直播脚本/排品 SOP，形成闭环**。 evidence: [T03-S005]
  - **CLI 化暗示 (v0.6+)**: 可抽进 `cli/workflow/post-live-review.sh`（如「这场复盘你看了 GPM 吗？」「你定位到具体是哪个时段/动作出问题了吗？」「复盘结论写回 SOP 了吗？」）。
- **来源** (≥ 3):
  - [Secondary] 蝉妈妈运营圈 千万 GMV 直播运营复盘技巧 — 五步复盘框架 + 新手 vs 资深差异: https://www.chanmama.com/yunyingquan/article/802.html （collected: 2026-05-14）
  - [Secondary] 数英〈三万字干货〉四象限复盘法: https://www.digitaling.com/articles/534841.html （collected: 2026-05-14）
  - [Reference] Wave 1 Track 06 glossary — GPM 进排序（T06-S013）；Track 04 canon — 尹晨书指标体系（T04-S010）
- **Last_updated**: 2026-05-14
- **Decay risk**: medium — 五步复盘框架本身稳定（low decay），但「复盘哪些指标」「复盘哪些场域」随平台机制变（GPM 进排序、全域），整体判 medium。

### 6. 直播间合规自查

- **One-liner**: 从「准备开播 / 监管或平台规则更新」到「产出一份过了红线检查、可安全开播的合规清单」的工作流 —— 强监管行业的必备工作流，「能不能做」先于「怎么做」。
- **Trigger**: 每场直播开播前（话术/资质自查）+ 监管或平台规则更新时（全面重校）。 evidence: [T03-S011, T03-S012]
- **Output**: 一份合规自查清单（绝对化用语 / 资质公示 / 禁限商品 / AI 标识 / 主播身份 / 售后承诺），全部过检后可安全开播。 evidence: [T03-S011, T03-S012]
- **入门 SOP（minimum viable steps）**:
  1. **绝对化用语自查**: 检查话术里有没有「全网最低价/最好/第一/唯一/史上最低」等《广告法》第九条禁用的绝对化用语，改为「直播间专享价」等。**跳过会发生什么**：被市监部门处罚（已多案被罚），账号也暴露「未受合规培训」。 evidence: [T03-S011, T03-S012]
  2. **资质与信息公示自查**: 检查账号主页/直播页面是否持续公示真实名称、社会信用代码、经营地址、联系方式（2026 新规强制）；专业领域（医疗/财经/法律/教育）主播是否持执业资质并报备。**跳过会发生什么**：违反 2026 新规信息公示义务，处罚 + 黑名单风险。 evidence: [T03-S011, T03-S012]
  3. **禁限商品 + 虚假宣传自查**: 检查所选商品是否在平台「负面目录」/地方禁限商品目录内；讲款话术有无夸大功效、虚假承诺。**跳过会发生什么**：售假/虚假宣传触发退一赔三、平台限流。 evidence: [T03-S011]
  4. **AI 数字人标识自查**: 若用 AI 生成虚拟主播，必须做显著标识并持续提示（2026 新规强制）。**跳过会发生什么**：违反 2026 新规 AI 标识义务。 evidence: [T03-S011]
  5. **售后承诺自查**: 检查直播间承诺的售后（退换/赔付）是否能落地「首问负责/先行赔付」。**跳过会发生什么**：售后承诺兑现不了引发投诉、违反 2026 新规售后追责。 evidence: [T03-S011]
- **资深路径（差异点）**:
  - **skip**: 无 —— **合规红线没有任何一步可以跳**，这是与其它 5 个 workflow 的本质区别。
  - **optimize**: 资深团队**把合规自查做成固定 checklist 前置到场前脚本环节**（workflow 1 step 2 就过一遍），新手是被罚了才知道有这些线。 evidence: [T03-S001]
  - **add**: ① 资深团队/MCN 会**预判主播的法律身份**（广告代言人 vs 广告发布者 vs 经营者 vs 导购，责任完全不同），新手以为「主播只是帮忙吆喝不担责」；② 资深团队**跟监管更新节奏校准**（2026-02 新规一出就重校 SOP），新手用旧规则裸奔。 evidence: [T03-S011]（Wave 1 Track 06 seed）
  - 差异点统计：optimize 1 + add 2 = 共 3 处，合格。
- **近期变化**（近 12 个月）:
  - **2026-02-01 起，《直播电商监督管理办法》（117 号令）施行**，自查清单新增：身份核验登记（至少每 6 个月更新）、信息持续公示、AI 数字人显著标识、首问负责/先行赔付售后制度、违规黑名单。 evidence: [T03-S011]
  - 触发事件类型：法规变化（国家级专门立法）。
  - 当前采用率：法规强制，所有从业者都要执行；但首年执法口径未定，自查清单的精确边界仍在落地中。
- **典型耗时**: 入门 SOP 每场开播前 10-30 分钟（checklist 化后）；规则更新时的全面重校约半天到 1 天。
- **关键工具（与 Track 02 关联）**: 抖音电商学习中心「规则中心」（对接监管要求，必备）、巨量百应（资质管理，必备）。
- **关键人物（与 Track 01 关联）**: 监管原文是 ground truth（Track 04 canon 收录《直播电商监督管理办法》《网络主播行为规范》）；律所长稿（中伦/锦天城）给出主播法律身份四分法（Track 06 glossary）。
- **常见失败模式**:
  - **直播间张口喊「全网最低价」**: 触发原因 —— 主播没受过合规培训、沿用老话术。后果 —— 《广告法》第九条违规被罚。怎么避免：**话术里所有绝对化用语（最低/最好/第一/唯一）全部替换为「直播间专享价」类合规表述，开播前逐句过一遍**。 evidence: [T03-S011, T03-S012]
  - **以为「主播只是帮忙吆喝、不担责」**: 触发原因 —— 不懂主播法律身份认定。后果 —— 出问题时被认定为广告代言人/经营者，承担行政处罚/连带赔偿/三倍赔偿。怎么避免：**开播前明确主播在本场的法律身份（是否亲自使用商品、是否与品牌直接签约），按身份预判责任**。 evidence: [T03-S011]
  - **AI 数字人直播不标识**: 触发原因 —— 不知道 2026 新规要求。后果 —— 违反 AI 标识义务被处罚。怎么避免：**用 AI 虚拟主播必须全程显著标识「AI 生成」**。 evidence: [T03-S011]
  - **用旧规则裸奔**: 触发原因 —— 不跟监管更新。后果 —— 2026-02 新规生效后还按老办法做，踩新增红线。怎么避免：**监管/平台规则一更新就重校自查清单，把抖音电商学习中心「规则中心」当固定信息源**。 evidence: [T03-S011]
  - **CLI 化暗示 (v0.6+)**: 可抽进 `cli/workflow/compliance-check.sh`，每条变 yes/no（如「你的话术里还有『全网最低价』这类词吗？」「用了 AI 数字人吗？标识了吗？」「这是按 2026-02 后的新规自查的吗？」）。这个 workflow 的 CLI 化价值最高 —— 红线检查最适合做强制 yes/no 确认。

## Phase 2 提炼提示

### 反复出现 ≥ 3 个 workflow 都包含的步骤（候选 playbook 通则）

- **「先确认承接/前端人货场，再谈引流/投放」** 出现于 workflows: 1（场中节奏前提是承接）/ 2（0→1 先把泛流量承接住）/ 3（投千川前先确认停留≥30s、留存≥30%）/ 4（起号靠停留和转化达标）→ **强候选「如果 X，则 Y」playbook**：如果「加了流量但 GMV/转化不涨」，则先查承接（停留时长 / UV 价值 / 品组），**不要继续加投流**。这与 Track 04 canon「承接 > 引流」心智模型完全互证。
- **「按数据指标定位问题，而非凭感觉」** 出现于 workflows: 1（四象限复盘）/ 5（五步复盘 + 时段异常值）/ 3（盯 ROI/GPM 优化）→ 候选 playbook：如果要诊断直播间问题，则拉「停留时长 / 转化率 / UV 价值 / GPM」这组指标 + 定位到具体时段/动作，不要笼统归因。
- **「分阶段、不要齐头并进」** 出现于 workflows: 1（127 排品分款型）/ 2（0→1→10→100 不同阶段不同打法）/ 4（起号期福利款占比 80%）→ 候选 playbook：如果商家问「该做什么」，先定位 ta 在哪个阶段，按阶段给单一重点（0→1 只追标签精准、不追 GMV），不要让 ta 同时追所有指标。
- **「合规先于打法」** 出现于 workflows: 1（场前脚本含合规自查）/ 6（独立合规自查）+ 隐含在所有 workflow（憋单/起号/矩阵都有合规边界）→ 候选 playbook：如果用户问任何「怎么做」，先用监管红线（绝对化用语 / 退货率 / AI 标识 / 资质公示）过一遍，再谈运营。与 Track 04 canon「合规是红线不是参考」心智模型互证。

### 入门 SOP 和资深路径之间最大的差距（候选 心智模型）

- 入门 SOP 平均长度 5-6 步，资深路径**步骤数没有明显压缩**（不像典型工程行业「资深人少做几步」）—— 这一行资深人的核心优势**不在「跳步」而在三件事**：
  1. **归因能力**：资深人能把「直播间不行」精准归因到人/货/场哪一环、哪个时段、哪个动作；新手只会笼统焦虑。→ **心智模型候选**：「这一行的资深能力 = 把模糊的『效果不好』翻译成具体的『哪个指标的哪个环节出了问题』」。
  2. **不踩坑**：6 个 workflow 的「资深差异」里，`add` 类型（资深人额外做的事）远多于 `skip` —— 资深人不是少做事，是**多做新手会忽略的防坑动作**（倒推账号结构 / 供应链前置 / 合规前置 / 超长周期低烧投放 / 不盲目矩阵化）。→ **心智模型候选**：「这一行新手和资深的差距，主要是『资深人知道哪些坑会让前面所有努力归零』—— 供应链、合规、心态、矩阵节奏」。
  3. **预期管理 / 长期主义**：资深人反复强调「心流管理」「初期降低预期」「长期布局」，新手把直播当「弯道超车」。→ 心智模型候选：「抖音直播带货不是弯道超车，是需要熬过 0→1 的长期生意；90% 的人卡在 0→1」。
- 差异类型分布统计（6 个 workflow 合计）：**skip 5 处 / optimize 12 处 / add 17 处** —— `add` 占比最高（约 50%），印证「资深 = 多做防坑动作」而非「资深 = 跳步」。这是本行业 workflow 维度最重要的一个结构性发现，Phase 2 心智模型和反模式节都要用到。

### 近期工作流变化的根本原因

- 触发变化的事件类型分布（6 个 workflow 合计 12 个近期变化）：
  - **新工具驱动 8 个**：全域推广（2025-05，影响 workflow 1/2/3/4/5）、千川·乘方 AIGC（2025-12，影响 workflow 1/3）、GPM 进排序（影响 workflow 5）。
  - **法规变化驱动 3 个**：《直播电商监督管理办法》2026-02（影响 workflow 1/4/6）、税务查账征收（影响 workflow 2）。
  - **行业事件驱动 1 个**：货架场战略崛起（与全域推广交织）。
- **主要驱动力 = 字节平台自身的产品/战略迭代（尤其千川）+ 监管收紧**。这两股力量是本行业的「外部驱动力」：
  - 平台驱动力的方向是**「去手动化、AI 化、全域化」** —— 商家/投手的工作正从「手动操纵参数」转向「给系统喂目标和内容标签」。Phase 2 反模式节应警示：把旧的「手动搭计划、手动定向」当核心技能教，会很快过时。
  - 监管驱动力的方向是**「持牌经营、全链路追责」** —— 「合规能力」从边缘变成核心。
- → Phase 2 识别行业「外部驱动力」时，这两条直接进 心智模型 + 反模式 + 诚实边界。

### 冷僻 / 信号薄弱

按 03-workflows.md Step 6 自检，逐条结果：

- **workflow 数 < 4？** ✅ 否（6 个，全 KEEP / high）。
- **一手 source < 50%？** ⚠️ **是**。14 条 source 中 verified_primary 2 + surrogate_primary 2 = 约 29% 一手，**低于 50% 阈值**。原因与 Track 05 sources 完全一致 —— 这一行**没有英文圈那种「工程师写的 we built X」式一手工作流长文档**：操盘手真实 SOP 要么锁在付费课程里（不可追溯、且 70%+ 是「7 天速成」噪音被本 track 狠筛掉），要么以媒体访谈稿形式存在（二手转述）。一手只能靠平台官方课程（surrogate）+ 监管原文（verified）。**这是行业真实状态，不是调研不充分。**
- **资深差异点缺失 > 50% workflows？** ✅ 否（6/6 workflow 都填了 ≥3 处资深差异，100% 达标）。
- **结论**：触发「一手 source < 50%」一条。Phase 2.8 诚实边界必须标注：
  > 「本 skill 的 workflow 节，一手来源占比偏低（约 29%）。抖音直播带货行业**不存在公开、可追溯、高质量的一手工作流文档** —— 操盘手的真实 SOP 要么在付费课程里（且行业 70%+ 是『速成』噪音），要么在媒体访谈中被二手转述。本 track 已狠筛营销号、只保留操盘手长稿 + 财经科技媒体深度稿 + 平台官方课程 + 监管原文，但用户应知道：workflow 节的步骤是『多个二手转述交叉印证 + 平台官方文档 + 监管原文』拼出来的，不是直接从某个顶级操盘手嘴里逐字记录的。各步骤的『大方向』可信（多源一致），但『精确参数』（如停留时长目标 2 分钟、预算 300-500 元、127 排品比例）是行业经验值、各家略有出入，应作参考而非铁律。」
- **额外的 high decay 警告**（03-workflows.md「全是 high-decay workflows」失败处理）：6 个 workflow 中 **4 个标 high decay**，仅 1 个 medium、1 个 low。本行业 workflow 模块**衰减极快**，Phase 2.8 必须明确写：
  > 「本 skill 工作流模块预计是全 skill 衰减最快的部分 —— 千川产品近 12 个月已迭代两次（全域推广 2025-05、千川·乘方 2025-12），监管 2026-02 刚出新规且首年执法口径未定。**强烈建议每 3 个月跑一次 update**，重点刷新 workflow 1/3/4（直播流程 / 千川投放 / 起号）和 workflow 6 的合规清单内容。」
