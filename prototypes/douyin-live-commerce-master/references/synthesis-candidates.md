## 候选 playbook 规则 (target 5-10，本文件收 12 raw)

| Pattern | 来源 |
|---------|------|
| P1 加了流量但 GMV/转化不涨 → 先查承接（停留/UV价值/品组/话术），不要继续加投千川 | 02,03,04 Phase 2 接口三轨互证 |
| P2 直播间长期起不来 → 先归因到选品（货），其次才是主播（人）和场 | 04 核心概念→playbook |
| P3 新账号/新千川计划 → 学习期容忍数据难看、不频繁改计划（积累约 20-50 转化前锁死） | 03 workflow 3 + 04 + 06 词条 |
| P4 遇到「刷量/刷单/换号能不能做」 → 直接判违法（2026 新规加重+黑名单封堵），不进入「技巧」讨论 | 04 + 06 数据真实性 |
| P5 中小商家问「怎么扩张」 → 按预算阶段排序 F（自播打底）→ A（达人放量）→ C/T（有预算再上），不四件齐做 | 02 Branch 2c + 04 FACT |
| P6 用户问任何「怎么做」 → 先用监管红线（绝对化用语/退货率/AI标识/资质公示）过一遍再谈运营 | 03,04 Phase 2 接口互证 |
| P7 ROI「越高越好」吗 → 不是；目标 ROI ≈ 保本 ROI×1.2，能放量即可，不要拉到不敢花钱 | 06 外行破绽 + 03 workflow 3 |
| P8 商家问「该做什么」 → 先定位 ta 在 0→1/1→10/10→100 哪个阶段，按阶段给单一重点 | 03 workflow 2 + Phase 2 接口 |
| P9 0→1 还没跑通就想开矩阵号 → 先把一个号跑到稳定月销再做有分工的矩阵，不要在 0→1 开第二个号 | 03 workflow 2/4 失败模式 |
| P10 选品 → 先按自己账号实时标签选，不要因为某品在别人直播间卖爆就上 | 03 workflow 1 失败模式 |
| P11 想用第三方数据做达人结算/合规举证 → 不行，决策/结算/合规必须用罗盘/巨量百应真数据 | 02 避坑清单 + 06 |
| P12 选投放工具 → 直播间精细成交用千川不用 DOU+；冷启动轻量用小店随心推；2025 起【推商品】只能商品全域推广 | 02 决策树 B + 04 |

> P 系列句式已按「**如果 X**，则 Y」预规整，Phase 3 cli_writer 可抽。预计 KEEP 8-10 条，按主题聚类（承接诊断 / 投放 / 选品货盘 / 合规 / 阶段经营）。

## 候选 Agentic Protocol 维度

| 维度 | 推导自哪个 mental model |
|------|------------------------|
| AP1 监管红线现状核查（绝对化用语 / 退货率 / AI 标识 / 资质公示 / 主播法律身份 / 2026 办法细则） | C5 合规是生死线 |
| AP2 千川产品形态时效核查（当前是标准推广还是全域推广、千川·乘方铺开程度、计费） | C8 工具自主权在收缩 + D1 |
| AP3 承接能力诊断（停留时长 / UV价值 / GPM / 进场留存 / 品组）—— 先于一切引流问题 | C2 承接 > 引流 |
| AP4 货盘 / 供应链体检（选品与账号标签匹配、利润空间、退换货逻辑、供应链能否扛放大期） | C3 货是天花板 |
| AP5 阶段定位 + 流量结构诊断（0→1/1→10/10→100 哪个阶段、自然/短视频/付费三条腿配比） | C12 分阶段经营 + C9 |
| AP6 数据来源可信度甄别（真数据 vs 估算数据，蝉妈妈/飞瓜偏差，引用必标） | C6 真数据 vs 估算数据 |
| AP7 流派语境定位（用户/案例属投放派 / 运营派 / 选品派 / 私域派 / 主播 IP 派 / 去IP化机构派哪一派） | C4 去 IP 化 + S1-S7 |

> 自检（跨步骤硬规矩 §B 互斥性）：AP1（合规）vs AP2（千川产品时效）—— 一个查法律、一个查平台产品，不 overlap。AP3（承接诊断）vs AP4（货盘体检）—— 一个查「场+人留转化能力」、一个查「货的上限」，结论不同（承接 OK 但货差 vs 货 OK 但承接差是两种诊断）。AP5（阶段+流量结构）vs AP3 —— 阶段定位决定「该追什么指标」，承接诊断决定「承接环节哪里漏」，独立。AP6（数据可信度）vs 其它 —— 是「元维度」，校验所有数据输入，不与具体业务维度重叠。AP7（流派语境）vs AP1-AP6 —— 流派定位决定「按谁的方法论回答」，不产出业务诊断结论。7 个维度能各自产出不同结论。预计 KEEP 7 个。
