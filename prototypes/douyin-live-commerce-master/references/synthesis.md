## 1. 心智模型 (6 个)

> 三重验证 (extraction-framework § 一)：验证 1 跨场景复现 / 验证 2 生成力 / 验证 3 排他性。candidates 13 个走验证后 KEEP 6：C9「大爆发逻辑」生成力结论与 C1 重叠 → 并入 1.1；C10+C11「归因 + 防坑」两个单轨候选合并为 1.6；C13「不适合 90% 的人」排他性强但生成力偏弱 → 降级到 §8 诚实边界 + §2 playbook。C5「合规是生死线」三验证全 PASS 但本质是「红线约束」而非「思维框架」，按行业放大版 PARTIAL 保留为 1.5。

### 1.1 算法生意：直播间是做给算法看的，不是做给用户看的

- **一句话**：抖音直播带货的本质是「向推荐算法证明这个直播间值得被推流」——理解算法如何用停留 / 转化 / GPM 这组指标分配免费流量，比话术技巧更底层；流量是非线性「大爆发」的，核心变量是流量 × 选品。
- **应用方式**：面对任何「直播间为什么不行 / 怎么做起来」的问题，先翻译成「算法在用什么指标评分、这个直播间在算法那里的得分卡在哪一项」，而不是先想话术 / 人设。看到「3000 块做到 3000 万」这类爆发叙事时，知道这是算法流量池逐级跃迁的结果，不是营销奇迹。
- **局限**：仅适用于「自然推荐流量占比仍有意义」的阶段与品类；纯达人分销 (FACT 的 A 阵地) / 纯货架场「人找货」场景里「算法博弈」的解释力下降。且「GPM 进千川推荐排序的程度」是操盘手观察推断、非官方公布的算法细节 (T06 GPM 词条)，不能当精确公式用。
- **evidence**: [T01-S001, T04-S010, T01-S022]　# 李牧人「大爆发逻辑」长访谈 + 尹晨书「算法生意」+ 跨 Track 04 canon 共识

### 1.2 承接 > 引流：流量是放大器不是发动机

- **一句话**：直播间能不能做起来，发动机是「承接能力」(UV 价值 / 停留时长 / GPM / 品组 / 话术)，流量 (自然 + 投流) 只是把这台发动机的输出放大——承接是负数时，加流量等于加速亏钱。
- **应用方式**：遇到「加了流量但 GMV / 转化不涨」，第一反应是诊断承接 (停留、进场留存、UV 价值、品组)，而不是继续加投千川。判断一个增长方案是否成立，先问「承接这台发动机是正的吗」。
- **局限**：「承接达标线」(如平均停留 ≥ 30 秒、进场留存 ≥ 30%、人均停留 ≥ 2 分钟) 是行业经验值、各家与各品类略有出入 (服饰退货率 40-60% vs 食品低，承接逻辑不同)，是参考不是铁律 (T03 workflow 1/3)。
- **evidence**: [T02-S012, T03-S007, T04-S010]　# 千川操盘手长文 + workflow 3「投千川前先确认停留/留存」+ 尹晨书「承接不行白搭」，02/03/04 三轨独立互证

### 1.3 货是天花板：选品决定上限，人和场只能在上限内优化

- **一句话**：人货场三要素里「货 (选品)」是天花板——选品错了，再强的主播和再精细的场也救不回来；MCN / 操盘手的真本事在「组货 / 供应链」，不在「话术」。
- **应用方式**：一个直播间长期起不来，先归因到选品 (货与账号标签是否匹配、利润空间、供应链能否扛放大期)，其次才查主播 (人) 和场。评估一门直播生意能做多大，先看供应链能给什么。
- **局限**：「货是天花板」对「靠内容 / 人设信任本身成交」的知识型超头 (董宇辉式) 解释力弱——这一派的天花板是内容审美和个人天赋，不是货 (见 §7 智识谱系分歧 1)。属行业放大版 (零售「人货场」的特化)，放大点在「抖音的算法分发让选错品的惩罚被即时放大」。
- **evidence**: [T01-S022, T04-S010, T03-S008]　# 尹晨书「货是天花板」+ Track 04 canon 强共识 + workflow 2「99% 死在供应链短板」

### 1.4 去 IP 化：个人 IP 是高风险资产，机构护城河在供应链 + 可复制矩阵 + 利益分配

- **一句话**：单一头部主播 IP 是高波动、高风险资产 (会塌房、会单飞、会焦虑)；机构能长期活下来靠的是「重供应链 + 可复制的小号矩阵 + 合理利益分配机制」，不是某一个人。
- **应用方式**：面对「该不该 all in 一个大主播 / 要不要做矩阵 / MCN 怎么估值」，先看护城河是不是建立在「人」之外的可复制资产上。看到超头主播出事，理解这是「赌单一 IP」这条路径的结构性风险兑现，不是偶然。
- **局限**：「小号可复制」本身存疑——交个朋友的小号矩阵仍高度依赖其供应链和流量积累，普通商家照搬难成 (T01-S001 批评)；这是「去 IP 化机构派」的主张，与「超头 IP 派」是未抹平的流派分歧 (见 §7)。
- **evidence**: [T01-S001, T01-S004, T01-S002]　# 李牧人「大网红不可复制」+ 黄贺「做号不做人」+ 雷彬艺「找对路+分好钱」，5 个 figure 命中 (罗永浩/黄贺/李牧人/奥利/雷彬艺)，Track 01 最强候选

### 1.5 合规是生死线：「能不能做」先于「怎么做」(行业放大版)

- **一句话**：这一行的动作空间由「立法 + 平台规则」双重框定，「能不能做」是先于「怎么做」的过滤器——绝对化用语、退货率、AI 标识、资质公示、主播法律身份、刷量刷单，每一条都是「能不能活」而非「要不要做」。
- **应用方式**：用户问任何「怎么做」，先用监管红线过一遍 (绝对化用语 / 退货率 / AI 数字人标识 / 资质公示 / 主播身份)，再谈运营。遇到「刷量 / 刷单 / 换号能不能做」直接判违法，不进入「技巧」讨论。
- **局限**：放大版的通用道理 (「合法经营」每行都成立)，放大点在「直播带货是 2026-02 才有国家级专门立法、首年执法口径未定、且超头被罚案密集」的强监管 + 高不确定环境。《直播电商监督管理办法》2026-02-01 才施行 (T03-S011 / T04-S003)，细则与首批执法案例会重塑「合规」的精确边界——这条心智模型本身稳定，但「红线在哪」高频变。
- **evidence**: [T04-S003, T03-S011, T01-S013]　# 117 号令原文 + workflow 6 合规自查 + 小杨哥案 (月饼虚假宣传被罚 6894.91 万，T01-S013/S014 行政通报级)

### 1.6 资深 = 归因力 + 防坑，不是少做步骤

- **一句话**：这一行的资深人和新手，差距不在「资深人少做几步」(典型工程行业的特征)，而在两件事——① 归因力：能把模糊的「效果不好」精准翻译成「哪个指标的哪个环节、哪个时段、哪个动作出了问题」；② 防坑：多做新手会忽略的防坑动作 (供应链前置 / 合规前置 / 不盲目矩阵 / 超长周期低烧投放)。
- **应用方式**：诊断问题时，拒绝「直播间不行」这种笼统判断，强制落到具体指标 + 具体环节。给建议时，资深视角是「加上新手会漏的防坑动作」，不是「砍掉步骤求快」。判断一个人是不是资深，看他会不会归因、会不会主动防坑。
- **局限**：来自 Track 03 workflow 维度的统计性发现 (6 个 workflow 合计 skip 5 / optimize 12 / add 17，add 占比约 50%，T03 Phase 2 接口)——是「workflow 维度的结构性观察」，跨 track 第二证据较弱 (Track 01 figures 侧零星印证「倒推账号结构」「心流管理」)，置信度 medium。
- **evidence**: [T03-S001, T03-S005]　# workflow 1「倒推账号结构 + 合规前置」资深差异 + workflow 5「时段异常值精准定位」资深差异

## 2. 标准 Playbook (9 条决策规则)

> 来源：Step 1 降级候选 + Track 03 跨 workflow 共同步骤 + Track 01 figures 决策标准 + Track 04 核心概念暗示。句式按 `**如果 X**，则 Y` 规整，Phase 3 cli_writer.py 按主题聚类 (承接诊断 / 投放 / 选品货盘 / 阶段经营 / 合规)。

1. **如果「加了流量但 GMV / 转化不涨」**，则先查承接 (停留时长 / UV 价值 / 品组 / 话术)，不要继续加投千川。
   - **聚类**: 承接诊断
   - **案例**: 千川操盘手长文与 workflow 3 均把「投千川前先确认平均停留 ≥ 30 秒、进场留存 ≥ 30%」当达标线；遥望给贾乃亮投流 23.04 亿、占采购 53.87% 仍亏 (T01-S016)，是「承接 / 结构没解决就堆流量」的反面教材。
   - **evidence**: [T02-S012, T03-S007, T01-S016]

2. **如果一个直播间长期起不来**，则先归因到选品 (货)，其次才是主播 (人) 和场。
   - **聚类**: 承接诊断
   - **案例**: 尹晨书「人货场中货是天花板」；workflow 1 失败模式「选品与账号标签不符 → 流量进来不转化、UV 价值低」。
   - **evidence**: [T04-S010, T03-S001]

3. **如果是新账号 / 新千川计划**，则学习期 (积累约 20-50 个转化前) 锁死计划不动、容忍数据难看，给算法学习时间。
   - **聚类**: 投放
   - **案例**: workflow 3「学习期频繁改计划 → 模型一直学不会、永远过不了学习期」；Track 06「学习期 / 跑量」词条同义。
   - **evidence**: [T03-S007]

4. **如果有人问「ROI 是不是越高越好」**，则回答不是；目标 ROI 设为「保本 ROI × 1.2」左右、能放量即可，不要拉到不敢花钱的程度。
   - **聚类**: 投放
   - **案例**: Track 06 外行破绽 top「ROI 越高越好 = 高 ROI 做不大」；workflow 3 资深差异「追的是能放量的可接受 ROI」。「保本 ROI×1.2」是行业经验值、各家有出入，非官方口径。
   - **evidence**: [T03-S007]

5. **如果要选投放工具**，则：直播间精细成交用千川、不用 DOU+ (DOU+ 是弱内容加热)；冷启动轻量起步用小店随心推 (单场预算约 300-500 元，行业经验值)；2025-05-22 起【推商品】场景只能用商品全域推广 (标准推广已下线)。
   - **聚类**: 投放
   - **案例**: Track 02 决策树 B；亿邦动力〈千川"标准推广"5月下线〉(T02-S012)。
   - **evidence**: [T02-S012, T03-S007]

6. **如果要选品**，则先按自己账号的实时标签和历史成交人群选，不要因为某品在别人直播间卖爆就上。
   - **聚类**: 选品货盘
   - **案例**: workflow 1 失败模式「跟风大主播选品、不看自己账号画像 → 流量进来不转化」。
   - **evidence**: [T03-S001, T03-S002]

7. **如果商家问「该做什么 / 怎么扩张」**，则先定位 ta 在 0→1 / 1→10 / 10→100 哪个阶段，按阶段给单一重点 (0→1 只追账号标签精准、不追 GMV 与利润率)；扩张按预算阶段排序 F (自播打底) → A (达人放量) → C/T (有预算再上)，不要 FACT 四件齐做。
   - **聚类**: 阶段经营
   - **案例**: workflow 2 三阶段路径；Track 02 决策树 Branch 2c + FACT 白皮书「按预算和阶段配比」。
   - **evidence**: [T03-S006, T02-S008]

8. **如果 0→1 还没跑通就想开矩阵号**，则先把一个号跑到稳定月销、验证 SOP 可复制，再做有明确分工的矩阵；不要在 0→1 阶段开第二个号。
   - **聚类**: 阶段经营
   - **案例**: workflow 2 失败模式「同主体账号在同一用户池互相抢量、每个直播间流量稀薄、花很多钱都干不成」(CBNData 抖音品牌自播四大坑)。
   - **evidence**: [T03-S008]

9. **如果用户问任何「怎么做」**，则先用监管红线过一遍 (绝对化用语 / 退货率 / AI 数字人标识 / 资质公示 / 主播法律身份)，再谈运营；遇到「刷量 / 刷单 / 螺旋起号刷数据能不能做」直接判违法 (2026 新规数据真实性条款 + 黑名单封堵换号)，不进入「技巧」讨论。
   - **聚类**: 合规
   - **案例**: workflow 6 合规自查是 6 个 workflow 里唯一「无步骤可跳」的；《广告法》第九条绝对化用语已多案被罚 (T03-S011/S012)。
   - **evidence**: [T04-S003, T03-S011, T03-S012]

## 5. 表达 DNA

> 不模拟某个具体 figure，模拟「这一行的资深人聚一起讨论时的 register」。多人融合。来源：Track 06 outsider-tell + 厂商话术拒绝 + Track 01 长访谈 register。

### 5.1 高频用语 / 黑话 (内行秒懂，外行听不懂或会用错)

起号 / 螺旋起号 / 单品起号；憋单 vs 逼单 (外行最常搞反)；过款 / 过品；投流 / 撬自然流 (不是「买观众」是「撬杠杆」)；跑量 / 学习期 / 计划起量 / 素材衰减；控成本 vs 放量 vs 通投拉满；留人款 / 利润款 / 福利款 (选品结构黑话)；坑产 / 坑位费 (达人合作只认坑产)；公域 / 私域 / 私域回流；起飞 / 爆单 / 掉权重 / 限流；GPM / UV 价值；人货场；FACT (F 自播 / A 达人 / C 活动 / T 头部大 V)；中控 / 场控。

### 5.2 严肃 register (长访谈 / conference talk 的口气，非 Twitter 短稿)

- **操盘手 register (最有信息量)**：用「大号 / 小号 / 起号 / 交付 / 阵地」这套行话，讲「趋势判断 + 结构决策」，不讲情怀。例：「既然大号已经是下滑的趋势……不如把多阵地搞起来。」
- **MCN 创始人 register**：把复杂经营压缩成 2-4 字口号 (「找对路 + 分好钱」「服务者」)。
- **头部主播 register**：理性克制 (罗永浩「把生意和情怀分开讲」) 或情绪外露的文人式自我剖白 (董宇辉「想站着赚钱」)——两种截然相反。
- **平台官方 register (要识别并标注)**：排比、抽象、「全域 / 生态 / 协同 / 内容本质」高频词。**外行容易把这套腔调当成行业真相**——它是「平台想让你怎么理解这个生意」的官方口径，不代表一线操盘语言。

### 5.3 内 vs 外沟通差异

- 内部 (同行间)：允许「起号 / 憋单 / 通投拉满 / 坑产」等黑话直给，讲「算法博弈、投流脏活、灰色操作真实有效」。
- 对外 (解释给非从业者 / 商家客户)：把黑话展开 (「憋单 = 故意不上库存留住人」)，且会把「能不能做 (合规)」放在「怎么做」前面讲。

### 5.4 外行破绽 (insider-only spotting tells，直接用)

1. 「憋单」当成催促下单 (实为故意不上库存留人)；2. 「ROI 越高越好」(高 ROI = 做不大)；3. 「先涨粉再带货」(直播和短视频两套算法，0 粉能起号)；4. 「不投流就是做自然流」(自然流要付费流量撬)；5. 「场观 10 万 = 10 万人在看」(场观是累计人次，看 PCU)；6. DOU+ 和千川混为一谈 (量级和逻辑完全不同)；7. 「GMV = 赚到的钱」(扣退货 / 佣金 / 投流 / 坑位费后差很远)；8. 「中控 = 助播」(中控是不出镜的后台操盘)；9. 张口喊「全网最低价」(广告法红线 + 没受过合规培训的标志)；10. 「付坑位费就能卖爆」(内行只认坑产 + 达人品类匹配度)。

### 5.5 厂商话术拒绝 (拒绝 → 反映行业价值观 + 反模式，直接用)

1. 「7 天速成主播 / 3 天见收益 / 15 天赚回学费」——课程营销最经典话术，资深从业者一律视为割韭菜信号；2. 「我们是抖音官方授权 / 官方合作的培训机构」——抖音不存在这种「官方委托培训」授权；3. 「投流就能卖爆 / ROI 保 X」——把千川神化成可复制公式，没人能「保 ROI」；4. 「全网最低价 / 全球最低价」——既违法 (《广告法》第九条) 也是不专业标志，内行 2024 后改用「直播间专享价」；5. 「核定征收帮你合法节税 / 找税收洼地园区」——2022 三部门意见 + 薇娅案 (13.41 亿) 后此话术 = 高危违法。
- 补充：FACT 是抖音官方框架，但课程方大量二次包装成「秘籍」售卖——行业对「把平台官方方法论术语化兜售」保持警惕。

## 6. 质量基准 + 反模式

### 6.1 什么算「好」(具体可验证的基准)

1. **复盘看「算法评分代理」指标，不只看 GMV**：好的复盘拉「GPM / UV 价值 / 停留时长 / 转化率」这组指标，并定位到具体时段 / 动作；只报 GMV 总量的复盘是不合格的 (靠投流堆出的高 GMV 可能 GPM / UV 价值很低、不可持续)。来源：尹晨书 + workflow 1/5。
2. **承接达标线先于投流**：好的判断是「投千川前先确认直播间平均停留 ≥ 30 秒、进场留存 ≥ 30%」(行业经验值)；承接不达标先修承接。来源：workflow 3。
3. **流量结构健康 = 三条腿**：好的直播间流量结构是「自然流 + 短视频 + 付费」三条腿，不是单靠直播推荐流量 (单一来源抗风险弱)。来源：workflow 2。
4. **分阶段单一重点**：好的经营是 0→1 只追账号标签精准、不追 GMV 与利润率，按阶段给单一重点；让商家同时追所有指标是不合格的建议。来源：workflow 2 + Track 04 canon。
5. **合规自查前置成 checklist**：好的团队把绝对化用语 / 资质 / AI 标识自查做成固定 checklist、前置到场前脚本环节；「被罚了才知道有红线」是不合格的。来源：workflow 6。

### 6.2 反模式 (外行 / 入门常犯错，来源：Track 06 outsider-tell + Track 02 避坑 + Track 03 失败模式)

1. **承接没跑通就加投千川** = 加速亏钱 (UV 价值 / 话术 / 品组不行时投流，效果由素材 / 选品 / 承接共同决定)。
2. **把蝉妈妈 / 飞瓜估算数据当真数据**做决策 / 结算 / 合规举证 (与抖音真实数据偏差 10-30%，T02-S021；决策 / 合规 / 达人结算必须用罗盘 / 巨量百应真数据)。
3. **把 DOU+ 当千川用**做精细成交 (DOU+ 是弱内容加热工具，量级和逻辑完全不同)。
4. **喊「全网最低价」等绝对化用语** (《广告法》第九条红线，已多案被罚)。
5. **「ROI 越高越好」** (高 ROI = 不敢花钱 = 做不大)。
6. **「先涨粉再带货」** (直播和短视频是两套算法，0 粉能起号，涨粉是副产物)。
7. **0→1 没跑通就盲目矩阵化** (同主体账号在同一用户池互相抢量、每个直播间流量稀薄)。
8. **复盘只看 GMV 总量**，不看 GPM / UV 价值 / 停留时长。
9. **供应链短板 —— 爆单即翻车** (线下厂商尤其盲目自信，「99% 死在供应链」)。
10. **选品跟风大主播、不看自己账号标签** (流量进来不转化)。
11. **把 FACT 当「四件都要满分的清单」/ 当工具 / 当投放技巧** (FACT 是顶层经营框架、是平台官方话术，多数中小商家只做得动 F+A)。
12. **学习期频繁改计划** (模型一直学不会、永远过不了学习期)。
13. **把「螺旋起号」刷数据骗流量池当冷启动技巧** (2026 新规数据真实性违规 + 黑名单封堵换号)。
14. **信「7 天速成主播 / 投流保 ROI / 抖音官方授权培训」厂商话术** (割韭菜信号)。
15. **把旧的「手动搭计划、手动定向」当核心技能教** (平台正在去手动化，标准推广已下线，很快过时)。

## 7. 智识谱系

> 保留分歧而非抹平。来源：Track 01 figures 流派分裂 + Track 02 工具流派 + Track 04 canon 智识谱系种子 + Track 06 标准之争。
> **行业特殊性**：这一行**没有学术奠基、没有公认方法论奠基人**。谱系是「平台官方派 + 一线操盘手实战派 + 监管派 + (边缘) 学术派」的拼合。

### 7.1 四大「派系底座」

| 派系 | 奠基文本 / 来源 | 当前代表 (figures) | 角色 |
|------|----------------|--------------------|------|
| **平台官方派 (最强势的「正统」)** | FACT 经营矩阵白皮书 (抖音电商 × 贝恩，2021-05) +「兴趣电商 → 全域兴趣电商」定位 | 魏雯雯 (历任抖音电商总裁)、林旭灵 (巨量引擎策略侧)、贝恩咨询直播电商团队 | 定义规则与叙事；发声高度受控、几乎只有大会主题演讲 |
| **一线操盘手 / 实战派** | 尹晨《抖音直播带货实操指南》(2022) —— 当前最接近「公认必读」的一本 | 尹晨 (最稳 sub_skill 候选)、黄贺、李牧人、雷彬艺、奥利 | 承认官方框架 (用 FACT / 人货场词汇)，但更讲「算法博弈、投流脏活」 |
| **监管派 (ground truth)** | 《网络直播营销管理办法 (试行)》2021 → 《直播电商监督管理办法》2026-02；薇娅税案 (2021，13.41 亿) 是行业「judicial canon」 | 网信办 / 市场监管总局 / 税务总局 / 地方市监局 | 红线由立法定，「能不能做」的最终裁判 |
| **学术派 (最弱，边缘)** | 无单一奠基，散见传播学 (互动仪式链)、营销学 (准社会互动)、信息系统 (扎根理论) | 武大《新闻与传播评论》、清华 CJIS 等核心期刊零散论文 | 滞后且样本偏淘宝；给实践提供「机制解释词汇」而非方法 |

### 7.2 核心分歧 (保留，不抹平)

**分歧 1 — 运营 / 投放派 vs 知识 / 内容信任派**：直播间的发动机是「算法 × 投流 × 选品」的工程，还是「内容 / 人设信任本身」？
- A 派 (投放 / 运营工程派)：尹晨「算法生意 + 千川校准」、李牧人「流量 + 选品」。
- B 派 (知识 / 内容信任派)：董宇辉「全程不介绍产品也成交」。
- 张力：B 派的可复制性是他自己都没解决的问题 (Track 01 标注「这是一种 register / 审美，不是可批量执行的 SOP」)；这也是为什么心智模型 1.3「货是天花板」对 B 派解释力弱。

**分歧 2 — 去 IP 化 / 机构矩阵派 vs 超头 IP 派 / 明星派**：人是高风险资产要去 IP 化，还是整个商业模式就建立在「人」上？
- A 派 (去 IP 化机构派)：罗永浩、黄贺、李牧人 (交个朋友体系)。
- B 派 (超头 IP 派 / 明星派)：董宇辉 (知识型超头)、贾乃亮 (明星)、小杨哥 (娱乐 IP)。
- 张力：**B 派三个代表近 2 年全部出过问题** (董宇辉焦虑 / 翻车、贾乃亮背后遥望 2021-2023 累计亏约 20 亿、小杨哥月饼虚假宣传被罚 6894.91 万 + 停播半年)——间接给 A 派背书；但 A 派「小号可复制」本身也存疑 (高度依赖交个朋友的供应链和流量积累)。

**分歧 3 — 精细操盘派 (手动控) vs 系统智能派 (托管)**：投放是手艺、操盘手价值在精细调控，还是投放交给算法、人的价值在素材和选品？
- A 派 (精细操盘派)：千川标准推广、手动控成本 / 放量 / 定向；一线投手类操盘手。
- B 派 (系统智能派)：全域推广、千川·乘方千策整店托管、千意 AIGC；巨量引擎产品团队 (平台主推)。
- 张力：**2025 年有新变量** —— 平台用「标准推广【推商品】场景 2025-05-22 下线 + 千川·乘方 0.6% 费率优惠」强行把行业从 A 推向 B。由此推出心智模型 1.6 的延伸：工具自主权在收缩，资深的标志从「会精细操盘」变成「会喂好素材 + 选对货 + 看懂真数据」。

**分歧 4 — 公域 vs 私域 (未解争议)**：抖音生态下私域到底有没有价值？
- 私域沉淀派「公域只是入口」vs 投放派认为抖音生态私域价值被高估。Track 04 标为「未解争议」，不假装有标准答案。

**分歧 5 — 官方叙事 vs 实战现实 (canon 内部的「明线 vs 暗线」)**：
- 官方课程讲「合规、健康、长期、内容本质」；实战派知道「灰色操作 (矩阵 / 切片 / 憋单) 真实存在且有效」。这不是「谁对谁错」，是用户读材料时必须意识到的双层结构。

**次要 (互补非对立) —— 官方真数据派 vs 第三方估算派**：罗盘 / 巨量百应 (真、闭、仅本人可见) vs 蝉妈妈 / 飞瓜 / 新抖 (估算、开放、可看竞品)。不是对立是互补，但「把估算当真数据」是高频反模式。

## 8. 诚实边界

> 信息截止 2026-05-14。按 extraction-framework § 七 + § 八。

### 8.1 信源失衡 —— 一手占比实测 33.3%，是行业结构性问题，不是调研缺漏

- **全 manifest 一手占比 33.3% (111 条 total / 37 first-hand)，几乎全中文，营销号 + 课程营销噪音估 70%+ (intake warning #2)**。这不是衰减、是「本来就薄」。
- **对哪些组件影响最大**：
  - **figures 维度最薄 —— 一手占比仅约 8%** (24 条里只有尹晨的书 T01-S022/S023 是 `verified_primary`)。根本原因：头部主播几乎不写书 / 不开 podcast，操盘手长访谈多是「媒体对话体」(verifier 保守判 secondary)，平台高管只有官方通稿。**直接后果**：心智模型里凡挂 figures evidence 的 (尤其 1.1 / 1.4)，更多是「从媒体对操盘手的访谈里提炼」，不是「从大佬本人的系统著述里提炼」。
  - **表达 DNA 受冲击最直接**：§5.X 对话样本库 `voice_confidence: medium`——只采到「客户版」2 段、「同业版」4 段较实，「监管 / 专业版」全是转述，本行业 figures **没有公开的「私下 / 内训」对话语料**。Phase 3 写 SKILL.md §5 时，部分 register 的精确语感靠 Track 06 黑话表 + Track 03 工作流话术补，**主 SKILL.md 风格输出在「监管 / 专业 register」上有一部分靠 LLM 默认而非真行业语料**。
  - **工具栈维度**：一手占比 ~32% 且全部是 `surrogate_primary` (vendor 官网)、无一条独立 `verified_primary`；必备工具 100% 是单一供应商 (字节跳动) 的闭源产品；「实际使用工作流」从行业媒体二手长稿 + B 站操盘手视频还原。
  - **工作流维度**：一手占比 ~29%，行业不存在公开可追溯的一手工作流文档；步骤是多源二手交叉印证 + 平台官方课程 + 监管原文拼出来的，「精确参数」是行业经验值。
  - **canon 维度**：没有学术意义上的 canon。真·必读书严格意义上只有尹晨 1 本达标；学术论文只稳收 3 篇且滞后、样本偏淘宝。
  - **sources 维度**：行业没有成熟的 newsletter / podcast 生态 (Newsletter retained = 0，Podcast retained = 1 且为平台占位)。一手高度依赖字节官方单一渠道。

### 8.2 数据可信度 —— 蝉妈妈 / 飞瓜是估算数据

- 蝉妈妈 / 飞瓜 / 新抖与抖音内部真实数据有 **10-30% 偏差** (intake warning #6 + T02-S021 第三方测评，业内估计 / 第三方观察)。直播场景下「一个商品多直播间在带 + 临时改价 / 防拍价」会让偏差放大 (案例：同链接 30 天热推达人，飞瓜显示 34 人、蝉妈妈显示 547 人，T02-S021 单案例)。本文件凡引用蝉妈妈 / 飞瓜口径数据已标注；**真数据唯一来源是字节自家的罗盘 / 巨量百应 (仅商家本人可见)**。
- 「GPM 进千川推荐排序的程度」是操盘手观察推断、非官方公布的算法细节 (T06 GPM 词条)。
- 平台口径数据 (如「全域推广 GMV +20% / 操作 -65%」T02-S014、魏雯雯演讲「商城 GMV +277%」T01-S019) 是平台官方报喜口径、不可 cross-check，引用时已标。

### 8.3 行业是强监管 + 高时效，工具栈 / 工作流衰减最快

- **工具栈 / 工作流是全 skill 衰减最快的部分**：千川产品形态年度甚至季度被改写——标准推广【推商品】2025-05-22 下线、千川·乘方 2025-12-11 发布，过去 12 个月千川迭代两次。6 个 workflow 中 4 个 high decay。**强烈建议每 3 个月跑一次 `update 大师 douyin-live-commerce`**，重点刷 workflow 1/3/4 + workflow 6 合规清单 + 工具栈节 (去巨量学 / 千川官网核对当前产品形态)。
- **法规近期变化的预期影响**：①《直播电商监督管理办法》(117 号令) 2026-02-01 才施行，首年执法口径未定，实施细则 / 各平台配套规则 / 首批典型执法案例会重塑「合规话术」边界。② MCN《网络信息内容多渠道分发服务机构相关业务活动管理规定》2025 仍在草案征求意见，大概率 12 个月内定稿。心智模型 1.5「合规是生死线」本身稳定，但「红线在哪」高频变。
- **平台叙事在转移**：从「兴趣电商 / FACT」向「全域兴趣电商 / 货架场」演进；心智模型层稳定，但叙事重心 (尤其 1.1「算法生意」在货架场「人找货」场景的解释力) 在变。
- 衰减慢的部分 (1-2 年才动)：心智模型、智识谱系、表达 DNA 核心黑话 (起号 / 憋单 / 中控等)。

### 8.4 行业自我认知 —— 不适合 90% 的人 (intake 明确要求诚实写)

头部 0.1% 主播占行业 GMV ~30%+ (intake warning #4，业内估计)，腰部 + 尾部生存困难；约 90% 的人卡在 0→1 上不去 (T03 workflow 2)。这是一门「需要熬过 0→1 的长期生意」，不是「弯道超车」。本 skill 不写「成为头部主播秘籍」，写「行业怎么运转 + 你大概在哪个位置 + 这位置有什么具体打法」，绝不夸大成功率。

### 8.5 海外不可迁移

信息几乎全中文。海外 livestream commerce (Amazon Live / Whatnot / TikTok Shop) 生态完全不同，本 skill 的工具栈 / 工作流 / 合规框架**不可直接迁移**到海外 (intake warning #1)。

## 9. Agentic Protocol — 研究维度

> 从每个心智模型反推「这个模型让人首先关注什么」。7 个维度，按行业复杂度 (宽 + 工具栈强平台依赖 + 流派多) 取标准偏上。
> **互斥性自检 (跨步骤硬规矩 §B)**：AP1 (查法律) vs AP2 (查平台产品) 不 overlap；AP3 (查「场 + 人」留转化能力) vs AP4 (查「货」的上限) 结论不同 (承接 OK 但货差 vs 货 OK 但承接差是两种诊断)；AP5 (阶段决定该追什么指标) vs AP3 (承接环节哪里漏) 独立；AP6 是校验所有数据输入的「元维度」；AP7 (流派定位决定按谁的方法论回答) 不产出业务诊断结论。7 维度能各自产出不同结论。
> Phase 3 `cli_writer.py` 会把本节维度编排进 `cli/protocol/agentic.sh`，每维度一个分节问询——「看什么 / 在哪看 / 输出格式」三段为脚本必填字段。

### 9.1 监管红线现状核查 (推导自心智模型 1.5)

- **看什么**：当前绝对化用语执法口径、退货率红线、AI 数字人显著标识要求、资质 / 信息公示义务、主播法律身份四分法 (代言人 / 发布者 / 经营者 / 导购)、《直播电商监督管理办法》(117 号令) 实施细则与首批典型执法案例。
- **在哪看**：网信办 (cac.gov.cn) + 市场监管总局官网监管原文；抖音电商学习中心「规则中心」(school.jinritemai.com)；地方市监局合规指引 (如《北京市直播带货合规指引》)。
- **输出格式**：1-2 句结构化结论 ——「当前 X 行为的合规状态是 [可做 / 高危 / 已违法]，依据 [法条 / 案例]；2026-02 后新增红线为 [...]。」

### 9.2 千川产品形态时效核查 (推导自心智模型 1.6)

- **看什么**：当前是标准推广还是全域推广 (哪些场景标准推广还残存)、千川·乘方 (千策 / 千意 / 千寻) 铺开程度与计费 (技术服务费率)、商品全域推广的体裁覆盖。
- **在哪看**：千川官网 (qianchuan.jinritemai.com) + 巨量学 (school.oceanengine.com)；亿邦动力 / IT 之家 / 36 氪对千川大会的报道；千川大会官方发布信息。
- **输出格式**：1-2 句 ——「截至 [日期]，千川【推商品 / 推直播间】场景当前默认模式是 [...]，标准推广 [已下线 / 残存于 X 场景]，千川·乘方处于 [发布 / 铺开] 阶段、费率 [...]。」

### 9.3 承接能力诊断 (推导自心智模型 1.2)

- **看什么**：直播间平均停留时长、进场留存率、UV 价值、GPM、品组结构 (引流 / 福利 / 主推 / 利润 / 形象款配比)、话术六板块完整度——先于一切引流 / 投放问题。
- **在哪看**：抖音电商罗盘 (compass.jinritemai.com，真数据，四大核心指标：人气 / 互动 / 商品 / 订单)；巨量百应 (达人合作真数据)；不用蝉妈妈 / 飞瓜 (估算)。
- **输出格式**：1-2 句 ——「承接发动机是 [正 / 负]：停留 [X 秒 vs 达标线 30 秒]、进场留存 [X% vs 30%]、UV 价值 [...]；漏在 [品组 / 话术 / 节奏] 环节。结论：[先修承接 / 可以加流量]。」

### 9.4 货盘 / 供应链体检 (推导自心智模型 1.3)

- **看什么**：选品与账号实时标签 / 历史成交人群的匹配度、利润空间、退换货逻辑 (品类退货率，如服饰 40-60%)、供应链能否扛放大期的量级和发货时效。
- **在哪看**：巨量算数 (trendinsight.oceanengine.com，行业趋势 / 垂类热度，定方向) → 蝉妈妈 / 飞瓜 (查具体商品 / 竞品 / 达人，标注是估算) → 巨量百应 (自己商品被带的真数据)。
- **输出格式**：1-2 句 ——「货的天花板是 [高 / 低]：选品与账号标签 [匹配 / 错配]、利润空间 [...]、供应链扛量能力 [能 / 不能扛月销翻 10 倍]。结论：[换品 / 货可以、问题在别处]。」

### 9.5 阶段定位 + 流量结构诊断 (推导自心智模型 1.1)

- **看什么**：商家 / 案例在 0→1 / 1→10 / 10→100 哪个阶段 (对应该追什么指标)、自然流 / 短视频 / 付费三条腿的配比是否健康、是否只靠直播推荐单一来源。
- **在哪看**：抖音电商罗盘的流量来源构成 + 阶段数据 (月销量级)；抖音电商学习中心「商家成长路径」课程对阶段的官方分层。
- **输出格式**：1-2 句 ——「当前在 [0→1 / 1→10 / 10→100] 阶段，本阶段单一重点是 [标签精准 / 拉升 / 全域放大]；流量结构 [三条腿健康 / 单一来源风险]，缺 [短视频 / 付费] 这条腿。」

### 9.6 数据来源可信度甄别 (推导自心智模型「真数据 vs 估算数据」，C6 —— 元维度)

- **看什么**：手上每个数字是「真数据」还是「估算数据」——真数据 = 罗盘 / 巨量百应 (仅本人可见)；估算 = 蝉妈妈 / 飞瓜 / 新抖 (偏差 10-30%)；以及是否为「平台官方报喜口径」(不可 cross-check)。
- **在哪看**：核对数字出处——决策 / 合规 / 达人结算只认罗盘 / 巨量百应；竞品对标用蝉妈妈 / 飞瓜但标估算；平台大会数据标官方口径。
- **输出格式**：1-2 句 ——「本结论引用的数据是 [真数据 / 估算 (偏差 10-30%) / 平台官方口径]，来源 [...]；用于 [决策 / 结算 / 合规] 的部分 [是否] 满足『必须用真数据』要求。」

### 9.7 流派语境定位 (推导自心智模型 1.4 + 五大流派分歧)

- **看什么**：用户 / 案例属于哪一派——投放 / 运营工程派、知识 / 内容信任派、去 IP 化机构派、超头 IP / 明星派、选品派、私域沉淀派、平台官方派；以及他引用的方法论是「官方明线」还是「实战暗线」。
- **在哪看**：用户的表述 register (黑话密度 / 是否讲「算法博弈、投流脏活」) + 引用的代表人物 / 框架 (FACT / 兴趣电商 = 官方派；千川校准 = 投放派；做号不做人 = 去 IP 化派)。
- **输出格式**：1-2 句 ——「用户语境属 [X 派]，对应方法论侧重 [...]；该派与 [Y 派] 在 [分歧点] 上立场相反，回答应 [按 X 派方法论 / 同时呈现分歧、不抹平]。」
