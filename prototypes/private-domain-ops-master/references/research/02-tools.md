## 必备层

### 1. 企业微信（WeCom）

- **One-liner**: 私域 2024+ 的 default 主战场——腾讯官方的 to-B 客户连接工具，私域用户池的合规底座，所有 SCRM 都长在它的 API 上。
- **Tier**: 必备
- **Maintainer / Owner**: 腾讯（https://work.weixin.qq.com/）
- **License**: proprietary（免费使用，部分高级接口 / 互通能力按规则开放）
- **Maturity signals**:
  - GitHub stars: N/A（闭源平台，无开源仓库）
  - First public release: 2016（企业微信 1.0）；2019 起开放「客户联系」能力，私域用法才成型
  - Last update: 持续迭代（企业微信生态大会年度发布；2026 微信公开课 PRO 仍在推公私域协同）(evidence: [T02-S001])
  - Activity: healthy（腾讯战略级产品）
- **不适合 / 替代方案**: 不适合「想绕开平台规则做激进自动化」的玩法——企微对外挂 / 异常频率 / 虚假裂变持续收紧。替代品理论上是个人微信号，但 2024 起个人号已被行业共识压缩为辅助角色（intake warnings 明确），不构成「主战场」级替代。海外 WhatsApp Business / LINE OA 概念接近但生态完全不同，不可迁移。
- **生产案例**:
  - 瑞幸：被 2026 微信公开课 PRO 引为案例，称 90% 订单来自私域客户（数据有 vendor PR 成分，需打折看）(evidence: [T02-S001])
  - 行业共识：完美日记 / 名创优品 / 屈臣氏等头部品牌私域均以企微为承接主体（GMV 口径被广泛质疑夸大，见 intake warnings）
- **维护者背景**: 腾讯微信事业群——与 Track 01「平台派」figure（腾讯 / 企业微信官方）直接关联。
- **近期变化**（近 12 个月）: 持续推「公私域协同 / 全域」叙事；2025 传出将上线「信用分」体系对违规企业阶梯处罚、AI 质检覆盖率提升（evidence: [T02-S023]，注：此为第三方转述腾讯安全口径，未见官方原文，可信度 medium）。

### 2. 企微 SCRM（微伴助手 / 探马 / 微盛 / 尘锋 / EC / 句子互动 等同质工具群）

> **本行最重要的选型节点。** intake 明确警示「SCRM 类工具多且功能高度同质」——这里把它们作为**一个工具品类**整体卡片化，差异化选型放到下方「选型决策树」处理，避免把 6 张几乎一样的卡片堆给读者。

- **One-liner**: 在企业微信之上加「标签 / SOP / 群发 / 渠道活码 / 朋友圈 / 数据看板」的运营层——企微给你「客户连接」，SCRM 给你「规模化运营客户」的能力。
- **Tier**: 必备（品类级；≥80% 有一定规模的私域团队会装其中一个）
- **Maintainer / Owner**: 各为独立厂商。微伴助手（https://www.weibanzhushou.com/）、探马（https://www.tanmasoft.com/）、微盛·企微管家（https://www.weishengyun.com/，腾讯投资）、尘锋（https://www.dustess.com/）、EC SCRM（https://www.workec.com/）、句子互动（https://www.juzibot.com/）、腾讯企点（https://qidian.qq.com/，腾讯官方）。
- **License**: proprietary。商业模式分两类：**免费+增值**（微伴助手主打「安全免费」，60+ 功能免费，高级功能 / 坐席收费）vs **按坐席/年订阅**（探马 / 尘锋 / 微盛 / EC 等，几千到几万/年不等）(evidence: [T02-S002], [T02-S018])。
- **Maturity signals**:
  - GitHub stars: N/A（全部闭源 SaaS）
  - First public release: 2019-2020 集中出现（企微开放「客户联系」API 后催生的一代工具）
  - Last update: 全品类 healthy——2025-2026 集体在加 AI 能力（见新兴层）(evidence: [T02-S002], [T02-S003])
  - Activity: 品类 healthy；个别长尾厂商（艾客 / 粮仓 / 数云 / 沃丰 等）信号弱、未单独 retain
  - 采用率信号：被 36氪 两篇盘点 + IT之家选型文 + 各 vendor 自报 ≥4 个独立 source 反复点到「微伴 / 探马 / 尘锋 / 微盛」为头部 (evidence: [T02-S017], [T02-S018], [T02-S023])
- **典型使用场景**（决定选哪家的关键，越具体越能进决策树）:
  - **B2B / 重销售过程管理**（线索跟进、销售漏斗、BI 报表、员工管控）→ 探马 / EC 这类「销售型 SCRM」更顺手 (evidence: [T02-S004], [T02-S007], [T02-S017])
  - **零售 / 电商 / 教培、要全功能但预算敏感** → 微伴助手（免费起步、功能覆盖广）或微盛（13 万+ 企业、泛零售方案成熟）(evidence: [T02-S002], [T02-S006])
  - **医美 / 快消、要行业化会员运营模板** → 尘锋（医美是其重点行业）(evidence: [T02-S005])
  - **重 AI 自动应答 / 智能客服机器人** → 句子互动（AI 对话路线，见场景特化层单独卡片）(evidence: [T02-S003])
  - **强合规 / 受监管行业（金融 / 保险），要会话存档 + 敏感词拦截** → 优先看「合规资质齐全 + 会话存档」能力，微伴等头部均已支持 (evidence: [T02-S023])
- **不适合 / 替代方案**: 不适合「以为买了 SCRM 就有私域」的团队——SCRM 是底座，运营动作（SOP / 内容 / 人）才是内容，没有运营团队时 SCRM 的 80% 功能会闲置（evidence: [T02-S023]）。也不适合用 SCRM 去做平台明令禁止的自动化（批量加好友 / 模拟操作）——那是外挂红线，见避坑清单。替代方案：超小规模（几百好友）私域可先用企微原生功能（标签 / 群发 / 客户朋友圈）裸跑，不急着上 SCRM。
- **生产案例**:
  - 微盛·企微管家：vendor 自报服务 13 万+ 企业、3 万+ 付费企业，泛零售 / 电商 / 泛金融 / 教培方案 (evidence: [T02-S006]，vendor 自报数字需脱水)
  - 探马 / 尘锋 / 微伴：被 36氪 独立盘点列为头部企微 SCRM，覆盖教育 / 金融 / 医美 / 快消 (evidence: [T02-S017])
  - 注：SCRM 厂商的「服务 N 万企业」是行业惯例话术，按 intake warnings「ROI 不透明」原则一律打折，**没有一家给出可独立验证的私域 GMV production case**——这是本品类证据的结构性弱点。
- **维护者背景**: 微盛 / 腾讯企点与腾讯资本 / 官方有关联；其余为独立厂商。无与 Track 01 figures 的强关联（SCRM 厂商创始人不在 intake figures 列表）。
- **近期变化**（近 12 个月）: 全品类 2025-2026 集体「AI 化」——智能打标 / 话术推荐 / 会话总结 / 合规质检成为新卖点，「生态为王、AI 致胜」是厂商共同叙事（evidence: [T02-S002], [T02-S003], [T02-S023]）。

- **维护者背景**: 与 Track 01 figures 无直接关联。
- **近期变化**（近 12 个月）: 处于「AI+私域」风口中心；这一定位的 decay 同时受「AI 能力进步」和「微信对 AI 自动化的合规态度」双重影响。

  - [Primary] 句子互动官网: https://www.juzibot.com/ (collected: 2026-05-14)
  - [Reference] Track 05 sources「新 tools 发现」回传：句子互动与「AI 营销 / 私域智能化」强关联 (evidence: [T05 sources Phase 2 接口])
- **可信度**: medium（定位清晰、有跨 track 交叉佐证；但功能细节缺中立 source）
- **Decay risk**: high（AI 对话赛道演进极快 + 微信合规变量；新兴能力默认 high）

## 新兴 / 实验层

> **重要说明**：本行没有「2024 年后从 0 冒出来的全新工具」这一物种——私域工具盘子（企微 + SCRM + 电商 SaaS + 编辑器 + 数据）2019-2020 就基本定型，近 12 个月的「新」全部是**存量工具长出的 AI 能力层**，而非新工具。所以新兴层记录的是「能力」而非「独立产品」，且全部 `stability: experimental`。

## 选型决策树

> 不是「工具 A 适合 X、B 适合 Y」的清单，而是从「你的核心问题是什么」往下分支。**8 个节点。** 每个分支可追溯到上方研究笔记 + Source Manifest。
> **前置硬规则（不进树、所有分支共享）**：① 主战场默认企业微信，不要再问「要不要用个人号」——2024 起已定局（evidence: [T02-S001], [T02-S019]）；② 任何工具选型先过「合规资质」门槛，没有齐全合规资质 / 会话存档能力的工具，再便宜也不选（evidence: [T02-S023]）。

### 决策树 A：你现在的核心瓶颈是什么？

#### Branch 1：刚启动私域 / 0 到 1，还没有稳定流量和复购模型
→ **推荐**：企业微信原生功能（标签 / 群发 / 客户朋友圈 / 渠道活码）先裸跑，验证「能不能加到人、加到人有没有用」。
→ **推荐（轻量）**：要排版就 135 编辑器 / 秀米二选一；团队 SOP 用腾讯文档 / 飞书。
→ **不推荐**：此阶段就上重型企微 SCRM——没有运营团队和 SOP 时，SCRM 的 80% 功能会闲置（evidence: [T02-S023]）。
→ **不推荐**：此阶段就上微盟 / 有赞重型电商 SaaS——商城是承接器不是引流器，没流量灌进来就是空壳。
→ **不推荐**：此阶段就上小裂变做裂变——不懂微信裂变红线时，裂变是封号高发区（evidence: [T02-S008] + Track 04/06 封号红线）。
→ 真实案例：`[no clean public case found, based on stack-pattern analysis]` —— 中小商家「先裸跑再上工具」的路径在 Track 05 见实 / 群响语境中被反复提及，但缺乏可独立验证的具名案例，本 branch 可信度降为 medium。

#### Branch 2：已有一定私域规模，瓶颈在「运营动作没法规模化执行」（标签 / SOP / 群发 / 数据）
→ 进入企微 SCRM 选型子树（决策树 B）。这是本行最高频、最纠结的选型节点。

#### Branch 3：瓶颈在「私域里的流量转化不成交易」
→ **推荐**：上电商 SaaS——微盟 / 有赞**二选一**（不要同时上，功能高度重叠）。
→ **子分支 3a**：追求稳定性 + 全渠道 + 支付一体化 → **有赞**（有支付牌照，「重技术」）(evidence: [T02-S021], [T02-S022])。
→ **子分支 3b**：重营销能力、线上线下一体化的大品牌 / 重度线下品牌 → **微盟**（「重营销」，垂直行业方案多）(evidence: [T02-S021], [T02-S022])。
→ **替代**：交易体量很小可先用微信小商店 / 群接龙裸跑；技术强的大厂可自研小程序商城。
→ 真实案例：微盟服务智慧零售 / 餐饮 / 酒店等重线下大品牌；有赞被老客户认可功能与稳定性（evidence: [T02-S022]）。

#### Branch 4：瓶颈在「拉新增长」，要做老带新裂变
→ **推荐（先看）**：你用的企微 SCRM 是否已内置裂变 / 活码功能——微伴 / 探马等头部 SCRM 都有基础裂变，规模不大时不必单独上工具。
→ **推荐（专门做）**：核心打法就是裂变 → 小裂变（任务宝 / 活码 / 抽奖 / 盲盒获客）(evidence: [T02-S008])。
→ **强制前置**：上裂变前先核对微信「诱导分享 / 诱导关注」当前红线，3-6 个月前的裂变模板默认作废重验（Track 04/06 封号红线 + 玩法保鲜期）。
→ **不推荐**：把裂变当私域常规引擎——增长派 vs 平台/合规派在这点上分歧明确（Track 04 已识别），裂变是高风险灰区。

#### Branch 5：瓶颈在「内容触达 / 选号 / 数据」
→ **子分支 5a**：要做公众号 / 视频号内容运营、选号投放、竞品监测 → **新榜 / 新视**（公众号 + 视频号数据榜单）(evidence: [T02-S013])。
→ **子分支 5b**：私域流量从抖音 / 快手公域引来，要做公转私上游的达人筛选 / 投放分析 → **飞瓜数据**（注意：飞瓜在公域侧，是私域引流的输入工具，不是私域运营本身）(evidence: [T02-S014])。
→ **子分支 5c**：只要把公众号图文做好看 → 135 编辑器（全功能）/ 秀米（强设计感）二选一 (evidence: [T02-S024])。

### 决策树 B：企微 SCRM 怎么选？（Branch 2 的展开 —— intake 警示「工具多且高度同质，rationale 是重点」）

> 前提：6 家头部（微伴 / 探马 / 微盛 / 尘锋 / EC / 句子互动）功能高度同质，**90% 的基础能力（标签 / SOP / 群发 / 活码 / 朋友圈 / 数据）都有**。差异在「行业模板深度 + 偏运营还是偏销售 + 商业模式 + AI 路线」。先问下面 4 个问题，不要陷入逐项功能对比。

#### Branch B1：你的业务是「重销售过程管理」（线索跟进 / 销售漏斗 / 员工管控 / BI 报表）吗？
→ **是** → **探马 / EC** 这类「销售型 SCRM」更顺手——探马打通销售管理全流程 + BI 定制报表，EC 是老牌销售型 SCRM（教育 / 金融）(evidence: [T02-S004], [T02-S007], [T02-S017])。
→ **否（是「重运营」：社群 / 内容 / 触达 / 会员）** → 看 B2。

#### Branch B2：你的行业有强行业化模板需求吗？
→ **医美 / 快消** → **尘锋**（医美是其重点行业，行业化会员运营模板深）(evidence: [T02-S005])。
→ **泛零售 / 电商 / 教培、要成熟行业方案** → **微盛·企微管家**（13 万+ 企业，泛零售 / 教培方案成熟，腾讯投资）(evidence: [T02-S006])。
→ **行业不特殊 / 要通用全功能** → 看 B3。

#### Branch B3：预算敏感吗？
→ **是（中小团队 / 想低成本起步）** → **微伴助手**（主打「安全免费」，60+ 功能免费，高级功能 / 坐席再付费）(evidence: [T02-S002], [T02-S018])。
→ **否（愿为服务 / 稳定性付费）** → 探马 / 微盛 / 尘锋等按坐席订阅的均可，回到 B1/B2 按业务属性定。

#### Branch B4：核心需求是「AI 自动接待 / 智能客服机器人」吗？
→ **是** → **句子互动**（AI 对话机器人路线最专）(evidence: [T02-S003])；或看现有 SCRM 的 AI 模块（微伴 AI / 各厂 AI Agent，2025-2026 已普遍内置，见新兴层）。
→ **否** → 不必为 AI 单独选工具，回到 B1-B3。

#### Branch B5（特例分流，不在主链）：你是有线上线下多渠道的中大型品牌，要「全域统一客户视图」吗？
→ **是** → 跳出通用企微 SCRM，看 **群脉 SCRM**（CDP+CEP 品牌型 SCRM，联合利华 / 波司登级客户）(evidence: [T02-S015])——但中小商家 / 单渠道商家不要进这个分支，重型 CDP 架构对你是浪费。
→ **否** → 留在 B1-B4 的通用企微 SCRM 选型。

> **决策树 B 的元规则（最重要的一条 rationale）**：6 家头部企微 SCRM 选谁，**对私域成败的影响 < 选不选得对「主战场（企微）+ 有没有运营团队和 SOP」**。工具同质化意味着「选型纠结的边际收益很低」——别花两周对比 SCRM，那两周拿去搭 SOP 更值。且任何「哪家最好用」的结论保鲜期只有 3-6 个月（Decay risk: high）。

## 避坑清单

> 抓自「业内人吐槽 / 失败案例 / 厂商红线提醒 / 平台处罚标本」。≥5 条。本行避坑的核心母题是 **合规** 和 **「工具 ≠ 私域」**。

- ❌ **不要用个人微信号外挂（自动加好友 / 多开 / 批量群发 / 自动回复）做私域**：WeTool 是这一坑的标本——2020-05-25 被微信以「破坏平台生态、为恶意营销提供便利」永久封禁，使用者账号「完全没有先兆」地连带被封，有运营团队一夜 3 万用户失联、3000 个微信号被封（evidence: [T02-S019], [T02-S020]）。2024-2025 趋势更严：企微违规账号封禁量同比增 47%，私域营销场景占 68%（evidence: [T02-S023]）。任何「模拟最终用户手动操作」的程序都被微信定义为外挂，无论挂在个人号还是企微上。

- ❌ **不要以为「买了 SCRM 就有私域」**：SCRM 是底座，运营动作（SOP / 内容 / 人）才是内容。很多企业买了通用工具，最后发现 80% 的功能用不上（evidence: [T02-S023]）。Track 04 canon 反复出现的共识：「没有运营动作的池是死水」。先有 SOP 和团队，再上 SCRM；不是反过来。

- ❌ **不要只看低价 / 功能数量选 SCRM，忽略合规底线和行业适配**：随着数据安全法规全面实施，合规已是私域生命线——没有齐全合规资质的产品再便宜也会带来监管风险（evidence: [T02-S023]）。2026 选型逻辑已从「看功能数量」转向「看落地能力 / 合规安全 / 长期价值」。还有一个反向信号：对续费率自信的厂商不会自降身价推「多年使用权+打折」，看到这种促销要警惕（evidence: [T02-S023]）。

- ❌ **不要同时上微盟和有赞（或多家电商 SaaS）**：两家功能高度重叠，「重技术 vs 重营销」是定位差异不是互补（evidence: [T02-S021]）。工具派操盘手的共识是「两家对比着看、单跟一家足够」（呼应 Track 05 对两家年度大会的判断）。同理，6 家企微 SCRM 也只装一个。

- ❌ **不要把裂变工具当私域的常规增长引擎**：裂变是微信「诱导分享 / 诱导关注」处罚的高发区，玩法保鲜期只有 3-6 个月，3 个月前的裂变模板默认作废（Track 04/06 封号红线 + 玩法保鲜期）。增长派把裂变当核心引擎，但平台派 / 合规派视之为高风险灰区——Track 04 已识别这是行业明确分歧。用裂变工具（小裂变等）前必须先核对微信当前红线。

- ❌ **不要把飞瓜 / 新榜这类公域数据工具当「私域工具」**：飞瓜在抖音 / 快手公域侧，新榜在公众号 / 视频号半公域侧——它们的产出是私域引流的**输入**，不是私域运营本身。私域内部的转化 / 复购数据要看 SCRM 后台。混淆「公域数据工具」和「私域运营工具」会导致工具栈配置错位。

- ❌ **不要给中小 / 单渠道商家上群脉这类品牌型 CDP SCRM**：CDP 中台的价值在「多渠道数据打通」，渠道单一时这套重型架构是纯浪费——成本高、上手重、功能用不满。中小商家用通用企微 SCRM（微伴 / 探马等）即可。反过来，有线上线下多渠道的大品牌用通用企微 SCRM 又会缺「统一客户视图」——选型要匹配自己的渠道复杂度。

- ❌ **不要期待 AI SCRM 能力「全自动替代私域运营人」**：2025-2026 各厂 AI 能力（智能打标 / 话术推荐 / 会话总结）真实有效，但定位是「提效 / 降下限」不是「替人」。「客户意图识别准确率 99.2%」这类数字是厂商 / 测评口径，按 intake「ROI 不透明」原则大幅打折看（evidence: [T02-S023]）。更要警惕：AI 自动应答 / 自动外联若做得激进，同样可能撞微信对「模拟操作 / 异常频率」的红线。

- ❌ **不要轻信 vendor / 头部品牌的私域 GMV 数字做选型依据**：本行 ROI 严重不透明——SCRM 厂商的「服务 N 万企业」、瑞幸「90% 订单来自私域」这类数字都有 PR 成分（evidence: [T02-S001], [T02-S006]，intake warnings 明确点名头部私域 GMV 多被疑夸大）。选工具看「合规 / 行业适配 / 自己团队能不能用起来」，不看 vendor 宣传的成功学数字。

### 反复出现 ≥ 3 source 都强调的「工具选型原则」（候选 playbook 规则）

- **「主战场默认企业微信，不要再纠结个人号」** —— 出现于：T02-S001（企微官网 + 2026 公开课公私域协同）/ T02-S019（WeTool 封禁反证个人号外挂死路）/ T02-S023（2026 SCRM 选型默认企微为底座）+ Track 04/06 多处。2024 起已定局，任何「个人号无脑加好友 / 多开」式建议直接判过时。
- **「工具是底座，运营动作（SOP / 内容 / 人）才是内容——先有 SOP 再上工具」** —— 出现于：T02-S023（80% 功能用不上）+ Track 04 canon 多本书「没有运营动作的池是死水」+ Track 05 见实 / 群响语境。这是把「外行」和「内行」分开的判据。

- **「同质工具只装一个，别花两周做选型对比」** —— 出现于：T02-S021（微盟 vs 有赞二选一）/ T02-S017+S018（6 家企微 SCRM 功能高度同质）/ Track 05（两家 SaaS 大会单跟一家足够）。选型纠结的边际收益低，时间拿去搭 SOP 更值。
- **「工具 / 玩法保鲜期 3-6 个月，结论要标时效、要高频复验」** —— 出现于：T02-S008（裂变赛道）/ T02-S023（SCRM「生态为王、AI 致胜」新阶段）+ Track 04 canon 反复 + Track 06 封号红线。这条直接决定 SKILL.md tools 节是 update 流程刷新频率最高的一格。

### 显著的工具流派分裂（候选 智识谱系条目）

- **「工具派 / SCRM 自动化驱动」 vs 「人本派 / 关系运营驱动」**：
  - 工具派（代表工具：企微 SCRM 全家桶、句子互动 AI、AI-native SCRM 能力；代表人物：与 Track 01 鉴锋 / SCRM 厂商关联）——信「SOP + 自动化 + AI 跑一切」。
  - 人本派（代表立场：见实 / 品牌操盘手）——信「私域核心是人对人的温度，工具只降下限」。
  - 这正是 Track 04 canon 已识别的「工具能多大程度替代人」分歧，在 tools 维度的具体投影。
- **「增长 / 裂变派」 vs 「平台 / 合规派」**：
  - 增长派（代表工具：小裂变、各 SCRM 的裂变模块）——把裂变当核心引擎。
  - 平台 / 合规派（代表立场：微信官方规则、企微生态大会）——视裂变为高风险灰区。
  - 工具选型上表现为「敢不敢上裂变工具、敢上多激进」。
- **「通用企微 SCRM 派」 vs 「品牌型 CDP SCRM 派」**：
  - 通用派（微伴 / 探马 / 微盛 等）——轻、快、便宜，服务腰部商家。
  - 品牌 CDP 派（群脉 等）——重、全、贵，服务有多渠道的中大型品牌。
  - 对应 Track 04「复购/会员（品牌）派」与「增长/工具派」的工具侧分野。

### 新兴工具信号

- **当前活跃 / 上升的新「工具」数：0 个真正的新产品**——本行工具盘子 2019-2020 已定型，近 12 个月没有从 0 冒出来的新工具。
- **当前活跃 / 上升的新「能力」数：2 类**——AI-native 企微 SCRM 能力（智能打标 / 话术推荐 / 会话总结 / 合规质检）+ 电商 SaaS 的 AI First 套件（微盟 Weimob Admin Skills / WAI）。全部 `stability: experimental`。
- **出现 → 主流 速度估计**：AI 能力层约 **6-12 个月**会从「卖点」沉淀为「标配」（2025 已普遍内置，2026 已成第三方评估维度）——比新工具从出现到主流快，因为它是长在存量工具上的，没有获客 / 教育成本。最大变量不是技术，是**微信对 AI 自动化的合规态度**。

### 冷僻 / 信号薄弱（Phase 2.8 诚实边界须标注）

**本 track 触发多条冷僻 / 信号薄弱信号，Phase 2.8 必须标注「工具栈维度信号结构性偏弱」**：

- **必备层 < 3 个？** 否（5 个），但其中 1 个（腾讯文档 / 飞书）缺私域专门 source、靠 intake 点名 + 行业常识，1 个（企微 SCRM）是「品类卡片」而非单一产品。
- **必备层证据 < 50% 采用率 / 缺 GitHub stars 基准？** ✅ 触发。本行工具几乎全闭源 SaaS，**无 GitHub stars 这一信号**，SKILL.md 原定的「stars 中位数 ×2」自适应基准在本行不可用，已改用「≥3 独立 source 点名 + 行业 default 共识」替代。采用率数据全是 vendor 自报（「13 万企业」「2000+ 企业」「543 万商户」）或第三方盘点口径，**无一是可独立验证的行业 survey**。
- **新兴层 0 个真新工具？** ✅ 触发（新兴层 2 条都是「能力层」非「独立产品」）。结论：这不是调研失败，是行业进入稳态——工具盘子定型、变量在 AI 能力层。
- **production case 普遍缺失？** ✅ 严重触发。**几乎没有一个工具有可独立验证的私域 GMV / ROI production case**——vendor 给 logo 墙和「服务 N 万企业」，第三方盘点带厂商立场，瑞幸「90% 私域订单」这类数字被 intake 明确点名为「疑夸大」。本行 ROI 不透明是结构性的。
- **一手来源占比**：manifest 27 条中 verified_primary 0 / surrogate_primary 16（全是 vendor 官网，按规范升级）/ secondary 11 / blacklisted 0。**若按「vendor docs = surrogate_primary 算一手」口径，一手占比约 59%（16/27）**；但需诚实说明：这里的「一手」全部是**厂商自己的官网**，不是中立的行业 survey 或操盘手实测——vendor 一面之词的 mental model 必须 ≥2 source 对冲。**真正中立的第三方一手（行业 survey / 独立评测）几乎为零。**
- **黑名单的结构性损耗**：本行工具对比内容的主阵地是知乎 / CSDN / 人人都是产品经理（woshipm）/ 百度百科——全部黑名单。搜索命中里这类 URL 占比极高（每次搜索 5-7 成），**全部丢弃未进 manifest**，导致「第三方中立对比」这一维度的可用 source 被结构性砍掉一大半。这也是多个工具卡片（群脉 / 新榜 / 飞瓜）source 偏薄的直接原因。

**结论给 Phase 2**：工具栈维度信号结构性偏弱，根源 = (a) 工具全闭源、无 GitHub 等开放信号；(b) ROI / 采用率数据全是 vendor 口径，行业无中立 survey；(c) 第三方对比内容主阵地全在黑名单平台，按规则丢弃后中立 source 所剩无几；(d) SCRM 类工具高度同质，「选型」本身的信息价值就低于「主战场 + SOP + 团队」。**生成 SKILL.md 时，工具栈节应诚实写明：「本行工具盘子已定型（企微 + SCRM + 电商 SaaS + 编辑器 + 数据），近 12 个月变量集中在 AI 能力层；工具选型对私域成败的影响小于战场选择和运营体系；所有‘哪家最好用’结论保鲜期 3-6 个月，且缺乏中立 production case 支撑，需高频复验。」**
