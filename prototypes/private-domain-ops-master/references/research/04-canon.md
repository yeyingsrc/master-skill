### 📖 1. 增长黑客（创业公司的用户与收入增长秘籍）

- **Author / 作者**: 范冰（XDash）（→ Track 01 figure 候选，intake 已点名）
- **Year**: 2015 / 后有《增长黑客实战》（范冰 + 张溪梦）
- **Type**: popular science / 方法论
- **One-liner**: 私域「增长」二字的上游祖本——把「靠技术 + 数据做增长」的理念引入中文圈，私域的裂变 / 漏斗 / AARRR 话语都从这里继承。
- **核心论点 (3-5)**:
  1. 增长是技术 + 营销之间的新角色，不靠砸钱买量
  2. AARRR 海盗模型（拉新→激活→留存→变现→自传播）
  3. 数据驱动、A/B 测试、快速试错是增长的方法论内核
  4. 病毒传播 / 自传播机制设计是低成本增长的关键
- **读完得到什么**: 一套「漏斗 + 实验 + 自传播」的通用增长心智，是理解私域裂变 / 复购话术的前置。
- **难度**: 入门
- **Endorsement evidence**:
  - `[type: blog_secondary]` 综合运营 / 增长书单长期收录，豆瓣 2015 经管畅销榜 top5 (evidence: [T04-S003])
  - `[type: figure_short]` 滴滴柳青 / 罗振宇 / 金山张宏江等公开推荐 (evidence: [T04-S003])
  - `[type: blog_secondary]` 黄有璨《运营之光》体系将增长 / 流量与运营并列讨论，同源谱系 (evidence: [T04-S004])
- **是否被新版 supersede**: 部分。《增长黑客实战》（2018，范冰 + 张溪梦）更新更系统，但 2015 原版的「概念启蒙」教学路径仍有独立价值——它是中文增长话语的起点。保留原版。
- **替代品**: 《硅谷增长黑客实战笔记》（曲卉）；Sean Ellis《Hacking Growth》英文原典
- **如果可以只读 1 章**: AARRR 模型 + 病毒传播章
- **可信度**: medium（综合书，非私域专书；增长 ≠ 私域，迁移时需注意场景差异）
- **Decay risk**: low（方法论稳定，但「增长黑客」一词在国内已部分过气）

### 📖 2. 运营之光 3.0（我的互联网运营方法论与自白）

- **Author / 作者**: 黄有璨（三节课联合创始人）（→ Track 01 figure 候选，intake 已点名）
- **Year**: 初版 2016 / 3.0 版 2022-06（电子工业出版社）
- **Type**: monograph / 方法论 + 自白
- **One-liner**: 中国互联网运营的「通识教科书」级作品，3.0 版自我颠覆、只保留原作 <20% 内容，补入短视频 / 直播 / 私域流量章节。
- **核心论点 (3-5)**:
  1. 系统梳理中国互联网「流量—变现」格局的演化与底层规律
  2. 运营是「四两拨千斤」，靠对人性 / 细节的把握而非资源堆砌

  4. 强调运营者的「价值观坚守」与长期主义
- **读完得到什么**: 把私域放回「整个互联网运营」的坐标系里理解——私域不是孤立技能，是流量变现链条的一段。
- **难度**: 入门-进阶
- **Endorsement evidence**:
  - `[type: course_syllabus]` 三节课运营课程体系的核心配套教材 (evidence: [T04-S004])
  - `[type: figure_long]` 黄有璨本人 15 年从业 + 连续创业，本书即其长篇方法论自述 (evidence: [T04-S004])
  - `[type: blog_secondary]` 「影响一代互联网人」的通识书，运营研究社等派系长期引用 (evidence: [T04-S004])
- **是否被新版 supersede**: 3.0 即最新版，已 supersede 1.0 / 2.0（作者明确「只保留 <20% 旧内容」）。旧版 DROP。
- **替代品**: 运营研究社体系的运营书；私域专题则看《超级用户增长》

### 📖 3. 超级用户增长（低成本实现私域用户持续复购）

- **Author / 作者**: 晏涛（社会化营销专家）（→ Track 01 figure 候选）
- **Year**: 2020-12（中国友谊出版公司）
- **Type**: 方法论 / 商业管理
  > 注：intake.json 把作者写成「陈历 / 江南春」，调研更正为 **晏涛**；江南春是《抢占心智》作者，与本书无关。
- **One-liner**: 私域复购 / 会员体系方向流传最广的框架书——「IP 化 → 连接 → 促活 → 分层 → 复购」五步法则。
- **核心论点 (3-5)**:
  1. 私域的终点是「超级用户」：高忠诚、高复购、能影响他人
  2. 五步运营法则：IP 化、连接、促活、分层、复购
  3. 上篇讲私域兴起背景与布局，中篇讲五步法，下篇讲会员体系 + 数据化运营
  4. 私域营销闭环 = 公域转私 + 精细化分层 + 会员体系沉淀
- **读完得到什么**: 一套可直接套用的「私域复购 / 会员体系」操作框架，是私域人讨论复购时的共同语言。
- **难度**: 入门
- **Endorsement evidence**:
  - `[type: blog_secondary]` 私域书单高频收录，「五步法则」被大量二手文章复述 (evidence: [T04-S002])
  - `[type: blog_secondary]` 豆瓣有专门书评讨论「如何构建私域流量池 / 寻找超级用户」 (evidence: [T04-S002])
  - `[type: conf_citation]` 「超级用户 / KOC」概念与见实大会、SaaS 厂商话语高度重叠 (evidence: [T04-S002], [T04-S006])
- **是否被新版 supersede**: 无显著新版，保留。
- **替代品**: 《私域流量池》（更偏工具）；《增长黑客》（更偏拉新而非复购）
- **如果可以只读 1 章**: 五步法则中的「分层」+「复购」两章
- **可信度**: medium（框架清晰，但案例 ROI 数据按行业惯例需打折看）
- **Decay risk**: medium（框架稳定，但 2020 年成书，企微为主战场的变化未充分覆盖）

### 📖 4. 私域流量池

- **Author / 作者**: 刘翌（私域 SaaS 厂商创始人系）
- **Year**: ~2020（机械工业出版社系）
- **Type**: 方法论 / 厂商背书手册
- **One-liner**: 工具派视角的「流量池」搭建手册，自称基于服务 3 万 + 企业的经验。
- **核心论点 (3-5)**:
  1. 私域 = 把流量蓄进「池子」反复运营，对抗公域流量越来越贵
  2. 流量池搭建 = 工具底座（SCRM）+ 标准化运营动作
  3. 强调数据化、自动化、SOP 化运营
- **读完得到什么**: 工具派 / SaaS 视角下的私域搭建路线图。
- **难度**: 入门
- **Endorsement evidence**:
  - `[type: blog_secondary]` 私域书单常列，被描述为「权威实操教科书」（注：此类描述多带营销色彩）(evidence: [T04-S005])

  - `[type: blog_secondary]` 与《增长黑客》的「流量池」思想同源 (evidence: [T04-S003])
- **是否被新版 supersede**: 信息不足，暂保留。
- **替代品**: 《超级用户增长》（同主题、非厂商背书视角）
- **如果可以只读 1 章**: 流量池工具底座章

## 💡 Core Concepts / 核心概念

> 24 个，按 tier 分组。来源说明：本行概念无单一「奠基论文」，多源自 (a) 见实《私域流量运营词汇手册》
> （1.0 版，105 词，行业最接近「标准词表」的产物，见 [T04-S014]）；(b) 商业畅销书框架；
> (c) 从「增长黑客 / 关系营销 / CRM」迁移而来的上游概念。Track 06 (glossary) 会做更细的术语处理，
> 此处只取「进入 canon 级、讨论方法论时绕不开」的概念。

### tier-1（核心，所有从业者必懂）

#### 💡 1. 私域流量
- **Tier**: tier-1
- **One-liner**: 沉淀在企业 / 品牌自有渠道、基于信任关系、可反复免费触达的用户流量。
- **来源**: `[primary]` 见实《私域流量运营词汇手册》行业定义 + 2019 行业造词；`[significant follow-up]` 各畅销书各有措辞
- **关联概念**: 公域（对立）、流量池（容器）、触点（手段）
- **行业内当前理解 vs 原始定义**: 原始更偏「微信个人号好友」；当前理解扩展为「企微 + 公众号 + 视频号 + 小程序 + 群」全生态可触达用户。
- **为什么进入 canon**: 解决「公域流量越来越贵、买一次用一次」的痛点——私域是「买一次反复用」。
- **常见误用**: 把「公众号粉丝数」「APP 注册用户」都叫私域；真私域的判据是**能否直接 1v1 / 1vN 主动触达**。
- **Endorsement evidence**: (evidence: [T04-S014], [T04-S001], [T04-S013])

#### 💡 2. 公域 / 私域（二分）
- **Tier**: tier-1
- **One-liner**: 平台算法分发、用一次付一次的流量（公域） vs 自主掌握、可反复触达的流量（私域）。
- **来源**: `[primary]` 行业共识，2019 私域造词时同步成型
- **关联概念**: 公转私 / 私转公、全域（近年提法，强调公私协同）
- **行业内当前理解 vs 原始定义**: 近年从「公私对立」演化为「全域协同」——见实 2024 大会圆桌即谈「全域团队」。
- **为什么进入 canon**: 这是整个行业的世界观基础，不理解这对二分无法讨论任何私域问题。
- **常见误用**: 把私域当成「不花钱的流量」——其实私域的人力 / 工具 / 内容成本很高。
- **Endorsement evidence**: (evidence: [T04-S014], [T04-S006], [T04-S013])

#### 💡 3. 公转私 / 私转公
- **Tier**: tier-1
- **One-liner**: 把公域流量（抖音 / 小红书 / 视频号）引导沉淀到私域，及用私域反哺公域内容 / 数据的双向动作。
- **来源**: `[primary]` 行业共识
- **关联概念**: 私域漏斗（公转私是漏斗第一步）、引流承接
- **行业内当前理解**: 公转私的「承接」环节（加好友话术 / 福利钩子）是漏斗最易漏的一环。
- **为什么进入 canon**: 私域不会凭空有人，公转私是私域的「入水口」。
- **常见误用**: 只重引流不重承接，导致加粉后大量流失。
- **Endorsement evidence**: (evidence: [T04-S014], [T04-S001])

#### 💡 4. 流量池
- **Tier**: tier-1
- **One-liner**: 把流量蓄存起来反复运营的「池子」心智模型。
- **来源**: `[primary]` 《增长黑客》范冰引入「池」思维；`[significant follow-up]` 《私域流量池》刘翌专门化
- **关联概念**: 私域流量（池里的水）、SCRM（管理池子的工具）
- **行业内当前理解**: 从「蓄水」延伸到「养鱼」——不只是存量，更是经营。
- **为什么进入 canon**: 提供了私域最核心的隐喻框架。
- **常见误用**: 以为建了群 / 加了好友就有「池」——没有运营动作的池是死水。
- **Endorsement evidence**: (evidence: [T04-S003], [T04-S005])

#### 💡 5. 触点
- **Tier**: tier-1
- **One-liner**: 触达用户的具体渠道——公众号、微信社群、小程序、企业微信、朋友圈。
- **来源**: `[primary]` 见实《私域流量运营词汇手册》明确定义
- **关联概念**: 全触点运营、UnionID（打通多触点身份）
- **行业内当前理解**: 强调「多触点协同」而非单点——一个用户应被多触点覆盖。
- **为什么进入 canon**: 私域操盘的本质就是「在对的触点、对的时间、发对的内容」。
- **常见误用**: 只用单一触点（只发公众号 / 只发群），触达效率低。
- **Endorsement evidence**: (evidence: [T04-S014])

#### 💡 6. SCRM / 企微 SCRM
- **Tier**: tier-1
- **One-liner**: 社会化客户关系管理工具；企微 SCRM 是基于企业微信 API 的 SCRM 形态，私域的工具底座。
- **来源**: `[primary]` CRM 概念迁移 + 企业微信开放生态催生；见实词汇手册收录
- **关联概念**: 用户标签 / 分层、企业微信、UnionID
- **行业内当前理解**: 2024 起「SCRM」基本等同于「企微 SCRM」——个人号 SCRM 因合规风险被边缘化。
- **为什么进入 canon**: 没有 SCRM，标签 / SOP / 数据归因都无法规模化执行。
- **常见误用**: 以为买了 SCRM 就有私域——工具是底座，运营动作才是内容。
- **Endorsement evidence**: (evidence: [T04-S014], [T04-S009])

#### 💡 7. 用户标签 / 分层
- **Tier**: tier-1
- **One-liner**: 给用户打标签，并按价值 / 生命周期阶段分群做差异化运营。
- **来源**: `[primary]` 《超级用户增长》晏涛「五步法则」中的「分层」；CRM 上游概念
- **关联概念**: RFM、用户生命周期、SCRM（执行载体）
- **行业内当前理解**: 标签体系的搭建质量直接决定私域精细化程度——粗放打标 = 没打标。
- **为什么进入 canon**: 私域的「精细化」全靠分层落地。
- **常见误用**: 标签维度过多过杂、无法指导动作；或全员一套话术不分层。
- **Endorsement evidence**: (evidence: [T04-S002], [T04-S014])

#### 💡 8. 1v1 触达 SOP
- **Tier**: tier-1
- **One-liner**: 加好友后按标准化话术 + 节奏（如首次话术 / 7 日触达节奏）做私聊触达。
- **来源**: `[primary]` 操盘手实践共识，无单一出处
- **关联概念**: 朋友圈 SOP、社群 SOP、SCRM（话术库 / 自动化）
- **行业内当前理解**: SOP 是「下限保障」——保证新手也能达到及格线，高手在 SOP 之上做个性化。
- **为什么进入 canon**: 私域是「规模化的关系运营」，没有 SOP 就无法规模化。
- **常见误用**: SOP 僵化成骚扰式群发，反而触发用户反感 / 拉黑。
- **Endorsement evidence**: (evidence: [T04-S001], [T04-S006])

#### 💡 9. 私域漏斗
- **Tier**: tier-1
- **One-liner**: 引流 → 沉淀 → 触达 → 转化 → 复购 → 裂变 的私域转化链路。
- **来源**: `[primary]` 《私域流量》冯平等「六层框架」；与《增长黑客》AARRR 同源
- **关联概念**: AARRR（上游模型）、公转私（漏斗入口）、复购（漏斗后段）
- **行业内当前理解**: 漏斗各环节的「转化率」是私域诊断的核心抓手。
- **为什么进入 canon**: 提供了私域工作的整体流程地图。
- **常见误用**: 只盯前端引流，忽视后端复购 / 裂变——而后端才是私域 ROI 的来源。
- **Endorsement evidence**: (evidence: [T04-S001], [T04-S003])

#### 💡 10. 复购 / 会员体系
- **Tier**: tier-1
- **One-liner**: 用等级 / 积分 / 储值 / 老带新机制驱动用户重复购买。
- **来源**: `[primary]` 《超级用户增长》晏涛下篇专论；零售会员制上游
- **关联概念**: 超级用户、LTV、用户分层
- **行业内当前理解**: 私域的核心价值主张就是复购——不能带来复购的私域是伪私域。
- **为什么进入 canon**: 区分「私域」与「一次性流量收割」的分水岭。
- **常见误用**: 把会员体系做成复杂积分游戏，用户算不清也不在乎。
- **Endorsement evidence**: (evidence: [T04-S002], [T04-S013])

## Phase 2 接口

### 反复出现 ≥ 3 个 canon 都讨论的核心 idea（候选行业心智模型）

- **「流量池 / 蓄水养鱼」** —— 出现于：《增长黑客》（引入「池」思维）/《私域流量池》（专门化）/《超级用户增长》（连接 + 促活即「养」）/ 见实大会话语 → **强候选心智模型**：私域人看一切流量的镜片是「能不能蓄进池、能不能反复用」，而非「这次能转化多少」。
- **「公私二分 / 流量自主权」** —— 出现于：见实词汇手册 /《私域流量》/ 艾瑞白皮书 / 几乎所有来源 → **强候选心智模型**：私域人的世界观底层是「流量是租的还是自己的」，焦虑来自「公域流量越来越贵且用一次付一次」。
- **「精细化分层 / 千人千面」** —— 出现于：《超级用户增长》（分层是五步法核心）/ 见实词汇手册（标签 / 触点）/ SCRM 厂商话语 → **候选心智模型**：私域人默认「用户不是一群人，是 N 个分层」，反对全员一套话术。
- **「SOP 化 = 规模化关系运营」** —— 出现于：《私域流量》（六层框架含 SOP 意识）/ 见实大会操盘手分享 / SCRM 厂商 → **候选心智模型**：私域人把「关系」当成可被流程化、可被工具放大的对象，这是它区别于「纯销售」和「纯客服」的地方。
- **「漏斗 / 后端复购才是 ROI」** —— 出现于：《私域流量》六层框架 /《增长黑客》AARRR /《超级用户增长》复购专论 → **候选 playbook 而非心智模型**（生成力够但排他性弱，AARRR 是通用增长概念）。

### 智识谱系种子

本行**没有学术谱系**，是从「上游商业方法论 + 微信平台演化」拼接出的实践谱系。五大流派（intake warnings 已识别），与 canon 的对应：

- **增长 / 工具派**
  - 奠基：《增长黑客》（范冰，2015）—— 提供「漏斗 + 实验 + 自传播」底层方法论
  - 当前代表：→ Track 01 figure 鉴锋（零一裂变）、各 SCRM 厂商；canon 对应《私域流量池》（刘翌）
  - 主张：私域 = SCRM + 自动化 + 数据驱动一切
- **运营通识派**
  - 奠基：《运营之光》（黄有璨，2016 / 3.0 2022）—— 把私域放进「互联网运营」整体框架
  - 当前代表：→ Track 01 figure 黄有璨、运营研究社体系
  - 主张：私域不是孤立技能，是流量变现链条的一段
- **复购 / 会员（品牌）派**
  - 奠基：《超级用户增长》（晏涛，2020）—— 「IP→连接→促活→分层→复购」五步法
  - 当前代表：→ Track 01 完美日记 / 名创优品 / 屈臣氏等品牌操盘手；见实大会品牌分会场
  - 主张：私域服务品牌长期价值，慢但稳，终点是「超级用户」
- **媒体 / 操盘手派**
  - 奠基：见实科技（徐志斌）—— 以「私域大调研」+ 年度大会持续输出一手案例
  - 当前代表：→ Track 01 figure 徐志斌；群响（刘思毅）
  - 主张：私域无固定真理，靠持续拆解标杆案例 + 操盘手共创
- **平台派**
  - 奠基：微信官方（微信公开课 PRO）+ 企业微信官方
  - 当前代表：→ Track 01 腾讯 / 企业微信官方
  - 主张：私域必须在平台规则内做，企微为 default 战场
- **主要分歧（still-debated within the canon）**：
  1. **私域是「销售放大器」还是「品牌资产」**？销售派要激进话术快变现，品牌派要慢养信任——同一问题给完全相反建议。
  2. **工具能多大程度替代人**？工具派信「SOP + 自动化跑一切」，媒体 / 品牌派信「私域的核心是人对人的温度」。
  3. **裂变玩法的边界**：增长派把裂变当核心引擎，平台派和合规派视之为高风险灰区。
  4. **私域是否人人能做**：canon 普遍鼓吹「人人私域」，但 intake 和行业实况显示「头部 vs 腰部断层」「不适合 70% 商家」——这是 canon 本身的盲区，Phase 2 须用诚实边界对冲。

### 核心概念 → 候选 playbook

- **「公私二分」暗示**：遇到「流量贵 / 投放 ROI 下降」类问题 → 先问「这部分流量能不能沉淀成私域反复用」，而非继续加投放预算。
- **「流量池」暗示**：遇到「加了很多好友 / 建了很多群但没产出」 → 判断「有没有运营动作」，无动作的池是死水，先补 SOP 再谈规模。
- **「精细化分层」暗示**：遇到「私域转化率低 / 用户反感」 → 先查「是不是全员一套话术」，按标签 / 生命周期分层是第一刀。
- **「个人号 vs 企微」暗示**：遇到「该用什么承接 / 现有个人号矩阵要不要迁」 → 默认推企微为主战场，个人号辅助；任何「个人号无脑加好友」式建议直接判过时。
- **「封号红线 / 玩法保鲜期」暗示**：遇到任何「现成裂变 / 自动化玩法」 → 先问「这玩法多新、踩不踩微信当前红线」，3-6 个月前的玩法默认要重新验证。
- **「漏斗」暗示**：遇到「私域 ROI 不行」 → 不要笼统归因，按引流→沉淀→触达→转化→复购→裂变逐环节看转化率，定位是哪一环漏。
- **「复购才是 ROI」暗示**：遇到「私域值不值得投入」类问题 → 用 LTV / 复购视角而非首单 ROI 评判，私域价值在长期。
