### Step 1 三重验证预判（候选 → 评级）

- **KEEP 为心智模型（✅✅✅ 或 ✅✅⚠️）**：C1 公私二分（验证 3 ⚠️ 放大版）、C2 流量池（✅✅✅）、C3 精细化分层（✅✅✅）、C5 玩法保鲜期（✅✅✅ 强排他）、C8+C9 合并为「克制与算账」（✅✅✅）。
- **C4 SOP 化**：✅✅⚠️ —— SOP 是通用管理概念，但「把关系流程化 + 工具放大」是私域特化 → KEEP 但描述捕捉放大程度。
- **C6 IP 化**：✅✅⚠️ —— 个人 IP / 人设是泛内容行业概念，私域放大版（账号人格化是触达前提）→ KEEP 放大版。
- **降级 playbook**：C7 复购/LTV（排他性弱）、C10 漏斗（Track 04 已明示）、C11 承接漏损、C12 企微 default、C13 钩子精准度、C14 触点协同、C17 不适合 70%。
- **进 §7 谱系不进心智模型**：C16 运营通识派主张、C18 小群效应（单人）。
- **进 §5 表达 DNA**：C15 真私域判据（「拒绝伪私域」）。
- 合并同类项：C8+C9 → 一个心智模型「克制 + 算账（私域是有成本、有红线、数据会骗人的经营）」；C10+C11 → 一条 playbook。
- **最终心智模型预计 6 个**：公私二分、流量池/蓄水养鱼、精细化分层、SOP 化（关系工程化）、玩法保鲜期/合规半衰期、克制与算账。

## 候选 playbook 规则 (target 10-20，实收 13)

| # | Pattern | 来源 |
|---|---------|------|
| P1 | 如果流量贵 / 投放 ROI 下降 → 先问「这部分流量能不能沉淀成私域反复用」，而非继续加投放预算 | T04 公私二分暗示 |
| P2 | 如果加了很多好友 / 建了很多群但没产出 → 判断「有没有运营动作」，无动作的池是死水，先补 SOP 再谈规模 | T04 流量池暗示 + T02 避坑「买了 SCRM≠私域」 |
| P3 | 如果私域转化率低 / 用户反感 → 先查「是不是全员一套话术」，按标签 / 生命周期分层是第一刀 | T04 精细化分层暗示 + T03 W6 |
| P4 | 如果要选承接工具 / 现有个人号矩阵要不要迁 → 默认企微为主战场，个人号辅助；任何「个人号无脑加好友」式建议直接判过时 | T04 个人号vs企微 + T02 决策树前置硬规则 + T06 |
| P5 | 如果要用一个现成裂变 / 自动化玩法 → 先问「这玩法多新、踩不踩微信当前红线」，3-6 个月前的玩法默认重新验证 | T01/T03/T04/T06 玩法保鲜期 |
| P6 | 如果私域 ROI 不行 → 不要笼统归因，按引流→沉淀→触达→转化→复购→裂变逐环看转化率，定位漏损最大的一环（含 C11 承接） | T03/T04/T06 漏斗 |
| P7 | 如果判断「私域值不值得投入」→ 用 LTV / 复购视角而非首单 ROI，且先查品类适配性（高复购 / 高客单 / 决策链长更适合）+ 三个前置条件（稳定公域来源 / 能持续产内容 / 客单价覆盖人力） | T04 复购暗示 + T03 W1 资深 optimize + intake C17 |
| P8 | 如果面对「要不要一次性做全 / 做大 / 做完美」→ 默认先用最小版本验证跑通，再放大（企微 1v1 + 朋友圈两件套先验证品类和话术） | T03 W1-W8 资深 skip，「极强候选」 |
| P9 | 如果要选企微 SCRM / 电商 SaaS → 同质工具只装一个，别花两周做选型对比，时间拿去搭 SOP 更值 | T02 决策树 B 元规则 + 避坑「不要同时上微盟和有赞」 |
| P10 | 如果选工具 → 先过合规门槛（合规资质 + 会话存档），不看低价 / 不看功能数量；看到「多年使用权+打折」促销要警惕 | T02 避坑 + Phase 2 接口 |
| P11 | 如果要用户做一个动作（加好友 / 迁移 / 转化）→ 先想清楚「对用户的理由 / 钩子是什么」，且钩子要「高感知价值、低成本、与产品强相关」（钩子越猛留存越差） | T03 Phase 2 接口 + T06 钩子词条 |
| P12 | 如果一个私域动作要规模化 → 先确保用户被打标、能分层；标签体系反推自「要做哪些差异化动作」，没有触达策略对应的标签是死标签 | T03 Phase 2 接口 + T06 标签词条 |
| P13 | 如果对内 / 对老板汇报私域成绩 → 用脱水后的真实 ROI（留存 / 复购 / 真实互动交叉验证），不跟风「瑞幸 90% 私域订单」式头部夸大数据；KPI 不设「唯加粉量」「唯 GMV」 | T03 W7 资深 add + intake ROI 不透明 |

## 候选工具流派分裂 (智识谱系)

| Split | A 派代表 | B 派代表 | 来源 |
|-------|---------|---------|------|
| 工具能多大程度替代人 | 工具派：企微 SCRM 全家桶、句子互动 AI、鉴锋（agency「SOP+自动化可规模化」） | 人本派：徐志斌 / 刘思毅（「私域核心是人对人 / 小群真实关系」） | T01 分歧 + T02 流派分裂 + T04 分歧 2 |
| 私域是「销售放大器」还是「品牌资产」 | 销售派 / 操盘手 IP 派：刘思毅（快变现、强「搞钱」直给） | 复购 / 会员（品牌）派：晏涛 / 完美日记·名创（慢养信任、终点「超级用户」） | T01 分歧 + T04 分歧 1 |
| 裂变玩法的边界 | 增长 / 裂变派：鉴锋、小裂变、各 SCRM 裂变模块（裂变当核心引擎） | 平台 / 合规派：腾讯企微官方、微信公开课（视裂变为高风险灰区） | T01 分歧 + T02 流派分裂 + T04 分歧 3 |
| 私域是专门赛道还是运营整体一段 | 媒体 / 操盘手派：徐志斌（私域无固定真理、靠拆案例 + 操盘手共创） | 运营通识派：黄有璨（脱离运营整体框架单看私域是舍本逐末） | T01 较弱分歧 + T04 谱系 |
| 通用企微 SCRM vs 品牌型 CDP SCRM | 通用派：微伴 / 探马 / 微盛（轻快便宜、服务腰部） | 品牌 CDP 派：群脉（重全贵、服务多渠道中大型品牌） | T02 流派分裂 |
| 「私域」一词定义本身 | —— 定义混乱：有的指企微社群、有的指公众号粉丝、有的指自营 APP 用户 | —— SKILL.md 须先锁定「私域 = WeChat 生态内可直接 1v1/1vN 触达的用户」 | intake warnings + T06 「定义混乱是最大术语坑」 |

> 五大流派（intake）= 品牌派 / 销售派 / 内容派 / 工具派 / 社群派 + 平台派（T04 补充为第 6 派）。各派术语偏好不同（T06 提供语言层硬证据）。
