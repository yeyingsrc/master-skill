#### 维度 1: 平台合规红线与玩法保鲜期核查
- 看什么: 用户打算用的具体玩法 / 动作（裂变形态、自动化加好友、群发频次、朋友圈打卡、个人号多开）踩不踩微信当前红线；该玩法距上次验证多久（> 3-6 个月默认作废）；引用的平台规则条款编号是否仍是官网当前版本。
- 在哪看: 微信外部链接内容管理规范原文（weixin.qq.com/agreement/...）、企业微信用户账号使用规范（work.weixin.qq.com/nl/norm）、企业微信违规场景及处罚规则；微信公开课 PRO / 企业微信生态大会近一年政策发布；不看课程文章和 3 个月前的裂变模板。
- 输出: 「该玩法当前合规状态：白帽 / 灰区（具体踩哪条边缘）/ 黑帽（违反规范第 X 条）；保鲜期评估：还可用 / 需重验 / 已失效；引用条款核对日期 YYYY-MM-DD」。

#### 维度 2: 流量自主权与公转私漏斗诊断
- 看什么: 用户当前流量的所有权归谁（自有 / 平台租用）；公转私漏斗逐环转化率（曝光 → 扫码 → 添加 → 通过 → 破冰留存）哪一环漏损最大；承接 SOP 是否存在（钩子是否兑现、欢迎语是否打标）；是否分渠道活码（能否归因）。
- 在哪看: 用户自己的企微 / SCRM 后台数据（各渠道加粉量与成本、7 日留存率）；T03 W2 公转私漏斗 SOP 的步骤骨架；见实大会的品牌承接案例（比 vendor 中立）；不看「加粉总数」这种单一数字。
- 输出: 「流量自主权评级：高 / 中 / 低（含理由）；漏斗最大漏损环：{引流/承接/触达/转化/复购}，转化率 X%；承接 SOP 状态：完整 / 缺失；归因能力：分渠道可归因 / 一个共享码不可归因」。

#### 维度 3: 品类适配性与前置条件评估
- 看什么: 品类是否适合私域（复购频次 / 客单价 / 决策链长度——高复购、高客单、决策链长更适合）；三个前置条件是否齐备（稳定公域流量来源、能持续产内容的人、客单价能否覆盖私域人力成本）；是否属于「不适合私域的 70% 商家」。
- 在哪看: 用户自己的品类复购数据和成本结构；T03 W1 0→1 冷启动的「品类判断 + 前置条件」资深路径；晏涛《超级用户增长》的品类适配讨论；intake warnings「头部 vs 腰部断层」；不看「人人都能做私域」的课程话术。
- 输出: 「品类适配性：适合 / 勉强 / 不适合（含复购·客单·决策链三维判断）；前置条件缺口：{公域来源/内容产能/人力成本覆盖} 缺哪些；建议：硬启动 / 先补前置条件 / 不建议做私域」。

#### 维度 4: 运营体系成熟度（工具 vs 运营 vs 人）
- 看什么: 用户是「买了工具就以为有私域」还是「有 SOP / 团队 / 标签分层」；池子是死水还是有「蓄水—养鱼—捕鱼」循环；是否过度铺触点 / 过早求全（该用最小版本验证的阶段在做全套）；私域是否绑死在一个人身上（有没有 SOP 文档化）。
- 在哪看: 用户的实际运营动作清单（有没有 1v1 SOP / 社群 SOP / 朋友圈内容表 / 分层动作）；T03 八个 workflow 的「入门 SOP vs 资深路径」对照；T02 避坑「买了 SCRM≠私域」；见实 / 群响语境的「先裸跑再上工具」路径。
- 输出: 「运营体系成熟度：工具阶段（只有工具没动作）/ SOP 阶段（有动作没体系）/ 体系阶段（可复制、不依赖单人）；过度设计风险：有（具体哪里铺太大）/ 无；下一步重点：补 SOP / 拆组织 / 收缩摊子」。

#### 维度 5: 流派定位识别
- 看什么: 用户当前打法的术语组合和价值取向，反推他属于哪个流派——增长/工具派（密集说 SCRM / 自动化 / 中台）、复购/会员（品牌）派（LTV / 用户资产 / 长期价值）、媒体/操盘手派（拆案例 / 操盘手共创）、社群/IP 派（群活跃 / 破冰 / 私域 IP）、平台派（合规优先 / 企微 default）、运营通识派（流量变现链条）；用户的诉求和他实际该走的路是否匹配。
- 在哪看: 用户自己描述需求时用的词（T06 给的「多流派术语偏好」语言层证据）；T01 figures 的流派归属 + T04/T07 智识谱系；不看「我们要做私域」这种没有流派信息的笼统表述。
- 输出: 「用户流派定位：{流派名}（依据：用了哪些术语 + 什么价值取向）；该流派的盲区：{对应分歧里这一派容易忽略的}；建议补充视角：{对立派的哪条主张值得参考}」。

#### 维度 6: 工具栈选型与同质化避坑
- 看什么: 主战场是否企微（不是个人号）；企微 SCRM / 电商 SaaS 是不是只装一个（同质工具别同时上）；工具是否过了「合规资质 + 会话存档」门槛；AI 能力层处于「卖点」还是「标配」阶段；是否把公域数据工具（飞瓜 / 新榜）误当私域工具。
- 在哪看: T02 选型决策树 A/B + 前置硬规则 + 避坑清单；各 vendor 官网（surrogate_primary，需对冲）+ 36氪 / IT之家第三方盘点（带厂商立场）；不看知乎 / CSDN / woshipm 的工具对比（黑名单）。
- 输出: 「主战场：企微 / 个人号（过时）；SCRM 选型建议：{具体一家 + 理由，或『现有够用不必换』}；同质工具冗余风险：有（同时装了 X 和 Y）/ 无；合规门槛：通过 / 不通过；提醒：该结论保鲜期 3-6 个月，需复验」。

## 心智模型

> 6 个。每个经 [extraction-framework § 一](../../../references/extraction-framework.md) 三重验证（跨场景复现 / 生成力 / 排他性）。`figures:` 行列 ≥ 2 个支持该模型的具体 figure；`evidence:` 挂 ≥ 2 个 source_id（来自六轨 Source Manifest，跨 track 优先）。

### 1.1 公私二分 / 流量自主权（mental model）

- **一句话**：私域人看一切流量的底层镜片是「这部分流量是租来的还是自己的」——租来的（公域）用一次付一次、平台控分发，自己的（私域）可反复免费触达；近年这对二分从「对立」演化为「全域协同」，但「自主权」这条轴没变。
- **figures:** 徐志斌 / 腾讯·企业微信官方
- **evidence**: [T04-S014, T04-S006, T01-S002, T01-S023] —— 徐志斌《小群效应》对「私域 = 企业可自主掌控的流量」的定义（T01-S002）+ 见实词汇手册「公域/私域二分」(T04-S014) + 见实 2024 大会圆桌谈「全域团队」(T04-S006) + 企业微信官方「公私域协同」叙事 (T01-S023)。
- **应用方式（when to invoke）**：遇到「投放 ROI 下降 / 流量越来越贵」类问题，先问「这部分流量能不能沉淀成自己的、反复用」，而非继续加投放预算；遇到任何渠道决策，先判断「这个触点的流量所有权归谁」。
- **局限**：① 验证 3（排他性）为 ⚠️ PARTIAL —— 「自有流量 vs 付费流量」是泛营销概念，本模型的排他性在「放大程度」：微信生态把「自主权」具体到「能否直接 1v1/1vN 触达」这个可操作判据，是这一行的特化；脱离微信生态（如 DTC 独立站）这条轴存在但形态不同。② 「全域协同」是 2025-2026 的演化方向，把私域和公域硬切成「二选一」是过时理解。

### 1.2 流量池 / 蓄水养鱼（mental model）

- **一句话**：私域人把流量当成「要蓄进池子、反复经营」的对象而非「这次能转化多少」的一次性曝光——但「建了群 / 加了好友」只是蓄了水，**没有运营动作的池是死水**；价值在「蓄水—养鱼—捕鱼」的循环，不在池子大小。
- **figures:** 范冰 / 晏涛
- **evidence**: [T04-S003, T04-S005, T01-S004, T01-S001] —— 范冰《增长黑客》引入「池」思维 (T01-S004, T04-S003) + 《私域流量池》专门化 (T04-S005) + 晏涛《超级用户增长》「连接 + 促活」即「养」(T01-S001)；T04 流量池词条明示「以为建了群就有池——没有运营动作的池是死水」。
- **应用方式（when to invoke）**：遇到「加了很多好友 / 建了很多群但没产出」，先判断「有没有运营动作」——无动作先补 SOP 再谈规模；遇到「私域池子多大」类汇报，把注意力从「数量」拨到「质量与可触达性」（僵尸好友是负资产，还占企微外部联系人付费额度）。
- **局限**：① 「蓄水」隐喻容易让人「只进不出」地囤好友——必须配「捕鱼（转化）」才闭环。② 这一模型是「工具≠私域」「先有 SOP 再上工具」的母题，但它不告诉你 SOP 怎么写（那是 §4 工作流的事）。③ 对超小规模（几百好友）私域，「池」的工程化反而是过度设计——先验证能不能加到人、加到人有没有用。

### 1.3 精细化分层 / 用户是 N 个分层（mental model）

- **一句话**：私域人默认「用户不是一群人，是 N 个分层」——按生命周期阶段 × 价值 × 意向 × 活跃度多维分群，对不同层用不同动作；**全员一套话术是这一行最高频的死法**。
- **figures:** 晏涛 / 完美日记·名创优品 / 黄有璨
- **evidence**: [T01-S001, T01-S021, T01-S005, T04-S002] —— 晏涛五步法核心步骤「分层」(T01-S001, T04-S002) + 完美日记/名创优品「用户标签体系」「50 亿标签」(T01-S021) + 黄有璨「运营即精细化连接用户」(T01-S005)；W1/W2/W3/W6/W7 五个 workflow 全流程都有打标分层环节。
- **应用方式（when to invoke）**：遇到「私域转化率低 / 用户反感」，先查「是不是全员一套话术」——按标签 / 生命周期分层是第一刀；遇到「该建什么标签」，反推自「我要做哪些差异化动作」，而非先打标签再想用途。
- **局限**：① 分层有成本——中小商家数据量不够、消费链路太短时 RFM 跑不出有效分层，是「看起来专业但用不起来」的典型（T06 RFM 词条）。② 标签维度过多过杂、无对应触达策略 = 死标签，比不打更糟。③ 分层是动态的（用户跨层流动），打完一次不更新就脱离真实用户。

### 1.4 SOP 化 = 把关系当可流程化、可被工具放大的对象（mental model）

- **一句话**：私域人把「关系运营」当成可被流程化（带时间轴的动作序列）、可被 SCRM 自动放大的对象——这是它区别于「纯销售」（靠个人话术天赋）和「纯客服」（被动应答）的地方；但 **SOP 是「下限保障」不是「上限」**，保证新手达及格线，高手在 SOP 之上做个性化。
- **figures:** 鉴锋 / 徐志斌
- **evidence**: [T01-S009, T01-S011, T04-S001, T03-S014] —— 鉴锋把「裂变 / 私域代运营」做成可复用 SOP 产品而非一次性活动 (T01-S009, T01-S011) + 《私域流量》六层框架含 SOP 意识 (T04-S001) + 微伴助手「企微 SOP = 带时间轴、可被 SCRM 自动触发的动作序列」(T03-S014)；T04「1v1 触达 SOP」词条：SOP 是下限保障，高手在 SOP 之上做个性化。
- **应用方式（when to invoke）**：遇到「私域怎么规模化 / 怎么让新人也能做」，先想「哪些动作能固化成带时间轴的 SOP（第 1 天破冰、第 3 天发福利、第 7 天促单）+ 挂进 SCRM 半自动化」；遇到「为什么我的私域离不开某个人」，先 SOP 文档化。
- **局限**：① 验证 3 为 ⚠️ PARTIAL —— SOP 是通用管理概念，本模型的排他性在「放大程度」：私域把 SOP 特化为「可被 SCRM 按标签 / 时间自动触发的动作序列」+「关系」这种本来最反流程化的东西被流程化，是这一行的特化。② SOP 僵化成「全员同一套话定时群发」会直接触发用户反感 / 拉黑——每条话术须留个性化变量。③ SOP 保鲜期只有 3-6 个月（见 1.5），过度精细的 SOP 是包袱。

### 1.5 玩法保鲜期 / 合规半衰期（mental model）

- **一句话**：成熟私域人默认「任何玩法都有半衰期」——3-6 个月前的打法（尤其裂变 / 自动化）默认要重新验证；这一行真正的「硬约束」不是行业标准，而是「平台即裁判」的微信 / 企微规则 + PIPL，且规则滚动修订、常不预告。这种「玩法会过期」的焦虑是私域区别于其他营销领域的独特心智。
- **figures:** 鉴锋 / 腾讯·企业微信官方
- **evidence**: [T01-S009, T01-S023, T06-S001, T06-S002] —— 鉴锋早期赖以起家的「个人号裂变」打法已大面积过时、业务被迫从裂变开发转向代运营 (T01-S009) + 企业微信官方「平台规则即边界」(T01-S023) + 企微账号使用规范禁外挂 / 禁裂变诱导原文 (T06-S001) + 微信外链规范禁诱导分享 / 关注原文 (T06-S002)。
- **应用方式（when to invoke）**：遇到任何「现成裂变 / 自动化 / 引流玩法」，先问「这玩法多新、踩不踩微信当前红线」——3-6 个月前的玩法默认作废重验；做裂变第一步是「合规审查」不是「设计海报」；引用任何平台规则条款编号（如企微规范 3.x）须以使用时官网当时版本为准。
- **局限**：① 衰减的是「具体玩法」，不是「合规优先」这个原则本身——原则稳态，玩法 high decay。② 本模型不告诉你「当前红线具体是什么」（那需要 Agentic Protocol §9.1 实时核查），只告诉你「必须去核查」。③ 对「白帽运营」（企微官方功能 + 用户授权打标 + 靠服务做复购）这条路，保鲜期焦虑较低；它主要约束灰帽 / 黑帽边缘的玩法。

### 1.6 克制与算账（私域是有成本、有红线、数据会骗人的经营）（mental model）

- **一句话**：这一行的资深人和初学者最大的分野是——初学者把私域当「免费流量 + 加粉游戏」，资深人把它当「有成本、有合规约束、数据会注水、ROI 需要脱水核算的经营」；资深判断力一半体现在「不做什么」（拒绝过度铺触点 / 过早求全 / 全量迁移 / 唯加粉量），一半体现在「初学者系统性忽略的那些事」（钩子成本核算、ROI 诚实核算、合规预检、触达频次克制、屏蔽率监测、防数据造假）。
- **figures:** 徐志斌 / 完美日记·名创优品
- **evidence**: [T03-S008, T01-S002, T01-S021, T02-S001] —— T03 八个 workflow 的资深路径 skip / add 项高度集中（「先验证小闭环、拒绝过度设计」+「私域是要算账、有红线、数据会骗人的」，T03-S008 操盘手视角稿）+ 徐志斌「私域大调研」年年重做、不信「固定真理」(T01-S002) + 头部品牌私域 GMV 公认存疑（完美日记「4800 万私域」、名创「50 亿标签」，T01-S021）+ 瑞幸「90% 订单来自私域」类 vendor 数字需打折 (T02-S001)。
- **应用方式（when to invoke）**：遇到「要不要一次性做全 / 做大 / 做完美」，默认「先用最小版本验证跑通再放大」；遇到「私域成绩怎么汇报」，用脱水后的真实 ROI（留存 / 复购 / 真实互动交叉验证），不跟风头部夸大数据；遇到任何「私域是免费流量」「7 天速成」「人人能做私域」的说法，本能地反驳「私域 CAC 不为零、有前置条件、不适合 70% 商家」。
- **局限**：① 「克制」不等于「不作为」——私域最大的死法不是做得不够，是摊子铺太大；但对「该铺的基础（企微主体 + 1v1 承接 + 朋友圈）」也克制就是另一种错。② 「算账」依赖数据，而这一行 ROI 数据结构性不透明（见 §8）——能做到的是「不被表面数字骗 + 用交叉指标逼近真实」，不是「算出精确 ROI」。③ 这一模型对「头部品牌 / 高客单 / 高复购品类」和「腰部 / 低复购品类」的「账」算法完全不同，不可一刀切。



## 标准 Playbook

> 行业内的「快速决策规则」。每条形如「**如果 X**，则 Y」（粗体 + 逗号分隔），含具体案例 + evidence。来源：Step 1 降级的决策启发式 + Track 03 跨 workflow 反复出现的步骤模式 + Track 02 选型决策原则。

1. **如果流量越来越贵 / 投放 ROI 持续下滑**，则先问「这部分流量能不能沉淀成可反复免费触达的私域」，而不是继续加投放预算——把「买一次用一次」换成「买一次反复用」是私域存在的理由。
   - **案例**：见实 / 群响语境里中小商家启动私域的标准触发点就是「公域 ROI 下滑 / 平台流量越来越贵 / 老板要把客户沉淀下来反复触达」（T03 W1 Trigger）。
   - **evidence**: [T04-S014, T03-S008]

2. **如果加了很多好友 / 建了很多群但没产出**，则先判断「有没有运营动作（SOP / 内容 / 人）」，无动作的池是死水——先补 SOP 和团队再谈规模，不是反过来先买工具。
   - **案例**：很多企业买了通用 SCRM，最后发现 80% 功能用不上（T02-S023）；Track 04 canon 反复出现的共识「没有运营动作的池是死水」。
   - **evidence**: [T02-S023, T04-S005]

3. **如果私域转化率低 / 用户反感 / 被屏蔽删好友**，则先查「是不是全员一套话术」，按标签 + 生命周期分层是精细化的第一刀；且标签体系要反推自「我要做哪些差异化动作」——没有触达策略对应的标签是死标签。
   - **案例**：W6 标签+复购+会员 workflow 跳过「分层」这步的后果就是「高价值用户和羊毛党被同等对待，转化率低、用户反感」（T03 W6）；群应用 SCRM「用户标签体系搭建 + 分层运营 SOP」（T03-S004）。
   - **evidence**: [T03-S004, T04-S002]

4. **如果要选私域承接的主战场 / 现有个人号矩阵要不要迁**，则默认企业微信为主战场、个人号为辅助——2024 起已成定局；任何「个人号无脑加好友 / 多开矩阵」式建议直接判过时（撞封号红线、客户不可继承）。
   - **案例**：WeTool 2020-05-25 被微信永久封禁，使用者「完全没有先兆」连带被封，有运营团队一夜 3 万用户失联、3000 个号被封（T02-S019, T02-S020）；W8 个人号迁企微是 2024 起的「历史包袱清算」工程。
   - **evidence**: [T02-S001, T02-S019, T03-S001]

5. **如果要用一个现成的裂变 / 自动化 / 引流玩法**，则先核对微信当前「诱导分享 / 诱导关注 / 外挂」红线，3-6 个月前的玩法默认作废重验——做裂变第一步是合规审查，不是设计海报。
   - **案例**：鉴锋早期赖以起家的「个人号裂变」打法已大面积过时，零一数科被迫从裂变开发一路转向私域代运营 / 视频号代播（T01-S009）；微信外链规范 2018-05 起禁朋友圈打卡返现、持续禁诱导分享（T06-S002）。
   - **evidence**: [T06-S002, T01-S009]

6. **如果私域 ROI 不行**，则不要笼统归因，按「引流 → 沉淀 → 触达 → 转化 → 复购 → 裂变」逐环看转化率，定位漏损最大的一环——尤其先查「承接」（引流和承接是两段，引来 1000 个、加好友通过率 30%、破冰留存 50%，实际只剩 150，承接漏损往往比引流本身更致命）。
   - **案例**：W2 公转私漏斗「只重引流不重承接」失败模式——把 KPI 定成「加粉量」，应改为「加粉后 7 日留存数」（T03 W2）。
   - **evidence**: [T04-S001, T03-S008]

7. **如果要判断「私域值不值得做」**，则用 LTV / 复购视角而非首单 ROI 评判（私域经济模型是「首购可能亏、复购才赚」），且先过品类适配性 + 三个前置条件——品类高复购 / 高客单 / 决策链长才适合；稳定公域流量来源、能持续产内容的人、客单价能覆盖私域人力成本，三者缺一就先补前置条件、不硬启动。
   - **案例**：W1 0→1 冷启动跳过「品类适配性判断」的后果是「投入大量人力但品类本身复购低，做成赔本赚吆喝」；intake 明确「不适合 70% 商家」（T03 W1 + intake warnings）。
   - **evidence**: [T03-S011, T04-S002]

8. **如果面对「要不要一次性把全套做全 / 做大 / 做完美」**，则默认先用最小版本验证跑通再放大——0→1 阶段先用「企微 1v1 + 朋友圈」两件套验证品类和话术，跑通了再加触点 / 加渠道 / 加标签维度 / 做归因体系。私域最大的死法不是做得不够，是摊子铺太大、每个都半成品。
   - **案例**：Track 03 八个 workflow 的资深路径几乎都有这一刀——skip「全触点一次性铺齐 / 全渠道铺开 / 复杂标签体系 / 等体系完美再复制 / 全量迁移」（T03 W1/W2/W6/W7/W8 资深路径）。
   - **evidence**: [T03-S002, T03-S008]

9. **如果要选企微 SCRM 或电商 SaaS**，则同质工具只装一个、别花两周做选型对比——6 家头部企微 SCRM 90% 基础能力相同，微盟 / 有赞功能高度重叠（「重技术 vs 重营销」是定位差异不是互补）；选型纠结的边际收益很低，那两周拿去搭 SOP 更值。
   - **案例**：T02 决策树 B 元规则「6 家头部企微 SCRM 选谁，对私域成败的影响 < 选不选得对主战场 + 有没有运营团队和 SOP」；澎湃 / 36氪「有赞 vs 微盟二选一」（T02-S021, T02-S022）。
   - **evidence**: [T02-S021, T02-S017]

10. **如果选私域工具**，则先过合规门槛（合规资质齐全 + 会话存档能力）再看别的，不看低价、不看功能数量——没有齐全合规资质的产品再便宜也是监管风险；反向信号：对续费率自信的厂商不会自降身价推「多年使用权 + 打折」，看到这种促销要警惕。
    - **案例**：随数据安全法规全面实施，2026 选型逻辑已从「看功能数量」转向「看落地能力 / 合规安全 / 长期价值」；企微违规账号封禁量同比增 47%、私域营销场景占 68%（T02-S023）。
    - **evidence**: [T02-S023, T06-S001]

### 3.1 三层结构（Track 02 retained 16 个，自适应基准说明见下）

- **必备层（5 个）**：① 企业微信（私域 2024+ 的 default 主战场、合规底座，所有 SCRM 长在它的 API 上）；② 企微 SCRM 同质工具群（微伴 / 探马 / 微盛 / 尘锋 / EC / 句子互动——在企微之上加标签 / SOP / 群发 / 渠道活码 / 数据的运营层）；③ 微盟 / 有赞二选一（私域承接交易闭环：小程序商城 + 营销玩法 + 会员体系）；④ 135 编辑器 / 秀米二选一（公众号排版，内容回路基础设施）；⑤ 腾讯文档 / 飞书（团队 SOP 协同载体）。
- **场景特化层（5 个）**：小裂变（裂变窄切口 SaaS）/ 群脉 SCRM（品牌型 CDP SCRM，服务多渠道中大型品牌）/ 句子互动（AI 对话机器人路线）/ 新榜·新视（公众号 + 视频号数据）/ 飞瓜数据（短视频-直播数据，公转私上游）。
- **新兴层（2 个，全是「能力」非「独立产品」）**：AI-native 企微 SCRM 能力（智能打标 / 话术推荐 / 会话总结 / 合规质检）+ 微盟 Weimob Admin Skills / WAI（AI First 套件）。

### 3.2 选型决策树（Track 02 提供 8 节点，前置硬规则 + 决策树 A/B）

- **前置硬规则（不进树、所有分支共享）**：① 主战场默认企业微信，不再问「要不要用个人号」（2024 定局）；② 任何工具选型先过「合规资质 + 会话存档」门槛，没有的再便宜也不选。
- **决策树 A（按核心瓶颈分流）**：Branch 1 刚启动 0→1 → 企微原生功能裸跑；Branch 2 运营动作没法规模化 → 进决策树 B；Branch 3 流量转化不成交易 → 微盟 / 有赞二选一；Branch 4 拉新增长 / 老带新裂变 → 先看现有 SCRM 内置裂变，专门做才上小裂变（强制前置合规核对）；Branch 5 内容触达 / 选号 / 数据 → 新榜·新视 / 飞瓜 / 编辑器分流。

### W1 0→1 冷启动（生命周期主线）

- **skip（资深跳过）**：跳过「全触点一次性铺齐」—— 先用「企微 1v1 + 朋友圈」两件套验证品类话术，跑通再加触点。
- **optimize（资深优化）**：优化 step1 品类判断 —— 不只看品类，还看「三个前置条件」（稳定公域来源 / 能持续产内容的人 / 客单价覆盖人力成本），缺一先补不硬启动。
- **add（资深额外做、入门系统性忽略）**：① 钩子 / 福利的成本核算（0→1 就把「送出去多少钱换一个私域用户」算清）；② 合规红线预检（起盘前确认引流 / 裂变话术不踩当前红线，不抄半年前玩法）。

### W2 公转私漏斗（核心子 SOP）

- **skip**：跳过「全渠道铺开」—— 先押 1-2 个与用户最匹配的渠道，跑出正 ROI 再扩。
- **optimize**：① 优化钩子 —— 用「产品强相关的轻钩子」替代「通用大额券」，宁可加粉慢不要羊毛党；② 优化承接 —— 欢迎语顺势做轻分层提问，把承接和打标合一。
- **add**：承接话术的 A/B 测试（同渠道跑两版引导话术比加粉率，入门只用一版拍脑袋话术）。

### W3 1v1 触达 SOP（核心子 SOP，high decay）

- **skip**：跳过对「明显高意向用户」的标准 7 日节奏 —— 加好友第一句就问价问货的直接进促单话术，不走慢培育。
- **optimize**：把 SOP 当下限不当上限 —— 标准话术保证「不发错」，但每条留个性化变量（带用户上次需求 / 称呼），避免僵化成骚扰式群发。
- **add**：① 触达频次的克制管理（主动控总频次，宁可少发不被拉黑）；② 话术库的版本迭代（定期拿转化数据淘汰无效话术）。

### W4 社群运营 SOP（核心子 SOP）

- **skip**：对「短期目标群（快闪 / 活动群）」跳过长期内容节奏 —— 直接「3-5 天高强度内容 + 转化 + 解散」，不在注定短命的群上投长期人力。
- **optimize**：① 优化内容节奏的「价值占比」—— 控制转化类内容比例（约 ≤ 1/3，经验值），用促活 / 干货养活跃度；② 优化「群分层」—— 按价值 / 阶段建不同群、各用不同 SOP 模板，不一个大群通吃。
- **add**：死群止损（定期盘点群健康度，对救不活的死群果断解散 / 合并，不让僵尸群占运营注意力）。

### W5 朋友圈 SOP（核心子 SOP）

- **skip**：跳过「逐条原创」—— 建素材库 + 模板（人设类素材可批量囤），精力集中在产品 / 活动类关键圈。
- **optimize**：① 优化内容比例 —— 压低产品硬广占比、抬高用户价值 + 人设内容占比；② 优化「分组可见」—— 按用户标签分组发圈，减少无效曝光和屏蔽。
- **add**：屏蔽率监测（定期自查 / 用工具看朋友圈触达情况，某类内容掉触达就调整；入门不知道自己已被大面积屏蔽）。

### W6 标签 + 复购 + 会员（生命周期主线，1→10）

- **skip**：跳过「一步到位的复杂标签体系」—— 先用「能直接指导动作」的少数核心标签（生命周期阶段 + 价值层），跑顺再加。
- **optimize**：① 优化 RFM 阈值 —— 按自己品类真实购买周期校准，不套搬零售通用阈值；② 优化会员体系的「可感知性」—— 做成用户「算得清、感受得到」的，不是复杂积分游戏。
- **add**：标签的动态更新机制（确保标签随用户行为实时 / 高频更新，不是打完一次就静态不动）。

### W7 规模化 / 团队 / 归因（生命周期主线，10→100）

- **skip**：跳过「等体系完美再复制」—— 先用「够用的归因口径」让复制跑起来，在复制中迭代体系。
- **optimize**：优化 KPI 的「防造假设计」—— 私域数据极易注水，KPI 设计就规避「唯加粉量 / 唯 GMV」单一指标，用留存 / 复购 / 真实互动交叉验证。
- **add**：① 诚实的 ROI 核算（对内 / 对老板用脱水后的真实 ROI 沟通，不照搬头部夸大数据）；② 合规体系化（把「玩法保鲜期 / 红线变动」做成团队级定期审查机制）。

### W8 个人号矩阵迁企微（一次性迁移工程）

- **skip**：跳过「全量迁移」—— 按盘点只迁高价值 / 活跃客户，长尾泛粉直接放弃，不为「迁移率」虚荣指标耗成本。
- **optimize**：优化迁移引导的「给理由」—— 引导不是「请重新加我」，而是「企微侧有个人号没有的实在好处」（专属权益 / 更好服务），让客户有动力主动迁。
- **add**：① 迁移前的合规与规则预读（吃透企微继承机制的分配次数 / 保护期 / 标签同步范围限制）；② 迁移节奏控制（分批小步迁、每批观察流失率再调话术，不一夜全迁）。



## 表达 DNA

> 模拟「这一行的资深人聚一起讨论时的 register」——多人融合，不模拟某个具体 figure。来源：Track 06 outsider-tell + 厂商话术拒绝 + Track 01 长访谈 register。

### 5.1 高频用语 / 黑话（秒懂 = 内行，听不懂 / 用错 = 外行）

- **公转私 / 私转公**：描述流量流向的基本动词，不说「引流到微信」这种外行话。
- **触点 / 触达**：严格区分载体（触点，名词静态）和动作（触达，动词），且「触达」隐含「用户实际看到」——发了没看到不算有效触达。
- **SOP**（1v1 SOP / 社群 SOP / 朋友圈 SOP / 公转私 SOP）：念字母 S-O-P，默认指「带时间轴、可被 SCRM 自动触发的动作序列」，不是静态文档。
- **裂变**（任务宝 / 拼团 / 砍价 / 分销 / 邀请有礼）：判据是「自传播闭环」（A 拉的 B 又能拉 C），且开口就带「合规吗、保鲜期多久」的本能。
- **企微 / 个人号**：用「企微」简称，默认企微为主战场——还在主推「个人号矩阵」暴露停留在 2022。
- **承接 / 蓄水**：把「引流」和「接住 / 储备」拆成独立环节。
- **打标签 / 分层 / 千人千面**：标签是颗粒度、分层是动作，强调「死标签」「动态分层」。
- **复购 / 留存 / LTV**：谈成绩谈复购和 LTV，不谈加粉数；「加了就死」是行话。
- **钩子 / 诱饵 / 羊毛党**：谈拉新必谈钩子精准度，警惕「钩子越猛留存越差」。
- **操盘手**：圈层身份认同词，对「速成操盘手」有强烈鄙视链。
- 其他高频：漏斗（每层算转化率、找漏损最大的那层）、破冰、KOC、私域流量池、屏蔽率 / 删粉率、会话存档、渠道活码。

### 5.2 严肃 register（长 podcast / 大会 talk 的口气，非 Twitter 短稿）

- **口语化、案例驱动、不端着**：见实大会 / 《温柔一刀》里资深人讲话高度口语化，靠「我操盘 XX 时遇到的具体情况」说话，不堆理论。
- **谈分歧不回避**：内行讨论「私域该多激进」「工具能不能替代人」「裂变该不该做」时直接亮立场，不和稀泥。
- **数字带 caveat 的本能**：听到亮眼私域 GMV，第一反应是「归因口径是什么、含不含自然复购」，不是「哇好厉害」。
- **不学术**：除非引上游概念（AARRR / RFM）会标「这是从增长黑客 / CRM 借的」，不自创学术黑话。

### 5.3 内 vs 外沟通差异

- **对内（同行间）**：用「企微 / SOP / 公转私 / 钩子」缩写和黑话，默认对方懂「死群率」「加了就死」「羊毛党」这些行业现实，直接谈漏损 / 保鲜期 / 红线。
- **对外（解释给非从业者 / 甲方老板）**：把「私域」展开成「不用每次买广告就能直接联系到的自家客户」，把「SOP」说成「第几天发什么、谁来发的标准流程脚本」，并主动管理预期——「这不是免费流量」「不适合 70% 商家」「玩法 3-6 个月会过期」。

### 5.4 外行破绽 top 10（insider-only spotting tells，Track 06 直接用）

1. 把「公众号 / 抖音 / APP 粉丝」都叫私域（真私域 = 能直接、免费、反复 1v1 触达；其余是弱连接 / 伪私域）。
2. SCRM 念成「scrum」/ 当成传统 CRM（SCRM 重心在社交触达和内容运营，传统 CRM 重心在销售漏斗和客户档案）。
3. 「私域 = 建群」「社群 = 微信群」（私域是触达体系，社群是有运营目标的用户组织）。
4. 还在教「个人号无脑加好友 / 多开」（2024 后企微为主，个人号自动化 = 封号红线）。
5. 把「7 天速成私域操盘手」当真 / 觉得「考个私域运营师证」有用（本行无任何有公信力的执业认证）。
6. 汇报只看「加粉数」，不看留存 + 复购 + LTV。
7. 把外挂 / 多开当「行业常规工具」（企微规范明文定义并禁止，是黑帽）。
8. 「触点」「触达」混用。
9. 以为企微加客户无限免费（外部联系人超 2000 需按规模付费，来源 T06-S062 外部联系人词条；具体阈值以企微官网当时版本为准）。
10. 把私域 GMV 当利润 / 当真实数据（GMV 是流水，头部公开数据普遍被质疑夸大）。

### 5.5 行业拒绝的厂商话术 top 5（拒绝什么词 = 表达 DNA 的另一半）

1. **「装了 SCRM = 做了私域」** —— SCRM 厂商最常见的收编话术。拒绝它反映价值观：**运营 > 工具**。
2. **「7 天搭建私域 / 月入 5 万 / 人人能做私域」** —— 课程营销话术。拒绝它反映价值观：**私域有前置条件、不适合 70% 商家、玩法保鲜期只有 3-6 个月**。
3. **「私域是免费流量 / 零成本获客」** —— 把「触达免费」偷换成「获客免费」。拒绝它反映价值观：**引流、运营、企微外部联系人额度都有成本，私域 CAC 不为零**。
4. **平台方把站内粉丝包装成「平台私域」（抖音私域 / 支付宝私域）** —— 拒绝它反映价值观：**真私域的判据是流量所有权 + 能否直接反复 1v1 触达，不是「有个粉丝列表」**。
5. **「私域 1.0/2.0/3.0」「私域中台」「数字化营销中台」等 X.0 / 中台叙事** —— 厂商 PR 和课程高频包装。内行用它做叙事但不当严格标准（disputed），反映价值观：**警惕被滥用的宏大概念，看实操**。

### 5.6 对话样本库（industry voice 实战语料）

> **⚠️ 重要诚实标注**：Track 01 figures 轨的 voice_samples **绝大多数是「转述」，几乎没有「原话」** —— 私域行业 figure 的代表性长内容（长访谈 / 长 talk / 原话片段）几乎全在微信公众号（黑名单）+ 付费社群 / 付费课程（半闭源）里，本调研按黑名单规则不收录公众号 URL，导致「客户解释样本」「同业讨论样本」这两个最能体现 voice DNA 的 register 在原始材料里基本只有「转述」。下方样本均如实标注来源与「原话 / 转述 / 推断」。**voice_confidence = low**（计算见末尾）。

#### 5.6.1 客户版（面客解释 / 教育非从业者）

- 「你别把公众号粉丝、抖音粉丝都当成私域。真私域的判据就一条——你能不能不花钱、直接、反复地一对一联系到这个人。公众号推送有频次限制、打开率常常不到 5%，那最多算个弱连接。」（source: T06-S004 见实词汇手册「私域」词条 + T01-S002 徐志斌私域定义，转述，客户场景：纠正甲方对「私域」的理解；「打开率不到 5%」为业内估计，见 §8.2）
- 「做私域不是不花钱。触达是免费的，但你把人引到企微要成本、运营维护要人力、企微外部联系人超过 2000 还要按规模付费。所以私域的账要算 LTV 比 CAC，不是看加了多少粉。」（source: T06-S042 CAC 词条 + T06-S062 外部联系人词条，转述，客户场景：管理老板对「私域 = 免费流量」的预期）

#### 5.6.2 同业版（私下 / 内训 / 同业访谈）

- 「私域不是把人加进来就完了。是 IP 化建人设、连接建关系、促活、再分层，最后才是复购，五步缺一不可——多数人只做了第一步加好友。」（source: T01-S001 晏涛《超级用户增长》五步法则，转述，同业场景：讲私域复购方法论）
- 「裂变不是做一场活动，是把一套玩法做成可复用的产品，每个行业线套不同的 O+O 增长模型。但裂变保鲜期短、合规风险高，所以我们一路从裂变开发往私域代运营、视频号代播转。」（source: T01-S009 + T01-S011 鉴锋，转述/推断，同业场景：agency 视角谈裂变工程化与业务演进）
- 「用户在 4-6 人的小群里最活跃，大群是死的、小群才有传播力。」（source: T01-S003 徐志斌《小群效应》核心论点，转述，同业场景：谈社群运营的小群效应）

#### 5.6.3 监管 / 专业版（公开场合谈标准 / 平台规则 / 数据）

- 「微信外部链接不得通过利益诱导用户分享、传播外部链接或关注公众号；企业微信不得使用未经许可的外挂、插件进行自动化操作。」（source: T06-S001 / T06-S002 平台规则原文，转述自规则原文，专业场景：解读平台合规红线）
- 「超级用户不等于买得多的人。他得同时满足『高忠诚高复购』和『能影响他人』两个条件，少了后一半只是大客户、不是超级用户。」（source: T01-S001 晏涛《超级用户增长》，转述，专业场景：定义 KOC / 超级用户标准）
- 「私域不是一个孤立的新技能，它是『流量—变现』整条链路里的一段，脱离运营整体框架单看私域玩法是舍本逐末。」（source: T01-S005 黄有璨《运营之光 3.0》私域章定位，转述，专业场景：把私域放回运营整体坐标系）

#### 5.6.4 反例版（这一行的资深人绝不会这样说的话）

- 「装了我们的 SCRM，你就有私域了。」（source: T06 厂商话术拒绝 #1，why 反例：把工具等同于运营体系——SCRM 是底座，运营动作 SOP / 内容 / 人才是内容，80% 功能会闲置。运营派对工具派的核心反驳。）
- 「7 天搭建私域，人人都能做，私域操盘手月入 5 万。」（source: T06 厂商话术拒绝 #2 + intake warnings，why 反例：私域有品类适配性和前置条件、不适合 70% 商家、玩法保鲜期只有 3-6 个月——内行听到「速成」会直接判定对方是被课程营销污染或在割韭菜。）
- 「我们私域 GMV 占比 90%，私域 ROI 1:10。」（source: T06-S029 GMV 词条 + 瑞幸案例 T02-S001，why 反例：内行听到亮眼私域 GMV 第一反应是「归因口径是什么、含不含自然复购」——头部品牌公开私域 GMV 业内普遍认为有夸大，GMV 是流水不是利润。）

## 质量基准 + 反模式

### 6.1 什么算「好」（质量基准，可验证）

1. **闭环跑得通且每一环可归因**：好的私域不是「加了多少粉」，是「引流 → 沉淀 → 触达 → 转化 → 复购」每一环都有转化率数据、能定位漏损最大的一环；公转私用分渠道活码（不是一个共享码）才能归因。来源：T03 W1/W2 资深路径 + T04 漏斗概念。
2. **私域价值用 LTV / 复购衡量，不是首单 ROI 或 GMV 表面数**：好的私域经济模型是「首购可能亏、复购才赚」，用 LTV:CAC 比值判断健康度；ROI 数字是脱水后的真实数（留存 / 复购 / 真实互动交叉验证），不跟风头部夸大数据。来源：T06 LTV/复购/GMV 词条 + T03 W7 资深 add。
3. **SOP 是「下限保障 + 留个性化变量」，不是「全员同一套话定时群发」**：好的 1v1 SOP 保证新手达及格线，同时每条话术留个性化变量（带上用户上次说过的需求 / 称呼），并严控触达频次（宁可少发不被拉黑）。来源：T04 1v1 SOP 概念 + T03 W3 资深路径。
4. **标签体系反推自「要做哪些差异化动作」**：好的标签是「能直接指导动作」的少数核心标签（生命周期阶段 + 价值层），不是埋头打几百个用不上的死标签；RFM 阈值按自己品类的真实购买周期校准、不套搬零售通用阈值。来源：T06 标签/RFM 词条 + T03 W6 资深路径。
5. **默认白帽，灰帽明确标注、黑帽只作反模式陈列**：好的私域走「企微官方功能 + 用户授权打标 + 合规裂变 + 真实内容种草 + 靠服务和信任做复购」；判据 = 不违反微信平台规则 + 不违反 PIPL + 不欺骗用户。来源：T06 白帽/灰帽/黑帽区分。

### 6.2 反模式（外行 / 入门常犯错，Track 06 outsider-tell + Track 02/03 失败模式）

1. **用个人微信号外挂（自动加好友 / 多开 / 批量群发 / 自动回复）做私域**——WeTool 标本：2020-05-25 被永久封禁，有运营团队一夜「3 万用户失联、3000 个号被封」(T02-S019, T02-S020，媒体报道口径)。任何「模拟用户手动操作」的程序都被微信定义为外挂，挂在个人号还是企微上都是黑帽。
2. **以为「买了 SCRM 就有私域」**——工具是底座，运营动作（SOP / 内容 / 人）才是内容；很多企业买了通用工具发现 80% 功能用不上。先有 SOP 和团队，再上 SCRM。
3. **0→1 阶段一次性铺齐全触点**——同时做企微 + 视频号 + 小程序 + 公众号 + 社群，结果五个半成品、没有一个验证过的转化闭环，浪费 1-2 个月。应先用「企微 1v1 + 朋友圈」两件套验证。
4. **只重引流不重承接 / KPI 定成「加粉量」**——举例说明漏损叠加（数字为示意、非实测）：引来 1000 个、加好友通过率 30%、破冰留存 50%，实际只剩 150；承接漏损往往比引流更致命。KPI 应改为「加粉后 7 日留存数」。
5. **1v1 SOP 僵化成骚扰式群发 + 不分层热客户被养凉**——把 SOP 理解成「全员同一套话定时群发」会触发用户反感 / 拉黑；对加好友第一句就问价的高意向用户还走 7 天慢培育，会把热的人养凉。
6. **建群即死群 / 广告群**——没定位、没规范、没角色、没内容节奏的群迅速劣化；转化类内容占比过高（经验值：应控制在约 1/3 以内，来源 T03 W4 / T03-S013）把群发死；攒一堆僵尸群不止损，占用运营注意力。
7. **朋友圈高频硬广刷屏被屏蔽 + 多账号同素材被判批量**——发圈频率经验值是每天 3-5 条为佳（来源 T03 W5 / T03-S005），高频硬广（如 8 条以上全是产品广告）会让用户大面积屏蔽（被屏蔽 = 这个触点对该用户永久死亡）；多个矩阵号同发一套素材被微信判定为批量营销、触发限流。
8. **直接套搬零售 RFM 阈值不结合品类购买周期 + 会员体系做成复杂积分游戏**——快消品和耐用品的「沉睡」定义天差地别，套搬阈值导致召回活动打偏；会员机制要「用户算得清、感受得到」，不是自嗨的复杂积分。
9. **私域绑死在一个人身上 + GMV 归因口径放水**——没拆组织、没 SOP 文档化，人走即崩；GMV 归因口径混乱或刻意注水，要么说不清贡献被砍预算，要么虚报被质疑。
10. **跟风「人人都要做私域」硬上品类不适配的项目**——低复购、低客单品类硬投私域人力，ROI 永远算不正，做成赔本赚吆喝。私域不适合 70% 商家，启动前先过品类适配性 + 三个前置条件。

### 7.2 核心分歧（still-debated，给完全相反建议）

1. **私域是「销售放大器」还是「品牌资产」**？销售 / 操盘手 IP 派（刘思毅）要激进话术、快变现、强「搞钱」直给；复购 / 会员（品牌）派（晏涛、完美日记·名创）要慢养信任、终点是「超级用户」。同一个「私域话术该多激进」给完全相反建议。
2. **工具能多大程度替代人**？工具派（鉴锋的 agency 视角、各 SCRM 厂商、句子互动 AI）信「精细化 SOP + 自动化 + AI 跑一切」可规模化；人本派（徐志斌、刘思毅）信「私域的核心是人对人 / 小群里的真实关系，工具只降下限」。AI SCRM 化（2025-2026 主线）让这个分歧更尖锐。
3. **裂变玩法的边界**？增长 / 裂变派（鉴锋、小裂变、各 SCRM 裂变模块）把裂变当核心引擎、追求拉新成本最低；平台 / 合规派（微信官方、企微生态大会）视激进裂变为「诱导分享红线」要封号。同一个「任务宝该不该做」给完全相反答案。
4. **私域是「需要持续拆案例的专门赛道」还是「运营整体框架里的一段」**？媒体 / 操盘手派（徐志斌）靠「私域大调研 + 年度大会」持续输出一手案例、认为私域无固定真理；运营通识派（黄有璨）认为脱离运营整体框架单看私域是舍本逐末。（较弱分歧）
5. **通用企微 SCRM vs 品牌型 CDP SCRM**？通用派（微伴 / 探马 / 微盛）轻快便宜、服务腰部商家；品牌 CDP 派（群脉）重全贵、服务有多渠道的中大型品牌。这是「复购/会员（品牌）派」与「增长/工具派」在工具侧的分野。
6. **私域是否人人能做**？canon 普遍鼓吹「人人私域」，但 intake 和行业实况显示「头部 vs 腰部断层」「不适合 70% 商家」——**这是 canon 本身的盲区**，本 skill 在 §8 诚实边界对冲。

### 7.3 一个未抹平的术语分歧

「私域」一词本身定义混乱——有的指企微社群、有的指公众号粉丝、有的指自营 APP 用户；平台方（抖音 / 支付宝 / 小红书）把站内粉丝包装成「平台私域」，业内普遍称之为「伪私域」。**本 skill 锁定「私域 = WeChat 生态内可直接 1v1/1vN 触达的用户」**（见文首定义），但须诚实告知读者：这个锁定是本 skill 的取舍，行业内并无统一定义。



## 诚实边界

> 信息截止 2026-05-14。以下局限是这一行的**结构性问题**，不是调研不努力——如实写明。

### 8.1 一手来源占比 46.6%，低于 50%——结构性而非调研缺漏

- 全六轨 manifest 共 103 条 source，verified_primary / surrogate_primary 中真正算「一手」的约 48 条，**一手占比 46.6%**。
- **根因（结构性）**：这一行核心内容的主阵地是 **微信公众号（信源黑名单）+ 付费社群 / 付费课程（半闭源）**。见实 / 群响 / 鉴锋 / 怒马 / 操盘手手记的代表性长内容、操盘手第一人称复盘、客户场景原话——几乎全在公众号长文 + 知识星球 + 付费课里，按调研黑名单规则不收录公众号 URL。能进 manifest 的「一手」很有限：出版书的 douban / weread entity 页、见实大会官方页、企微官方文档、vendor 官网（surrogate_primary）。
- **分轨更极端**：Track 03 workflows 一手占比仅约 13%（15 条中 verified_primary 仅 2 条）；Track 05 sources verified_primary 为 0。Track 02 工具一手占比按「vendor docs 计入」约 59%，但**全部是厂商自己的官网**，不是中立的行业 survey 或操盘手实测。
- **对本 skill 的影响**：SOP「步骤骨架」「核心概念」「工具品类」因跨多个二手来源高度一致而可信；但「具体话术模板」「量化效果数字」「工具功能对比」单源不可尽信，已在引用处标注。**这不是可以靠『再多搜一轮』补上的——是行业知识载体本身的结构性特征。**

### 8.2 ROI / 数据不透明是结构性的

- 头部品牌公开私域 GMV（完美日记「4800 万私域」、名创「50 亿标签」、瑞幸「90% 订单来自私域」）**业内普遍认为有夸大**（intake warnings 点名）；中小商家私域真实 ROI 数据极少公开；SCRM 厂商的「服务 N 万企业」「客户意图识别准确率 99.2%」是 vendor / 测评口径，无一可独立验证。
- 本 synthesis 所有 `%` / 具体数字 / 价格均带 source_id 或 caveat：如「企微违规账号封禁量同比增 47%、私域营销场景占 68%」(T02-S023，第三方转述腾讯安全口径，未见官方原文，可信度 medium)、「公众号打开率常低于 5%」(T06 公众号词条，业内估计)、「群响社群约 ¥2000/年、私董约 ¥5 万」(T05-S001 vendor 自报)。**引用任何私域 ROI 数字必标口径，头部品牌数字默认打折看。**

- **figures 维度**：真·私域专门 + 有自洽思想体系 + 可调研的 figure 只有 5 个（徐志斌 / 鉴锋 / 黄有璨 / 刘思毅 / 晏涛），其余 6 个是机构型（腾讯企微官方）/ 案例型（完美日记·名创）/ 上游跨界型（范冰 / 刀姐 / Spenser）/ 纯占位（怒马，一手取证为零）。

- **全中文，海外无对应物**：WhatsApp Business / LINE OA / Discord 概念接近但生态完全不同，强行迁移会失败（intake warnings）。本 skill locale = zh-CN，不假装有跨语言覆盖。

### 8.5 时效衰减 + update 建议

- **衰减最快的模块（每 3-6 月建议 update）**：工具栈（§3）+ 工作流（§4）。其中 W3 1v1 触达 SOP 标 high decay、12 个月内大概率显著过时（AI SCRM 正在改写）；「哪家 SCRM / 电商 SaaS 最好用」结论保鲜期仅 3-6 个月；AI 能力层 6-12 个月从「卖点」沉淀为「标配」。
