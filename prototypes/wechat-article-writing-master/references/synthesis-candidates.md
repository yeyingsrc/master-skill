## 候选 Agentic Protocol 维度 (target 5-7 互斥)

| 维度 | 推导自哪个 mental model | 输出 |
|------|------------------------|------|
| 「这个选题 在我的号 fit 不 fit」 | 我的号 fit 校验 + voice 壁龛 + 账号-as-公司 | yes/no + 角度 |
| 「这一议题 当前微信生态热度」 | 数据是反馈 + 候选 + 自检 | 蹭与否 + 差异化角度 |
| 「同 vertical TOP 5 KOL 已写过的视角」 | voice 壁龛 + 长期 IP | 我的 unique 视角 |
| 「voice + 结构 选择」 | voice 壁龛 + 候选 + 自检 | voice + 结构 + 大纲 |
| 「平台规则自审」 | 平台是地基 + 短期套利反噬 | 是否安全 + 修补点 |
| 「变现路径校准」 | 内容是漏斗 + 商业化多元 | 商业 placement |
| 「时效 + 复用 / 系列化」 | 节奏 hard-stick + 长期 IP | 系列化与否 + 重发节奏 |

7 个候选; 互斥性自检 — 「voice + 结构选择」 vs 「同 vertical 已写过」 可能重叠 (后者输入前者), 但前者输出 = voice / 结构 / 大纲, 后者输出 = unique 视角, 视角是 INPUT 到 voice/结构, 不重叠. 「时效 + 复用」可能弱, 与 voice 选择无 overlap (前者是分发节奏决策, 后者是写作骨架决策). 保留 5-6 个进 §9.

最终拟 5 维度入 §9: (1) 选题 fit (2) 议题热度 + 同行扫描 (3) voice + 结构选择 (4) 平台规则自审 (5) 变现 + 时效校准 (合并原 6 + 7, 都是分发后链路决策).
