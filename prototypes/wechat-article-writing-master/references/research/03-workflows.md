## A. 单篇 SOP — 一篇推文 0 到 推送

> 7 stages, 共 38 条 actionable steps. 每 stage 含: 子步骤 / 工具 / 校验 / 失败信号.

### Stage A1. 选题决策 (5 子步骤)

**Trigger**: 今天 (或本周) 要排发一篇推文 / 编辑会上选题缺位.
**Output**: 1 个 confirmed 选题 + 角度 + 选题模板归类 + 头条 / 次条 定位.
**典型耗时**: 入门 1-2 h / 资深 15-30 min.

1. **A1.1 双轨候选 brainstorming**: 同时跑两路 — **自上而下** (热点榜 / 节日 / 政策 / 产品发布) + **自下而上** (后台留言精选 / 后台搜索词 / 私域群讨论 / 朋友圈反馈). 入门 SOP 各列 5-10 个候选, 资深通常自下而上权重 > 自上而下 (evidence: [T03-S033, T03-S034, T03-S035]).
   - **跳过 后果**: 只走自上而下 → 蹭热点但读者不买账 (常见新手坑); 只走自下而上 → 错过破圈机会.
2. **A1.2 热度 + 时效 检验**: 候选选题用 **微信指数 + 百度指数 + 微博热搜 + 抖音热榜 + 知微事件库** 交叉验证 — 若全平台都已经发酵 ≥ 24 h, 公众号窗口几乎错过 (公众号是「慢平台」, 不抢首发) (evidence: [T03-S001, T03-S043]).
   - **跳过 后果**: 写完发现热点已过 / 大号已霸屏 → 阅读漏斗弃.
3. **A1.3 「我的号 fit」校验**: 候选选题对照 **(a) 我的号定位** + **(b) 我读者画像** + **(c) 我的 voice / 主张** — 三层都过才进入下一步. 这一步是新手最常省的 (90% 新手坑) (evidence: [T03-S033, T03-S038]).
   - **跳过 后果**: 内容与读者脱节 → 取关率上升, 算法画像被污染, 后续推荐流也受损.
4. **A1.4 选题模板 归类**: 把候选选题归类到模板 — **时效热点 / 经典翻新 / 反差观点 / 数据解读 / 案例拆解 / 工具教程 / 人物深度 / 系列连载**. 不同模板对应不同结构 + 不同标题策略 (evidence: [T03-S034, T03-S036]).
   - **跳过 后果**: 写到一半才发现「这选题应该用 case study 结构, 现在写成评论了」, 返工成本高.
5. **A1.5 头条 / 次条 / 三条 排位**: 一次推送多条时, 头条 = 主打选题 (流量预期最高), 次条 = 类目挖深 / 系列连载, 三条 = 试错位 (新方向 / 数据型). 头条吃 70-80% 流量, 不要把好选题挤头条 (evidence: [T03-S001, T04-S013 cross-ref]).
   - **跳过 后果**: 4 个好选题挤头条 → 次条永远没数据, 永远不知道哪个能爆.

**资深路径差异**:
- **skip A1.1 上半** (自上而下): 已建立稳定读者池的资深号, 90% 选题来自自下而上 + 自有素材库, 几乎不蹭外部热点 — 因为热点选题对账号定位的反噬大于流量收益 (evidence: [T03-S005, T03-S033]).
- **optimize A1.3**: 资深用「我读者会在朋友圈转吗 / 留言会说什么」内化为 30 秒判断, 不再列表逐条对照.
- **add A1.6**: 资深会做 **「假装写完了」的预拟**, 在选题阶段先写 3 行 imagined-headline + imagined-opening, 卡住即枪毙, 不进入写作 (evidence: [T03-S030, T03-S043] 半佛 / 刘飞 提及预拟法).

**关键工具 (Track 02 关联)**: 微信指数 / 百度指数 / 微博热搜 / 抖音热榜 / 知微事件库 / 新榜热榜 / 公众号后台「数据 → 用户分析 → 来源关键词」.

**关键人物 (Track 01 关联)**: 刘润 / 万维钢 系列代表「自下而上」+ 长期主义选题; 半佛仙人 代表「热点 + 段子化深度」复合; 咪蒙 (反面) 代表「极端热点 + 情绪化」选题.

**Last_updated**: 2026-05 (新榜 2024 复盘 + 36kr 2025 推荐流改版).
**Decay risk**: medium — 选题决策框架稳定, 但热点 surface 工具 (微信指数 vs 抖音热榜权重) 12 月内会调整.

### Stage A2. 标题工程 (5 子步骤)

**Trigger**: 选题 confirmed.
**Output**: 1 个 final 标题 + 同候选标题库 (3-10 条) 留档复盘.
**典型耗时**: 入门 30 min-1 h / 资深 10-20 min.

1. **A2.1 起 3-10 个候选标题**: 强制下限 3 个, 入门期推荐 ≥ 8 个. 起标题时分别对应不同模板: **数字 / 反差 / 悬念 / 利益 / 情绪 / 名人借势 / 反问 / 列表 / 数据 / 故事化** (10 种基本范式) (evidence: [T03-S034, T03-S035]).
   - **跳过 后果**: 只起 1 个 → 锚定第一直觉, 错过更好的角度.
2. **A2.2 候选标题自检 — 信息标 vs 标题党边界**: 每个候选问 4 问 — (a) **能 deliver 吗** (内容能兑现这个承诺?), (b) **会被监管定性「不实信息 / 标题党」吗** (evidence: [T03-S012, T03-S022, T03-S023]), (c) **会被读者举报 / 取关吗**, (d) **复制到信息流卡片预览, 前 18-22 字能完整露出吗** (公众号信息流卡片标题字符限制).
   - **跳过 后果**: 标题党触发限流 / 删文 / 处罚, 严重的封号 (咪蒙 reference, 见反模式).
3. **A2.3 候选 ranking + 选 1**: 从候选库选 1, 决策依据 — (a) **打开率预期** (信息流场景假想测试), (b) **完读 / 在看一致性** (标题鼓励的预期是否能在文中兑现), (c) **账号长期 voice 一致性** (这个标题像我吗?) (evidence: [T03-S033]).
   - **跳过 后果**: 凭感觉选 → 数据复盘时无对照组.
4. **A2.4 准备摘要 + 封面 + 标题三件套**: 信息流卡片是 **「封面图 + 标题 + 摘要 (前 50-80 字)」** 三位一体, 不能割裂处理 — 摘要前 30 字是「标题之外的钩子」, 不要写「本文将介绍」之类的元话术 (evidence: [T03-S004, T03-S007, T03-S034]).
   - **跳过 后果**: 摘要拉跨 → 标题再好也救不回打开率.
5. **A2.5 A/B 假测试 (公众号无原生 A/B, 用 next 篇反推)**: 公众号原生不支持单篇 A/B, 资深做法 — 留下未选用的 1-2 个候选标题, 下一篇同类目选题时换不同模板, 跑 3-5 篇后对照打开率反推自己的标题模板偏好 (evidence: [T03-S043, T03-S044]).
   - **跳过 后果**: 标题永远凭感觉, 不积累自己的标题模板库.

**资深路径差异**:
- **optimize A2.1**: 资深从「起 N 个」变成「先想清楚要哪个情绪 / 角度, 再起 1 个 final + 1 个 backup」, 起的少但更准.
- **add A2.4'**: 资深会同步在标题中**留「金句钩子」**— 标题里就埋一句可截图传播的句子, 双重 amplify.
- **skip A2.5** (新号): 流量小时 A/B 反推无统计意义, 直接靠直觉 + 复盘头部账号标题库.

**关键工具**: 公众号后台「群发预览」/ 西瓜数据 / 新榜爆款标题库 (vendor 看板).
**Decay risk**: medium-high — 标题党边界随平台监管 + 网信办专项行动调整, 2023-2025 共 3 次专项收紧 (evidence: [T03-S021, T03-S022, T03-S023]).

### Stage A3. 开篇钩子 (前 50-150 字) (4 子步骤)

**Trigger**: 标题敲定, 准备落笔.
**Output**: 1 段 50-150 字开篇, 信息流卡片 + 文章首屏使用.
**典型耗时**: 入门 30-60 min / 资深 10-20 min.

1. **A3.1 钩子模板 选 1**: 从 8 种钩子模板选 1 — **数据冲击 / 反常识断言 / 个人故事 / 痛点共鸣 / 悬念抛出 / 借势热点 / 名人引用 / 问题清单** (evidence: [T03-S033, T03-S034]).
   - **跳过 后果**: 用「最近大家都在讨论...」「在这个 X 的时代」「我们都知道」这类元话术开头 → 信息流卡片露出元话术 → 0 打开 (常见新手坑).
2. **A3.2 前 50 字硬约束**: 公众号信息流 / 摘要默认露 前 50-80 字 (设备而异), 这 50 字必须独立成钩子, 不能是「需要看到第二段才理解」的伏笔 (evidence: [T03-S004, T03-S034]).
   - **跳过 后果**: 信息流卡片露出的是非钩子段, 浪费 inventory.
3. **A3.3 兑现 + 跳读测试**: 钩子写完后做两个 check — (a) **兑现**: 这个钩子在文中第 1-2 屏给出兑现 (悬念 → 揭秘, 痛点 → 共鸣描述, 数据 → 数据来源), 不能拖到尾部; (b) **跳读**: 把开篇 50 字单独给陌生人看, 问「你会继续读吗 / 你期待看到什么」, 反向校准.
   - **跳过 后果**: 钩子张力大但兑现晚 → 完读率断崖, 影响推荐流分配.
4. **A3.4 与标题去重 + 协同**: 开篇不能复述标题, 但要与标题 voice 一致 — 标题是冷峻数据 + 开篇是煽情故事 = 风格断裂 (evidence: [T03-S033]).
   - **跳过 后果**: 读者点开发现「不是标题答应的那个调」→ 跳出.

**资深路径差异**:
- **optimize A3.1**: 资深直接对应 voice 选模板 — 半佛派惯用「反常识断言 + 数据冲击」, 故事派惯用「个人故事 + 悬念」, 知识系统派惯用「问题清单」, 不再每次重新选.
- **add A3.5**: 资深会做 **「开篇前置 5 屏」mental walkthrough** — 在脑里把信息流卡片 + 第 1 屏 + 第 2 屏 + ... 顺序过一遍, 检查每屏给读者的「奖赏」是否递进 (evidence: [T03-S030, T03-S043]).

**关键人物**: 半佛 (反常识 + 数据冲击 范式) / 六神磊磊 (古文反差 + 借古喻今 范式) / 万维钢 (问题清单 + 学术引用 范式).
**Decay risk**: low — 钩子学是稳定 craft, 5+ 年范式不变.

### Stage A4. 结构 + 行文 (6 子步骤)

**Trigger**: 钩子定稿, 进入正文.
**Output**: 完成全文初稿.
**典型耗时**: 入门 4-12 h / 资深 1-4 h (含 1-2 轮自改).

1. **A4.1 结构范式 选 1**: 三种主流 — **短平快 (500-1500 字, 情感号 / 资讯号)** / **长深度 (3000-8000 字, 知识号 / 商业号)** / **故事化 (1500-3000 字, 人物 / case study)** (evidence: [T03-S033, T03-S037, T03-S038]).
   - **跳过 后果**: 结构与选题错配 (财经分析硬写成故事 → 失去信任; 个人故事拉长到 8000 字 → 读者疲劳).
2. **A4.2 骨架 templates 选 1**: 结构内的具体骨架 — **三段论 / 总分总 / 倒金字塔 / 主张-论证-反驳 / 时间线 / 对比**. 主张 - 论证 - 反驳是商业咨询派 / 知识系统派 的「鲁棒模板」 (evidence: [T03-S036, T03-S037]).
   - **跳过 后果**: 写成 wiki 词条 (列举无主张) → 读者无 emotional hook → 完读率低.
3. **A4.3 voice register 锁定**: 5 种主流 voice — **段子化 (半佛系) / 知识系统 (万维钢系) / 文化考据 (六神磊磊系) / 情感生活 (情感号 / IP 号) / 商业咨询 (刘润系)**. 每种 voice 有自己的 register markers (用语 / 句长 / 段落节奏 / 比喻习惯) (evidence: [T03-S030, T03-S037, T03-S038, T03-S036, T03-S033]).
   - **跳过 后果**: 模仿多个不同 KOL → 自己 voice 不稳定, 读者画像分裂.
4. **A4.4 段落 + 句长 微观节奏**: 段落 2-3 行 max (移动阅读优化), 长短句交替 (每段至少 1 个短句作 ground), 关键句独立成段. 这不是审美选择, 是 mobile reading microformat (evidence: [T03-S034, T03-S035]).
   - **跳过 后果**: 段落长 + 无节奏变化 → 移动端阅读弃读 (跳出率最常见原因).
5. **A4.5 金句 设计**: 文中刻意设计 2-5 句可独立传播的「金句」 (适合截图 / 引用 / 朋友圈 caption 的句子). 金句不是文学修辞, 是二次传播抓手 (evidence: [T03-S033, T03-S036]).
   - **跳过 后果**: 文章主旨无人能 1 句话复述 → 无人转发 / 在看的隐性原因.
6. **A4.6 自改 1-2 轮**: 初稿完成后强制自改 ≥ 1 轮 — 主要改 (a) 开篇是否够 hook, (b) 中段是否有 dead zone, (c) 结尾是否给出 CTA / 行动指令 / 留白. 「初稿就发」是新手最常犯的省事 (evidence: [T03-S033, T03-S034]).
   - **跳过 后果**: 错字 / 逻辑断 / 节奏松 → 头部读者识别后取关.

**资深路径差异**:
- **skip A4.2** (流派稳定时): 资深的骨架是肌肉记忆, 不再每次选模板, 直接进入 A4.3.
- **optimize A4.4**: 资深的句长 + 段落节奏在写作时自然出, 不需要排版后再调.
- **add A4.7**: 资深会做 **「读 1 遍出声」**(self-read aloud) — 出声读 1 遍, 拗口的段落必改, 是检测节奏的最快方法 (evidence: [T03-S043, T03-S044]).
- **add A4.8 (高级)**: 资深会用 **AI 改稿** (Claude / DeepSeek) 跑 1 轮「润色 + 找逻辑漏洞」, 但**不直接用 AI 生成正文** — 边界是 craft control (evidence: [T03-S024, T03-S025] 政策推动 + 业内共识).

**关键工具**: Notion / 飞书文档 (草稿) / Word / Typora / Obsidian (markdown) → 后转 135 编辑器 / 秀米 (排版).
**关键人物**: 万维钢 (长深度 + 知识系统 范式) / Spenser (个人 IP 漏斗 + 自传 + 商业 范式) / 半佛 (段子深度 + 财经 + 多平台 范式) / 六神磊磊 (文化考据 + 借古喻今 范式).
**Decay risk**: low — 结构 + 行文 craft 5-10 年稳定, AI 写作冲击的是「效率」不是「范式」.

### Stage A5. 排版 + 配图 + 合规自审 (5 子步骤)

**Trigger**: 初稿完成 + 自改通过.
**Output**: 排版后的最终预览版本, 准备进入推送队列.
**典型耗时**: 入门 1-2 h / 资深 20-40 min.

1. **A5.1 排版工具 + 模板 选 1**: 三大主流工具 — **135 编辑器 / 秀米 / 壹伴**. 入门用 135 编辑器现成模板 (品类齐全 + 学习成本低), 资深定 1-2 个自家模板套到底 (品牌识别度) (evidence: [T03-S034, T03-S035]).
   - **跳过 后果**: 每篇换模板 → 视觉无品牌识别度.
2. **A5.2 排版六件套**: (a) 段落 2-3 行 (b) **关键句 + 关键词加粗** (c) **引用块** for 金句 / 引文 (d) 段间空行 (e) 每 500-800 字一张 **配图** (头图必, 文中过渡图 0-3 张) (f) emoji / 表情符号 谨慎使用 (头部知识号慎用, 情感号合用) (evidence: [T03-S034, T03-S035]).
   - **跳过 后果**: 排版不规范 → 移动端跳出率高 (即使内容好).
3. **A5.3 配图 license + 来源合规**: 配图来源选择 — (a) **摄图 / 包图 / Pixabay / Unsplash (商用 OK, 注意 license 细则)**, (b) **视觉中国 / 东方 IC (高侵权风险, 资深避免)**, (c) **截图 (要 cite 来源)**, (d) **AI 配图 (CC 政策不一, 2025-09 起 AI 内容含图必标识)** (evidence: [T03-S024, T03-S025] + 行业 best practice).
   - **跳过 后果**: 视觉中国 / 未授权图 → 收律师函 / 被索赔 (头部号常见踩坑); AI 配图不标 → 触发 2025-09 起的 AI 内容标识法规.
4. **A5.4 三 view 预览 check**: 排版后用公众号后台「预览」功能, 同时 check **移动端 + PC 端 + 折叠态 (订阅号入口预览)** 三个 view — 不同 view 的字号 / 行距 / 图片缩放表现不同 (evidence: [T03-S007, T03-S034]).
   - **跳过 后果**: PC 端漂亮但移动端跑版; 移动端漂亮但订阅号入口卡片露出错位.
5. **A5.5 合规自审清单 (推送前 last gate)**: 对照《微信公众平台运营规范》(evidence: [T03-S012]) 跑 8 项 — (a) 标题 (是否标题党?), (b) 配图 (是否侵权?), (c) 链接 (外链 → 微信内白名单内才能跳?), (d) 二维码 (诱导关注?) (evidence: [T03-S013, T03-S014]), (e) CTA 话术 (「不转不是中国人」「分享解锁」黑名单?), (f) 涉政 / 涉黄 / 涉医疗 红线 (evidence: [T03-S021, T03-S022, T03-S023]), (g) 软广 / 商单 是否标「广告」(evidence: [T03-S020]), (h) AI 生成内容 是否标识 (2025-09 起强制) (evidence: [T03-S024, T03-S025]).
   - **跳过 后果**: 单条违规 = 删文 + 警告; 多次违规 = 限流 / 限能力 / 失原创权; 严重违规 = 封号.

**资深路径差异**:
- **skip A5.1 重新选模板**: 资深已锁定 1-2 个自家模板.
- **optimize A5.4**: 资深的「三 view check」简化为「移动端 + 订阅号入口卡片」, PC 端不再单独 check.
- **add A5.6**: 资深会做 **「关键字段双人复核」** — 标题 + 摘要 + 封面图 + 第一段, 由非作者本人 (编辑 / 实习生 / 配偶) 校 1 遍, 把错字率从 80% 降到 5%.

**关键工具**: 135 编辑器 / 秀米 / 壹伴 / 公众号后台预览 / 摄图 / Unsplash / 视觉中国 (避免).
**Decay risk**: medium — 排版 craft 稳定; 但合规自审清单随网信办专项 + AI 标识法规 12 月内会更新.

### Stage A6. 推送 时间窗口 + 队列 (4 子步骤)

**Trigger**: 排版预览通过 + 合规自审通过.
**Output**: 推送成功, 数据开始累积.
**典型耗时**: 入门 10-15 min / 资深 5 min.

1. **A6.1 时间窗口 选定**: 按 vertical 差异 — **资讯 / 早读类 早 7-9** / **商业 / 知识类 午 12-13** / **情感 / 故事类 晚 19-22** / **周末整体下移** (evidence: [T03-S034, T03-S035, T03-S001]).
   - **跳过 后果**: 凌晨 / 工作极忙 时段推 → 阅读漏斗弃 (打开率减半).
2. **A6.2 服务号 vs 订阅号 规则适配**: **订阅号** 日 1 次群发 (可 1 次发 8 篇), 折叠在订阅号入口; **服务号** 月 4 次群发, 消息直达对话框. 二者推送 SOP 完全不同, 个人 / 自媒体 99% 用订阅号 (evidence: [T03-S007, T03-S034]).
   - **跳过 后果**: 服务号用订阅号 SOP → 浪费月 4 次推送 inventory; 订阅号用服务号 SOP → 频次过低.
3. **A6.3 头条 / 次条 / 三条 final 排序确认**: 选题阶段已定位 (A1.5), 此处复检 — 头条 = 主打, 次条 = 类目挖深, 三条 = 试错. **不要把广告 / 软广放头条** (除非该篇就是高质量广告内容) (evidence: [T03-S001, T03-S034]).
   - **跳过 后果**: 头条留给广告 → 这一波推送的自然流量被广告浪费.
4. **A6.4 推送前 last-check 清单**: 30 秒过 7 项 — **段落 / 错字 / 链接可点 / 配图加载 / 标题 / 摘要 / 封面图**. 是 SOP 最后 gate, 推送后再改要重新发 (老粉重复推送提醒, 体验差).
   - **跳过 后果**: 错字 / 链接错 / 封面图破损 → 推送后才发现, 删文重发掉粉.

**资深路径差异**:
- **skip A6.1 重新选时间**: 资深已锁定 1-2 个固定推送时段 (例 11:30 / 19:30), 让读者形成预期, 不再每次选.
- **optimize A6.4**: 资深的 7 项 check 内化为 30 秒动作, 不再列表.
- **add A6.5**: 资深推送后立即 **朋友圈 + 私域群 同步分发**, 不依赖订阅号入口的自然流量.

**关键工具**: 公众号后台「群发」/ 后台「定时发送」.
**Decay risk**: high — 推送窗口数据随平台改版 + 用户行为 12 月内会变 (例 2021 信息流改版后早高峰红利消失, 2024 推荐流权重提升后窗口效应再弱).

### Stage A7. 推送后 24 小时 + 复盘 (5 子步骤)

**Trigger**: 推送成功.
**Output**: 数据复盘 + 留言精选 + 二次分发 + 入库.
**典型耗时**: 入门 1-3 h 分散 24 h / 资深 30-60 min 分散.

1. **A7.1 数据监控 + 阈值 alert (1 / 6 / 24 h)**: 监控 — **打开率 / 完读率 / 在看 / 转发 / 留言 / 收藏 / 涨粉 - 取关**. 头部号通常 1 小时内反馈即定生死 (推荐流权重 6 h 内确定) (evidence: [T03-S001, T03-S003, T03-S004]).
   - **跳过 后果**: 24 h 不看数据 → 迭代慢, 选题 + 标题 + voice 优化基线不建立.
2. **A7.2 留言精选 SOP**: 在留言出现后 1-3 h 内选 5-10 条高质量留言公开, 优先 — (a) **延伸主旨的** (b) **作者可回复制造互动二次内容的** (c) **代表读者画像的「类型样本」**. 留言精选本身是文章本体延伸 (evidence: [T03-S034, T03-S035]).
   - **注意**: 2018-08-15 后新注册号无留言功能 — 是隐性资产分水岭 (evidence: [T06-S001 cross-ref]).
   - **跳过 后果**: 留言区无管理 → 错过二次传播抓手 + 错过读者反馈信号.
3. **A7.3 二次分发 (朋友圈 + 私域 + 视频号 + 跨平台)**: 推送后 1-2 h 内 — (a) **朋友圈** 分享 (附作者一句话 caption), (b) **私域群** 分享 + 引讨论, (c) **视频号** 联动 (公众号底部挂视频号卡片 / 视频号短视频引流回公众号) (evidence: [T03-S042]), (d) **跨平台 cross-post** (改版后发知乎 / 小红书 / 即刻, voice 需适配各平台).
   - **跳过 后果**: 只靠订阅号入口自然流量 → 中尾部号永远涨不动.
4. **A7.4 复盘分类 — 爆款 / 平款 / 滑铁卢**: 24 h 后归类 — **爆款** (数据 > 自己均值 2x) / **平款** (在均值 ±50%) / **滑铁卢** (< 均值 0.5x). 每类列假设原因 — 选题 / 标题 / 钩子 / 结构 / 时间窗口 / 外部事件 (evidence: [T03-S033, T03-S043, T03-S044]).
   - **跳过 后果**: 不分类 → 迭代无方向, 永远「凭运气」.
5. **A7.5 数据入库 (Excel / Notion / 飞书)**: 单篇数据入库 — 选题模板 / 标题模板 / 钩子模板 / 结构 / voice / 时间窗口 / 配图风格 / 阅读 / 在看 / 转发 / 留言 / 涨粉. 3-6 月累积后做模板偏好分析 (evidence: [T03-S033, T03-S034]).
   - **跳过 后果**: 累积 1-2 年仍凭直觉, 无法识别自己 voice 的 actual 数据偏好.

**资深路径差异**:
- **optimize A7.1**: 资深的 alert 阈值是肌肉记忆 (1 h 打开率 < X% 即知滑铁卢, 不需要事后算).
- **add A7.6**: 资深会做 **「精选评论 → 下一篇 selection bias 校验」**— 把精选评论本身作为下一篇选题候选的「读者投票」.
- **add A7.7 (头部团队)**: 资深团队会做 **「投流测试 + 互推」** — 数据较好的爆款付费投流 / 找互推扩大, 中尾部 SOP 不在此.

**关键工具**: 公众号后台「数据 → 群发分析 / 用户分析」/ 新榜有数 / 西瓜数据 / 飞瓜 / 自营 Excel / Notion / 飞书.
**Decay risk**: medium — 数据指标稳定, 但推荐流权重背后的算法 12 月内会调整, 复盘 heuristic 需要 refresh.

## B. 长期运营 SOP — 一个号 0 到 商业化 (4 阶段)

### 阶段 B1. 0 → 第一篇 (Week 1-2)

**Trigger**: 决定做公众号.
**Output**: 账号开通 + 定位 + 第一篇发出.
**典型耗时**: 1-2 周.

1. **B1.1 定位 5 件套**: (a) **vertical (垂类)**: 财经 / 知识 / 情感 / 文化 / 商业 / 行业 / 工具 / 资讯 ...; (b) **受众画像** (年龄 / 性别 / 城市 / 收入 / 阅读偏好); (c) **内容形态** (短平快 vs 长深度 vs 故事化); (d) **voice 流派** (从 A4.3 5 种 voice 选 1); (e) **商业化路径假设** (流量主 / 互选 / 知识付费 / 私域 / 多元) (evidence: [T03-S033, T03-S034]).
   - **跳过 后果**: 没有定位 → 选题杂 → 算法画像不清 → 推荐流不友好.
2. **B1.2 账号基建**: 名字 + 头像 + 简介 + 自动回复 + 菜单结构. **名字 + 简介 SEO 化** (含 vertical 关键词, 利搜一搜); **头像 + 简介品牌化** (识别度) (evidence: [T03-S034, T03-S035]).
   - **跳过 后果**: 名字平庸 → 搜一搜不出来 → 错过长尾流量.
3. **B1.3 第一篇 = 定位声明 + 深度内容**: 第一篇通常是「我是谁 / 我要写什么 / 给谁看」+ 一篇本垂类的代表性深度内容. 不要发寒暄式「大家好新号开张」(无信息 + 无 hook) (evidence: [T03-S033]).
   - **跳过 后果**: 第一篇敷衍 → 后续吸引来的读者无定位锚.
4. **B1.4 启动种子流量**: 亲友圈 + 已有跨平台 (知乎 / 微博 / 抖音 / 朋友圈) 引流. **不要买粉** (违规 + 算法识别 + 假粉污染画像) (evidence: [T03-S012, T03-S034]).
   - **跳过 后果**: 0 启动流量 → 算法判定「无人关心的号」→ 推荐流权重低.
5. **B1.5 节奏承诺**: 公开声明 (or 内部明确) 一个 hard 节奏 — 例 **「每周二 / 五 早 8 点」**. 节奏的稳定性比频次重要 (evidence: [T03-S033, T03-S034]).
   - **跳过 后果**: 第一周 5 篇下一周 0 篇 → 算法判定不稳定 → 推荐流降权.

---

### 阶段 B2. 1 → 1000 粉 (Month 1-3)

**Trigger**: 第一篇已发, 进入持续运营.
**Output**: 1000 粉, 内容 + 涨粉 + 互动节奏稳定.
**典型耗时**: 1-3 个月 (头部团队 1 月, 个人 3-6 月).

1. **B2.1 持续节奏**: 3-7 篇/周起步, 选 1 个周-周节奏 hard-stick. 中尾部公众号最常犯「日更冲刺一周, 然后断更」(evidence: [T03-S039, T03-S034]).
   - **跳过 后果**: 节奏断 = 算法降权 + 读者流失.
2. **B2.2 选题矩阵 (3-5 大类 sub-vertical)**: 在主 vertical 内定 3-5 个 sub-vertical (子专题), 让选题不重复, 让读者预期不混乱 (evidence: [T03-S033, T03-S036]).
   - **跳过 后果**: 选题杂 → 老读者迷惑「这号到底写啥的」.
3. **B2.3 涨粉渠道 多源**: 5 路并行 — (a) **内容自然涨粉** (慢), (b) **互推** (同行同体量号互换尾部 banner), (c) **跨平台 cross-post** (知乎 / 小红书 / 抖音 / 即刻), (d) **朋友圈 / 微信群 自然分享**, (e) **视频号联动** (2024-2025 主推) (evidence: [T03-S003, T03-S042, T03-S033]).
   - **跳过 后果**: 单一渠道依赖 → 渠道 1 失效全盘崩.
4. **B2.4 留言 / 互动 启动**: 积累 5-20 留言用户, 培养 super readers (会留长评 + 转发的). 这是私域的种子池 (evidence: [T03-S034]).
   - **跳过 后果**: 无互动池 → 后续做私域漏斗时, 入口空.
5. **B2.5 数据监控 + 月度复盘**: 每月做 1 次月度复盘 — 选题命中率 / 标题模板偏好 / 涨粉来源 / 流失原因 (evidence: [T03-S033, T03-S043]).
   - **跳过 后果**: 凭直觉运营 3 个月 → 无任何可优化的数据基线.

---

### 阶段 B3. 1000 → 1 万 粉 (Month 3-12)

**Trigger**: 已破 1000 粉.
**Output**: 1 万粉, 内容深度 + 涨粉加速 + 第一笔收入.
**典型耗时**: 3-12 个月 (头部 3 月, 中部 6-9 月, 长尾可能 12-24 月或停在 3000-5000).

1. **B3.1 内容升级 — 深度 + voice 稳定 + 系列**: 单篇深度提升 (字数 / 论证 / 案例厚度), voice 沉淀 (不再模仿头部), 至少 1 个「标志性系列」(每周固定栏目 — 例《六神磊磊读金庸》《刘润 5 分钟商学院》范本) (evidence: [T03-S038, T03-S036]).
   - **跳过 后果**: 内容平庸 → 涨粉曲线放缓.
2. **B3.2 涨粉加速 (微信内 + 联动)**: (a) **微信推荐流权重提升** (持续输出 + 完读率高 + 在看高), (b) **搜一搜 SEO** (标题 + 摘要 + 标签优化), (c) **视频号联动** (推视频号短视频引流公众号) (evidence: [T03-S003, T03-S042]), (d) **引爆 1-2 篇 10w+** (一篇 10w+ 通常涨 500-5000 粉, 业内估计) (evidence: [T03-S001]).
   - **跳过 后果**: 只靠自然涨粉 → 1 万粉天花板.
3. **B3.3 原创 + 白名单 必拿**: 申请原创权 (注册满 X 天 + 持续输出 + 无违规); 配置白名单转载策略 (允许 / 黑名单 哪些号转载) (evidence: [T03-S015, T03-S033]).
   - **跳过 后果**: 没原创 = 无流量主资格 + 易被洗稿 + 转载无署名分成.
4. **B3.4 流量主准入**: 粉丝过门槛 (2024 起 100 粉 / 500 粉 分级) + 内容合规 → 开通流量主, 收入 first dime — 月入 **业内估计 ¥100-¥3000 不等** (强度依赖 CPM + 阅读规模, 中尾部多在 ¥100-500/月) (evidence: [T03-S016, T03-S019, T03-S026]).
   - **跳过 后果**: 不开流量主 → 第一笔收入永远 0.
5. **B3.5 第一笔商业接单 (互选广告)**: 粉丝过 5000-1 万后, 可入腾讯互选广告, 接 ¥500-5000/单 互选 (中部号区间, 业内估计) (evidence: [T03-S017, T03-S018]). 抽成 2023-07 后为 5% (evidence: [T03-S018]).
   - **跳过 后果**: 只做流量主, 收入天花板永远低.

### 阶段 B4. 1 万 → 商业化 (Year 1-3)

**Trigger**: 已破 1 万粉, 内容 + 商业模式稳定.
**Output**: 多元商业化 + 团队化 + 跨平台 IP 化.
**典型耗时**: 1-3 年, 终点不是一个 fixed 数, 而是「稳定的月收入 / 团队规模 / 平台分布」.

1. **B4.1 矩阵化 (子号 / 类目号)**: 主号 + 2-5 个子号 (sub-vertical 拆分 or 不同 IP 共主理人). 矩阵互导 + 广告分摊. **2023 自媒体 13 条 + 2025 不实信息整治** 限制了同质矩阵 → 资深矩阵化要差异化 (evidence: [T03-S021, T03-S023, T03-S034]).
   - **跳过 后果**: 单号商业化天花板 vs 矩阵差一个数量级; 但乱矩阵 → 网信办盯上.
2. **B4.2 商业化多元 (5-7 路)**: (a) **流量主 (CPM 广告分成)** (基本盘), (b) **互选广告 / 软广** (主收入), (c) **知识付费 funnel** (公众号 → 课程 / 训练营) (Spenser / 万维钢 / 刘润 范本) (evidence: [T03-S033, T03-S036, T03-S037, T03-S032]), (d) **私域 funnel** (公众号 → 企微 / 个人号 → 卖货 / 卖课) (evidence: [T03-S033, T03-S047]), (e) **直接卖货** (自营商城 / 视频号小店) (evidence: [T03-S042]), (f) **出版 + 演讲**, (g) **付费阅读 / 付费合集** (灰度, 小众) (evidence: [T03-S040]).
   - **跳过 后果**: 只做流量主 → 月入 ¥几百-几千 天花板; 收入与读者 trust 脱钩.
3. **B4.3 团队化**: 编辑 (1-3 人) + 设计 (排版 + 配图) + 数据 (后台 + 选题) + 商务 (互选 / 软广 / 课程) + 私域运营 (中头部团队配置). 个人创作者 1 人 cover 5 个角色, 头部团队 5-50 人 (evidence: [T03-S028 反例 + T03-S046]).
   - **跳过 后果**: 个人单干天花板低 + 倦怠.
4. **B4.4 跨平台 IP 化**: 公众号 → 视频号 → 抖音 → 小红书 → 知乎 → 即刻 → B 站 + 出版 + 课程 + 大会演讲. voice 跨平台需各适配 (B 站长视频 vs 小红书短图文 vs 公众号长文) (evidence: [T03-S030, T03-S031, T03-S033]).
   - **跳过 后果**: 单平台 → 平台政策变化时一夜归零 (2018 头条系 / 2019 咪蒙 / 2022 短视频电商 整治 各有案例).
5. **B4.5 长期主义 + 节奏调整**: 1-3 年后, 节奏可能从「日更」降到「周 1-3」, 但单篇深度提升 + 商业化多元 弥补. 长期主义比短期套利 valuable (evidence: [T03-S008, T03-S008, T03-S038]).
   - **跳过 后果**: 短期套利 → 长期反噬 (咪蒙 reference).

## C. 变现 specific SOPs (3 子流程)

### C.1 流量主 接入 + 优化 SOP (6 步)

**Trigger**: 粉丝达门槛 + 内容合规 + 想开通被动收入.
**Output**: 流量主月收入 + 月度优化复盘.
**典型耗时**: 接入 1-3 天 (审核), 持续优化 monthly.

1. **C.1.1 准入审核**: 公众号后台 → 流量主入口 → 提交申请. 门槛 2014: 5000 粉 / 2020: 500 粉 / 2024: 100 粉 (返佣商品) + 500 粉 (程序化广告) (evidence: [T03-S016, T03-S026]). 需通过内容合规审核 (无近期违规).
2. **C.1.2 广告位设置**: 文末 / 文中 / 视频后 等位置 — 中间插入 vs 文末 vs 互选位 各有 CPM 差异. 入门全开, 资深选择性开 (避免广告位过多伤读者体验).
3. **C.1.3 屏蔽类目**: 后台可屏蔽不想出现的广告类目 (例屏蔽医美 / 借贷 / 游戏 / 短剧, 保护账号调性). 资深屏蔽清单是动态优化的核心动作 (evidence: [T03-S019]).
   - **跳过 后果**: 不屏蔽 → 高 CPM 但读者反感 / 取关.
4. **C.1.4 收入跟踪**: 后台「数据 → 财务结算」每日 / 每月看 — CPM / eCPM / 总收入 / 屏蔽影响. **业内估计公众号流量主 CPM ¥3-¥30** (内容向号), **头部可到 ¥50+** (evidence: 行业 secondary, 平台无官方公告). caveat: 数字是 ranged estimate, 平台无官方披露.
5. **C.1.5 月度复盘 + 屏蔽策略微调**: CPM 变化原因 — 广告主预算季节性 / 屏蔽过多压价 / 内容质量算法权重 / 推荐流流量结构. 屏蔽策略每月微调 (evidence: [T03-S019, T03-S041]).
6. **C.1.6 风险 monitor — 异常涨粉 / 涨阅读**: 流量主开通后, 警惕被「灰产挂广告主刷流量」连带封号 (微信对异常流量敏感) (evidence: [T03-S012]).

---

### C.2 互选广告 / 软广 接单 SOP (7 步)

**Trigger**: 粉丝过 5000-1 万 + 有广告主联系 / 主动 pitch.
**Output**: 1 单互选 / 软广完整闭环.
**典型耗时**: 单次接单 1-4 周 (从 pitch 到 settle).

1. **C.2.1 报价计算**: 基础 — **粉丝数 × 头条阅读 ÷ X** + 类目溢价 (金融 / 美妆 / 母婴 高, 文化 / 文学 低). **业内估计中部号互选报价 ¥3000-30000/篇头条, 头部 ¥10万+** (evidence: [T03-S017, T03-S041, T03-S033], caveat 数字 ranged estimate, 头部多不公开).
2. **C.2.2 平台 互选 vs 直签 选择**: 2023-06-30 后微信强制商单走腾讯互选 (合规 + 透明 + 抽成 5%) (evidence: [T03-S018]). 直签 = 违规 — 但仍有部分号灰色操作, 风险 = 互选广告身份被取消.
3. **C.2.3 广告主评估**: (a) 类目是否冲突 (医疗 / 金融 / 招商 强监管) (evidence: [T03-S017]), (b) 品牌方调性 (是否伤账号 voice), (c) 品牌方过往口碑 (是否假货 / 投诉多), (d) 投放预期 (data ROI 是否合理).
   - **跳过 后果**: 接「调性冲突 / 品牌差」广告 → 短期赚钱 + 长期掉粉.
4. **C.2.4 合同 + 软广 ratio cap**: 单月软广 ratio 自控 ≤ 20-30% (头部多有此自规则). **必须标「广告」二字** (《互联网广告管理办法》2023-05-01 起强制) (evidence: [T03-S020]).
   - **跳过 后果**: 不标 → 市场监管罚款 (最高 ¥10 万) + 平台限能力.
5. **C.2.5 软广写作 — 故事化 + 品牌融入**: 软广 ≠ 硬广, 资深做法 — (a) **品牌嵌入故事 / 痛点 / 解决方案**, (b) **保持账号 voice 不变**, (c) **末尾 CTA + 跳转链接** (微信内白名单).
   - **跳过 后果**: 软广像硬广 → 读者反感; 软广无故事 → 兑现率低品牌方不复购.
6. **C.2.6 上线后数据复盘 → 反馈广告主**: 24-72 h 内出阅读 / 在看 / 点击数据, 主动 share 给广告主. 数据 transparent 是复购的关键.
7. **C.2.7 复用客户 + 累积报价库**: 单一广告主复购 vs 一次性 — 资深倾向培养 5-10 个复购客户而非追新; 报价库随阅读 + 涨粉每季度升 (evidence: [T03-S033, T03-S041]).

---

### C.3 知识付费 funnel SOP (7 步)

**Trigger**: 已有稳定内容资产 (≥ 1 年) + 想做高 ARPU 变现.
**Output**: 公众号 → 私域 → 课程 / 训练营 / 付费社群 完整漏斗, 月入 ≥ ¥万 (中部) / ≥ ¥10 万 (头部, 业内估计).
**典型耗时**: 漏斗搭建 1-3 月; 首期课/训练营完整跑通 3-6 月.

1. **C.3.1 公众号文章 → 私域引流**: 文章中 / 文末 引导加企微 / 个人号 — 用「**资料包 / 内部群 / 案例库 / 1v1 咨询**」做钩子 (evidence: [T03-S033, T03-S047]).
   - **灰区**: 「资料包 + 二维码」型引流是「诱导关注」灰边界 (evidence: [T03-S013, T03-S014]); 暴力引流 → 微信封号.
2. **C.3.2 私域培育 (1-3 周)**: 加好友后 1-3 周做培育 — (a) 每天朋友圈 1-3 条 (生活 + 内容 + 价值), (b) 群内不发广告只发讨论 + 答疑, (c) 1v1 私聊不主动推销, 等用户问 (evidence: [T03-S047, T03-S033]).
3. **C.3.3 漏斗结构设计 — 免费 → 低客单 → 高客单**: 三层漏斗 — (a) **免费内容 / 资料包** (吸引), (b) **¥99-¥999 入门课 / 引流课** (筛选 + 信任), (c) **¥1999-¥29999 训练营 / 高客单课 / 私董会** (变现) (evidence: [T03-S033, T03-S047]).
4. **C.3.4 课程 / 训练营 售卖**: 报名页 + 朋友圈 + 私域群 + 公众号软文 4 路同步 — 通常做 7-14 天「报名季」+ 限时优惠 + 早鸟价 (evidence: [T03-S033]).
5. **C.3.5 漏斗转化率 监控**: 关键指标 — **公众号 → 加微信 转化率** (**业内估计 0.5%-5%** 因钩子设计差异, evidence: secondary), **加微信 → 报名 转化率** (**业内估计 1%-20%** 因私域培育强度差异), **入门课 → 高客单 转化率** (**业内估计 5%-30%** 因漏斗设计, evidence: [T03-S033, T03-S047]). 数字 caveat: ranged estimate, 头部不公开真实数据.
6. **C.3.6 售后 + 复购**: 课程交付 + 社群运营 (老学员复购率往往 > 新拓 ROI). 头部知识付费 (得到 / 罗辑思维 / Spenser) 50% 收入来自复购.
7. **C.3.7 整体合规**: (a) 不夸大宣传 (不承诺收入 / 不承诺月入 X 万), (b) 7 天无理由退款 (消费者权益保护法), (c) 课程内容不涉违法 (传销 / 金融骗局边界).
   - **跳过 后果**: 夸大宣传 → 市场监管罚; 知识付费课程 = 「教别人赚钱的课」最高风险, 多数被监管点名 (evidence: [T03-S032 cross-ref + 网信办整治]).

## 反模式 / 常见失败 (12 条)

### F1. 选题没有「我的号 fit」校验 → 蹭热点但读者不买账
- **现象**: 看到全网在讨论 X, 自己也写 X — 不管自己的号定位是不是 X.
- **触发原因**: 新手急于流量, 跳过 Stage A1.3.
- **后果**: 蹭来的流量是「一次性陌生流量」, 取关率 > 涨粉率, 算法画像污染.
- **如何避免**: A1.3 必跑 — 三层 (定位 + 读者画像 + voice) 都过才进入下一步 (evidence: [T03-S033, T03-S034]).

### F2. 标题党 → 流量高但完读 / 在看低 → 触发限流
- **现象**: 标题极端 / 误导, 内容无法兑现, 读者打开后 1 屏即跳出.
- **触发原因**: 短期 KPI 压力, 标题模板偏向「极端情绪 + 悬念」.
- **后果**: 单篇阅读高 / 完读 + 在看低 → 推荐流权重降 → 后续推送被压. 系统化标题党 → 平台明文禁止 (evidence: [T03-S012]), 触发删文 / 限能力 / 封号.
- **如何避免**: A2.2 4 问自审; 完读 / 在看 / 阅读 比例做监控 (evidence: [T03-S012, T03-S022, T03-S023]).

### F3. 开篇 50 字用「最近大家都在讨论」/「在这个 X 的时代」/「我们都知道」→ 信息流划过
- **现象**: 开篇是元话术 / 寒暄 / 主题前奏, 信息流卡片 + 摘要露出的是这段.
- **触发原因**: 新手按传统作文「先引入 后展开」结构, 不知公众号信息流约束.
- **后果**: 0 打开 — 信息流时代最常见的隐性扣分.
- **如何避免**: A3.1 用 8 钩子模板; A3.2 强制前 50 字独立成钩子.

### F4. 结构没有主张, 像 wiki 词条 → 读者无 emotional hook
- **现象**: 文章罗列事实 / 数据 / 案例, 没有作者的「主张 / 立场 / 推论」.
- **触发原因**: 模仿百科 / 论文风格; 害怕表达立场.
- **后果**: 完读率低; 无金句可截图传播; 在看 / 转发数据差.
- **如何避免**: A4.2 优先「主张 - 论证 - 反驳」骨架; A4.5 强制 2-5 个金句设计.

### F5. voice 模仿多个不同 KOL → 自己 voice 不稳定
- **现象**: 这周像万维钢, 下周像半佛, 下下周像 Spenser.
- **触发原因**: 新手「学谁就模仿谁」, 缺自己的 voice 锚.
- **后果**: 读者画像分裂, 老读者疏远, 算法画像混乱.
- **如何避免**: 阶段 B1.1 定位时选 1 个 voice 流派 (A4.3 5 种), B3.1 内容升级时强化 voice 稳定性 (evidence: [T03-S033, T03-S037, T03-S038]).

### F6. 排版段落长 + 无加粗 + 无配图 → 移动端弃读
- **现象**: 文章一坨黑字, 没有空行 / 加粗 / 配图.
- **触发原因**: 把公众号当 word 文档写, 忽视 mobile reading microformat.
- **后果**: 跳出率高; 数据差; 推荐流降权.
- **如何避免**: A5.2 排版六件套 (短段落 + 加粗 + 引用块 + 空行 + 配图 + emoji 谨慎) (evidence: [T03-S034, T03-S035]).

### F7. 推送时间选凌晨 / 工作极忙时段 → 阅读漏斗弃
- **现象**: 凌晨 2 点推送, 或 上班 9-11 / 下班 18-19 黄金时段错过.
- **触发原因**: 「写完就发」, 不考虑读者作息.
- **后果**: 打开率减半 (相比黄金时段).
- **如何避免**: A6.1 按 vertical 选时间; B1.5 节奏承诺 (周固定时段).

### F8. 24 小时没看数据 / 不复盘 → 迭代慢
- **现象**: 推送后不看数据, 不复盘, 凭直觉写下一篇.
- **触发原因**: 写作累, 没精力 follow up; 没有数据复盘的工具习惯.
- **后果**: 1-2 年累积仍凭直觉, 无法识别自己的模板偏好; 涨粉曲线无方向.
- **如何避免**: A7.1 + A7.5 强制 数据 monitor + 入库.

### F9. 长期节奏不稳 (今天 5 篇下周 0 篇) → 算法不喜
- **现象**: 周更 vs 日更 vs 断更 不稳定.
- **触发原因**: 灵感型创作; 工作主业冲突; 倦怠.
- **后果**: 算法对不稳定账号的推荐流权重降 (evidence: [T03-S039]).
- **如何避免**: B1.5 节奏承诺 + 内容储备 (常备 2-3 篇库存).

### F10. 商业化只做流量主 → 收入天花板 + 不与读者建立 trust
- **现象**: 只开流量主, 不做软广 / 互选 / 知识付费 / 私域.
- **触发原因**: 想「省心 + 不打扰读者」; 缺商业化技能.
- **后果**: 月入 ¥几百-几千 卡死; 错过与读者深度互动的机会.
- **如何避免**: B4.2 商业化多元 (5-7 路并行) (evidence: [T03-S033, T03-S041]).

### F11. AI 直接生成 + 复制粘贴 → 读者识别 + 留言投诉
- **现象**: ChatGPT / DeepSeek / Claude 直接生成全文 + 不改 + 不标识 → 发.
- **触发原因**: 想快速量产; 不掌握 AI 边界使用.
- **后果**: (a) 读者识别 AI 语气 (套路 / 缺细节 / 平庸结构) → 投诉 / 取关, (b) **2025-09-01 起《人工智能生成合成内容标识办法》强制标识** — 不标识 = 违规 (evidence: [T03-S024, T03-S025]).
- **如何避免**: A4.8 — AI 用于选题 / 大纲 / 改稿 / 润色, 不直接生成正文; 若用 AI 全文, 必须标识.

### F12. 抄袭 + 洗稿 → 原创权不保 + 投诉 + 封号
- **现象**: 改写他人原文表达, 保留思想 + 结构 (洗稿); 或直接抄袭 (字面挪用).
- **触发原因**: 想省创作成本; 想蹭爆款.
- **后果**: 微信原创合议判定 → 失原创权 + 限能力 + 严重封号 (evidence: [T03-S015, T06-S016, T06-S017, T06-S030 cross-ref]).
- **如何避免**: A4.x 写作时引用必标; 借鉴时改写到「不可被还原原版」程度 + 在文末 cite 来源.

## 历史反模式案例 (2 个)

### 案例 H1. 咪蒙现象 (2015-2019) — 短期 effective / 长期 toxic / 已禁

**时间线** (evidence: [T03-S027, T03-S028, T03-S029])
- 2015: 马凌化名咪蒙开公众号
- 2016-2018 巅峰期: 粉丝过千万, 报价行业第一梯队 (头条 ¥50-100 万 业内估计), 团队孵化矩阵号 (「才华有限青年」等)
- 2019-01-29: 「才华有限青年」发《一个出身寒门的状元之死》, 因虚构嫌疑 + 煽动焦虑被群嘲, 当晚被微信屏蔽
- 2019-02-01: 咪蒙团队道歉, 公众号停更 2 月, 微博永久关停
- 2019-02-21: 咪蒙公众号注销, 头条 / 微博 / 知乎 / 凤凰 等**全网同步封禁**

**方法论核心 (反面)** (evidence: [T03-S027, T03-S028, T03-S029]):
- **「热点 + 金钱 + 性 + 暴力」选题公式** (流量极强)
- **标题「态度鲜明 + 一定冒犯性 + 引起读者站队」** — 标题党升级版
- **4 阶段情绪曲线** (焦虑 → 共鸣 → 解决方案 → 行动指令)
- **写作团队化 / 流水线化** (主笔 + 数据 + 选题分工)
- **矩阵分发** (同方法论复制多号)

**为什么 frame 为「短期 effective / 长期 toxic / 已禁」**:
1. **短期 effective**: 2016-2018 头部稳定, 流量 + 报价 + 商业化都是行业第一梯队
2. **长期 toxic**: (a) 内容真实性失守 (虚构嫌疑), (b) 价值观短期投机 (贩卖焦虑 / 性别对立), (c) 监管 + 平台 + 公众三重反弹
3. **已禁**: 2019-02 全网封禁是行业最大反面案例, 任何后来者必须知道这条红线在哪 — 不是单一平台偶发处理, 是行业共识

**SKILL.md 不可软化 — 必须明确写**: 「咪蒙方法论短期有效是真实的, 但长期反噬 + 终被封禁是结构性. 4 阶段情绪曲线的有效性与其情绪操纵 / 价值观投机内核绑定, 解耦后失效. 不要尝试 (evidence: [T03-S027, T03-S028, T03-S029])」.

### 案例 H2. AI 批量产号灰产 (2024-2025) — 短期可走 / 长期被收紧 / 即将变黑

**现象**: 2024+ 用 ChatGPT / DeepSeek / Claude 批量生成公众号文章, 矩阵化注册数百号自动发布, 主要靠流量主 / 互选广告分成. 单号内容质量低, 但因量大 + 长尾, 短期能收益.

**典型操作**:
- 单团队注册 50-500 个公众号 (矩阵化)
- AI prompt 模板批量生成 5000 字 / 篇
- 自动排版 + 自动推送
- 流量主 + 互选广告自动接单
- 公众号头像 / 名字 微差异化 + 内容相似

**短期 + 中期 reality (2024-2025)**:
- **2024**: 部分团队月入 ¥5-50 万 (业内估计, 头部灰产)
- **2025-09-01 起《人工智能生成合成内容标识办法》强制标识** (evidence: [T03-S024, T03-S025]) — 平台正在搭检测能力
- **2023-07 自媒体 13 条** + **2024-04 「无底线博流量」专项** + **2025-07 「不实信息」专项** 均点名矩阵号 + 同质化 + AI 内容 (evidence: [T03-S021, T03-S022, T03-S023])
- **2026-01 微信单月封禁 2119 账号** (滥用原创 / 抄袭洗稿) (cross-ref T06-S036+)

**长期 reality (2025-2026+)**:
- 平台对「AI 痕迹明显 + 矩阵化 + 异常涨粉」联合识别 → 推荐流降权 + 大规模封号
- AI 标识办法的 enforcement 2025-09 起逐步落地
- 给整个行业制造噪音, 拉低读者对「公众号原创」的信任度 — 是公共资源的负外部性

**SKILL.md 不可软化 — 必须明确写**: 「AI 批量产号在 2024-2025 是灰区, 2025-09 起政策已就位, enforcement 在加强. 短期能赚 ≠ 可持续 — 平台 + 监管 + 读者 三重压力. 不要把账号信誉押在这里」(evidence: [T03-S024, T03-S025, T03-S021]).

## Workflow Summary (parser entry — for cli_writer)

> 摘要形式, parser 友好 (`### N.` + `**入门 SOP**:` + numbered list). 完整 workflow 仍在上面 A / B / C 节. 此节为 Phase 3 cli_writer 抽取 `cli/workflow/*.sh` 用.

### 1. 单篇推文 0 → 推送 (Single-Article Production)

- **One-liner**: 从选题决策到推送的 7 stage 单篇 SOP, 总耗时入门 8-12h / 资深 2-4h.
- **入门 SOP**:
  1. 选题决策 (双轨 + fit 三层校验; 微信指数 + 同 vertical TOP 5 同题扫描)
  2. 标题工程 (起 5-10 候选 + 4 问自检 + 选 1; ≤ 30 字 + 数字/反常识/悬念/利益)
  3. 开篇钩子 50 字 (8 模板选 1, 禁元话术「最近大家都在讨论」)
  4. 结构 + 行文 (3 结构 + 6 骨架 + 5 voice 选 1)
  5. 排版 + 配图 + 合规自审 (135 / 秀米 / 壹伴 + Unsplash / 摄图 + 8 项自审清单)
  6. 推送时间窗口 + 队列 (vertical 不同时段不同; 头条 / 次条 / 三条 排位)
  7. 推送后 24 小时数据监控 + 留言精选 + 二次分发 + 复盘入库
- **资深路径差异**:
  1. 模板化 stage 4-5 (结构 + 行文 + 排版固化为 2-3 模板), 把精力压回 stage 1-2 (选题 + 标题)
  2. Stage 6 时间窗口选择 + 头条 / 次条排位决策 凭直觉 < 5 min
  3. Stage 7 数据反馈 6h 内判生死, 即时调整下篇方向
- **典型耗时**: 入门 8-12h / 资深 2-4h (单篇)
- **关键工具**: 公众号后台 + 135/秀米/壹伴 + 微信指数 + 新榜 + DeepSeek/Kimi + Unsplash/摄图
- **常见失败模式**:
  1. F1 选题没做 fit 校验 (蹭热点但读者不买账)
  2. F3 开篇 50 字用元话术 (信息流划过)
  3. F8 24h 没看数据 / 不复盘 (迭代慢)
- **来源**: T03-A1 ~ A7 (38 actions 已展开在上方)
- **Last_updated**: 2026-05

### 2. 一个号 0 → 商业化 (Long-term Account Growth)

- **One-liner**: 从开号到稳定商业化收入的 4 阶段长期 SOP, 总耗时 1-3 年.
- **入门 SOP**:
  1. 0 → 第一篇 (Week 1-2): 定位 5 件套 (vertical + 受众画像 + voice + 名字 + 头像) + 第一篇深度内容
  2. 1 → 1000 粉 (Month 1-3): 持续节奏 (周 3-7 篇 hard-stick) + 5 渠道涨粉 (互推 + 跨平台 + 朋友圈 + 微信群 + 视频号联动)
  3. 1000 → 1 万 粉 (Month 3-12): 内容升级 + 涨粉加速 + 原创权拿到 + 流量主准入 (粉丝 ≥ 500 / 内容合规)
  4. 1 万 → 商业化 (Year 1-3): 矩阵化 + 5-7 路并行变现 (流量主 + 互选广告 + 软广 + 知识付费 funnel + 私域 funnel + 卖货 + 出版引流) + 团队化 + 跨平台 IP 化
- **资深路径差异**:
  1. B1 定位 5 件套 30 min 完成 (新手要 1-2 周纠结)
  2. B3.2 涨粉用 视频号联动 + 朋友推荐 红利 (新手只会 公众号内 SEO)
  3. B4.5 长期主义 — 节奏可从 日更 降到 周 1-3 但单篇深度提升 + 商业化多元
- **典型耗时**: B1 1-2 周 / B2 1-3 月 / B3 3-12 月 / B4 1-3 年
- **关键工具**: 公众号后台 + 新榜数据 + 飞瓜视频号联动 + 135/秀米 + AI 写作 + 知识星球
- **常见失败模式**:
  1. F9 长期节奏不稳 (今天 5 篇下周 0 篇, 算法不喜)
  2. F10 商业化只做流量主 (收入天花板 + 不与读者建立 trust)
  3. F2 标题党路径短期涨粉, 但触发 平台限流 + 监管整治 (咪蒙 case)
- **来源**: T03-B1 ~ B4 (22 actions 已展开在上方) + T01-S020 Spenser + T01-S025 罗振宇
- **Last_updated**: 2026-05

### 3. 变现路径选择 + 接入 (Monetization Pipeline)

- **One-liner**: 5-7 路变现选择 + 流量主 / 互选 / 知识付费 funnel 三大子流程接入 SOP.
- **入门 SOP**:
  1. 评估账号变现 phase (B 流当前阶段) + vertical 适配的 1-2 路主变现
  2. 流量主接入 (粉丝 ≥ 500 申请 + 广告位设置 + 屏蔽类目 + 月度复盘)
  3. 互选广告 / 软广 接单 (粉丝 ≥ 1 万 + 报价计算 + 平台互选 vs 直签 + 软广写作合规标识)
  4. 知识付费 funnel (公众号文章 → explicit CTA 引流私域 → 私域 1-3 周培育 → 3 层漏斗销售 → 售后复购)
  5. 月度商业化复盘 + 调整下季度多元化配比
- **资深路径差异**:
  1. 5-7 路并行 (不单押任何 1 路, 抗平台依赖)
  2. 互选客户复购运营 (5-10 个高质量复购 > 30 个一次性)
  3. 知识付费漏斗用 explicit CTA + 私域培育 (不是软引导)
- **典型耗时**: 单路接入 1-4 周 / 5-7 路稳定 6-12 月
- **关键工具**: 公众号后台 (流量主 / 互选入口) + 企微 / 知识星球 (私域承接) + 草料二维码 (引流入口) + 新榜 (报价校准)
- **常见失败模式**:
  1. F10 只做流量主 (CPM 极低, 收入天花板)
  2. 软引导知识付费 (转化率 < 0.1%, 必须 explicit CTA)
  3. 软广不标「广告」(违《互联网广告管理办法》, 平台处罚 + 商务复购流失)
- **来源**: T03-C1 ~ C3 (20 actions 已展开在上方) + T06-S015 广告法 + T01-S020 Spenser + T06-S006/007/008 互选 / 流量主 vendor docs
- **Last_updated**: 2026-05

## Phase 2 提炼提示

### 反复出现 ≥ 3 个 workflow 都包含的步骤 (候选 playbook 通则)

- **「我的号 fit 校验」三层 (定位 + 读者画像 + voice)** 出现于 A1.3 + A4.3 + B1.1 + F1 + F5 → 候选 playbook: **「每个内容决策最终 gate 是『这个内容像我吗 / 我读者要这个吗 / 我 voice 能驾驭吗』, 三问全过才推进」**
- **「合规自审 + 红线 check」** 出现于 A2.2 + A5.5 + C.2.4 + F11 + F12 + H1 + H2 → 候选 playbook: **「每次推送前跑 8 项合规自审 (标题党 / 配图侵权 / 链接合规 / 二维码诱导 / CTA 话术 / 涉政涉黄涉医 / 软广标识 / AI 标识), 是 last gate 不可省」**

### 入门 SOP 和资深路径之间最大的差距 (候选心智模型)

- **入门 SOP 平均 5-7 步 / stage, 资深路径压缩到 2-3 步**, 主要靠 (a) 模板内化为肌肉记忆, (b) skip 重复决策步骤, (c) add 资深特有的「预拟 / 出声读 / 三层 fit」深度校验
- 候选心智模型 1: **「公众号写作的本质是『写出来好像随手就能写好的样子, 实则每个微决策都经过 4 问自检』— craft 是看不见的训练量」**
- 候选心智模型 2: **「不要把 craft (写作技艺) 和 ops (流量 / 涨粉 / 商业化) 错位 — 入门期 craft : ops = 70 : 30, 中期 50 : 50, 资深期 craft : ops = 30 : 70 (因为 craft 已肌肉记忆)」**
- 候选心智模型 3: **「公众号是产品不是文章 — 单篇 A1-A7 SOP 必须服务于账号长期定位 (B1.1), 短期套利 (单篇爆款) 与长期 IP (账号资产) 是真分歧, 必须选边」**(evidence: [T03-S008, T03-S008, T03-S033, T03-S008] 张小龙长期主义 + cross-flow 共识)

## 诚实边界 (closing 段落)

大量操作级细节 (尤其头部 KOL 的具体技法演化 / 内部 prompt 库 / 私域 funnel 完整模板 / 互选广告私下议价 / 矩阵号互导 SOP / AI 工具链具体 stack) 住在三个闭环渠道:

1. **公众号本体** (调研黑名单, 不进 source 池)
2. **知识星球 / 微信群 / 付费社群** (部分公开, 大部分付费墙 + 群闭环 — 群响 / Spenser 商学院 / 何加盐 知识星球 / 见实 沙龙 等)
3. **私下饭局 + 操盘手互助 + MCN 内部** (完全闭环)

本 Track 03 覆盖的是 **externally-observable 层** — 平台官方表态 (微信公开课 PRO + 微信开放社区) + 监管原文 (网信办 + 市场监管总局 + 国家版权局 + 税务总局) + 出版书 (Spenser / 万维钢 / 刘润 / 六神磊磊 / 秦阳 + 秋叶 教材系) + KOL 在第三方平台长访谈 (36kr / 虎嗅 / 晚点 / 数英 / 澎湃 / IT 之家 / B 站 / 小宇宙) + 第三方数据机构 (新榜 / 新榜有数 / 西瓜数据 / 飞瓜).

这是「外部 + 公开 + 可追溯」的 SOP 池, 不是「头部 KOL 真实操作的全集」.

**头部 KOL 的真实 SOP 受 (a) 商业秘密保护 (训练营卖钱) + (b) 平台红线管控 (互选广告抽成数字 / 私下议价 / 矩阵号互导) + (c) 行业内默契 不全部公开**. SKILL.md 用户使用本 SOP 时, 预期是 **「框架级 + 入门级 + 中部级 actionable, 头部级 craft 细节需要进闭源圈子 + 实战练手」**.
