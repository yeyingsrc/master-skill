# Track 03 — Workflows (跨境电商运营)

> Mini-viable for T8 prototype.

### 1. 新品上架 (Amazon EU)

- **One-liner**: 从「我决定卖 X」到「listing 跑稳定单量」
- **Trigger**: 选品 done + 供应链 ready
- **Output**: 上架 + 首批单量 + 评价积累
- **入门 SOP**:
  1. 注册站点 + 完成合规 (VAT / 包装法 / GS-CE 检测报告)
  2. 写 listing (关键词研究 + 主副图 + A+ 内容)
  3. 第一批 FBA 入仓 + 单 ASIN 上架
  4. brand search + auto campaign + 联盟客优惠码组合投流
  5. 监控前 30 天数据, 调价 / 调图 / 调描述
- **资深路径**:
  - skip 完整投流 brand search + auto campaign 双开, 资深可只跑 SP-Auto + 合理 bid
  - optimize 关键词调研用 Helium 10 + 卖家精灵互相验证, 不只信一家
  - add 评价矩阵 + Q&A 预埋, 不等差评再补
- **常见失败模式**:
  - 合规材料没准备好就上架, 第一周被举报下架, 整个 ASIN 报废
  - 关键词调研只用一家工具, 数据偏差导致 listing 错位
  - 不监控前 30 天数据, 错过冷启动黄金窗口
- **Last_updated**: 2025-12-15
- **Decay risk**: high

### 2. Listing 优化 cycle

- **One-liner**: 从 listing 上线到持续提升转化率 / 排名
- **Trigger**: 新品上架后 30 天 + 季度优化
- **Output**: 转化率 / 排名持续提升
- **入门 SOP**:
  1. 拉数据 (Brand Analytics + Helium 10)
  2. 找差距 (vs 头部卖家 / vs 类目均值)
  3. 改一个变量 (主图 OR 标题 OR 价格 OR bullet) + A/B
  4. 验证 7-14 天数据
  5. 决定保留 / 回滚 / 继续
- **资深路径**:
  - skip 一次只改一个 (新手严格遵守); 资深可同时改 listing 多处但分阶段验证
  - optimize 用 Sellerise 监控反向工程头部卖家变化
  - add 季度做一次完整 listing 重写, 不只是局部 A/B
- **常见失败模式**:
  - 一次改太多变量, 不知道哪个有效
  - 不等数据稳定就回滚
  - 抄头部卖家的 listing 而不分析他们为什么这样写
- **Last_updated**: 2025-11-01
- **Decay risk**: medium
