## 心智模型 (4 个 PASS)

### 1. Selling on Amazon Is About Understanding Shoppers, Not Gaming the Algorithm

**一句话**: 亚马逊运营不是玩算法, 是理解买家想要什么, 然后把它精准呈现.

**它说的是**: 黑科技 / 套路 / SEO 操控这些都是在算法和你之间打仗. 真正长期赢的卖家是在「买家心智 → 产品 → listing」这条线上深耕. 算法升级总有一天会把套路一锅端, 但贴近买家的产品 + listing 永远 work.

**应用方式**:
- listing 撰写时问: 「这个买家在搜索这个词时, 真正想解决什么问题?」
- 用 Brand Analytics SQP 看真实搜索词 (不是工具反查的猜测词)
- 主图测 — 直接看 CTR 数据告诉你买家眼睛被什么抓住

**局限**:
- 极早期阶段还没数据, 必须有部分直觉 + 假设 + A/B
- 部分商品类目 (商品本身没差异) 算法因素仍权重高

### 2. Data-Driven 选品 + Listing 是亚马逊的真杠杆

**一句话**: 全栈数据驱动 — 从选品到关键词到 listing 到 PPC, 都该有数据支撑.

**它说的是**: 每个亚马逊运营决策都有官方数据 (Brand Analytics SQP) + 第三方数据 (Helium 10 Cerebro / Black Box / Adtomic) + 自家 BI 数据可参考. 不用数据 = 跟数据派对赌.

**应用方式**:
- **选品**: Helium 10 Black Box + Brand Analytics SQP 三源交叉
- **关键词**: Cerebro 反向 ASIN + SQP 真实数据
- **Listing**: Frankenstein / Scribbles 关键词覆盖检查
- **PPC**: Adtomic 自动化 + SQP 数据驱动调整

**局限**:
- 工具厂商立场会推用工具, 实际 data quality 要交叉验证
- 数据派也有偏见 (过度优化短期指标, 忽略长期品牌)

### 3. Cold Start (冷启动期) 是关键, 别打断系统学习

**一句话**: 新品上线前 14-30 天是亚马逊算法的扶持期 (Honeymoon Period), 这阶段策略错误等于浪费整个 launch.

**它说的是**: 算法在这阶段试人群 + 累积转化数据. 你硬调出价 / 突然加大预算 / 频繁改 listing 都会重置算法学习. 多数失败的新品都是冷启动期被「太着急」搞坏的.

**应用方式**:
- 冷启动期 14-30 天: Auto + Manual SP 双轨, 不动出价 / 不加预算 / 不改 listing
- Vine 同步上 — 30 个免费评价加速信任度积累
- 数据稳定 (≥ 1000 曝光 + 稳定 ACOS) 后才开始优化

**局限**:
- 极小预算 (< $300/月) 不够触发完整 honeymoon
- 旺季 (Q4) 节奏不同, 部分 honeymoon period 缩短

### 4. Multi-Channel + AI Era 让方法论持续演化 (2025-2026)

**一句话**: 2025-2026 多平台 (Walmart / TikTok Shop / Temu) + AI 工具 + 算法升级 是常态, 必须保持持续学习.

**它说的是**: AWD 普及 / Pan-EU 重新成为关键议题 / 关税不确定性 / TikTok Shop / Walmart 200k+ 卖家增长 — 这些 trend 让 Amazon 卖家也必须看其他平台.

**应用方式**:
- Q1 / Q3 复盘多平台 GMV 占比, 持续调整
- 关注 Helium 10 / Pacvue 等工具厂商对 Walmart / TikTok Shop 支持的扩展
- 跟 Weekly Buzz 节目 (周更政策解读)

## 决策规则 (7 条)

### 1. 如果新品冷启动期 (前 14 天)
**则**不动出价 / 不加预算 / 不改 listing — 让系统跑探索期.

### 2. 如果想做关键词调研
**则**用 Helium 10 Cerebro (反向 ASIN top 5 同类) + Brand Analytics SQP (真实数据) 交叉验证.

### 3. 如果做品牌路径
**则**第一天做 Brand Registry — 解锁 A+ Content / SQP / Vine / Sponsored Brands.

### 4. 如果月广告费 ≥ $5000
**则**上 Helium 10 Adtomic (基础) 或 Pacvue (大账户) 自动化, 但仍人工把关大额计划.

### 5. 如果 listing 转化率 < 类目均值
**则**先看主图 + 标题 (CTR), 再看 Bullet + A+ (CVR).

### 6. 如果 AI 生成 listing 内容
**则**必须人工编辑 + 加真实使用场景, 否则 Helpful Content Update 限流.

### 7. 如果 IPI < 200
**则**清库存 / 降价 / 转促销, 否则 FBA 入仓限制.

## 应用示例

### 用 Bradley Sutton 视角思考: 「我新品上 14 天 ACOS 50%, 自然单 0, 该不该砍?」

**Bradley 可能的回答框架**:
1. **冷启动期是不是还在?** 14 天可能刚出 honeymoon, 别急着砍.
2. **CTR 怎么样?** < 类目均值的话不是出价问题, 是主图 / 标题问题.
3. **CVR 怎么样?** < 5% 的话 listing (A+ / Bullet) 是问题.
4. **关键词覆盖准吗?** 用 Cerebro 反查 top 同类 ASIN 看你的 listing 有没有 cover 它们的核心词.
5. **Brand Registered 了吗?** 没的话先备案, 解锁 SQP 看真实搜索词.

**结论倾向**: ACOS 50% 在新品 14 天是正常的 (Honeymoon 期), 不要立刻砍. 先看 CTR / CVR / 关键词覆盖, 找出真正瓶颈再调.
