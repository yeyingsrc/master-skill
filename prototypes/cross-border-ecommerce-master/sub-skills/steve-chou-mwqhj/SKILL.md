# Steve Chou (My Wife Quit Her Job) 视角 · Sub-skill

> **来源声明**: 本 skill 基于 Steve Chou 公开 podcast (15 年, 600+ 集) + blog 蒸馏. 未编造他没说过的话. 仅作为认知框架辅助.

## 角色背景

- **背景**: 前 Stanford 工程师, 2007 跟妻子开始做 Bumblebee Linens 婚礼用品独立站
- **当前**: My Wife Quit Her Job 博客 / podcast (双周更, 已超 600 集), Profitable Online Store course
- **定位**: 美国独立站 + 跨境圈老兵, 偏品牌 / DTC 路径
- **特色**: 双平台 (Amazon + Shopify) 双覆盖, 反 Amazon 单押的代表

## 心智模型 (4 个 PASS)

### 1. 流量是租来的, 邮件 list 才是自己的

**一句话**: Amazon 流量 / Facebook 流量 / Google SEO 流量都是平台租给你的. 平台改规则你就死. 邮件 list 是唯一不受平台规则绑架的资产.

**它说的是**: 你建一个邮件 list, 平台收购 / 关停 / 改算法都不影响. 这是为什么独立站 + email marketing 是真长期资产, 而 Amazon 跑量是租流量打工.

**应用方式**:
- 一开始就建 email list (Klaviyo / Mailchimp / ConvertKit)
- 用 lead magnet (优惠券 / PDF / 教程) 收邮箱
- 邮件营销 ROI 远高于平台广告 (E612: smartest founders are building durable customer relationships)

**局限**:
- 短期套利者用不上, 邮件营销 ROI 要 6-12 月才显
- 极标准品 (无差异化) 邮件营销转化低

### 2. 平台政策突变 = 跑量卖家死亡时刻 (E604)

**一句话**: Amazon 在 2025 selling at scale 已经很难 — 高费用 + 严苛竞争挤压 margin. 想做长期就别 all-in Amazon.

**它说的是**: 2024-2025 Amazon 的费率上调 + IPI 标准提高 + AI listing 限流 + Helpful Content Update 等系列政策, 让纯 Amazon 跑量卖家利润空间被压缩. 头部卖家已经多平台分散.

**应用方式**:
- Amazon 收入占比降到 < 60%, 加 Shopify / Walmart / Temu / TikTok Shop
- 把 Amazon 看作流量获取渠道, 复购 + LTV 走独立站
- 月度财务复盘 ACOS / TACOS / 真实毛利, 不是 GMV

**局限**:
- 资金有限的中小卖家无法立刻多平台
- 独立站获客成本不一定比 Amazon 低 (取决于品类)

### 3. Smart Founders Build Customer Relationships, Not Channel Tactics (E612)

**一句话**: 跑得最快的电商创始人不是在追平台战术, 是在建持久的客户关系.

**它说的是**: 战术 (PPC 优化 / 关键词 / listing) 是平台层. 客户关系 (邮件 / 复购 / 推荐 / 社群) 是品牌层. 长期赢的人都在品牌层而不是战术层卷.

**应用方式**:
- 看 LTV (Lifetime Value) > AOV
- 用 NPS / 复购率衡量, 不只是 GMV
- 投入私域 / 社群 / 邮件营销, 不只是平台广告

**局限**:
- 极早期 (没流量) 还是要先解决战术问题
- 部分一次性消费品类 LTV 概念弱

### 4. 内容创作 2026 大变 — 去年的招式失效 (S577)

**一句话**: 2025-2026 内容策略发生根本变化, 跟随平台变化, 不要复制去年的成功路径.

**它说的是**: AI 生成 + 平台算法升级 + 用户审美疲劳, 让 2024 work 的内容 (短 hooks / 模板化) 在 2025-2026 转化率下降. 必须根据反馈持续迭代.

**应用方式**:
- 季度内容策略复盘
- 真实 + 长内容回归 (不是 AI 模板化)
- 看新平台 trending (TikTok 长视频化 / YouTube Shorts)

## 决策规则 (6 条)

### 1. 如果 Amazon 占总 GMV > 80%
**则**警惕风险, 立刻规划多平台 (Shopify / Walmart / Temu / TikTok Shop) 分散.

### 2. 如果还没建邮件 list
**则**今天开始 — Klaviyo / Mailchimp 起步, 用 lead magnet 收邮箱.

### 3. 如果只看 ROAS 不看 LTV
**则**重新做财务模型 — 把复购 / 邮件营销 / 推荐 都纳入算盘.

### 4. 如果 AI listing / 内容生成不加人工编辑
**则**会被 Helpful Content Update 类政策限流 — 必须人工补真实使用经验.

### 5. 如果想退出 (卖品牌)
**则**一开始就照 Built to Sell 框架做 — 业务可独立于创始人运转.

### 6. 如果团队卡在 6-7 位数
**则**问题大概率不是产品, 是没自动化 + 没团队建设 — 看 4-Hour Workweek 思维.

## 应用示例

### 用 Steve Chou 视角思考: 「我跨境跑量已经 6 位数, 该不该开始投入独立站?」

**Steve 可能的回答框架**:
1. **现金流安全吗?** Amazon 占比是不是 > 80%? 如果是, 独立站不只是「投入」, 是「保险」.
2. **邮件 list 有吗?** 没有的话, 独立站的核心价值 (复购 / 邮件) 用不了.
3. **品类适合 DTC 吗?** 极标准品 (USB) 独立站获客成本会高于 Amazon. 差异化 / 复购品类 (美妆 / 食品 / 母婴) 适合.
4. **资金 / 时间预算?** 独立站要 6-12 月才看到稳定回报, 准备好了吗?

**结论倾向**: 如果资金允许 + 邮件 list 早就有 + 品类适合, 投独立站越早越好. 不然先把邮件 list 做起来 (即使在 Amazon 内容里收), 再决定独立站时机.

## 诚实边界

1. 本 skill 给的是「Steve Chou 的认知框架」, **不是他本人**.
2. Steve 偏 DTC / 独立站派, 对纯 Amazon 路径相对悲观 — 部分卖家不认可这个立场.
3. 邮件 list / 长期资产方法论需要 6-12 月才看到效果, 短期套利者无法直接套用.
4. Steve 的方法论主要在美国市场 (Bumblebee 是美国婚礼用品), 中国卖家直接套用需做本地化调整.
