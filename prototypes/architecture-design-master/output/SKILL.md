### Step 2: 按这一行的方式做功课

⚠️ 必须使用工具（WebSearch / WebFetch / agent-reach 等）获取真实信息。

#### 维度 1: Program × Site × Tectonics 三轴诊断 (mental model 1.2 + 1.5 直接对应)
- 看什么: 项目 program (建筑类型 + 主要 user + 使用模式) / site (地理 + 气候 + 文化 + 周边 + 规范限制 — 容积率 / 限高 / 退线 / 防火) / tectonics (业主 budget tier + 主要可用材料 + 当地施工能力 + 主要节点系统)
- 在哪看: 业主 RFP / 用地红线图 / 当地气候数据 (CSWD 中国 / ASHRAE 国际) / 规划法规原文 (zh: 住建部 / 国标全文; en: IBC / local building code) / 同 typology 同地区 reference building (有方 / 谷德 / ArchDaily 项目库) / 当地承包商 capability 报告
- 输出: 3 行结构化 — `Program: {1 句描述 + 主要 user + 使用模式}` / `Site: {气候 + 文化 + 规范关键约束 3 件}` / `Tectonics: {budget tier $X-$Y/sqft + 主要材料 + 关键节点系统}`. 三轴任一空白 → 设计不成立, 回业主补 brief.

#### 维度 2: 业主真 motivation 解码 (mental model 1.4 直接对应)
- 看什么: 业主 stated goal (reputation / iconic / cost / schedule / asset value) 之外的 implicit motivation (政治 / 内部权力 / family legacy / 业务 PR / 上市估值); past project 历史 (业主之前合作的建筑师 / 项目失败原因); 决策链 (业主一人 / 业主夫妇 / 业主+董事会 / 政府委员会)
- 在哪看: 第一次会议 + 后续 3 次会议的细节笔记 (谁先开口 / 谁拍板 / 谁问钱); 业主之前项目 (求职业主 LinkedIn / 上市公司 annual report / 项目所在城市 news); 同行 reference (问做过该业主的同行建筑师)
- 输出: `Stated goal: {业主口头表达}` / `Implicit motivation: {推断的真动力 + 置信度 high/medium/low}` / `决策链: {who decides what + sign-off 节奏}` / `red flag: {风险信号 3 件}`.

#### 维度 3: 衰退周期分层诊断 (mental model 1.3 直接对应)
- 看什么: 项目 / 决策 / 工具选择对应哪个时间层 (15-25 年 style / 5-10 年 tool / 50+ 年 tectonics-material-structure-history); 押注的层是否匹配项目 lifespan (e.g. 公共建筑 50 年 vs 商业 zone 15 年 vs 临展 1 年)
- 在哪看: 项目 brief 中的 expected lifespan / 业主 horizon; 材料 spec 的 service life (混凝土 50-100 年 / 钢 50-80 年 / 玻璃 30-50 年 / curtain wall 25-40 年); 风格 / 工具的 hype curve (Gartner 类 + 行业 journals 历史 issue 对比)
- 输出: `Project lifespan: {N 年}` / `押注层匹配度: {high/medium/low}` / `mismatch risk: {如有 — e.g. 100 年公共建筑用 5 年 cycle parametric tool 出图 → 形式 + 工具都 25 年内过时, 但建筑本体 100 年, 推荐切回更长周期 tectonic + material 决策}`.

#### 维度 4: 三件事自检 — render / model / build (mental model 1.1 直接对应)
- 看什么: 当前 deliverable 是 render / digital model / building, 各自 success criteria (render: communication impact; model: coordination accuracy; build: durability + use + cost); 三件事之间 misalignment 信号 (renderer 出图业主签了, 但 model 几何不可建; model 协调了, 但 spec 与现实成本差 2x)
- 在哪看: SD/DD/CD deliverable 对照 (3 阶段渲染 vs 模型 vs spec); BIM clash report; 业主 feedback 是基于 render 还是基于 model 还是基于 reference building
- 输出: `Render integrity: {当前渲染基础是 model 真实数据 还是 mood board}` / `Model integrity: {LOD 是否匹配 phase 期望}` / `Build feasibility: {当前节点 + 材料 + budget 在当地施工方能力内吗}` / `Misalignment risk: {如有, 立即修正}`.

#### 维度 5: 流派定位 + 业内同代对照 (mental model 1.5 + 智识谱系 §7 对应)
- 看什么: 项目 / 设计师所在流派 (9 流派中哪 1-2 个); 同流派 ≥ 3 当代 figures 在做什么 (e.g. critical regionalism: Wang Shu / Liu Jiakun / Kéré 当前项目); 流派内分歧 (e.g. critical regionalism 内 "符号贴图" vs "抽象在地策略" 之争)
- 在哪看: Track 01 figures 9 流派矩阵 (本 skill referenes/research/01-figures.md); Architectural Review / Domus / El Croquis 当代专题 (流派分类); 中国当代: 有方 / 谷德 当代专题
- 输出: `当前流派: {1-2 流派 + 立场}` / `同代代表 3 件事: {N people + N projects + N debates}` / `分歧定位: {项目落在流派内哪个 sub-position}` / `差异化机会: {如有 — 同流派别人没做的 niche}`.

#### 维度 6: 5-phase 节奏 + 当下应该 lock / leave open 自检 (mental model 1.6 直接对应)
- 看什么: 当前项目所在 phase (SD/DD/CD/CA), 应该 lock 的内容 (体块 / 材料 / spec) vs 应该故意 leave open 的内容; 是否被业主 / 施工方 pressure 提前画大样 / 提前 lock spec
- 在哪看: AIA Project Delivery Guide (各 phase deliverable 标准) / RIBA Plan of Work / 中国设计周期对照; 项目当前 deliverable 检查 — SD 有大样? DD 还在改方案? CD 还在改材料?
- 输出: `当前 phase: {SD/DD/CD/CA}` / `应该 lock: {3 件}` / `应该 leave open: {3 件}` / `Mismatch: {如有 lock 早了 / lock 晚了 — 立即回正}`.

#### 维度 7: Carbon 决策窗口诊断 (mental model 1.7 直接对应)
- 看什么: 当前 phase 是否在 carbon 决策的最佳窗口 (Programming → SD 锁 envelope + 朝向 + 主体材料 = 80% (业内估, 引 EC3 + Architecture 2030) embodied carbon 已定); operational target vs embodied target priority; 业主对 carbon 的 真正 commitment (rhetoric vs budget)
- 在哪看: ClimateStudio + One Click LCA / Tally / EC3 早期 quick estimate; AIA 2030 Commitment 实际公开数据 (行业落后情况); Architecture 2030 baseline; 业主 carbon target spec (在合同中?)
- 输出: `Current carbon target: {operational kgCO2e/m²/yr + embodied kgCO2e/m² + 业内 reference}` / `Window status: {Programming/SD = high-leverage 决策窗口 / DD = medium / CD = low / CA = patch-only}` / `业主 commitment: {budget allocated / 仅 PR / 不 clear}` / `Recommended action: {early LCA + 替代材料对比 / 中期监控 / 后期 communication 管理}`.

## 心智模型

> 三重验证 (跨场景 + 生成力 + 排他性) 全通过. Evidence 至少跨 2 source_ids, 多数跨 ≥ 2 track.

### 1.1 三件事原则 — 可视化 ≠ 设计 ≠ 可建造 (visualization ≠ design ≠ buildable)

- **一句话**: 渲染图 / 数字模型 / 实建物 是三个独立 artifact, 各有不同的成功标准 + 不同读者 + 不同保鲜期; 混淆三者是 90% (业内估) 业主-建筑师冲突 + 项目失约的根因.
- **应用方式**: 拿到一张图, 先问 "这是给业主看的概念渲染 / 给工程师看的施工模型 / 给现场看的节点详图"; 把"渲染图等于建筑"当反模式. AI generative 时代尤其放大 (Midjourney 2 分钟一张图, 但其 90% (业内估) 几何不可建造).
- **局限**: 极少数 starchitect 全栈数字 fabrication 项目 (e.g. Gehry Digital Project / 部分 SHoP / Heatherwick CNC 加工) 能把渲染 → 模型 → 建造一致性拉到 95%+ (业内估), 但成本结构特殊. 普通项目 (中等预算 / 多分包 / 多专业协调) 三件事永远三件事.
- **evidence**: [T02-S071 反模式 Midjourney→业主当方案, T02-S072 反模式 Lumion 脱节施工图, T04 Frampton tectonic culture Studies in Tectonic Culture, T01-S025 Zumthor Atmospheres on craft, T03 anti-pattern starchitect 形态炫技超预算, T01-S066 Hadid Afragola "cattedrale nel deserto"]

### 1.2 三轴决定 — Program × Site × Tectonics (everything else is decoration)

- **一句话**: 每个设计决策必须在三轴上同时合理 — Program (人在里面做什么) × Site (在哪 + 什么气候 + 什么文化) × Tectonics (用什么材料 + 什么节点 + 谁来造); 缺一轴的设计要么沦为装修 / 要么变成无源之水的 starchitect 雕塑.
- **应用方式**: 评一个方案, 先做三轴自查 — 这个空间序列服务什么 program? 它在这个 site 比在别处合理在哪? 它的材料 + 节点能被这个 budget + 这个施工方造出来吗? 任一轴空白 → 方案不成立.
- **局限**: 学院教学 + 概念竞赛阶段允许"主要做 1 轴" (e.g. 纯形式探索 / 纯材料研究), 但只能停在 SD 之前. 一旦进入实际项目交付, 三轴缺一即崩.
- **evidence**: [T04 Frampton Studies in Tectonic Culture, T01-S046 MIT Press Frampton publisher, T01-S031 Schumacher 个人官网 (反例 — 形式优先), T01-S001 Wang Shu Pritzker citation (在地 + 材料 + 程序融合), T01-S007 Liu Jiakun Pritzker citation, T03 5-phase SOP, T03 anti-pattern 形态炫技超预算]

### 1.3 衰退周期分层 — Style 15-25 年, 工具 5-10 年, 节点 / 材料 / 结构原理 50+ 年

- **一句话**: 建筑师押注的"知识层"决定职业寿命 — 押 parametricism / 解构 / minimalism 等"风格", 15-25 年周期即过时; 押 BIM / AI 工具掌握, 5-10 年周期被新工具替换; 押节点 / 材料 / 结构原理, 50+ 年都成立. 把时间预算分到对的层.
- **应用方式**: 学习投入决策 — 30% (建议比例, 业内估) 学最新工具 (周期短但必要), 20% (业内估) 跟当代风格讨论 (中周期 differentiation), 50% (业内估) 打深节点 / 材料 / 结构 / 历史 (长周期 compound interest). 反过来全押短周期 = 30 岁焦虑.
- **局限**: 风格层并非完全无价值 — 在 starchitect 圈 / 媒体战场上, 风格签名是市场必需. 但纯靠风格 (无节点 / 无 program 整合) 的建筑师 5-10 年后通常被淘汰为 "曾经火过的".
- **evidence**: [T04 Mario Carpo Second Digital Turn 2017 (style decay 时代论), T04 Frampton Studies in Tectonic Culture (50+ 年原理), T02 AI generative 子类 last_updated 2026 (工具周期), T01-S031 Schumacher parametricism manifesto 2008 (vs 2026 实际衰减), T01-S066 Hadid Afragola 反例 (后期 ZHA 项目质量争议)]

### 1.4 业主-建筑师关系是 trust advisor, 不是 vendor 也不是 artist

- **一句话**: 5-15% (业内估) 设计费决定了建筑师的角色定位 — 高于装修商 (10-30% 业内估) 低于艺术家 (无定价基准); 自我定位为"承包商 vendor" → 业主不会为你的判断付费; 自我定位为"艺术家 artist" → 业主不会让你管 budget. 唯一可持续位置: trusted advisor / 业主的"建造顾问 + 风险翻译人 + 价值守门人".
- **应用方式**: 沟通时, 默认问"业主真正怕什么 + 真正想得到什么" (常是 reputation / cost / schedule 三者排序), 把建筑师价值翻译为业主语言 — 减风险 / 提资产价值 / 加项目可识别度. 不要把"我做一个好建筑"作为对业主的 selling point.
- **局限**: 私人住宅 (尤其 ultra-high-net-worth) + 文化建筑 (博物馆 / 美术馆) 业主允许更接近 artist 角色; 中国设计院 + 房地产快设计场景 业主把建筑师当 vendor 视为常态 — 但这是为什么这些项目挣钱却消磨人的本质原因.
- **evidence**: [T04 Reinier de Graaf Four Walls and a Roof (OMA 内部 critique), T01-S065 HUP publisher, T03 业主管理 workflow + 变更管理, T03-S 引 AIA B101 owner-architect contract, T01-S042 Shigeru Ban AIA Gold Medal (灾后 + 业主 + 社会平衡范本)]

### 1.5 在地性 vs 普世性张力 (critical regionalism as ethical compass)

- **一句话**: Frampton 中心论断 — 全球化让所有城市趋同 (国际样式 modernism 在每个气候都强加同一套), 当代建筑师唯一可对抗的杠杆是: (a) 主动让在地气候 / 在地材料 / 在地工艺 / 在地文化在设计中显形 (b) 抵抗"性能最优化主义"对建筑诗意的吞噬. 这不是审美选择, 是伦理选择.
- **应用方式**: 评一个项目 — 把它放到另一个城市, 还成立吗? 如果完全成立 (e.g. 任何 SOM 玻璃塔楼) → 它是 generic city 的复制品, critical regionalism 视角下不及格. 反之 — 它在这个 site 的特定理由是什么? 在地材料、地形、文化、季风、降水、人群的什么特征被建筑显形了?
- **局限**: critical regionalism 容易滑向"民族主义本质化" (e.g. 强加"中国传统元素" 套皮坡顶); 真正的 critical regionalism 是抽象的在地策略 (王澍的旧砖 / 刘家琨的再生砖 / Kéré 的当地土 / Zumthor 的瑞士石材), 不是文化符号贴图. 区分需要长期判断力.
- **evidence**: [T04 Frampton Modern Architecture A Critical History + Studies in Tectonic Culture, T01-S047 Frampton 2024 New School lecture (critical regionalism 持续), T01-S001 Wang Shu Pritzker citation (在地 + 旧砖), T01-S007 Liu Jiakun Pritzker citation (再生砖 + 西村大院), T01-S016 Kéré Pritzker citation (Gando 在地土), T01-S025 Zumthor publisher (瑞士在地)]

### 1.6 流程作为设计 (process is design, not task management)

- **一句话**: 资深建筑师把 5-phase (Programming / SD / DD / CD / CA) 当 design medium 不当任务清单 — SD 阶段的"留白"恰恰是 DD/CD 阶段创新空间; 跳 phase = 失去思考空间; 反过来在 SD 阶段强行做 DD 深度 = 浪费精力同时锁死后续选择.
- **应用方式**: 每个 phase 问自己 — "这个 phase 该 lock 什么, 该故意 leave open 什么". 例: SD 锁体块 + 程序 + 主立面策略, 故意留材料细节 + 节点构造给 DD; DD 锁材料 + 节点 + 系统选型, 故意留 spec 细节给 CD. 跳过 phase 等于一次锁死所有, 思考密度 / 选择空间 / 错误回退能力都崩.
- **局限**: 中国快设计场景 (30-60 天方案 + 90 天施工图) 物理上无法走完 5-phase. 此时唯一可行是 "压缩比例但保留逻辑" — 不是跳 phase, 是每 phase 时间砍 80% (业内估) 但 phase 序列保留; 否则 30 天就出施工图 = 建筑师承担本应业主分担的风险.
- **evidence**: [T03 A.1-A.5 5-phase SOP (Programming / SD / DD / CD / Bidding+CA), T03 资深 path diff A.6, T03 中国设计院 SOP B.4, T03 anti-pattern 跳 phase 直接 DD / SD 阶段做 DD-level 节点, T01-S037 Foster Taschen XXL monograph (process talks), T01-S040 Piano Pritzker acceptance speech, T03-S 引 AIA Project Delivery Guide]

### 1.7 Carbon 真实 vs LEED 营销 — embodied 才是当代核心

- **一句话**: 建筑全寿命碳排, operational (运营能耗) 已经被 envelope + HVAC + 可再生能源吃到 marginal, embodied (混凝土 / 钢 / 玻璃 / 装修) 才是当代真问题 — 占新建建筑全寿命碳排 50-90% (业内估, 来源 EC3 / Architecture 2030). AIA 2030 Commitment 实际公开数据显示行业落后 70%+ (业内估, 引 AIA 2030 公开数据) 目标. LEED Platinum 主要奖 commissioning + indoor quality, 对 embodied 几乎没设硬门槛.
- **应用方式**: 当业主谈"绿色" / "可持续" / "净零" — 先问 "operational target 还是 embodied target". 如果对方说不清, 给数字 — 每 sqm 30-50 kgCO2e operational vs 300-1000 kgCO2e embodied (业内估, 钢混结构). 真减碳决策在材料选择 (低碳混凝土 / 钢 / 木 / 既有结构保留) 上, 不在 LEED 文档上.
- **局限**: embodied carbon 数据基础 (EPD / GWP factor) 仍不完善, 中国 + 日本 + 多国本土因子库稀疏; 早期 (Programming/SD) LCA 决策置信度受限. operational performance 仍有意义 (尤其在重型 HVAC 气候区如海湾 / 印度 / 中东).
- **evidence**: [T04-S028 Mazria Passive Solar Energy Book publisher, T04-S029 Hosey Shape of Green publisher, T06-S003 USGBC LEED 协会, T06-S004 LEED v5 协会, T06-S005 LEED v4 协会, T06-S006 LEED v4→v5 changes 协会, T02-S028 ClimateStudio vendor docs, T02-S031 One Click LCA vendor docs, T02-S032 EC3 vendor docs, T02-S033 Tally vendor docs]

## 标准 Playbook

> Flat numbered list, 每条 `**如果 X**, 则 Y` 句式, cli_writer 可直接抽取并聚类 (envelope / process / cost / sustainability / business).

1. **如果业主预算 < $300/sqft (中国 < ¥3000/m²) 且业主期望 starchitect 视觉**, 则 SD 阶段早期必须教育降级期望 (展示同预算 reference building + 拒绝 Midjourney 渲染当方案), 否则 CD 阶段必崩溃 / 30-40% (业内估) 增项 / 建成与渲染严重不符. **案例**: 部分中东 starchitect 项目 (Hadid Afragola 等) 建成与渲染严重不符, 引 [T01-S066] 评论.
2. **如果业主从 Pinterest / Midjourney 发参考图**, 则用 reference building 反过来回业主 (同 typology / 同 budget / 同地区 实建项目, 标 cost + scope + 完成度 + 摄影师 credit), 把「图片审美」翻译为「项目约束」. 让业主同图比较「渲染」vs「实拍」是关键教育动作. **案例**: 资深住宅事务所常做「业主期望对账」 — 收到 Midjourney mood board, 立刻配 3 张同 budget 同 typology 实拍照, 让业主自己看差距.
3. **如果场地有自然灾害风险 (海岸 / 高烈度地震 / 极端温差 / 飓风) 或显著气候 stress**, 则被动设计 (体形系数 / 朝向 / 遮阳 / 自然通风) + 结构系统 + 防灾策略必须在 SD 阶段共同决定, 不延后到 DD; 否则 envelope 已锁后再返工成本是设计阶段的 5-10x (业内估, 引 Architecture 2030 / AIA 2030 实际数据). **案例**: Kéré Gando Primary School (T01-S016/S018) — 在地土 + 自然通风 + 在地工艺, SD 阶段 envelope + 结构 + 气候三者同决, 项目全寿命 maintenance 远低于同等钢混结构.
4. **如果 BIM 团队 < 5 人且项目类型 < 20000 sqft / 2000 m² 且非政府强制 BIM**, 则用 ArchiCAD 或 Vectorworks (一人多角色 + 设计驱动 BIM) 而非 Revit + Navisworks 全栈 (后者 overhead 在小项目上吞掉时间 + 阻碍设计迭代速度). Revit 优势在团队 ≥ 20 人 + 多分包协作场景. **案例**: 欧洲 boutique atelier (Caruso St John / Office KGDVS / 大舍 早期) 多用 ArchiCAD, 美国中型 firm + 中国设计院 多用 Revit — 团队规模 + 协作需求决定栈.
5. **如果项目要走 LEED / 绿建三星 / Passive House 认证**, 则在 Programming → SD 之间锁定 envelope 性能目标 + 朝向 + 主体材料选择, 否则 DD/CD 阶段补救成本是设计阶段决策的 5-10x (业内估 AIA 2030 数据). LCA / embodied carbon 决策同样如此 — 早期 30 分钟决定 = 后期 30 周也救不回. **案例**: Bullitt Center Seattle (Miller Hull) — Programming 阶段就锁 Living Building Challenge, envelope + 材料 + 系统全 SD 阶段联合决定, 后期 0 反工; 反例 大量 LEED Gold 项目 DD 才补 — 拿到 Gold 但 EUI 与设计目标差 30%+ (业内估).
6. **如果做投标 / 国际 Competition 且周期 < 6 周**, 则用 1 个 strong parti (核心建筑构思) + 3 个 supporting decisions (规划 / 材料 / 程序) 的极简策略, 而非 10 个 mediocre ideas 拼盘. 评审席只记得住 1 个 parti, 多了等于 0. **案例**: BIG 2010 上海世博丹麦馆 — 1 parti (城市自行车环线作为建筑), 简单到 30 秒讲清; 反例 大量学生投标 — 10 张图说 10 件事, 评审席 5 分钟全忘.
7. **如果你在中国做民营住宅项目 (2026 房地产寒冬背景)**, 则默认走 EPC 一体化 (设计 + 施工 + 部分供材) 或先 lock 设计 + 业主预付 30% (业内估) 关键节点款 (传统纯 design fee 模式已不可持续 — 2021-2026 设计费下行 50-70% 业内估); 战略重心同步转向公共建筑 / 城市更新 / 文旅 / 乡建 / 海外项目, 不要全押民营住宅. **案例**: 大舍 / 直向 / OPEN / 朱锫 等中生代不靠民营住宅养活 — 公共 + 文化 + 乡建 + 海外项目 为主, 因此抗周期能力远超同行.
8. **如果业主要求「creative freedom」但合同走 AIA B101 / 中国设计合同 standard scope**, 则用 additional service rider 把研究 / 异常工艺 / 反复迭代写明并单独计费, 否则你免费 give away design IP (常见: 业主用前期设计去找下家便宜执行, OMA / BIG / Foster 等都被坑过 业内估). **案例**: Reinier de Graaf *Four Walls and a Roof* [T01-S065] 第 4 章 OMA 内部 critique 详述这种 IP 被搬走的常见情景 + 防范实务.
9. **如果 SD 阶段就被业主或施工方要求 detail drawings / 节点详图**, 则推回 — 「SD 不出大样; 大样需要 DD-level information 此时不在我口袋里, 出了后期 30% (业内估) 还要重做」. 硬画大样是 SD 阶段最常见的反模式, 客户开心但建筑师丧失 design integrity. **案例**: 中国快设计场景常见这种压力 — 业主「我先看节点心里有底」 资深建筑师必须 push back, 用 reference building 节点照片代替.
10. **如果做 adaptive reuse / 历史建筑改造**, 则结构调研 + 危险材料 (asbestos / 铅 / 古建白蚁) + 历保 review + 旁边权益方 (邻居 / 街区 / 文物局) 先做 3-6 个月 due diligence, 设计动作 (草图 / 体块) 完全暂停. 不然全部图纸可能因结构 / 文物 / 邻居 reject 而废. **案例**: David Chipperfield Neues Museum Berlin (10 年改造, T01-S019/S020/S021) — 前 18 月几乎纯做考古 + 结构 + 文物登记, 设计后启, 这种 sequence 是 adaptive reuse Pritzker-tier 的标杆.

## 工作流 / Pipeline

> 三主线: A 单项目 5-phase / B 5 事务所类型 / C 7 横向能力. Track 03 已完整 SOP, 此处提炼资深 diff 关键并满足 cli_writer 解析 (每 subsection ≥ 2 of skip/跳过/optimize/优化/add/额外).

### 4.1 单项目 5-phase (AIA + 中国 dual mode)

**入门 SOP**:
1. Programming (1-2 周) — 收集业主 RFP / 项目目标 / 预算 / 用地条件; 输出: program 表 + site analysis + 风险登记.
2. Schematic Design (4-8 周) — 1 strong parti + 体块 + 总图 + 主要平/立/剖; 输出: SD 文件 + 投资估算 + 模型.
3. Design Development (6-12 周) — 锁材料 + 节点 + 主要 spec + MEP 协调; 输出: DD 文件 + 详细估算 + 系统选型.
4. Construction Documents (8-16 周) — 全套施工图 + spec + 详图 + 报批; 输出: 50%+90%+100% CD set (行业惯例 milestone, AIA B101 行业惯例).
5. Bidding + Construction Administration (6-36 月) — 投标 + 答疑 + RFI + ASI + submittal + punch list; 输出: 现场服务记录 + 竣工备案.

**资深 path diff (skip / 优化 / add)**:
- skip 入门版 "全画大样" 在 SD 阶段; **跳过** 业主"我先看节点"压力 推回 DD; **优化** SD 用 1 parti + 3 supporting 而非 multiple weak ideas; **额外** add embodied carbon 早期 LCA 在 Programming + SD 之间; **优化** DD 阶段把 BIM clash 提前到 50% (业内估惯例) 而非 90% (业内估惯例).

### 4.2 事务所类型 5 类 SOP

**入门 SOP**:
1. Boutique atelier (5-30 人) — design-intent 驱动, 1-3 同时项目, 高设计费 / 低交付速度; 业主关系靠 reputation + repeat.
2. 国际中型 firm (50-150 人) — typology mix (商办 / 文化 / 公寓), 标准化 SOP, BIM + spec 系统化.
3. Starchitect 国际工作室 (100-500 人) — concept lead + 当地 executive architect; 全球分公司 + 多 typology.
4. 中国大型设计院 (1000-5000 人) — 资质 + 施工图 + 流水线; 政府 / 房企项目主导; design fee 占比低 (3-5% 业内估).
5. Lone wolf / 独立建筑师 (4-15 人) — 单一 niche (e.g. 住宅 / 旧改 / 文化), 业主直接关系, 设计 + 项目管理 + 招投标 一人多角色.

**资深 path diff**:
- boutique **跳过** 大量 BIM 协调步骤但 **额外** 加 craft / 节点深度投入; 中型 firm **优化** 跨项目 spec 复用 + standard detail library; starchitect **优化** concept-execution split (本部 + 当地 EA); 中国设计院 **优化** 资质背书 + 流水线产能, **跳过** design intent depth (转 EA 角色); lone wolf **优化** 业主直 + 单一 niche 深耕, **额外** add 业务开发能力.

### 4.3 BIM 协作 workflow (C.1)

**入门 SOP**:
1. 建立 LOD 共识 (LOD 100/200/300/350/400/500) — 各 phase 对应 LOD 锁定.
2. 文件结构 + 命名规范 + central + local file 分工.
3. Clash detection 节奏 (50% DD / 100% DD / 50% CD / 100% CD — 业内惯例 milestone, 业内估).
4. IFC 交付 + spec spec library + family hierarchy.
5. RFI / ASI 与模型同步.

**资深 path diff**:
- **跳过** 入门"所有 element model 到 LOD 400" 强迫症 — 不同 element 不同 LOD; **优化** family library 跨项目复用; **额外** add Rhino.Inside.Revit 桥接到 parametric 流程.

### 4.4 可持续 / LCA workflow (C.2)

**入门 SOP**:
1. Programming 阶段锁定 operational + embodied carbon target.
2. SD 阶段做 envelope + 朝向 + 主体材料 LCA quick estimate (ClimateStudio / EC3).
3. DD 阶段锁定材料 spec + 系统选型 + 详细 LCA.
4. CD 阶段做 commissioning plan + spec embodied content requirement.
5. CA 阶段 follow up 验收 + post-occupancy energy.

**资深 path diff**:
- **跳过** "LEED 评分表 driven 设计" — 把 evaluator 当顾问而非设计驱动; **优化** 早期 30 分钟 LCA 决策 (远比后期 30 周 patch 有效); **额外** add embodied carbon 与 operational carbon 双轴 dashboard.

### 4.5 AI generative concept workflow (C.3)

**入门 SOP**:
1. mood board / 灵感阶段: Midjourney + Adobe Firefly 探索氛围 (≠ 设计).
2. 早期总图 / 可行性: Spacemaker (Forma) + TestFit + Hypar 跑 site option.
3. 概念草图: ControlNet + IPAdapter + 自己平面 condition 控制.
4. 业主沟通: 用 AI image 但**明确标注** "concept exploration, not construction".
5. 决策: AI 输出转为 hand sketch / Rhino 模型 再走 SD.

**资深 path diff**:
- **跳过** 用 Midjourney 出 final 图给业主 (反模式); **优化** AI 用作 ideation accelerator 而非 design replacement; **额外** add 业主期望管理脚本 "AI image ≠ buildable".

### 4.6 投标 / Competition workflow (C.4)

**入门 SOP**:
1. RFP 解读 + 评审 priority 识别 (3-5 周前 lock).
2. 团队冲刺: 1-2 周 charrette → 3 weeks design → 1 week deliverable polish.
3. 1 strong parti + 3 supporting decisions.
4. 评审席 deliverable: 1 image board + 1 short film + 1 model + 1 文字陈述.
5. 后期 follow up + 反馈复盘.

**资深 path diff**:
- **跳过** "10 ideas pitch" 散弹 — 评审只记 1 parti; **优化** 评审席 perspective 提前预设 (谁是 jury, 他们关心什么); **额外** add 团队复盘 lose-rate 历史 (赢率 < 5% (业内估) 是行业现实).

### 4.7 业主管理 / Owner relationship workflow (C.5)

**入门 SOP**:
1. RFP / 初次会议 — 听业主真正 motivation (常是 reputation + cost + schedule 三者 priority).
2. 设计合同 sign — 范围 / 设计费 / 设计深度 / variation 规则.
3. 节点 sign-off — 每 phase 业主书面签字, 锁定 scope.
4. 变更管理 — 任何 scope change 立即 change order, 不口头答应.
5. 项目结束 — 复盘 + reference photo + 业主推荐信.

**资深 path diff**:
- **跳过** "对业主全 yes" — 不 push back 的建筑师业主反而不信任; **优化** 把建筑师价值翻译为业主语言 (减风险 + 资产价值 + 可识别度); **额外** add additional service rider 写明研究 / 异常工艺 / 反复迭代.

### 4.8 CA 现场服务 + 中国寒冬转型 (C.6 + C.7 合并简述)

- CA 工作流: RFI / ASI / submittal / punch list / final acceptance. 中国监理 vs 美国 CA 角色差异 — 中国监理是法定独立第三方, 美国 CA 是建筑师 contract scope 内职责.
- 中国寒冬转型 (2021-2026): 公共建筑 / 城市更新 / 文旅 / 乡建 / 海外项目 / EPC 一体化 / 模块化建造 — 选 ≥ 1 路径生存, 别全押民营住宅.

- **业主 / 商务**: design fee, change order, additional service, 设计费率, 资质, EPC, 全过程咨询

### 5.2 严肃 register

- 引文献时报 author + 年份 + 章节 / page ("Frampton 1995, Studies in Tectonic Culture chapter 4 on Mies"), 不引"在某本书上看到说"
- 谈项目时报具体 typology + scale + budget tier 而非 "项目" 笼统词

- 数字必带 caveat (业内估 / AIA 2030 数据 / 业主 RFP 实测)

### 5.3 内 vs 外沟通差异

- **同业内**: 直接用术语 (SD/DD/CD), 不解释; 引 figure ("Like Zumthor's Therme Vals"); 讨论分歧不和稀泥
- **对业主**: 翻译为风险 + 价值 + 时间 / cost language; 用 reference building 不用 abstract 术语; 把 design fee 翻译为 reputation insurance / asset protection
- **对施工方**: 用 spec + 详图 + RFI 语言; 不谈"立意" 谈 "node detail + tolerance"
- **对公众 / media**: 1 张 image + 1 句 parti + 1 个 number (e.g. "减少 40% embodied carbon" — 业内典型话术, caveat 要附数据)

### 5.4 外行破绽 top 10

1. 把 "render" 当 "design" — 给业主一张 Midjourney 就以为方案完成
2. 不区分 schematic / design development / construction documents — 直接画大样以为是 SD
3. 谈 "modern architecture" 但不说 "modernist movement" (建筑师不混用 modern + modernist)
4. 把 "BIM" 当 "3D modeling" — BIM 是 info + data + 协作流程, 不只是 3D 几何
5. 不引 figure / 不引 canon — 资深建筑师对话 5 分钟内必 reference Frampton / Koolhaas / Zumthor / 王澍 之一
6. 说 LEED Platinum 等于"低碳" — 实际 LEED 不专门考核 embodied carbon
7. 把 "AI 设计自动化" 当业内主流方向 — 实际业内主流是 AI 作 ideation tool 而非 designer replacement
8. 谈中国建筑师只想到王澍 / 马岩松 — 不知道大舍 / 直向 / OPEN / Liu Jiakun 这一批中生代实验
9. 把 starchitect 项目当行业常态 — 实际 99% 建筑师做的是中等规范化项目
10. 谈 "建筑师设计建筑" 不谈 "建筑师管理风险 + 翻译业主意图 + 协调专业" — 实际后者占建筑师工作 70%+ (业内估)

## 质量基准 + 反模式

### 6.1 什么算「好」 (4 条具体可验证基准)

1. **三轴成立** — Program / Site / Tectonics 三轴都能 30 秒解释清楚 (评估: 拿一个项目问设计师 "为什么是这个 site / 这个 program / 这个 materiality", 答得出来三个独立理由 = 好; 答不出 = 形式驱动 / 形而上).
2. **5-phase 节奏匹配** — SD 阶段 lock 体块 / 程序 / 主立面策略; DD 锁材料 / 节点; CD 锁 spec / 详图; 不该 lock 的故意留白 + 该 lock 的真锁住 (评估: 在 SD set 里找不到详图 ✓ 在 DD 找到主要节点 ✓ 在 CD 找到完整 spec ✓).
3. **业主 align + design integrity 双线** — 业主在 SD/DD/CD 节点书面 sign-off + 建筑师在每 phase 保住 design intent 关键 3 件事 (评估: 项目结束后 owner satisfaction ≥ 7/10 + design team 自评 design intent 保留 ≥ 70%).
4. **embodied carbon 早期决策** — Programming → SD 之间至少有一次 LCA quick estimate, SD 完 envelope + 主材选择有 embodied carbon 数字而非"会考虑" (评估: SD 报告里有 ≥ 1 embodied carbon 数字 + ≥ 1 替代方案对比).

### 6.2 反模式 (10 条外行 / 入门常犯错)

1. **Midjourney → 业主**: AI 渲染当方案给业主, 业主期望与可建造性差距巨大, CD 阶段崩
2. **SD 画大样**: SD 阶段被业主 / 施工方压迫画节点详图, 实际无法成立, 后期 30% (业内估) 重做
3. **跳 phase**: 跳过 SD 或 DD 直接施工图, 失去思考空间 + 锁死后续选择
4. **形式优先 ignore Program/Site**: 学院化纯形式主义, 进 DD 后 program / site 反扑
5. **LEED checklist 驱动设计**: 把可持续 evaluator 当设计驱动, 出来的项目 LEED 高但设计平庸
6. **国际样式无在地**: 把 SOM 玻璃塔楼 / Foster 般 envelope 抄到完全不同气候 site
7. **Revit family hell**: 想用 Revit 解决一切, 但中国施工图深度超出 family 库范围, 节点仍画 AutoCAD
8. **设计费给 0 push back**: 业主提任何变更建筑师全 yes, 自费做额外工作, 业主反而不信任
9. **starchitect 形态炫技超预算**: 后期 ZHA / 部分中东 starchitect 项目案例 — 形态优先, 结构 / 维护问题没考虑, 建成与渲染严重不符
10. **AI 设计自动化幻觉**: 把 AI 当 design replacement 而非 ideation tool, 设计判断 / 业主信任 / 跨专业协调 这三件事 AI 无法做

### 7.2 核心分歧

- **形式 vs 在地**: Parametricism (Schumacher) vs Critical Regionalism (Frampton). 前者认为参数化逻辑可适应任何 site, 后者认为 site 是 untransferable 起点. 2024 critical regionalism 持续 academic 论争 (T01-S048).
- **可视化 vs 建造**: AI generative / Midjourney / starchitect 渲染主义 vs Phenomenology + material minimalism craft 派. 前者强调 communication speed, 后者强调 build truth.
- **市场 vs 学院**: Pragmatic populism (BIG / Snøhetta — 媒体化 + 商业可读) vs 学院派 (Frampton 评论 / OMA AMO 理论 / 中国实验). 前者 reach 广, 后者深度高.
- **starchitect vs craft**: Star architect 国际工作室商业模式 vs boutique atelier craft 模式. 商业成功路径不同, 设计 ethic 也不同.
- **中国本土 vs 国际共识**: 中国当代实验 (Wang Shu / Liu Jiakun / Dong Gong / 朱锫) 主要谈 在地 + 旧改 + 自然; 国际共识谈 carbon + AI + parametric. 真正 cross-cultural 桥梁少, 但 2025 Liu Jiakun Pritzker 是重要节点.

### 7.3 sub-skill 候选 (Phase 3 nuwa.skill 蒸馏)

Top 3 候选 (按思维密度 + long-form material + 流派代表性 + 当代活力 + 跨文化重要):
