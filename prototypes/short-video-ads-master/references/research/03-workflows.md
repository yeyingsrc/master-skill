# Track 03 — Workflows (短视频投流)

> Mini-viable for prototype.

### 1. 新计划冷启动

- **One-liner**: 从计划上线到稳定跑量的标准动作
- **Trigger**: 新品上市 / 新场景 / 新人群拓展
- **Output**: 计划度过冷启动 + 进入稳定跑量阶段
- **入门 SOP**:
  1. 准备 3-5 条素材 (一组测试)
  2. 设定保守预算 (日 500-2000) 跑 24-72 小时
  3. 不动出价 / 不加预算 / 不调定向 (让系统跑探索期)
  4. 72 小时后看完播率 + ROAS, 留优秀素材关失败素材
  5. 优秀计划逐步加预算 (每次加 20-30%, 不超过)
- **资深路径**:
  - skip 一开始就大预算放量 (新手最常见错误)
  - optimize 测试期素材一次只换一个变量 (前 5s / 卖点 / 演员), 便于归因
  - add 准备好素材 backup, 一旦爆了 24 小时复刻 3-5 条
- **常见失败模式**:
  - 冷启动期单日数据差就关计划 → 系统刚找到人群就被打断
  - 一次性大幅加预算 → 打断学习, 计划反而衰减
  - 同时测太多变量 → 数据出来不知道是哪个起作用
- **Last_updated**: 2025-12-15
- **Decay risk**: high

### 2. 爆量计划日常优化

- **One-liner**: 计划进入跑量阶段后, 持续优化以延长跑量周期
- **Trigger**: 计划进入稳定跑量, 每天 ROAS 达标
- **Output**: 跑量周期延长 + 素材衰减节奏匹配产出节奏
- **入门 SOP**:
  1. 每天看素材衰减曲线 (完播率 / CTR 是否下滑)
  2. 衰减开始就预备新素材 (不要等死了再补)
  3. 加预算时小步加 (20% 一次), 不要 ALL-IN
  4. ROAS 突然异动先查素材 + 定向, 别动出价
  5. 周复盘: 哪些素材跑得好, 共同特征是什么 → 沉淀成 SOP
- **资深路径**:
  - skip 大幅调价 / 大幅改定向 (打断系统学习)
  - optimize 用爆款素材的脚本结构复刻新素材, 不是从 0 创意
  - add 多账户 / 多产品组合投放 (摊平单计划风险)
- **常见失败模式**:
  - 素材衰减不补新的 → 计划死了再来一波冷启动很贵
  - 数据异动就大幅调出价 → 系统模型混乱, 后续更难跑
  - 不做周复盘 → 重复犯错
- **Last_updated**: 2025-11-25
- **Decay risk**: medium
