## 学派分歧 (canon 三角)

**议题 1: 30% Apple Tax 是否合理**
- Apple 派 (T04-S001 §3.1.1, T04-S007): 平台投资 + 安全 + 分发的合理对价, 中小开发者 < $1M 已降到 15%
- 反 Apple (T04-S011 Epic, T04-S018 Spotify): 「supracompetitive operating margins」(2025 Rogers J. 用语), 反垄断
- Indie 中立 (T04-S027 Marco 2013): 30% 不是核心问题, App Store 发现机制 + top lists 失灵才是
- 国内视角 (T04-S035): 30% 之外另叠 ICP + 算法备案 + 8-10 应用市场分成, 实际 cost 远超 30%

**议题 2: Sideloading**
- Apple 派 (Cook 2020 testimony, T04-S012): 安全风险 + 用户保护; EU DMA (T04-S014) 下做 alt marketplace, 但收 Core Technology Fee
- 反 Apple (T04-S018): Apple DMA 合规「blatantly disregards」立法精神; CTF 是变相税
- 国内: 不存在 sideloading 议题, 因为本身就是 8-10 应用市场并行 + 大陆区 App Store 分立, 备案才是核心

**议题 3: IAP 强制 / Anti-Steering**
- Apple 派 (T04-S001 §3.1): 数字商品/服务必须 IAP; 2024-01 起在美允许 link-out (Epic 案 injunction)
- 反 Apple: Spotify EU €1.84B 罚款主因; Epic 2025-04 Rogers 裁定 Apple "willfully" 违反
- Indie (T04-S029 Smith / T04-S027 Marco): 实操 IAP 摩擦最低, link-out 财务收益常被合规 + 转化漏斗下降抵消

**议题 4: Apple docs 是不是「全部真相」**
- 官方派: docs 是唯一权威, rejection 都有 guideline 引用
- Indie 派 (T04-S027/S028/S029/S030): docs 写了但不执行 / 没写但 reviewer 在管的灰色地带占 rejection 大头, 真实经验靠 rejection corpus
- 国内派: 官方 docs 完全不覆盖大陆区流程 (备案 / 算法 / 版号 / 多市场), 必须叠加国内 .gov.cn + 鸟哥/七麦经验

## Phase 2 提炼提示

**反复出现 ≥ 3 个 canon 都讨论的核心 idea** (候选行业心智模型):
- 「Apple App Review 是黑盒, 任何 guarantee 通过审核都是假的」(T04-S001 + T04-S027 + T04-S029 + Indie corpus) → 心智模型: rejection 不是失败, 是迭代的一步
- 「政策变更比代码变更频繁」(T04-S010 + WWDC 年度 + DMA/DSA 法律 + 国内合规季度收紧) → 12 月以前的攻略多数过期
- 「合规叠加比合规单一更难」(国内 ICP+算法+版号 vs 海外 GDPR+CCPA+ATT vs DMA+DSA) → playbook: 先按 region 列合规清单再做 architecture
- 「IAP/订阅是基础设施而非 feature」(T04-S008 + T04-S032) → 订阅 infra 是 launch 第一周决策, 不是 monetization 阶段决策

**智识谱系种子** (5 流派):
- A (Apple 派, 官方权威): App Review team / WWDC / DPLA + Cook 2020 antitrust 听证 + Phil Schiller App Store leadership era
- B (Indie 派): Marco Arment / David Smith / Federico Viticci / John Gruber / Sebastiaan de With / Joe Cieplinski 长博客 + 长 podcast 路线
- C (反 Apple 派): Epic / Spotify / Basecamp HEY 反垄断阵营 + EU Commission 立法 — Sweeney / Ek / DHH
- D (ASO/数据派): Sensor Tower / RevenueCat / data.ai / Mobile Action / 鸟哥 / 七麦 — 数据驱动工具厂学派
- E (国内合规派): 工信部 / 网信办 / 出版署 + 国内出海创业者 (硬地骇客 / 张小珺) — 海外完全不覆盖
- 主要分歧: A vs C 法律对抗; A vs B 文化共生 + 操作分歧; D 横切前 3 派; E 平行宇宙

**冷僻 / 信号薄弱**:
- canon 35 ≥ floor 20 ✅, 一手率 86% ≥ 80% ✅, 三角 (Apple/反 Apple/国内) 齐 ✅
- 经典书维度偏弱 (ASO Bible 唯一系统化书, 其他都是长博客 corpus) — 行业特征: 「政策频变 + docs-first」, 系统化书一出版就过时, canon 主轴必然是 docs + 法律 + 长博客
- 国内合规 canon 几乎全靠 .gov.cn, 缺商业书 — 国内 SaaS 出海长访谈是仅有非官方一手, 其余 SEO 文章
