## 本行业工具栈的元判断(先读这个)

国内房产投资的「工具」不是软件,是**数据口径 + 查档渠道 + 算账模型**。三条贯穿全文的原则:

1. **官方网签 / 备案成交数据 > 中介挂牌价**。中介平台(贝壳 / 安居客)的「挂牌价」是卖方报价 + 平台撮合产物,系统性偏高且滞后;房管局网签备案是法律意义上的真实成交,是唯一可信价格锚。挂牌价只能看「卖家心理预期」,不能当成交价用。
2. **流动性指标(挂牌量 / 成交周期 / 去化率) > 价格指标**。2024-2025 深度下行期,价格阴跌但「卖不掉」才是真风险。一个城市/板块的二手房挂牌量暴增、成交周期从 3 个月拉到 12 个月,比房价指数掉 5% 信息量大得多。工具的使用重点是「读出流动性」。
3. **信息社区类工具(论坛 / 知乎 / 小红书 / 公众号)噪音极大,只作情绪与线索,绝不作决策依据**。这一行「大师」大量是利益相关方(卖课 / 分佣 / 喊单只晒对不晒错),社区工具用来「发现一个新政策传闻 / 一个板块的负面情绪」,然后必须回到官方数据和不动产登记中心查证。

> ⚠️ 红线工具:本文不收录任何服务于「经营贷置换房贷 / 首付贷 / 阴阳合同 / 假流水」的中介或工具 —— 这些是违规甚至违法操作,见「避坑清单」。

## 必备层(≥80% 从业者用)

### 1. 国家统计局 70 城房价指数

- **One-liner**:唯一全国可比的官方房价价格序列 —— 给整个市场定基调,判断「现在是周期什么位置」的起点。
- **Tier**:必备
- **Maintainer / Owner**:国家统计局(www.stats.gov.cn)
- **License**:proprietary(政府公开数据,免费查询,不可商用再分发)
- **Maturity signals**:
  - 数据序列:70 个大中城市住宅销售价格变动情况,2011 年起按月发布,延续至今
  - 发布节奏:每月中旬发布上月数据(2026-05 仍正常发布)
  - Activity:healthy —— 制度性月度发布,十余年无中断
- **怎么用它真正算账**:
  - 看「环比」而非「同比」—— 同比滞后一年,环比才反映当下拐点。连续 N 个月环比为负 = 趋势性下行,不是噪音。
  - 分城市能级看(一线 / 二线 / 三线分类汇总)—— 它本身就告诉你「没有全国房价」,一线企稳和三线还在跌可以同时发生。
  - 它是「指数」不是「价格」—— 不能用来算某套房值多少钱,只能判断方向和节奏。微观估值要用网签成交 + 中国房价行情网。
- **不适合 / 替代方案**:不适合微观估值(某小区某户型的价)、不适合判断流动性(它只有价格没有成交量/挂牌量)。流动性要看各地房管局网签系统。
- **生产案例**:任泽平、李迅雷、香帅等经济学家的房地产周期研究均以 70 城指数为价格基准序列(见 Track 01 / Track 04 关联);各研究院月报开篇几乎都引用它。
- **维护者背景**:政府统计机构,无利益关联 —— 这是它最大的价值,相对中介数据无「唱多」动机。
- **近期变化**:2024-2025 下行期数据持续走弱,统计局对样本与口径的微调需留意,但月度发布制度未变。
- **来源**:
  - [Primary] 国家统计局月度数据发布(evidence: [T02-S001],collected: 2026-05-14)
  - [Primary] 国家数据库历史序列查询(evidence: [T02-S002],collected: 2026-05-14)
- **可信度**:high
- **Decay risk**:low(24+ 个月内被显著改变概率 < 20% —— 制度性发布,行业基础设施)

### 2. 各地住建局 / 房管局网签备案系统

- **One-liner**:法律意义上的真实成交数据(成交价 + 成交量) —— 价格的唯一可信锚,也是流动性的第一手来源。
- **Tier**:必备
- **Maintainer / Owner**:各城市住建局 / 房管局(如北京市住建委 zjw.beijing.gov.cn、深圳市住建局 zjj.sz.gov.cn)
- **License**:proprietary(政府系统,各地开放程度不一 —— 有的公示日度网签量,有的只在交易时可查)
- **Maturity signals**:
  - 覆盖:每个有商品房交易的城市都有网签备案系统,是交易合法性的前置环节
  - 数据可得性:差异大 —— 一线城市多公示日度/月度网签套数;部分城市需到现场或通过中介调取
  - Activity:healthy —— 网签是法定环节,系统不会消失
- **怎么用它真正算账**:
  - 网签备案价 = 真实成交价,把它和贝壳挂牌价对比,差额就是「卖方预期 vs 市场现实」的缺口,下行期这个缺口能到 10-20%。
  - 日度/月度网签套数 = 流动性的直接读数。网签量同比腰斩 = 这个城市「卖不掉」,比房价指数更早预警。
  - 看「网签 - 备案」的时间差和撤销率 —— 大量网签后撤销 = 假成交/烂尾/贷款批不下来,是危险信号。
  - 二手房网签量 vs 新房网签量 分开看 —— 二手房网签量是真实换手需求,新房网签量混了开发商促销节奏。
- **不适合 / 替代方案**:不适合做城市间横向对比(各地口径不一)—— 横向对比用统计局 70 城指数。不适合趋势平滑分析(月度波动大)。
- **生产案例**:贝壳研究院、中指、克而瑞的城市月报中「成交套数/面积」数据底层即各地网签备案;严肃媒体报道城市市场冷热也以网签量为据。
- **维护者背景**:政府主管部门,无唱多动机。
- **近期变化**:2024-2025 多地为「稳预期」对网签数据公示频率/颗粒度有调整(有的减少公示),需逐城市确认当前可得性。
- **来源**:
  - [Primary] 北京市住建委(网签/政策公告样板,evidence: [T02-S014],collected: 2026-05-14)
  - [Primary] 深圳市住建局(evidence: [T02-S015],collected: 2026-05-14)
- **可信度**:high
- **Decay risk**:low(系统本身稳定;但「数据公示颗粒度」是 medium 风险 —— 可得性可能收紧)

### 3. 不动产登记中心查档(抵押 / 查封 / 产权人 / 性质)

- **One-liner**:核查一套房的产权真相 —— 有没有抵押、有没有被查封、产权人是谁、是住宅还是商办,尽调不可跳过的硬环节。
- **Tier**:必备
- **Maintainer / Owner**:各地不动产登记中心 / 自然资源主管部门(如上海市规划资源局 shgtj.gov.cn);全国一体化政务服务平台聚合查询入口
- **License**:proprietary(政府登记系统,查档需产权人本人或利害关系人,部分城市支持线上)
- **Maturity signals**:
  - 覆盖:不动产统一登记 2015 年起全国推行,现已基本全覆盖
  - 线上化:部分城市支持「带押过户」线上办理与线上查档,多数仍需现场
  - Activity:healthy —— 法定登记机构
- **怎么用它真正算账(尽调用法)**:
  - 查「他项权利」—— 有没有抵押、抵押给谁、抵押金额。带抵押的房交易要走「带押过户」或「先解押再过户」,流程和资金风险完全不同。
  - 查「查封」—— 被法院查封的房不能过户。法拍房尤其要查是否有「轮候查封」(多个债权人排队)。
  - 查「权利人」—— 核对卖方身份证与产权证一致,共有产权要所有共有人到场。常见骗局是「假房主」。
  - 查「规划用途 / 性质」—— 住宅 vs 商办(40 年产权)vs 划拨地。性质决定税费、落户、水电价格、再交易难度。
  - 查「面积与四至」—— 实测面积 vs 证载面积,挑出违建/赠送面积无产权的部分。
- **不适合 / 替代方案**:它只回答「产权状态」,不回答「值多少钱」「能不能涨」。也查不到学区政策、规划变更 —— 那些要去教育局/规划部门。
- **生产案例**:这是房产交易律师、专业中介、谨慎投资人的标准尽调动作;跳过查档导致买到抵押房/查封房的纠纷案例在裁判文书网大量存在(与 Track 03 workflow「估值与尽调」环节强关联)。
- **维护者背景**:政府登记机构,权威性最高。
- **近期变化**:2024-2025「带押过户」在多城市推广,显著降低了带抵押二手房的交易资金风险(见新兴层)。
- **来源**:
  - [Primary] 全国一体化政务服务平台(不动产登记查询入口,evidence: [T02-S013],collected: 2026-05-14)
  - [Primary] 上海市规划资源局(不动产登记主管样板,evidence: [T02-S016],collected: 2026-05-14)
- **可信度**:high
- **Decay risk**:low(法定登记制度,行业基础设施)

### 4. 自建持有成本 / 租金回报率 Excel(算账模型)

- **One-liner**:把「总持有成本 vs 总收益 vs 流动性」算清楚的核心工具 —— 这一行没有标准软件,资深人都靠自建表格,模型本身就是认知。
- **Tier**:必备
- **Maintainer / Owner**:无 —— 自建。市面房贷计算器只算月供,算不清完整持有账。
- **License**:N/A(自建电子表格)
- **Maturity signals**:
  - 这是「方法」而非「产品」,不存在版本/star/commit —— 但它是 ≥80% 谨慎投资人的实际工作方式
  - Activity:N/A
- **怎么用它真正算账(模型必须包含的项)**:
  - **买入端**:首付 + 契税(首套/二套不同)+ 增值税 + 个税 + 中介费 + 贷款手续费 —— 算「真实入场成本」,不是房价。
  - **持有端(年)**:月供利息部分 + 物业费 + 取暖费 + 维修 + 空置月份损失 + 资金占用机会成本(首付如果买理财能赚多少)。注意:房产税试点城市(上海/重庆)要加这一项,且全国推进节奏不确定,长期测算要留余量。
  - **收益端**:租金(扣空置率,真实出租周期常有 1-2 个月空窗)+ 资本利得(这一项在 2024-2025 要敢于填 0 甚至负数)。
  - **退出端**:卖出时增值税/个税 + 中介费 + **流动性折价**(下行期挂 12 个月卖不掉,要降价 X% 才能成交,这个折价必须进模型)。
  - **核心产出三个数**:净租金回报率(年净租金 / 总投入)、盈亏平衡持有年限、「急用钱时的可变现价值」。
  - **杠杆纪律**:模型必须跑「房价跌 20% / 租金跌 30% / 利率升 1%」三个压力场景 —— 杠杆放大收益也放大亏损,要看清现金流会不会断。
- **不适合 / 替代方案**:不能替代尽调(产权)和市场判断(城市/板块)。市面房贷计算器只能算月供,作为子模块输入可以,但不能当完整账。
- **生产案例**:香帅《钱从哪里来》系列强调把房产放进家庭整体资产负债表算「占比与流动性」;谨慎投资人社群普遍以「现金流压力测试表」作为是否出手的硬门槛(与 Track 01 香帅、Track 04 关联)。
- **维护者背景**:N/A(方法论)。
- **近期变化**:2024-2025 下行期,模型重点从「算回报率」转向「算流动性折价 + 现金流断裂风险」。
- **来源**:
  - [Primary] 央行货币政策司(LPR/利率 —— 模型杠杆成本输入,evidence: [T02-S003],collected: 2026-05-14)
  - [Primary] 全国银行间同业拆借中心 LPR 发布(evidence: [T02-S018],collected: 2026-05-14)
- **可信度**:high(方法论共识)
- **Decay risk**:low(模型框架稳定;具体税率/利率参数是 medium —— 需跟政策更新)

## 场景特化层(特定子方向用)

### 5. 贝壳找房 / 链家

- **One-liner**:房源 + 历史成交记录 + 带资金监管的交易服务 —— 看盘和找房的起点,但「挂牌价」不是「成交价」,要会去伪。
- **Tier**:场景特化(看房选房 / 二手房交易场景)
- **Maintainer / Owner**:贝壳找房(KE Holdings,贝壳与链家同属一个体系)
- **License**:proprietary(免费用,平台有撮合佣金动机)
- **Maturity signals**:
  - 市场地位:国内二手房交易撮合规模最大的平台,一二线城市房源覆盖最全
  - 数据特色:提供「成交记录」(部分城市可见历史成交价)和「带看量/关注量」
  - Activity:healthy(2026-05 仍是行业头部)
- **典型使用场景**:
  - **房源普查与板块比价**:同板块同户型横向看挂牌价分布,建立价格感。
  - **成交记录回溯**:某些城市贝壳显示历史成交价 —— 这比挂牌价可信,但仍要和房管局网签交叉验证。
  - **带看量/关注度读流动性**:一套房挂了半年带看个位数 = 价格虚高或硬伤;带看多但不成交 = 价格还在博弈。
  - **资金监管交易**:走贝壳的资金监管账户,房款进监管账户、过户完成才放款,不走私下交易。
- **不适合 / 替代方案**:**挂牌价不可当成交价用**(系统性偏高 + 滞后);单一平台房源有盲区,要和安居客交叉。城市间横向比价不可靠 —— 用统计局指数。
- **生产案例**:贝壳研究院的市场报告底层即平台交易数据;几乎所有一二线城市二手房投资人都以贝壳为第一看盘工具(广泛行业实践)。
- **维护者背景**:平台方,有撮合佣金动机 —— 它希望「促成交易」,所以对市场情绪的呈现要打折看。
- **近期变化**:2024-2025 下行期,部分城市贝壳调整了「成交价」展示口径;「真实房源」核验、VR 看房等持续迭代。
- **来源**:
  - [Surrogate-Primary] 贝壳找房平台(vendor docs,evidence: [T02-S007],collected: 2026-05-14)
- **可信度**:medium(数据有商业动机,需交叉验证)
- **Decay risk**:medium(平台本身稳定,但「数据展示口径」12-24 个月内可能再调整)

### 6. 安居客 / 房天下

- **One-liner**:贝壳之外的房源与挂牌量交叉验证渠道 —— 补盲区、对口径,避免被单一平台数据带偏。
- **Tier**:场景特化(房源交叉验证 / 挂牌量监测场景)
- **Maintainer / Owner**:安居客(58 同城体系)/ 房天下(原搜房网,SouFun)
- **License**:proprietary(免费,平台有引流/佣金动机)
- **Maturity signals**:
  - 市场地位:二线梯队房源平台,部分区域/城市覆盖与贝壳互补
  - 房天下:旗下「中指研究院」体系产出房地产指数与城市报告
  - Activity:healthy(运营中,体量小于贝壳)
- **典型使用场景**:
  - **挂牌量交叉验证**:同板块在安居客和贝壳的挂牌套数对比 —— 数字打架时说明某平台房源有水分(重复挂/虚假房源)。
  - **房源去重与防虚假**:同一套房在多个平台多个价格挂出,是中介引流套路,交叉看能识别。
  - **覆盖补盲**:某些三四线城市/郊区板块贝壳覆盖弱,安居客/房天下可能更全。
- **不适合 / 替代方案**:虚假房源问题比贝壳更明显,不能单独作数据源;同样挂牌价 ≠ 成交价。权威成交仍回房管局网签。
- **生产案例**:中指研究院(房天下关联)的城市报告被研究机构与媒体广泛引用;投资人用「多平台挂牌量」作流动性观察的标准做法。
- **维护者背景**:平台方,引流/佣金动机,虚假房源治理弱于贝壳。
- **近期变化**:房天下/搜房体系近年战略收缩,数据更新与覆盖较巅峰期下降。
- **来源**:
  - [Surrogate-Primary] 安居客平台(vendor docs,evidence: [T02-S008],collected: 2026-05-14)
  - [Surrogate-Primary] 房天下/中指研究院数据(机构数据,evidence: [T02-S006],collected: 2026-05-14)
- **可信度**:medium
- **Decay risk**:medium(房天下体系收缩,覆盖度有下行风险)

### 7. 央行 LPR + 各地房贷利率公告

- **One-liner**:杠杆成本测算的输入源 —— LPR 是基准、各城市加点是变量,月度变动,直接决定月供和持有成本。
- **Tier**:场景特化(房产金融 / 杠杆测算 / 提前还贷决策场景)
- **Maintainer / Owner**:中国人民银行(LPR 政策)/ 全国银行间同业拆借中心(LPR 报价发布)/ 各地住建局与人行分支(地方加点与首付比例公告)
- **License**:proprietary(政府/官方机构公开数据)
- **Maturity signals**:
  - LPR 机制:2019 年改革后成为房贷定价基准,每月 20 日左右发布 1 年期与 5 年期 LPR
  - 政策活跃度:2024-2025 多次下调,各地首付比例/利率下限频繁调整 —— 高频变动
  - Activity:healthy
- **典型使用场景**:
  - **月供与持有成本测算**:LPR + 本城本套加点 = 实际利率,喂进自建 Excel 算月供和利息总额。
  - **提前还贷决策**:对比「存量房贷利率 vs 当前理财/存款收益」—— 利率高于无风险收益且有闲钱,提前还贷是确定性收益。但要核对违约金条款。
  - **二套/首套利率差**:认房认贷政策决定按首套还是二套算,利率差可达 0.5-1 个百分点,影响整笔账。
  - **政策原文优先**:利率/首付调整以各地住建局、人行分支公告原文为准,不信自媒体解读。
- **不适合 / 替代方案**:LPR 只是基准,不等于你的实际利率 —— 必须叠加本城本套加点。具体放款政策(放款速度、流水要求)要问经办银行。
- **生产案例**:任泽平「短期看金融」框架的核心变量即利率与信贷;所有房贷测算和提前还贷分析都以 LPR 为锚(与 Track 01 关联)。
- **维护者背景**:央行 + 官方报价机构,权威无争议。
- **近期变化**:2024-2025 处于降息周期,存量房贷利率批量下调、首付比例下限多次降低 —— 这一项是当前变动最频繁的工具,必须定期刷新。
- **来源**:
  - [Primary] 央行货币政策司(evidence: [T02-S003],collected: 2026-05-14)
  - [Surrogate-Primary] 全国银行间同业拆借中心 LPR 发布(机构数据,evidence: [T02-S018],collected: 2026-05-14)
- **可信度**:high
- **Decay risk**:medium(机制稳定,但具体利率/首付数值高频变动,引用必须带日期)

### 8. 中指 CREIS / 克而瑞 CRIC / 易居研究院报告

- **One-liner**:城市层面的供求、去化、库存周期数据 —— 板块研判和「这个城市还有没有量」的判断依据,但带卖方/机构立场。
- **Tier**:场景特化(城市与板块研判 / 新房市场场景)
- **Maintainer / Owner**:中指研究院(CREIS,中指数据)/ 克而瑞(CRIC,易居企业集团旗下)/ 易居房地产研究院
- **License**:proprietary(部分报告免费,深度数据付费;面向房企/金融机构)
- **Maturity signals**:
  - 行业地位:三家是房地产数据研究的头部机构,房企拿地、银行授信、媒体报道高度依赖其数据
  - 数据特色:城市成交/供应/库存去化周期、房企销售榜、土地市场数据
  - Activity:healthy(持续产出月报/年报)
- **典型使用场景**:
  - **库存去化周期**:某城市「去化周期」(库存 / 月均成交)超过 18-24 个月 = 严重供过于求,慎入。这是判断三四线去库存风险的关键指标。
  - **土地市场温度**:看城市土拍流拍率、溢价率 —— 流拍多说明开发商不看好,是城市基本面的先行信号。
  - **房企销售榜与爆雷线索**:房企销售排名骤降是资金链预警,关联「买谁家的预售房安全」。
  - **新房供求结构**:新房 vs 二手房此消彼长,判断「打新」还是「买二手」。
- **不适合 / 替代方案**:**带机构立场** —— 这三家服务房企和金融机构,口径上有「稳市场」倾向,对下行的呈现可能偏温和;不可作为唯一判断源,要和统计局指数、网签量、严肃媒体报道(财新等)交叉。
- **生产案例**:房企投拓、银行房地产授信、券商地产研究普遍以这三家数据为底层;媒体报道城市市场冷热高频引用(行业广泛实践)。
- **维护者背景**:商业数据机构,客户是房企与金融机构 —— 有「行业向好」的隐性动机,与无利益的统计局形成对照。
- **近期变化**:2024-2025 这三家的城市报告对「分化」的强调增加,但整体基调仍偏建设性。
- **来源**:
  - [Surrogate-Primary] 房天下/中指研究院数据(机构数据,evidence: [T02-S006],collected: 2026-05-14)
  - [Surrogate-Primary] 克而瑞 CRIC 大数据(机构数据,evidence: [T02-S009],collected: 2026-05-14)
  - [Secondary] 易居/乐居(易居研究院关联媒体,evidence: [T02-S017],collected: 2026-05-14)
- **可信度**:medium(数据扎实但有机构立场)
- **Decay risk**:medium(机构稳定,但报告口径会随政策风向调整)

### 9. 中国房价行情网(分小区行情)

- **One-liner**:分城市、分板块、分小区的历史房价行情 —— 微观估值的参照系,补统计局指数「太宏观」的盲区。
- **Tier**:场景特化(微观估值 / 单套定价场景)
- **Maintainer / Owner**:中国房价行情平台(中指/房天下体系关联的房价数据产品)
- **License**:proprietary(基础数据免费,精细数据可能付费)
- **Maturity signals**:
  - 数据特色:可下钻到小区级别的均价与历史走势
  - 数据来源:聚合中介挂牌 + 部分成交,口径介于挂牌与成交之间
  - Activity:healthy(持续更新)
- **典型使用场景**:
  - **单套房估值锚定**:看目标小区近 12-24 个月均价曲线,判断当前报价处于历史什么分位。
  - **小区横向比价**:同板块多个小区均价对比,识别「被低估/被高估」的标的。
  - **下跌幅度量化**:某小区从高点跌了多少,给「抄底还是接刀」提供量化输入(但不替代判断)。
- **不适合 / 替代方案**:数据口径偏挂牌侧,**仍有系统性偏高** —— 最终成交价锚定必须回房管局网签。小区级数据样本量小时波动大。
- **生产案例**:投资人做单套尽调时的常规参照工具;与贝壳成交记录配合使用(行业实践)。
- **维护者背景**:商业数据产品,口径透明度有限。
- **近期变化**:下行期小区级数据「挂牌 vs 成交」背离扩大,使用时要更警惕。
- **来源**:
  - [Surrogate-Primary] 中国房价行情/中指(机构数据,evidence: [T02-S010],collected: 2026-05-14)
- **可信度**:medium
- **Decay risk**:medium

### 10. 阿里拍卖 / 京东司法拍卖

- **One-liner**:法拍房标的与竞拍平台 —— 看似「打折买房」,但法拍尽调难度远高于普通二手房,腾退/欠费/户口都是雷。
- **Tier**:场景特化(法拍房投资场景 —— 高难度子方向)
- **Maintainer / Owner**:阿里拍卖(司法频道,与各地法院对接)/ 京东司法拍卖
- **License**:proprietary(平台免费看,竞拍需缴保证金)
- **Maturity signals**:
  - 市场地位:法院司法拍卖几乎全部上网,阿里与京东是两大主平台
  - 数据特色:标的物详情、起拍价/评估价、拍卖公告、竞拍记录
  - Activity:healthy —— 2024-2025 法拍房数量明显增加(断供/债务案件增多),供应量上升
- **典型使用场景**:
  - **标的筛选**:按城市/价格/类型筛法拍房,看起拍价相对市场价的折让。
  - **拍卖公告精读**:公告里写的「现状交付」「税费由买受人承担」「不负责腾退」每一条都是成本/风险。
  - **结合不动产登记查档**:法拍房必须查是否有轮候查封、是否有抵押未清、户口是否能迁出。
- **不适合 / 替代方案(法拍专属风险,必须讲透)**:
  - **腾退风险**:房子里有人(原房主/租客/老人),法院不一定负责清场,可能要自己打官司腾退,耗时一两年。
  - **欠费风险**:物业费、水电费、取暖费可能积欠数年,过户后由买受人承担。
  - **户口风险**:原房主户口迁不出,影响落户和学区。
  - **税费风险**:有的法拍公告写明「所有税费由买受人承担」,可能包括卖方本应承担的增值税/个税,实际成本远超起拍价。
  - **贷款风险**:部分法拍房不能按揭,需全款。
  - → 法拍房不适合无尽调能力/无现金缓冲/无法律资源的投资人。「便宜」往往对应这些隐性成本。
- **生产案例**:专业法拍投资团队以阿里拍卖为标的来源,但配套专业尽调(实地、查档、问物业、查户口);裁判文书网有大量法拍房腾退纠纷案例(与 Track 03 workflow 关联)。
- **维护者背景**:平台对接法院,平台本身不对标的瑕疵负责 —— 风险全在买受人。
- **近期变化**:2024-2025 法拍房挂拍量上升、流拍率上升,「捡漏」窗口看似变多,但腾退/债务复杂度也上升。
- **来源**:
  - [Surrogate-Primary] 阿里拍卖司法频道(vendor docs,evidence: [T02-S011],collected: 2026-05-14)
  - [Surrogate-Primary] 京东司法拍卖(vendor docs,evidence: [T02-S012],collected: 2026-05-14)
- **可信度**:medium(平台可信,标的风险高)
- **Decay risk**:medium(平台稳定;法拍供应量与政策环境是变量)

## 新兴 / 实验层(近 12 个月起势)

### 11. 二手房「核验价 / 指导价 / 参考价」公示系统

- **One-liner**:部分城市住建/房管部门对二手房给出官方价格区间 —— 直接卡贷款额度,改变了「网签价随便填」的旧玩法。
- **Tier**:新兴(实验阶段,政策工具)
- **Maintainer / Owner**:各地住建局 / 房管局(城市间差异极大,有的暂停、有的强化)
- **License**:proprietary(政府系统)
- **Maturity signals**:
  - 出现:深圳 2021 年率先推「二手住房成交参考价」,后多地跟进/调整;属近年新政策工具,各地反复
  - 稳定性:`stability: experimental` —— 各城市做法不一,有的城市已淡化或停用,有的仍执行
  - Activity:政策属性,随调控方向变动
- **典型使用场景**:
  - **真实可贷额度测算**:银行可能按「参考价」而非成交价放贷,参考价低于成交价时首付要多掏 —— 必须查本城是否有此政策、目标小区参考价多少。
  - **识别价格管制扭曲**:有参考价的城市,挂牌价/成交价/参考价三套数据并存,要分清哪套对应哪个用途。
- **不适合 / 替代方案**:不是全国统一制度,只在特定城市特定时期有效 —— 必须逐城市、逐时点确认。不能用 A 城的逻辑套 B 城。
- **生产案例**:深圳 2021 年参考价政策曾显著影响二手房贷款与成交,是城市级调控工具的标杆案例(政策原文见住建局)。
- **维护者背景**:地方政府调控工具,目的是「稳预期」。
- **近期变化**:2024-2025 稳楼市基调下,部分城市淡化或暂停参考价限制 —— 高度不稳定。
- **来源**:
  - [Primary] 北京市住建委(地方住建政策公告样板,evidence: [T02-S014],collected: 2026-05-14)
  - [Primary] 深圳市住建局(参考价政策发源地,evidence: [T02-S015],collected: 2026-05-14)
- **可信度**:medium
- **Decay risk**:high(12 个月内被显著改变/取消概率 > 40% —— 政策工具,各地反复)

### 12. 政务平台「带押过户」线上办理

- **One-liner**:带抵押的二手房不用先还清贷款就能过户 —— 大幅降低交易中的资金垫付风险和「卖方解押后跑路」的隐患。
- **Tier**:新兴(实验阶段,流程创新)
- **Maintainer / Owner**:各地不动产登记中心 + 住建部门 + 银行联合推行;全国一体化政务服务平台部分支持线上
- **License**:proprietary(政府流程)
- **Maturity signals**:
  - 出现:2022-2023 起多地试点,2024-2025 加速推广,自然资源部曾发文鼓励
  - 稳定性:`stability: experimental` —— 各城市/各银行覆盖不一,流程仍在标准化
  - Activity:推广中
- **典型使用场景**:
  - **带抵押房交易**:卖方房子还有按揭未还清时,传统做法要买方先垫资帮卖方解押(资金风险大),带押过户让抵押权直接转移或衔接,省掉垫资环节。
  - **降低交易资金风险**:配合资金监管账户,减少「钱付了房没过户」「解押了卖方反悔」的风险敞口。
- **不适合 / 替代方案**:不是所有城市/所有银行都支持,跨行带押过户尤其受限 —— 交易前必须确认本城本行是否可办。不支持时仍需走传统「解押再过户 + 资金监管」。
- **生产案例**:多地住建/不动产登记部门 2024-2025 公布带押过户办理量上升;是降低二手房交易风险的实务进展。
- **维护者背景**:政府 + 银行联合推动,目的是促进二手房流通。
- **近期变化**:2024-2025 是其快速铺开期,但各地成熟度差异大。
- **来源**:

## 选型决策树

> 本行业「选工具」本质是「选数据口径」。每个节点的核心问题不是「用哪个 App」,而是「这个判断该信哪套数据」。节点共 9 个。

### 决策树 A:你当前在投资弧线的哪一步,该信哪套数据?

#### Branch 1:判断「现在是不是好时机 / 城市选哪个」(宏观与城市层)
→ **推荐**:国家统计局 70 城房价指数(定方向、看分化)+ 中指/克而瑞城市报告(看库存去化周期、土地市场)。
→ **不推荐**:贝壳挂牌价做城市横向比较 —— 各城口径不一、且是卖方报价,会得出错误结论;自媒体「XX 城要涨」判断 —— 利益相关噪音。
→ **关键动作**:先看统计局指数确认这个城市能级在涨还是在跌,再看克而瑞去化周期确认「还有没有量」。两个都过关再往下。
→ 真实案例:任泽平「长期看人口、中期看土地、短期看金融」框架,对应的就是先看人口/城市数据再看土地市场再看信贷,不从挂牌价入手(与 Track 01 关联)。

#### Branch 2:判断「这套房值多少钱」(微观估值层)
##### Branch 2a:要一个可信的成交价锚
→ **推荐**:各地房管局网签备案成交数据(法律意义的真实成交价)。
→ **替代**:贝壳「成交记录」(部分城市可见)作交叉验证,但优先级低于网签。
→ **不推荐**:贝壳/安居客挂牌价直接当估值 —— 系统性偏高 10-20%(下行期),会买贵。
##### Branch 2b:要看这个小区的历史价格走势和分位
→ **推荐**:中国房价行情网(小区级历史曲线)+ 贝壳同小区挂牌分布。
→ **注意**:这两个口径都偏挂牌侧,得出的「估值」要再用网签成交价校准。

#### Branch 3:判断「这个城市/板块卖不卖得掉」(流动性层 —— 2024-2025 最关键)
→ **推荐**:各地房管局网签备案的**成交量/套数**(流动性的第一手读数)+ 多平台(贝壳+安居客)挂牌量交叉 + 克而瑞去化周期。
→ **关键指标**:网签量同比、二手房挂牌量绝对值与变化、成交周期(从挂牌到成交多少个月)、去化周期(库存/月均成交)。
→ **不推荐**:只看房价指数判断流动性 —— 价格阴跌但「成交冰封」时,指数掉得慢,流动性却已经崩了。下行期「卖不掉」比「跌价」更致命。

#### Branch 4:核查「这套房产权干不干净」(尽调层 —— 不可跳过)
→ **推荐**:不动产登记中心查档(抵押/查封/产权人/性质/面积)—— 这是硬环节,任何房都要做。
→ **法拍房额外**:必须查轮候查封、问物业欠费、查户口能否迁出、精读拍卖公告的税费/腾退条款。
→ **不推荐**:信中介一句「房子很干净」就跳过查档 —— 买到抵押房/查封房的纠纷大量存在。
→ 真实案例:专业房产律师与谨慎投资人把「查档」作为出手前的标准动作(与 Track 03 workflow「尽调」环节关联)。

#### Branch 5:测算「杠杆和持有成本」(算账层)
→ **推荐**:自建持有成本/回报率 Excel,输入用央行 LPR + 各地房贷利率公告 + 本城税费政策。
→ **必做**:跑「房价跌 20% / 租金跌 30% / 利率升 1%」三个压力场景,看现金流会不会断。
→ **不推荐**:用市面房贷计算器的「月供」当成完整账 —— 它漏掉持有成本、空置、流动性折价、退出税费。

#### Branch 6:决定「走哪个交易渠道、怎么保资金安全」(交易层)
→ **推荐**:带资金监管账户的正规交易服务(贝壳/链家/正规地方中介都有)—— 房款进监管账户、过户完成才放款。带抵押房优先走「带押过户」(如本城本行支持)。
→ **不推荐**:为省中介费走私下交易、无资金监管 —— 「钱付了房没过户」「解押后卖方反悔」的风险敞口巨大。
→ **绝对不碰**:任何提出「阴阳合同避税」「经营贷置换房贷」「首付贷」「包装流水」的中介或「顾问」—— 见避坑清单。

#### Branch 7:跟踪「政策会不会变」(政策层)
→ **推荐**:各地住建局/房管局公告原文 + 央行/中国政府网政策原文 + 公积金中心。
→ **不推荐**:把自媒体/公众号的政策解读当政策本身 —— 解读会夹带立场和误读,以原文为准。
→ **原则**:政策原文优先于一切二手解读。读到「传闻」先回官方渠道证实。

#### Branch 8:用社区/论坛获取线索(情绪与线索层 —— 最低优先级)
→ **可用**:篱笆网、19 楼等地方论坛、知乎、小红书、公众号 —— 用来「发现」一个新政策传闻、一个板块的负面情绪、一个开发商的爆雷苗头。
→ **绝不**:把社区里的判断/喊单当决策依据。这一层信息**必须**回到 Branch 1-7 的官方数据和查档去证实,自身零决策权重。
→ 标注:`噪音极大,作情绪/线索,不作决策依据`。

### 决策树 B:你是哪类投资人,工具组合怎么配?

#### Branch 9:按投资人类型选最小工具集
→ **首次买房自住兼投资者**:统计局指数 + 房管局网签 + 不动产登记查档 + 自建 Excel + 正规资金监管交易 —— 五件套,缺一不可,不需要碰法拍。
→ **二手房改善/置换型投资人**:加贝壳/安居客做比价 + 中国房价行情网做小区估值 + 重点用 Branch 3 流动性工具(因为要先卖掉旧房)。
→ **法拍房投资人**:在五件套基础上,阿里/京东司法拍卖 + 强化的不动产登记查档 + 实地尽调能力 + 法律资源 + 现金缓冲 —— 不具备后三项不要做法拍。
→ **想做「城市轮动/抄底」的激进投资人**:这一行没有可靠工具支撑「精准抄底」—— 决策树到这里给的是警告而非工具:统计局指数 + 克而瑞去化能告诉你方向,但没有任何工具能告诉你「底在哪」。`[no public tool reliably supports market timing; based on cross-source consensus that timing is not data-tractable]`

## 避坑清单

- ❌ **不要把中介平台挂牌价当成交价用**:贝壳/安居客/房天下的挂牌价是卖方报价 + 平台撮合产物,系统性偏高、且滞后。下行期挂牌价和真实成交价能差 10-20%。用它估值 = 买贵。成交价锚定必须回各地房管局网签备案数据(evidence: [T02-S007, T02-S008, T02-S014])。
- ❌ **不要只看房价指数判断市场,忽略流动性指标**:2024-2025 的核心风险是「卖不掉」不是「跌一点」。价格阴跌时指数掉得慢,但二手房挂牌量暴增、成交周期从 3 个月拉到 12 个月 —— 流动性已经崩了指数还没反应。必须看网签成交量 + 挂牌量 + 去化周期(evidence: [T02-S014, T02-S009])。
- ❌ **不要信任何提供「经营贷置换房贷 / 首付贷 / 阴阳合同 / 包装流水」的中介或「顾问」工具**:这些是违规甚至违法操作,后果是银行抽贷、征信污点、法律责任,部分「技术流大师」公开教这些。本文不收录任何此类工具/渠道。合规融资只用商贷/公积金贷/组合贷,首付必须是自有资金(evidence: intake.json warnings #2)。
- ❌ **不要跳过不动产登记中心查档**:中介一句「房子很干净」不是尽调。不查档可能买到带抵押、被查封、产权人不符、实为商办性质的房,纠纷在裁判文书网大量存在。每一套房,无论一手二手法拍,都要查抵押/查封/产权人/性质/面积(evidence: [T02-S013, T02-S016])。
- ❌ **不要把法拍平台的「起拍价折让」当成「便宜」**:阿里/京东司法拍卖的低起拍价背后常埋着腾退(房里有人清不走)、欠费(物业水电积欠数年由你承担)、户口迁不出、税费全包(可能含卖方应缴的增值税/个税)、不能按揭。法拍的真实成本远超起拍价,不具备尽调能力/法律资源/现金缓冲的人不要碰(evidence: [T02-S011, T02-S012])。
- ❌ **不要用市面房贷计算器的「月供」当完整账**:它只算月供,漏掉物业/取暖/维修/空置损失/资金占用机会成本/房产税(试点城市)/退出税费/流动性折价。完整账要自建 Excel 并跑压力测试。月供能扛住 ≠ 这笔投资划算(evidence: [T02-S003])。
- ❌ **不要把中指/克而瑞/易居的报告当无立场的真相**:这三家服务房企和金融机构,有「稳市场/行业向好」的隐性倾向,对下行的呈现可能偏温和。它们数据扎实但要和无利益动机的统计局指数、房管局网签量、严肃媒体(财新等)爆雷报道交叉验证(evidence: [T02-S006, T02-S009, T02-S017])。
- ❌ **不要把论坛/知乎/小红书/公众号的判断当决策依据**:这一行「大师」大量是利益相关方(卖课、分佣、喊单只晒对不晒错)。社区工具只能用来发现线索(新政策传闻、板块情绪、爆雷苗头),发现后必须回官方数据和查档证实,自身零决策权重(evidence: intake.json warnings #4)。
- ❌ **不要把自媒体的政策解读当政策原文**:解读会夹带立场、简化甚至误读。限购/限贷/限售/利率/参考价以各地住建局、央行、政府网公告原文为准。读到政策「传闻」先回官方渠道证实再行动(evidence: [T02-S003, T02-S004, T02-S005])。

## Phase 2 提炼提示

**反复出现 ≥3 source 都强调的「工具选型原则」**(候选 playbook 规则):
- **官方网签/备案成交数据 > 中介挂牌价** —— 挂牌价是卖方报价,系统性偏高;真实成交价只在房管局网签系统(出现于:T02-S001/S007/S014,intake research_strategy_notes Track 02)。
- **流动性指标(挂牌量/成交周期/去化周期)> 价格指标** —— 2024-2025 下行期「卖不掉」比「跌价」更致命,价格指数滞后于流动性崩塌(出现于:T02-S009/S014,intake warnings #8)。
- **政策原文 > 自媒体解读** —— 强政策驱动行业,二手解读夹带立场与误读(出现于:T02-S003/S004/S005,intake research_strategy_notes Track 02)。
- **产权查档是不可跳过的硬环节** —— 不查档=踩雷,中介口头保证不算尽调(出现于:T02-S013/S016,intake warnings #9)。
- **算账要算「完整持有账 + 流动性折价 + 压力测试」,不是算月供** —— 工具的价值在模型完整性,不在 App(出现于:T02-S003,intake core_subdomains 房产金融与杠杆)。
- **社区/自媒体工具零决策权重** —— 只作情绪与线索,利益相关噪音极大(出现于:intake warnings #4,T02 信息社区类工具集体定性)。

**显著的工具流派分裂**(候选「智识谱系」条目):
- **「官方数据派」 vs 「中介平台派」**:前者以统计局指数 + 房管局网签 + 不动产登记为决策骨架,把中介平台只当看房入口;后者以贝壳挂牌价/成交记录为主要判断依据。本 track 立场明确站官方数据派 —— 中介数据有商业动机。这条分裂可直接喂 Phase 2.1 心智模型(「信源的利益结构决定数据可信度」)。
- **「数据驱动派」 vs 「喊单跟随派」**:前者用网签/去化/利率等可验证数据 + 自建模型决策;后者跟自媒体「大师」判断。与 Track 01 figures 的「经济学家框架派 vs 自媒体喊单派」分裂同构,可交叉印证。
- **「算账派」 vs 「叙事派」**:算账派把每笔投资跑成现金流压力测试表(与 Track 01 香帅的家庭资产负债表视角关联);叙事派靠「核心资产永远涨」「人口拐点全面承压」等宏大叙事下注。工具层面,前者重 Excel 模型,后者几乎不用算账工具。
