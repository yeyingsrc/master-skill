# Track 03 — Workflows (SEO 专家)

> Mini-viable for prototype.

### 1. 新站 SEO 上线

- **One-liner**: 从域名注册到第一个关键词进 top 10 的标准动作
- **Trigger**: 新建网站 / 新做品牌站
- **Output**: 站点被索引 + 至少 1 个目标词进 top 10
- **入门 SOP**:
  1. 域名 + 主机选型 (避免被惩罚的二手域名)
  2. 技术 SEO 基础 — robots.txt / sitemap / canonical / hreflang
  3. 关键词调研 — 用 Ahrefs 选 20-30 个目标长尾
  4. 内容生产 — 围绕目标词产 5-10 篇核心内容
  5. 外链建设 — 找 5-10 个相关性高的链接
- **资深路径**:
  - skip 大量初始外链建设, 资深做法是先把内容 + 技术 SEO 做扎实再外链
  - optimize 关键词调研用 SERP 反推, 不只看搜索量
  - add 第一周就装 Search Console + Analytics, 不等出问题再装
- **常见失败模式**:
  - 用二手被惩罚域名 → 怎么做 SEO 都起不来
  - 忽略技术 SEO 直接产内容 → 内容不被收录
  - 关键词只看搜索量不看竞争 → 选了打不动的词
- **Last_updated**: 2025-12-10
- **Decay risk**: high

### 2. 内容 SEO 单篇产出

- **One-liner**: 围绕一个目标关键词, 从选题到发布到优化的标准流程
- **Trigger**: 编辑日历 / 内容生产排期
- **Output**: 一篇能进 SERP top 30 的内容 (起始位置)
- **入门 SOP**:
  1. SERP 分析 — top 10 内容类型 / 长度 / 结构
  2. 大纲构建 — 覆盖 PAA + featured snippet 机会
  3. 内容创作 — 真人作者署名 + 真实案例
  4. On-page SEO — title / meta / 内链 / schema
  5. 发布 + Submit URL inspection
- **资深路径**:
  - skip 不必要的 SEO 老套路 (关键词密度堆), 资深关注语义覆盖
  - optimize 用 Surfer SEO 之类工具批量比对 top 10 内容信号
  - add 发布后 7-14 天根据 Search Console 数据做第二轮优化
- **常见失败模式**:
  - 不分析 SERP 就硬写 → 内容形式跟意图不匹配, 永远进不了 top 10
  - AI 写完直接发 → Helpful Content Update 干掉
  - 发布后不跟数据 → 错过早期优化窗口
- **Last_updated**: 2025-11-20
- **Decay risk**: medium
