## 心智模型 (4 个 PASS)

### 1. Technical SEO 是 baseline, 不是优势

**一句话**: Core Web Vitals + Mobile + Schema + Crawlability 是「不做就死」的舞台, 但内容 + 链接才是演员.

**它说的是**: 很多团队把 SEO 做成「技术 SEO 项目」 — 花几个月做 Core Web Vitals, 但内容跟链接都没做. 技术 SEO 是 baseline, 不会给你排名优势, 但缺了它谁都给不了你排名.

**应用方式**:
- Core Web Vitals 三件套: LCP ≤ 2.5s / INP ≤ 200ms / CLS ≤ 0.1
- Mobile-friendly + responsive (Mobile-first Indexing 已全量)
- Schema markup 全套 (Article / Product / FAQ / Organization)
- Crawlability 三件套: robots.txt / sitemap.xml / Canonical
- 这些都是 baseline 一次做好, 后续季度审计

**局限**:
- 单纯做技术 SEO 不带流量
- JS-heavy SPA 站点技术 SEO 复杂度高
- 跨平台 (Google / Bing / Baidu) 技术要求差异

### 2. 国际化 SEO (hreflang + 多语言) 是头部品牌必修

**一句话**: hreflang + 多语言版本 + 文化适配 + locale-specific URL 是真国际化 SEO, 不是 Google Translate 翻一下.

**它说的是**: Aleyda 服务 20+ 国家客户经验 — 多数国际化 SEO 失败是因为「翻译错误 + hreflang 配置错误 + 文化适配不到位」. 真正国际化 SEO 是工程级别的 work.

**应用方式**:
- hreflang 标签正确 (语言 + 地区 + canonical 全套)
- 多语言版本不靠 Google Translate, 用本地母语创作者
- 文化适配 (节日 / 习俗 / 度量衡 / 价格)
- locale-specific URL (subdomain / subdirectory / ccTLD 选择)
- CDN + locale-specific 服务器

**局限**:
- 国际化 SEO 起步成本高 (至少 5-10 国家市场才划算)
- hreflang 错误调试复杂
- 跨语言内容生产成本高

### 3. Mobile-first 是默认, 不是选项

**一句话**: Mobile-first Indexing 2019 全量后, 移动端是 Google 的真实索引版本. PC 体验只是辅助.

**它说的是**: 多数中文圈 / 部分 B2B 团队还在 PC 优先思维, 但 Google 早就改了. 你的移动端如果差, PC 端再好也排不上.

**应用方式**:
- 移动优先设计 (responsive)
- 移动端 Core Web Vitals 必须达标
- 移动端跟 PC 端内容一致 (不要 PC 多 / 移动少)
- 移动端体验 (字号 / 按钮 / 间距) 测试

**局限**:
- B2B / 工程类站点 PC 端使用率仍高
- 移动端 + PC 端两套设计成本高

### 4. 工具方法论 6-12 月衰减

**一句话**: SEO 工具 / 方法论 6-12 月一变. SEOFOMO 周更就是为了第一时间同步.

**它说的是**: Aleyda 的 SEOFOMO newsletter 35,000+ 订阅就是一手政策动向. 老工具方法论可能 6 个月就过时. 持续学习 + 跟最快是头部 SEO 操盘手底线能力.

**应用方式**:

**局限**:
- 中小团队时间 / 精力有限, 跟不上完整政策更新
- 信息过载也是问题, 关键是「跟该跟的」

## 决策规则 (6 条)

### 1. 如果做技术 SEO
**则**Core Web Vitals + Mobile + Schema + Crawlability 4 件套 baseline, 一次做好.

### 2. 如果做国际化
**则**hreflang + 多语言版本 + 文化适配 + locale-specific URL 工程级别 work, 不要 Google Translate.

### 3. 如果做移动 SEO
**则**Mobile-first 优先, 移动端必须达标, 不要 PC 优先思维.

### 4. 如果保持竞争力
**则**SEOFOMO 周更必读 + SMX / BrightonSEO 年度大会必看 + 持续学习.

### 5. 如果遇到 JS-heavy 站点
**则**SSR / SSG / hydration / Schema 注入, JS 渲染不验证 Google 没爬到.

### 6. 如果起步 SEO
**则**LearningSEO.io 路线图 (Aleyda 免费) 是入门最系统资源.

## 应用示例

### 用 Aleyda 视角思考: 「我跨境品牌想做 5 国市场 SEO, 怎么打?」

**Aleyda 框架**:
1. **国际化架构选哪种?** subdomain (xx.example.com) / subdirectory (example.com/xx) / ccTLD (example.xx) — 推荐 subdirectory (维护 + SEO 平衡好)
2. **hreflang 配置正确吗?** 语言 + 地区 + canonical 全套, 错了任意一个国际化 SEO 失败.
3. **多语言版本谁写?** 不要 Google Translate, 找本地母语创作者. 文化适配很重要.
4. **Core Web Vitals 跨地区 / 跨设备测了吗?** CDN + locale-specific 服务器 部署.
5. **每国本地化关键词研究做了吗?** 不能用英语直接翻译 — 当地用户搜的词跟英语完全不同.

**结论倾向**: 5 国国际化 SEO 是工程级别 6-12 月 project. 不要简单 Google Translate. 至少投入 5-10 人团队 (技术 + 内容 + 本地化).
