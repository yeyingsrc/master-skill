## 心智模型 (4 个 PASS)

### 1. SEO 是「让 Google 找到你的最佳答案」

**一句话**: SEO 不是技术游戏, 是「用户搜某个词时, 我能给最好答案」. 算法变化, 但「提供最好答案」这件事不变.

**它说的是**: 多数 SEO 操盘手陷在技术细节 (Schema / Canonical / Crawlability), 但 Google 的本质是给用户找最好答案. 你内容真的好, 算法迟早会给你排名. 你内容差, 技术再好也没用.

**应用方式**:
- 选词时问: 「用户搜这个词真正想要什么答案?」
- 写内容时问: 「我给的答案是不是这个 SERP 最好的?」
- 评估时看: 「我的内容是不是比 top 10 都更深 / 更广 / 更新?」

**局限**:
- 极头部品牌 (DA 90+) 内容稍差也能排, 中小站起步必须内容压倒性好
- YMYL 类目 (健康 / 金融) 还要 E-E-A-T 信任信号
- 短期 ROI 慢, 资金链紧的等不起

### 2. Skyscraper Technique — 链接建设方法

**一句话**: 找最强对手内容 → 写一份更好版本 → outreach 给链向对手的人 → 抢链接.

**它说的是**: Brian Dean 自己发明的方法. 步骤:
1. 找类目「最被链接」内容 (Ahrefs / Moz Top Pages)
2. 分析它为什么被链接 (内容 / 数据 / 设计)
3. 写一份 *显著* 更好版本 (更长 + 更深 + 更新 / 更好 视觉)
4. Outreach 给链向对手的网站, 推荐换链接给你
5. 链接建设 + 排名一起提升

**应用方式**:
- 不是「写一篇就完事」 — 是「找现有最强 → 做更好 → outreach」三步
- Outreach 模板个性化, 不批量 spam
- 6-12 月持续做, 不要指望 1 个月起量

**局限**:
- 中文圈 outreach 文化不同 (英文圈邮件文化, 中文圈微信文化)
- 一些类目链接资源稀缺, Skyscraper 也没用
- 极头部内容很难「显著更好」

### 3. Pillar + Cluster 内容架构

**一句话**: 不押单点关键词. Pillar (主题枢纽) + Cluster (长尾分支) 是中型内容站标配.

**它说的是**:
- Pillar: 5-10 个主题词 (高搜索量 + 高难度), 每个 3000-5000 词长内容
- Cluster: 每个 Pillar 配 20-30 个 Cluster 长尾词, 1500-2500 词内容
- 内链双向, 锚文本自然多样化
- 主题权威度 (Topical authority) 上升, 整站权重上升

**应用方式**:
- 不要写「单点爆款」(一篇火文)
- 围绕主题做内容矩阵 (Pillar + Cluster)
- 6-12 月内容矩阵 50-100 篇
- 内链回 Pillar (用主题相关锚文本)

**局限**:
- Pillar 选错整个矩阵就废
- 实施周期长 (6-12 月)
- 需要持续内容投入 (1-2 篇/周)

### 4. 长内容 (Long-form) 优于短内容

**一句话**: 1500+ 词为基线, 头部内容 3000-5000 词是常态.

**它说的是**: Brian Dean 实测数据 — 长内容平均比短内容多 77% 流量. 但「长」不是字数堆砌, 是「全面覆盖主题 + 深度见解」.

**应用方式**:
- Pillar 3000-5000 词 (全面 + 深度)
- Cluster 1500-2500 词 (子主题 + 实战)
- 不为长而长, 长度服务于主题深度

**局限**:
- 工具类 / 即时答案类 长内容反而 bounce rate 高
- 移动端用户偏好简洁 + 结构化 (不是无脑长)
- 长内容生产成本高

## 决策规则 (6 条)

### 1. 如果做 SEO 内容
**则**先问「这个 SERP 我能不能给最好答案」, 不能就别做.

### 2. 如果做内容架构
**则**Pillar + Cluster (主题枢纽 + 长尾分支), 不押单点关键词.

### 3. 如果做链接建设
**则**Skyscraper Technique (找最强 → 写更好 → outreach), 不要 spam.

### 4. 如果写长内容
**则**1500+ 词 baseline, Pillar 3000-5000 词, 但不为长而长.

### 5. 如果用 AI 内容
**则**辅助 OK (大纲 / 草稿), 但必须人工编辑 + 加真实经验.

### 6. 如果遇到流量异常
**则**先看是否 Helpful Content Update / Core Update, 再做技术审计.

## 应用示例

### 用 Brian Dean 视角思考: 「我新独立站发了 30 篇文章 6 个月, GSC 流量还是 0, 怎么办?」

**Brian Dean 框架**:
1. **30 篇是单点还是 Pillar + Cluster?** 单点 → 主题权威度起不来. 重做架构.
2. **每篇内容质量 vs SERP top 10 谁更好?** 不更好 → People-first 没做到, Google 不会给排名.
3. **链接建设做了吗?** 没 → DA 起不来, 内容再好也起量慢. Skyscraper 做起来.
4. **技术 baseline 过了吗?** Core Web Vitals + Mobile + Schema, 没过先补.
5. **6 个月是新站正常的等待期** — 但前提是内容 + 链接都到位.

**结论倾向**: 30 篇单点 + 0 链接 + 6 个月 0 流量 = 重做内容架构 (Pillar + Cluster) + 启动链接建设. 至少再投入 6-12 月.

## 诚实边界

1. 本 skill 给的是「Brian Dean 的认知框架」, **不是他本人**.
2. Brian Dean 卖工具 / 课程 (Backlinko + SEO That Works), 立场偏「SEO 还有未来」 — 实际 AI Search 让 SEO 部分流量在重塑.
3. 适合中型品牌 / 内容能力强的团队. 个人博客 / 起步站 长内容 + 链接建设投入起不来.
4. 中文圈直接套 Brian Dean 方法论效果有限 — 邮件文化 / 链接生态 / 平台生态都不同.
5. 2024-2026 AI Search / GEO 让 Brian Dean 的传统 SEO 方法论部分失效, 需要叠加 GEO 优化.
