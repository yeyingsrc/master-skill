## 候选 playbook 规则 (target 10-20)

| Pattern | 来源 |
|---------|------|
| 「写什么/增长不动」→ 先回 niche,窄到一句话能讲清 + 绑定不可替代经验 | workflow #1/#2/#3/#5, canon Niche T04-S018 |
| 还没稳定受众信任就想收费/接广告 → 先免费产出建信任,漏斗顶部够宽再收窄 | workflow #1/#2/#6, canon Free-vs-paid T04-S018, Lenny 9 个月 |
| 「靠哪个渠道增长」→ 同时跑 2-3 个验证有效的,不 all-in 任一个 | workflow #1/#5/#7, 避坑 |
| 纠结「什么时候/能不能变现」→ 不等 10k,在 1k-2k 用低价档/冷邮件前置验证付费意愿 | workflow #1/#2/#4/#6 资深 skip |
| 「要不要 all-in 某平台」→ 默认「不」,先建自有域名 + 可导出 email 列表 | workflow #7, canon Platform dependency T04-S018, 避坑 #4/#6 |
| 「XX 月入 N 万我能不能复制」→ 先问分布(中位数)、问样本(失败者在哪)再回答 | canon 头部效应 T04-S006, C4 |
| 「该怎么变现」→ 先问对方属于/想走哪一派(订阅/广告/课程漏斗/个人IP),四派答案不同 | intake 四大流派, canon T04 智识谱系 |
| 变现/平台/增长问题 → 先问 locale,海外 default subscription+Substack,国内 default 广告+知识付费+公众号 | 全 5 轨, intake warning |
| 平台选型 → 不只比抽成率,算总成本(抽成+月费+维护时间+迁移成本+退出门槛) | 避坑 #1, canon 平台抽成 T04-S021, tools Phase2 |
| 验证阶段 → 用最轻的工具(Substack 免费档),有 PMF 再按具体瓶颈迁移 | tools 决策树 B1→2→3, 避坑 #5 |
| niche/定价/增长渠道 → 当「发布后滚动校准的事」,不当「开局拍死的事」 | workflow #1/#3/#6 optimize (C8 降级) |
| 起步同 niche → 先逆向拆解 3-5 个成功账号当地图 | workflow #1/#3 资深 add, Chenell 方法 (C10 降级) |
| 赞助定价 → B2B/垂直小列表按受众价值定 flat-rate/溢价,不按订阅数线性 | workflow #4 optimize, glossary CPM/rate card |

## 候选工具流派分裂 (智识谱系)

| Split | A 派代表 | B 派代表 |
|-------|---------|---------|
| 一体化平台派 vs 解耦栈派 | Substack / beehiiv(写发增长变现读者发现一站式) | Ghost + Kit + Memberful + Plausible + Buzzsprout(各层选最优,0 抽成,自当系统集成商) |

| 海外订阅工具栈 vs 国内分发+知识付费+私域工具栈 | Substack / Kit / Ghost | 公众号 / 小报童 / 知识星球 / 得到 |
