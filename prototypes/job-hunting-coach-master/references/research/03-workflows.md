> Phase 1 wave 2 · Track 03 · industry=job-hunting-coach · locale=global · profile=practitioner · industry_type=coaching_practitioner
> 本文件 = 求职教练这一行实际怎么带客户走完一段求职 / 一场面试 / 一次谈薪 —— 「入门 SOP」+「资深路径（skip/optimize/add）」+「近期变化」+ 时效锚点。
> **本行业 workflow 的两条硬约束**：(1) **国内 vs 海外求职弧线差异极大**（简历格式、招聘渠道、面试形式、校招机制、谈薪文化都不同），必须分别画；(2) **资深差异不是「教人抄近道 / 兜售确定性」** —— 简历优化 vs 造假是硬边界，押题 / 背题库短期有用但有上限和风险；资深教练的 skip/optimize 是「把可控部分做到最好 + 帮求职者管理预期」。每个 workflow 标注「这是求职者的工作流」还是「教练的工作流」。
> **不 victim-blame**：求职结果受市场周期强烈影响，workflow 标注「教练能优化可控部分（定位 / 简历 / 面试 / networking），宏观周期 / 年龄 / 学历门槛是真实约束」。

### W1. 国内校招求职弧线

- **这是谁的工作流**: 求职者（应届生）的工作流；教练在每个节点上做诊断 + 陪跑。

- **One-liner**: 应届生从「不知道投什么」到「拿到并接受一个 offer」的全过程 —— 国内校招的特征是**工业化时间窗**（春招 / 秋招 / 提前批是硬节点，错过当季就要等下一季甚至失去应届生身份），主通道是「网申 + 笔试 + 多轮面试」而非 networking。

- **Trigger**: 应届生（或即将毕业者）进入校招时间窗 —— 秋招（约 8-11 月，主战场）、提前批（约 6-8 月，部分大厂 / 行业更早）、春招（约 2-4 月，补录性质，岗位少于秋招）。

- **Output**: 拿到并接受一个国内校招 offer；签三方协议（应届生身份的核心凭证）。

- **入门 SOP（minimum viable steps）**:
  1. **定位与岗位收敛**：确定目标行业 / 职能 / 城市 / 公司梯队（大厂 / 国企 / 外企 / 中小厂），列一份目标岗位清单。inputs：专业、实习经历、能力盘点；outputs：目标清单。**跳过会发生什么**：海投所有看到的岗位 → 笔面试时间冲突、简历无法针对性、面试讲不出「为什么是这家」，转化率全线崩。
  2. **简历定稿**：做一份校招简历（国内格式：常含照片 / 年龄 / 学校 / 实习 / 项目，一页为主），针对岗位类型微调。**跳过会发生什么**：用一份「万能简历」投所有岗位 → 网申初筛和面试都被「不匹配」刷掉。
  3. **网申 + 内推并行**：在公司官网 / 校招系统提交网申；同时通过牛客内推帖、师兄师姐、宣讲会要内推码（国内内推主要作用是「免笔试 / 简历直达 HR」，不是「保过」）。**跳过会发生什么**：只网申不内推 → 简历淹没在网申池；只等内推不网申 → 没内推渠道的公司全错过。
  4. **笔试准备 + 参加**：技术岗刷题（LeetCode / 牛客题库），非技术岗练行测 / 综合素质题型，研究目标公司历年笔试题型（牛客笔经）。**跳过会发生什么**：国内校招笔试是硬门槛，裸考被刷，连面试机会都没有。
  5. **多轮面试**：群面 / 行为面 / 专业面 / HR 面，逐轮推进。每轮前研究面经（牛客）、准备 STAR 故事。**跳过会发生什么**：不准备直接上 → 行为面讲成流水账、专业面露怯，被刷。
  6. **拿 offer + 决策 + 签三方**：比较手上的 offer（薪资 / 平台 / 城市 / 发展），做决策，签三方协议。**跳过会发生什么**：不比较直接签第一个 → 后悔但三方违约有成本（违约金 + 关系损耗）。

- **资深路径（差异点 — skip / optimize / add）**:
  - **skip**: 资深教练会让多数应届生**跳过「海量网申」里低性价比的部分** —— 不是不投，是把精力从「投 200 家」收敛到「30-50 家针对性投 + 重点公司深准备」。经验：校招笔面试时间高度冲突，投太多反而每家都准备不足，转化率不升反降。也跳过对「明显不匹配 / 明显凑数」岗位的投递。
  - **optimize**: 把「逐个研究面经」优化成「按公司梯队批量准备」—— 同梯队 / 同行业公司的笔面试题型和考察点高度相似，资深教练帮学生按「公司簇」准备一套，而非一家一套。把「提前批」当**低风险练兵 + 占坑**用（提前批不影响秋招正式批，且部分能直接锁 offer），而新手常因「提前批 = 还没准备好」而错过。
  - **add**: 资深教练会**额外做「应届生身份 / 时间窗管理」** —— 提前算清楚三方、应届生认定口径、各公司提前批 / 正式批时间线，做一张个人时间表；新手常因不懂应届生身份的价值（很多岗位只面应届生）和三方规则而踩坑。还会**额外做「秋招 = 主战场」的预期校准** —— 明说春招岗位远少于秋招，别把希望全压在春招上。

- **近期变化（近 12 个月）**:
  - **2023-2025 起，由国内青年就业市场承压 + 应届生规模持续高位驱动**：国内校招竞争加剧，「秋招主战场」属性更强（春招进一步萎缩），教练对应届生的预期管理权重上升 —— 「定位 + 早准备」比「广投」更被强调（evidence: [T03-S021]）。
  - **2024-2025 起，由招聘两端 AI 化驱动**：原 step 4「笔试」部分公司加入 AI 监考 / 在线 AI 测评；step 2「简历」可用 AI 辅助改写但「AI 味简历」开始被识别；step 5 部分大厂校招初筛环节出现 AI 视频初面。**这些是步骤内做法的变化，不是弧线本身的变化** —— 弧线骨架（定位→简历→网申/内推→笔试→面试→offer）稳态（evidence: [T03-S011], [T02-S012]）。
  - 当前采用率：无可靠量化数据；AI 辅助在应届生中已是普遍现象（行业现状观察）。

- **典型耗时**:
  - 入门 SOP（求职者侧）：一个完整秋招周期约 3-4 个月（8 月启动到 11-12 月拿 offer），含提前批可拉长到 6 个月。
  - 资深路径：周期不变（时间窗是外部硬约束，压不了），但「有效投入产出比」显著更高 —— 同样 3-4 个月，精准投 + 簇准备的转化率高于海投。

- **关键工具（与 Track 02 关联）**:
  - 牛客网（必备 —— 国内校招中枢：笔面经 + 内推 + 题库）
  - LeetCode / leetcode.cn（场景特化 —— 技术岗刷题）
  - 实习僧（场景特化 —— 找实习，实习是校招简历的核心素材）
  - 超级简历 / 五百丁（场景特化 —— 国内简历格式模板；**不需要 Jobscan**，国内 ATS 渗透率低）
  - ChatGPT / Claude（新兴 —— 简历改写、模拟面，强时效）

- **关键人物（与 Track 01 关联）**:
  - 本弧线**没有强对应的海外 figure** —— 国内校招的工业化节奏在海外无对应物。国内 figure（孙圈圈偏定位、Sean Ye 偏信息揭秘）有相关内容但主阵地在黑名单平台，一手信号结构性偏薄。
  - 国内校招培训机构（offer 先生 / 职优你 等）把这条弧线「工业化打包」成套餐 —— 作为**行业现状样本**，不作方法论背书（evidence: [T03-S022]）。

- **常见失败模式**:
  - **跳过定位直接海投**：触发原因 = 焦虑 + 「先投了再说」；后果 = 笔面试时间冲突、面试讲不出动机、全线低转化。怎么避免：投递前先花 1-2 周做 W5（定位），把目标收敛到 30-50 家。
  - **错过提前批 / 把希望压在春招**：触发原因 = 不懂校招时间窗结构（以为「提前批 = 还没准备好不该投」「春招还有机会」）；后果 = 提前批的低风险占坑机会浪费、春招岗位少导致 offer 荒。怎么避免：开学就做一张校招时间表，提前批当练兵 + 占坑，秋招当主战场。
  - **笔试裸考**：触发原因 = 低估国内校招笔试的硬门槛属性；后果 = 连面试机会都拿不到。怎么避免：技术岗按目标公司高频题刷，非技术岗练行测题型，提前看牛客笔经。
  - **把「付费内推 / 内推保过」当真**：触发原因 = 焦虑 + 机构营销话术；后果 = 花钱买的「内推」要么无效要么是绕过流程的灰色操作，背调有暴露风险。怎么避免：理解国内内推的真实作用是「免笔试 / 简历直达」不是「保过」，正规内推不收费。

### W2. 国内社招求职弧线

- **这是谁的工作流**: 求职者（在职 / 待业的有经验者）的工作流；教练做卡点诊断 + 谈薪 / offer 决策陪跑。

- **One-liner**: 有经验者从「想换工作」到「拿到 offer 并完成决策」的全过程 —— 国内社招的特征是 **BOSS 直聘聊天式直招**（求职者直接和 HR / 用人老板对话，对话本身就是触达 + 初筛），没有统一时间窗（rolling），背调环节比校招更实。

- **Trigger**: 在职者决定跳槽，或待业者开始找下一份工作；通常由「现岗发展受限 / 薪资倒挂 / 团队变动 / 被裁」触发。

- **Output**: 拿到 offer → 谈薪 → 通过背调 → 完成 offer 决策（接受 / 拒绝 / 多 offer 取舍）→ 离职交接。

- **入门 SOP（minimum viable steps）**:
  1. **定位 + 简历更新**：明确目标岗位 / 薪资区间 / 行业，把简历从「校招版」升级为「社招版」（社招简历重「成果与影响」不重「在校经历」，要量化）。**跳过会发生什么**：拿一份旧简历或目标模糊地投 → 聊天阶段 HR 一问「你想找什么」就露馅，匹配度低。
  2. **平台触达（聊天式）**：在 BOSS 直聘等平台主动「打招呼」，简洁说明匹配点，引导对话；同时可走脉脉弱关系、猎头（中高端岗位）。**跳过会发生什么**：只被动等 HR 来撩 / 只海投不主动开口 → 国内社招主动出击节奏，被动等的命中率低。
  3. **沟通 + 确认岗位真实性**：在聊天中确认 JD 细节、薪资范围、团队情况，筛掉「画大饼 / 不匹配 / 薪资不达标」的岗位。**跳过会发生什么**：不在聊天阶段筛 → 面试了一圈才发现岗位和预期差很远，浪费时间。
  4. **多轮面试**：专业面 + 主管面 + HR 面，逐轮推进；每轮针对性准备。**跳过会发生什么**：不准备 → 讲不清过往成果的「影响」，被刷。
  5. **谈薪**（详见 W7）：拿到口头意向后谈薪，争取合理涨幅。**跳过会发生什么**：直接接受首次报价 → 国内社招薪资有谈判空间，不谈通常拿不到最优。
  6. **背调 + offer 决策**：配合背调（社招背调查上家在职时间 / 职级 / 离职原因，比校招实），做 offer 横向决策，确认无误后走离职流程。**跳过会发生什么**：简历 / 面试有夸大 → 背调暴露，offer 撤回；不做横向决策直接跳 → 可能跳进一个更差的坑。

- **资深路径（差异点 — skip / optimize / add）**:
  - **skip**: 资深教练会让多数社招求职者**跳过「广撒网式打招呼」** —— 社招不是投得越多越好，目标明确者把精力放在「20-30 个高匹配岗位的深度沟通」上。也跳过对「明显薪资倒挂 / 明显画饼」岗位的投入 —— 在聊天第 3-5 句就该筛掉，不进面试。
  - **optimize**: 把「等 HR 主动」优化成「主动出击 + 精准开场白」—— 资深教练帮求职者写一句话开场白模板（不是群发话术，是「我做过 X，和你这个岗位的 Y 高度对得上」），把聊天式平台的「主动权」用起来。把「裸辞找工作」优化成「在职骑驴找马」—— 除非现岗不可忍，资深教练通常建议在职状态下找（议价权 + 心态都更好），新手常因冲动裸辞陷入「越急越谈不好」。
  - **add**: 资深教练会**额外做「背调前置体检」** —— 在投递前就和求职者核对：简历上的职级 / 在职时间 / 离职原因和真实情况是否一致，把会在背调暴露的隐患提前处理（要么改对，要么准备好如何解释），新手往往等到背调阶段才发现问题。还会**额外做「跳槽动机校准」** —— 先问清「为什么想走、走了能解决什么」，避免「逃离式跳槽」（只想离开现状、没想清要去哪），这是社招最常见的隐性失败。

- **近期变化（近 12 个月）**:
  - **2023-2025 起，由国内就业市场承压驱动**：社招岗位收缩、HR 议价权上升，教练对「谈薪空间」和「跳槽时机」的预期管理更保守 —— 「在职找、别裸辞」「涨幅预期要现实」被更强调（evidence: [T03-S021]）。
  - **2024-2025 起，由平台 AI 化驱动**：BOSS 直聘等加入 AI 简历诊断 / AI 岗位推荐，招聘方端也用算法做简历初筛 —— step 1「简历」和 step 2「触达」的具体做法在变，但**聊天式直招的弧线骨架稳态**（evidence: [T03-S010], [T02-S009]）。
  - 当前采用率：聊天式直招在国内社招已是绝对主流（多年稳态）；AI 辅助是新增层。

- **典型耗时**:
  - 入门 SOP（求职者侧）：在职找通常 1-3 个月（视岗位层级和市场冷热）；市场差时可拉长到 3-6 个月。
  - 资深路径：耗时不一定更短（市场是外部约束），但「精准沟通 + 背调前置体检」让「面到一半发现不匹配」「背调翻车」这类返工大幅减少。

- **关键工具（与 Track 02 关联）**:
  - BOSS 直聘（必备 —— 国内社招聊天式直招主力；对话记录天然是 tracker）
  - 猎聘 + 猎头（场景特化 —— 中高端岗位，总监级以上）
  - 拉勾（场景特化 —— 互联网技术岗）
  - 脉脉（场景特化 —— 职场社交 / 弱关系触达 + 职言区看公司情报）
  - 看准网（场景特化 —— 国内公司点评 + 面经 + 薪资交叉验证）

- **关键人物（与 Track 01 关联）**:
  - 本弧线**没有强对应的海外 figure** —— BOSS 直聘聊天式直招在海外无对应物。
  - Madeline Mann 的「招聘官视角」（理解 HR / 用人方怎么决策）对「沟通 + 谈薪」环节有跨 locale 的方法论价值，但她的具体战术（如「别先报数字」）需按国内场景变形（evidence: [T01-S017]）。

- **常见失败模式**:
  - **逃离式跳槽（动机不清）**：触发原因 = 只想离开现状、没想清去哪；后果 = 跳进一个新的坑，半年后又想走。怎么避免：跳槽前做动机校准 —— 「走了具体能解决什么、不能解决什么」。
  - **冲动裸辞**：触发原因 = 现岗情绪积压；后果 = 失去「在职」的议价权和心态，越急越谈不好，空窗还要解释。怎么避免：除非现岗不可忍，在职骑驴找马。
  - **背调翻车**：触发原因 = 简历 / 面试夸大了职级 / 在职时间 / 离职原因；后果 = offer 撤回 + 行业口碑受损。怎么避免：投递前做背调前置体检，简历与真实情况对齐。
  - **聊天阶段不筛，面到一半才发现不匹配**：触发原因 = 「先面着再说」；后果 = 时间浪费在无效岗位上。怎么避免：在聊天前 3-5 句就确认 JD / 薪资 / 团队，不达标的不进面试。

### W3. 海外求职弧线

- **这是谁的工作流**: 求职者（目标海外英语区）的工作流；教练做定位 / 简历 / networking / 谈薪全链路陪跑。

- **One-liner**: 求职者在海外英语区从「定位」到「谈定 offer」的全过程 —— 海外的特征是 **LinkedIn 为中枢 + recruiter screen + networking 权重高**（尤其科技 / 咨询 / 中高层级岗位），简历严格匿名化（无照片 / 年龄 / 婚育，反歧视），招聘 rolling 无统一时间窗。

- **Trigger**: 求职者目标地区是海外英语区（本土求职者、国际学生 / 海归求海外岗、外派 / 移民求职均属此）。

- **Output**: 拿到 offer → 谈薪（base + bonus + 股票总包）→ 接受 → （国际学生 / 移民）确认签证可行 → 入职。

- **入门 SOP（minimum viable steps）**:
  1. **定位与目标公司清单**（详见 W5）：收敛目标岗位 / 行业 / 地点，列一份目标公司清单（海外更强调「锁定少数目标公司深耕」而非广投）。**跳过会发生什么**：无差别海投 → 简历无法针对性、networking 没有方向、面试讲不出动机。
  2. **简历过 ATS + 匿名化**：做一份海外格式简历（匿名化：无照片 / 年龄 / 婚育；动词开头 + 量化成就），用 Jobscan / Teal 做关键词匹配度检查（详见 W6）。**跳过会发生什么**：简历被 ATS 解析失败 / 关键词不匹配 → 网申路径下连人都到不了；带照片 / 年龄反而触发反歧视流程问题。
  3. **networking / 内推**：在 LinkedIn 找目标公司在职者发起弱关系连接、做信息面谈（informational interview，铁律「第一次不要工作」）、争取内推。**跳过会发生什么**：海外科技 / 咨询 / 高层级岗位内推权重高，纯网申命中率低；hidden job market 完全进不去（evidence: [T03-S017]）。
  4. **网申 + recruiter screen**：通过公司官网 / LinkedIn 投递；通过初筛后是 recruiter screen（HR / recruiter 的初步电话面，过岗位匹配 + 期望薪资 + 时间线）。**跳过会发生什么**：没有投递入口；recruiter screen 没准备 → 期望薪资报错 / 讲不清动机，第一关就挂。
  5. **多轮面试**：行为面（STAR）+ 专业面（技术岗 coding / 系统设计、咨询 case、PM 产品题）+ 有时含 onsite / 多人轮面。**跳过会发生什么**：不按类型准备 → 行为面流水账、专业面露怯。
  6. **谈薪 + offer 决策 + 签证确认**（详见 W7）：海外谈薪空间通常比国内大，按 base + bonus + 股票总包谈；国际学生 / 移民还要确认 offer 能否支持签证（如美国 H1B / OPT、英国工签等）。**跳过会发生什么**：只比 base 不比总包 → 决策失真；不确认签证 → offer 拿到了但身份走不通。

- **资深路径（差异点 — skip / optimize / add）**:
  - **skip**: 资深教练会**根据「岗位层级 / 行业」决定要不要重投 networking** —— 不是所有岗位都 networking-first。高层级 / 科技 / 咨询岗 networking 权重高、值得重投；而大批量、流程化招聘的初级岗（如部分零售 / 客服 / 蓝领岗），networking 边际收益低，资深教练会让求职者**跳过重度 networking**，把精力放回简历 + 海投。也跳过「给每个岗位写一封长 cover letter」—— 多数场景定制简历 + 短 cover letter 即可，长信性价比低。
  - **optimize**: 把「海投 + 等回音」优化成「networking 触达优先于网申」（在 networking 权重高的赛道）—— Belcak 体系：与其把简历优化十遍丢进投递端口，不如研究目标公司在头疼的问题、用一份能帮上忙的小样品（Value Validation Project）直接触达招聘经理（evidence: [T03-S001]）。把「逐个公司临场准备」优化成「按面试类型批量准备」—— 行为面 STAR 故事库一次建好复用，技术岗按公司簇刷高频题。
  - **add**: 资深教练会**额外做「recruiter screen 专项准备」** —— 新手常把 recruiter screen 当「随便聊聊」，其实它是有筛选功能的一关（期望薪资在这里第一次被问，报错就出局）。还会**额外做「市场周期 + 签证现实」的预期校准** —— 明说 2023-2025 海外尤其科技业经历裁员潮、岗位收缩、签证政策收紧，「投了很久没回音」高度是结构性的、不是个人不行（不 victim-blame）；国际学生还要把签证可行性当**定位的前置筛选条件**，而非 offer 阶段才考虑（evidence: [T03-S020]）。

- **近期变化（近 12 个月）**:
  - **2023-2025 起，由全球科技业裁员潮 + 就业市场承压驱动**：海外岗位收缩、求职周期拉长、ghosting（投了 / 面了没下文）普遍 —— 教练对「求职是结构性艰难、不是你的错」的预期管理权重显著上升（evidence: [T03-S020], [T03-S007]，Alison Green 2025 多次公开谈「艰难求职市场」）。
  - **2024-2025 起，由招聘两端 AI 化驱动**：step 2「简历」可用 AI 改写但「AI 味」被招聘方识别成减分项；step 4 部分大厂校招 / 初筛环节用单向 AI 视频面试（候选人对摄像头答题、AI 打分）；招聘方简历初筛也在用 LLM。**这些是步骤内做法的变化，弧线骨架（定位→简历→networking→网申/screen→面试→谈薪）稳态**（evidence: [T03-S015], [T02-S025]）。
  - 当前采用率：networking-first 在海外科技 / 咨询求职是成熟主流方法（非新事物）；AI 工具在求职两端已是普遍现象。

- **典型耗时**:
  - 入门 SOP（求职者侧）：3-6 个月常见；2023-2025 市场承压期可拉长到 6-12 个月，尤其科技岗 / 国际学生。
  - 资深路径：耗时不一定更短（市场是外部约束），但「按赛道决定 networking 投入 + recruiter screen 专项 + 批量准备」让有效转化率更高、返工更少。

- **关键工具（与 Track 02 关联）**:
  - LinkedIn（必备 —— 海外求职 + networking + 公司研究中枢）
  - Jobscan / Teal（必备 —— 简历过 ATS 的关键词匹配度检查）
  - Glassdoor（场景特化 —— 公司面经 + 评价 + 薪资区间）
  - levels.fyi（场景特化 —— 科技岗谈薪精确数据 anchor）
  - 一亩三分地（场景特化 —— 北美华人 / 国际学生面经 + 时间线 + 签证讨论）
  - Pramp / interviewing.io（场景特化 —— 技术岗真人 mock）

- **关键人物（与 Track 01 关联）**:
  - Austin Belcak 在 Cultivated Culture 旗舰长文 + YouTube + 长 podcast 中详细谈过这条弧线的 networking-first 版本（锁定目标 → 价值邮件 → 信息面谈）（evidence: [T03-S001, T03-S002]）。
  - Liz Ryan 的 pain letter / human-voiced resume 是这条弧线「简历 + 触达」环节的「叙事派」打法（evidence: [T03-S003]）。
  - Andrew LaCivita 的「求职系统四步」（锁对雇主 → 找到对的人 → 发能收到回复的消息 → 建可重复计划）是这条弧线的另一套体系化表达（evidence: [T03-S005]）。
  - Mark Granovetter 的弱关系理论是整个 networking 环节的学术底座（evidence: [T03-S017]）。

- **常见失败模式**:
  - **无差别海投、零 networking**：触发原因 = 「投得多 = 努力」的误区；后果 = 在 networking 权重高的赛道命中率极低、hidden job market 完全进不去。怎么避免：先判断目标赛道的 networking 权重，权重高则 networking 触达优先于网申。
  - **信息面谈一开口就要工作 / 要内推**：触发原因 = 把 informational interview 当「变相求职」；后果 = 破坏信任、对方设防、关系做死。怎么避免：守「第一次不要工作」铁律，先建立关系和了解。
  - **简历带照片 / 年龄 / 把国内格式照搬**：触发原因 = 不懂海外匿名化反歧视惯例；后果 = 显得不专业，甚至给招聘方制造合规顾虑。怎么避免：海外简历严格匿名化，按目标地区惯例做。
  - **把 ghosting 解读成「我一定很差」**：触发原因 = 不了解 ghosting 高度结构性（市场差 + 招聘方流程问题）；后果 = 自我否定、放弃、心态崩。怎么避免：教练做预期校准 —— ghosting 是市场常态不是个人评判，把精力放回可控部分（W 系列各环节）。
  - **国际学生到 offer 阶段才查签证**：触发原因 = 把签证当「后面再说」；后果 = offer 拿到了但岗位 / 公司不 sponsor，前功尽弃。怎么避免：把签证可行性当定位的前置筛选条件。

### W4. 单次面试的结构（准备 / 临场 / 复盘）

- **这是谁的工作流**: 求职者的工作流；教练在「准备」和「复盘」两端介入最深（mock interview + debrief 是教练的核心交付物之一）。

- **One-liner**: 把一场具体面试从「拿到邀约」到「完成 + 复盘」走成一个可重复的闭环 —— 不是临场发挥，是「准备 → 临场 → 复盘」三段式，复盘喂回下一场。

- **Trigger**: 求职者拿到一场具体的面试邀约（任何弧线、任何轮次都适用，是 W1/W2/W3 内嵌的子流程）。

- **Output**: 完成这场面试 + 一份结构化复盘（哪些答得好 / 哪些卡住 / 下一场改什么）。

- **入门 SOP（minimum viable steps）**:
  1. **公司 + 岗位研究**：研究公司业务 / 近期动态、JD 关注点、面经（Glassdoor / 牛客 / 一亩三分地），判断这一轮考什么。**跳过会发生什么**：对公司一无所知 → 答「为什么是我们」时空洞、反向提问问不出有质量的问题，减分。
  2. **按面试类型准备内容**：行为面准备 STAR 故事（每段经历有可量化 Result）；专业面按岗位类型准备（技术岗刷高频题 / 系统设计、咨询 case、PM 产品题）。**跳过会发生什么**：行为面讲成流水账 / 职责描述、专业面没框架，被「讲不清楚」刷掉。
  3. **临场：结构化作答 + 双向评估**：把每个回答讲出结构（行为题套 STAR；技术题先澄清问题→举例→暴力解→优化→写→测）；同时记住面试是双向的，自己也在筛雇主，结尾做有质量的反向提问。**跳过会发生什么**：答案无结构（面试官跟不上）/ 不反向提问（显得不在意 + 错过筛雇主的机会）。
  4. **跟进**：面试后发一封简短得体的感谢 / 跟进邮件（海外更常见，国内视场景）。**跳过会发生什么**：在「礼仪重要 + 候选人接近」的场景下，少这一步可能微弱失分（但反复追问进度反而是负面 —— 跟进 ≠ 催）。
  5. **复盘**：趁记忆新，结构化记录 —— 哪些问题答得好 / 哪些卡住 / 面试官的反应 / 下一场要改什么。**跳过会发生什么**：只「面」不「复盘」→ 同样的错误下一场再犯，面试经验不累积。

- **资深路径（差异点 — skip / optimize / add）**:
  - **skip**: 资深教练 / 资深求职者会**跳过「为每一场从零做公司研究」** —— 同公司不同轮次、同行业不同公司，研究内容大量可复用，不重复造轮子。也跳过「准备一份逐字背诵的答案脚本」—— 背稿在临场被追问时会崩，且「背诵痕迹」本身减分；资深做法是准备「要点 + 故事素材」而非逐字稿。
  - **optimize**: 把「每场临时凑 STAR 故事」优化成「一次性建一个 STAR 故事库」—— 5-8 个覆盖不同能力维度的故事，面试时按问题调用、微调，而非现编。把「自己对着镜子练」优化成「真人 mock + 结构化 debrief」—— mock 的价值是把「会答」转成「能在压力下 + 被追问时讲清楚」，这是教练交付物里 ROI 最高的一项（evidence: [T03-S004, T03-S016]）。
  - **add**: 资深教练会**额外做「面试官视角校准」** —— 教求职者理解「面试官在按 rubric 评分、在找证据」，所以准备方向是「对着能力锚点准备真实证据」，不是表演术 / 读微表情 / 靠气场（结构化面试效度研究是这条的实证底座，evidence: [T03-S018]）。还会**额外做「失败场的情绪复盘」** —— 不止复盘「答得怎样」，还复盘「挂了之后怎么不被一场失败带崩心态」，把单场结果和自我评价解耦。

- **近期变化（近 12 个月）**:
  - **2024-2025 起，由招聘方 AI 化驱动**：部分公司（尤其海外大厂校招初筛）用单向 AI 视频面试 —— 求职者对摄像头答固定问题、AI 打分。这给「准备」环节加了一个新子类型：**应对 AI 初面**（结构化、关键词清晰、节奏稳、正视镜头）。这是「准备」环节内新增了一个分支，**三段式骨架（准备 / 临场 / 复盘）本身稳态**（evidence: [T03-S016], [T02-S025]）。
  - **2024-2025 起，由求职者侧 AI 工具驱动**：ChatGPT / Claude 用于生成练习题 + 追问、Yoodli 类工具用于「复盘」环节做表达分析（语速 / 填充词 / 眼神）。**红线**：AI 的合法位置是「赛前准备 + 赛后复盘」，不是「赛中实时喂答案」（AI 实时面试辅助 = 作弊，远程面试反制在收紧）（evidence: [T02-S026, T02-S027]）。
  - 当前采用率：AI 辅助练习 / 复盘在求职者中普遍；AI 视频初面在海外大厂校招初筛较多见。

- **典型耗时**:
  - 入门 SOP（求职者侧）：单场面试的准备 + 复盘约 2-6 小时（视轮次和岗位类型；技术岗专业面准备更久）。
  - 资深路径：单场准备时间显著下降（故事库 + 公司研究可复用），节省的时间投入「真人 mock + 深度复盘」。

- **关键工具（与 Track 02 关联）**:
  - Glassdoor / 牛客 / 一亩三分地（场景特化 —— 面经研究）
  - Pramp / interviewing.io（场景特化 —— 技术岗真人 mock）
  - Big Interview（场景特化 —— 行为面结构化训练 + AI 反馈）
  - Yoodli 类 AI 表达教练（新兴 —— 「复盘」环节的表达分析，强时效）
  - ChatGPT / Claude（新兴 —— 生成练习题 + 追问，强时效；红线：不用于赛中）

- **关键人物（与 Track 01 关联）**:
  - Gayle L. McDowell 在《Cracking the Coding Interview》中详细拆解了技术面试的「准备 + 结构化临场作答」流程（澄清→举例→暴力解→优化→写→测）（evidence: [T03-S004]）。
  - Andrew LaCivita 的「面试三动作」（卖自己 + 反向套情报 + close the interview）是「临场」环节的方法论代表（evidence: [T03-S005]）。
  - Alison Green 的 Ask a Manager 是「跟进礼仪 + 反向提问 + 面试是双向评估」这些环节的真实情境参照库（evidence: [T03-S007]）。

- **常见失败模式**:
  - **逐字背稿，被追问就崩**：触发原因 = 把面试当「背诵考试」；后果 = 临场追问 / 变体问题接不住，且背诵痕迹本身减分（显得不真诚）。怎么避免：准备「要点 + 故事素材」，不准备逐字稿；用 mock 练「被追问时还能讲清楚」。
  - **只「面」不「复盘」**：触发原因 = 面完就想休息 / 等结果；后果 = 经验不累积，同样的错误反复犯。怎么避免：面后趁记忆新做结构化复盘，喂回下一场。
  - **行为面讲成流水账 / 职责描述**：触发原因 = 不知道行为面要的是「证据」不是「经历叙述」；后果 = 答得再流畅也没有可评分的实质。怎么避免：套 STAR，每段经历必须有可衡量的 Result。
  - **把面试神秘化（读微表情 / 靠气场 / 话术征服）**：触发原因 = 营销话术 + 不懂结构化面试机制；后果 = 准备方向错，把精力花在表演术上。怎么避免：理解结构化面试按 rubric 评分，准备方向是「对着能力锚点准备真实证据」。
  - **被一场失败带崩**：触发原因 = 把单场结果等同于自我评价；后果 = 心态崩、影响后续面试。怎么避免：教练做情绪复盘，单场结果和自我评价解耦 —— 一场面试是一次匹配，不是一次审判。

### W5. 求职定位与职业叙事流程

- **这是谁的工作流**: 求职者的工作流；但这是**教练介入价值最高**的一段 —— 求职者最容易跳过、跳过代价最大。

- **One-liner**: 把「我想找工作」收敛成「我要找哪一类岗位 + 为什么是我」—— 求职的第一性问题。定位错了，后面简历 / 投递 / 面试全是高效地做错的事。

- **Trigger**: 求职者目标岗位 / 行业不清（「什么都能做 / 什么都想试」）、要转方向、应届生第一次求职、或求职「投了很多没回音」回头排查（往往根因在定位）。

- **Output**: 一份收敛的目标岗位清单（具体到职能 / 行业 / 层级 / 地点）+ 一条可复用的职业叙事（「为什么是你 / 为什么这条路」的连贯故事，能用在简历、cover letter、面试动机题、networking 自我介绍）。

- **入门 SOP（minimum viable steps）**:
  1. **自我盘点**：盘点能力 / 经历 / 兴趣 / 价值观 / 约束条件（薪资底线、地点、签证等）—— Bolles 的 Flower Exercise、DYL 的能量 / 心流日志都是这一步的工具。**跳过会发生什么**：跳过自我盘点直接定目标 → 定的是「听起来好」的目标而非「适合且能进」的目标。
  2. **目标收敛**：把「什么都能做」收敛到 1-3 个具体方向（职能 + 行业 + 层级），列目标岗位 / 公司清单。**跳过会发生什么**：目标不收敛 → 简历无法针对、海投、面试讲不出「为什么是这家」。
  3. **卖点提炼**：针对收敛后的目标，提炼「我相对这个岗位的核心卖点」（可迁移技能 + 量化成就 + 差异化）。**跳过会发生什么**：不提炼卖点 → 简历是「职责清单」、面试讲不清「为什么选我」。
  4. **故事线编织**：把零散经历串成一条连贯叙事 —— 「我从哪来、为什么走到这里、为什么这个方向是合理的下一步」。**跳过会发生什么**：经历是散点 → 招聘方看不出逻辑，尤其转行 / 有空窗 / 频繁跳槽的人会被「看不懂」直接刷。

- **资深路径（差异点 — skip / optimize / add）**:
  - **skip**: 资深教练对「已经定位清晰」的求职者会**跳过完整自我盘点** —— 定位清楚的人（如「我就是要做后端、目标是 XX 梯队公司」）不需要重做 Flower Exercise，直接进卖点提炼 + 故事线。把判断「需不需要做定位」本身当第一步，而不是对所有人都从头跑一遍。
  - **optimize**: 把「让求职者自己想清楚」优化成「教练用结构化提问 + 外部视角加速」—— 求职者常陷在「自己看自己」的盲区里（高估 / 低估某些经历、看不见可迁移性），资深教练用提问和行业认知帮他「看见」，把可能拖几个月的自我探索压缩到几次对话。把「先想清楚再行动」优化成「低成本原型 + 迭代」（DYL 的奥德赛计划）—— 不追求一次定到完美，先定个方向去试（投几份、做几次信息面谈），用反馈修正。
  - **add**: 资深教练会**额外做「约束条件显性化」** —— 主动问清薪资底线、地点、签证、家庭等真实约束，把它们当定位的硬筛选条件而非「后面再说」（新手常定一个忽略约束的「理想目标」，到后期才发现走不通）。还会**额外做「定位 vs 生计的现实平衡」** —— 不把「定位」做成「必须找到天命才能投」的奢侈品；有生计压力时，「先拿一个能接受的过渡 offer、同时继续定位」是合法策略，不是失败（career capital 视角，但不脱离当下生计 —— evidence: [T03-S023]）。

- **近期变化（近 12 个月）**:
  - **近 12 个月：弧线骨架无重大变化（行业稳态）**。「定位先于投递」是这一行最稳的方法论共识，从 Bolles（1970）到 DYL（2016）到当代 figures 一脉相承，5-50 年无重大变化（evidence: [T03-S009], [T03-S008]）。
  - **2024-2025 步骤内的小变化**：ChatGPT / Claude 被用于「卖点提炼」和「故事线编织」的草拟和角色扮演 —— 但这是工具层变化，定位流程本身稳态。**注意红线**：AI 能帮「梳理和表达」真实的经历和方向，不能帮「编一个不属于你的定位 / 故事」。
  - 当前采用率：AI 辅助梳理在求职者中普遍；定位流程方法论本身是长期稳态。

- **典型耗时**:
  - 入门 SOP（求职者侧）：自助做通常 2-6 周（自我探索是慢的，且容易拖）；有的人会拖几个月。
  - 资深路径：教练用结构化提问 + 外部视角加速，通常 2-4 次对话（1-3 周）能把方向收敛到「足够开始投递」的程度 —— 不追求完美，追求「可行动」。

- **关键工具（与 Track 02 关联）**:
  - 本流程**工具依赖低** —— 核心是结构化提问 + 自我盘点，不是软件。
  - ChatGPT / Claude（新兴 —— 卖点提炼 / 故事线草拟的辅助，强时效；红线：梳理真实经历，不编造定位）
  - LinkedIn / 牛客（场景特化 —— 看目标岗位的真实 JD、看在职者的真实路径，给定位提供「外部参照」）

- **关键人物（与 Track 01 关联）**:
  - Richard N. Bolles《What Color Is Your Parachute?》是「自我盘点 → 定位 → 渠道」流程的奠基者，Flower Exercise 是自我盘点的经典工具（evidence: [T03-S009]）。
  - Bill Burnett & Dave Evans《Designing Your Life》把「定位」现代化为「奥德赛计划 → 原型 → 迭代」的设计思维流程（evidence: [T03-S008]）。
  - Andrew LaCivita 的「create self-awareness *before* not after」强调定位 / 自我认知必须前置（evidence: [T03-S005]）。
  - 80,000 Hours 提供「career capital」等概念，把单次求职定位拉到长期职业视角（evidence: [T03-S023]）。

- **常见失败模式**:
  - **跳过定位直接改简历 / 海投**：触发原因 = 「定位很虚 / 浪费时间，先动起来再说」；后果 = 高效地做错的事 —— 简历无法针对、海投低转化、面试讲不清动机。怎么避免：把定位当强制前置步骤，目标不收敛不进 W6（简历）。
  - **定一个忽略约束的「理想目标」**：触发原因 = 不愿面对薪资 / 地点 / 签证等现实约束；后果 = 投到后期才发现走不通，前功尽弃。怎么避免：定位第一步就把约束条件显性化，当硬筛选条件。
  - **职业叙事 = 编一个完美故事**：触发原因 = 以为「叙事」就是「包装得好看」；后果 = 故事和真实经历脱节，面试一追问就崩、且越过造假红线。怎么避免：叙事是「把真实经历串成有逻辑的线」，不是虚构 —— 逻辑可以重新组织，事实不能编。
  - **把定位做成「找到天命才能投」的奢侈品**：触发原因 = 完美主义 + 不顾生计压力；后果 = 迟迟不投、生计危机加剧。怎么避免：有生计压力时，「先拿过渡 offer + 继续定位」是合法策略；定位追求「可行动」不追求「完美」。

### W6. 简历优化流程（JD 拆解 → 量化成就 → 关键词 → 格式）

- **这是谁的工作流**: 求职者的工作流；教练做 JD 拆解示范 + 量化追问 + **优化 vs 造假红线把关**。

- **One-liner**: 把一份简历针对一个 / 一类岗位优化到「机器读得顺、人看得清、且全部属实」—— 核心是「针对性」和「举证」，硬约束是**优化不是造假**。

- **Trigger**: 求职者要投一个具体岗位或一类岗位（W1/W2/W3 内嵌的子流程，定位 W5 完成后启动）。

- **Output**: 一份针对目标岗位的简历 —— 量化的成就（而非职责清单）、与 JD 对得上的关键词、机器和人都读得顺的格式、内容全部真实可验证。

- **入门 SOP（minimum viable steps）**:
  1. **JD 拆解**：把目标 JD 拆成「硬性要求 / 加分项 / 高频关键词 / 这个岗位真正在解决什么问题」。**跳过会发生什么**：不拆 JD 直接写 → 简历是「我有什么」而非「岗位要什么」，匹配度低、初筛（机器或人）都过不去。
  2. **经历 → 量化成就改写**：把每段经历从「负责 X」改写成「做到了 X（用 Y 衡量），靠做 Z」（Jeff Su 的 X-Y-Z 公式）—— 用数字 / 结果，不用职责描述。**跳过会发生什么**：简历是职责罗列 → 招聘方看不出你「干成了什么」，和所有同岗位候选人长一样。
  3. **关键词对齐**：把 JD 里的关键技能 / 工具 / 术语，在简历里用**真实经历**自然覆盖（海外网申场景用 Jobscan / Teal 做匹配度检查）。**跳过会发生什么**：海外网申场景关键词不匹配 → ATS 初筛 + 人眼初筛都判「不相关」。
  4. **格式与可读性**：海外格式严格匿名化（无照片 / 年龄 / 婚育）+ 简洁可解析（避免复杂表格 / 图片破坏机器可读性）；国内格式常含照片 / 学校 / 实习。**跳过会发生什么**：海外简历带照片 → 给招聘方制造合规顾虑；格式花哨 → ATS 解析失败 / 人眼疲劳。
  5. **真实性核对（红线步骤）**：核对每一条 —— 经历真实、数字可验证、头衔没改、能力没夸大。**跳过会发生什么**：编造经历 / 改头衔 / 虚构数字 → 背调和试用期会暴露，offer 撤回 + 职业污点。**这一步不可跳过、不可外包给 AI 决定**。

- **资深路径（差异点 — skip / optimize / add）**:
  - **skip**: 资深教练在 networking / 内推路径下会**降低 ATS 关键词优化的优先级** —— 内推简历多由人直接看，不经过 ATS 重度筛选，此时把精力从「堆关键词」转回「故事和成果讲清楚」。也会让求职者**跳过「一份简历投所有岗位」的幻想**和「为冷门小厂也做精细定制」的过度投入 —— 定制是分级的，重点岗位深度定制，长尾岗位轻度调整。
  - **optimize**: 把「逐字逐句重写每个版本」优化成「一个 master 简历 + 按岗位类型做 2-4 个变体」—— 不是每投一家重写一遍，而是按「岗位簇」维护少数几个版本。把「把 Jobscan 匹配度刷到越高越好」优化成「匹配度当参考不当 KPI」—— 资深教练知道堆关键词到 90%+ 反而是 keyword stuffing，被人眼看到减分，且 ATS 在语义化升级，纯关键词战术有效性在被稀释（evidence: [T03-S013], [T02-S028]）。
  - **add**: 资深教练会**额外做「优化 vs 造假的红线教学」** —— 主动和求职者划清楚：量化真实成就、ATS 关键词、措辞提升、用目标岗位语言重新包装真实技能 = 合法优化；编造经历、改头衔、虚构数字、把不具备的能力写成具备 = 造假。这条线新手最容易模糊（「美化一下而已」），资深教练必须硬划（evidence: [T03-S007]，intake warnings #2）。还会**额外做「AI 味检查」** —— 2024-2025 千篇一律的 LLM 措辞被招聘方识别成减分项，资深教练会让 AI 辅助后的简历再做「去 AI 味 + 个性化」改写。

- **近期变化（近 12 个月）**:
  - **2024-2025 起，由通用 LLM 普及驱动**：ChatGPT / Claude 用于「经历 → 量化改写」「关键词对齐」已是普遍做法 —— 但带来两个新问题：(1) **AI 味简历被招聘方识别**成减分项，需额外做去 AI 味改写；(2) LLM 会「顺着你美化到编造的程度」，**真实性核对（step 5）的把关压力上升**（evidence: [T02-S023, T02-S024]）。
  - **2024-2025 起，由招聘方 ATS 语义化升级 + AI 初筛驱动**：ATS 不再是纯关键词匹配、在向语义匹配升级，招聘方也用 LLM 做简历初筛 —— 「堆关键词」这一战术本身的有效性正在被稀释（evidence: [T02-S001, T02-S028]）。**流程骨架（JD 拆解 → 量化 → 关键词 → 格式 → 真实性核对）稳态，变的是「关键词」这一步的有效性和「真实性核对」的把关难度**。
  - 当前采用率：AI 辅助改简历在求职者中已是普遍现象；「AI 味识别」是招聘方端的新增反制。

- **典型耗时**:
  - 入门 SOP（求职者侧）：第一份针对性简历从零做约 4-10 小时；之后每个岗位定制约 0.5-2 小时。
  - 资深路径：master + 变体体系建好后，单岗位定制压缩到 15-40 分钟；省下的时间投入「真实性核对 + 去 AI 味」。

- **关键工具（与 Track 02 关联）**:
  - Jobscan / Teal（必备 —— 海外网申场景的 ATS 关键词匹配度检查；国内不需要）
  - 超级简历 / 五百丁（场景特化 —— 国内简历格式模板）
  - ChatGPT / Claude（必备但强时效 —— 经历改写 / 关键词对齐辅助；红线：不编造、输出要去 AI 味）

- **关键人物（与 Track 01 关联）**:
  - Jeff Su 的 X-Y-Z 公式（「做到了 X，X 用 Y 衡量，靠做 Z」）是「经历 → 量化成就改写」这一步的事实标准战术（evidence: [T03-S006]）。
  - Liz Ryan 的 human-voiced resume 是这条流程的「叙事派」对照 —— 主张简历要「像人写的」，与「为 ATS 堆关键词」的技术派形成直接张力（evidence: [T03-S003]）。
  - Gayle L. McDowell《Cracking the Coding Interview》也有简历章节（多数技术求职者直接刷题忽略这部分）（evidence: [T03-S004]）。

- **常见失败模式**:
  - **把「美化」和「编造」混为一谈**：触发原因 = 红线模糊 + 「大家都美化」；后果 = 越过造假线，背调 / 试用期暴露，offer 撤回 + 职业污点。怎么避免：教练硬划红线 —— 重新组织和包装真实经历可以，编造事实不可以。
  - **不拆 JD，写一份「万能简历」投所有岗位**：触发原因 = 嫌定制麻烦；后果 = 简历是「我有什么」而非「岗位要什么」，全线低匹配。怎么避免：每类岗位先拆 JD，至少做岗位簇级别的定制。
  - **把 ATS 匹配度刷到越高越好**：触发原因 = 把工具分数当 KPI；后果 = keyword stuffing，人眼看到反而减分，且 ATS 语义化后纯关键词失效。怎么避免：匹配度当参考不当目标，关键词要用真实经历自然覆盖。
  - **让 LLM 直接生成整份简历就投出去**：触发原因 = 图省事；后果 = AI 味措辞被识别成减分项，且 LLM 容易「美化到编造」。怎么避免：AI 辅助后必做「去 AI 味 + 个性化 + 真实性核对」。
  - **海外简历照搬国内格式（带照片 / 年龄）**：触发原因 = 不懂匿名化反歧视惯例；后果 = 不专业 + 给招聘方制造合规顾虑。怎么避免：按目标地区惯例 —— 海外严格匿名化。

### W7. 谈薪流程

- **这是谁的工作流**: 求职者的工作流；教练做信息搜集示范 + 话术演练 + 多 offer 博弈陪跑 + offer 横向决策。

- **One-liner**: 从「即将拿 offer」到「谈定并完成 offer 决策」—— 谈薪是有信息基础、有时机、有话术、多变量的博弈，不是「张口要个数字」；核心是「把球踢回去 + 比总包不比 base + 用真实杠杆」。

- **Trigger**: 求职者进入 offer 阶段，或面试推进到「即将被问期望薪资 / 即将拿口头意向」（W1/W2/W3 内嵌的子流程）。

- **入门 SOP（minimum viable steps）**:
  1. **信息搜集（定 anchor）**：查目标岗位 / 职级 / 地点的市场薪资 —— 海外科技岗用 levels.fyi、通用岗用 Glassdoor / Payscale；国内用招聘平台区间 + 看准网 + 脉脉职言交叉。定一个 anchor（目标值）和 walk-away（底线）。**跳过会发生什么**：没有市场数据 → 报价没有锚、不知道对方报的合不合理，要么贱卖要么报得离谱出局。
  2. **管理「先报数字」的时机**：被过早问期望薪资时，尽量把球踢回去（「想先了解这个岗位的预算区间 / 整体 package」），不在信息不足时第一个报数字。**跳过会发生什么**：过早报数 —— 报低了贱卖自己、报高了被直接 pass（Madeline Mann 的「别先报数字」核心）。
  3. **拿到 offer 后再正式谈**：口头 / 正式 offer 到手后才是谈判窗口（offer 没正式接受前都还能谈），用「我对这个机会很感兴趣 + 基于市场数据 / 我的经验，希望能在 X 上调整」的结构。**跳过会发生什么**：在没 offer 时硬谈 → 没有筹码；接了 offer 才想谈 → 窗口已过。
  4. **按总包谈，不止 base**：base、奖金、股票 / RSU、签字费、假期、提前 review、搬家补助、签证 sponsorship 等都是可谈筹码。**跳过会发生什么**：只盯 base 一个数字 → 放弃了其他可谈空间，且无法跨 offer 公平比较。
  5. **多 offer 横向决策**：有多个 offer 时，按总包 + 平台 + 发展 + 文化 + 地点 + 风险综合比较（不是只比钱），做决策。**跳过会发生什么**：只比 base 月薪 / 只比单一维度 → 决策失真，可能选错。

- **资深路径（差异点 — skip / optimize / add）**:
  - **skip**: 资深教练会**根据场景跳过「死守不先报数字」** —— 在已强制薪资透明立法的地区、或招聘流程强制先填期望薪资的场景，「别先报数字」这条战术要变形（此时给一个有数据支撑的区间，而不是僵硬地拒绝）。把「先报数字者输」当**默认而非铁律**，按地区 / 流程判断。也会让求职者**跳过「为 5% 的涨幅把关系谈僵」** —— 资深教练知道谈判的边际收益和关系成本要权衡，不是「能多谈一点就一定多谈」。
  - **optimize**: 把「凭感觉报价」优化成「用数据 + 真实杠杆构建报价」—— anchor 来自市场数据，杠杆来自**真实的**竞争性 offer / 当前薪资 / 稀缺技能，话术围绕数据展开而非情绪。把「单轮硬谈」优化成「多变量打包谈」（Lewis Lin 的思路：base 谈不动就谈股票 / 签字费 / review 时间，把谈判当多变量而非一个数字的拔河）。
  - **add**: 资深教练会**额外做「话术演练（角色扮演）」** —— 谈薪话术「知道」和「说得出口」是两回事，资深教练用角色扮演让求职者把话术练到不慌（ChatGPT / Claude 可做陪练）。还会**额外做「offer 决策的真实约束校准」和「编造 offer 的红线」** —— 提醒求职者：(1) offer 决策不是纯财务题，要把发展 / 风险 / 生计压力一起算；(2) 用竞争性 offer 做杠杆只能用**真实的** offer，编造不存在的 offer 是高风险（被识破直接出局，且行业小）。
  - 国内特别说明：国内薪资信息透明度明显低于海外（没有 levels.fyi 级别的精确数据），国内谈薪 anchor 精度天然弱于海外，资深教练对国内求职者的谈薪预期管理更保守（evidence: [T02-S013, T02-S020]，intake warnings #4）。

- **近期变化（近 12 个月）**:
  - **2023-2025 起，由就业市场承压驱动**：海外科技业裁员 + 国内就业承压，整体「谈薪保守化」—— 招聘方议价权上升，教练对「涨幅预期」的预期管理更现实（不是「不能谈」，是「预期要贴市场」）（evidence: [T03-S020], [T03-S021]）。
  - **2023-2025 起，由多地薪资透明立法驱动**：海外越来越多地区强制薪资透明（JD 必须标薪资区间 / 禁止问薪资史）—— 「别先报数字」这条经典战术在这些地区要变形。这是**流程步骤内的变化（step 2 的具体打法），谈薪弧线骨架（信息搜集 → 时机 → 话术 → 总包 → 多 offer 决策）稳态**（evidence: [T03-S019]）。
  - **2024-2025 起**：ChatGPT / Claude 用于谈薪邮件 / 话术草拟 + 角色扮演陪练 —— 工具层变化，流程稳态。
  - 当前采用率：「比总包不比 base」「offer 后才是谈判窗口」是成熟共识；薪资透明立法是地区性、持续扩散中。

- **典型耗时**:
  - 入门 SOP（求职者侧）：信息搜集 1-3 小时 + 谈判往返通常 2-7 天（含等公司内部审批）。
  - 资深路径：耗时相近，但「数据 + 真实杠杆 + 话术演练」让谈判更可控、结果更优，且避免「谈僵 / 编造 offer」这类高代价失误。

- **关键工具（与 Track 02 关联）**:
  - levels.fyi（场景特化 —— 海外科技岗精确薪资 anchor）
  - Glassdoor / Payscale（场景特化 —— 通用岗位薪资区间）
  - 看准网 / 脉脉职言（场景特化 —— 国内薪资交叉验证，精度弱于海外）
  - ChatGPT / Claude（新兴 —— 谈薪邮件 / 话术草拟 + 角色扮演陪练，强时效）

- **关键人物（与 Track 01 关联）**:
  - Madeline Mann 的「别先报数字 / offer 没接受前都能谈」是这条流程传播最广的战术体系（evidence: [T01-S017]）。
  - Lewis Lin《Decode and Conquer》的谈薪章节 + 「一分钟谈薪脚本」+「谈薪是多变量博弈」（base + bonus + 股权 + 假期 + 签证）是体系化的谈薪框架（evidence: [T04-S003]）。
  - Andrew LaCivita 的求职系统也含谈判环节（evidence: [T03-S005]）。

- **常见失败模式**:
  - **没有市场数据就开谈**：触发原因 = 嫌查数据麻烦 / 不知道去哪查；后果 = 报价没锚，要么贱卖要么离谱出局。怎么避免：先用 levels.fyi / Glassdoor（海外）或招聘平台 + 看准网（国内）定 anchor。
  - **过早第一个报数字**：触发原因 = 被问就答的本能；后果 = 报低贱卖 / 报高出局。怎么避免：信息不足时把球踢回去问预算区间（但薪资透明地区要变形 —— 给有数据支撑的区间）。
  - **只比 base 月薪**：触发原因 = base 最直观；后果 = 放弃其他筹码 + 跨 offer 比较失真。怎么避免：按总包谈、按总包比，base 之外都是可谈空间。
  - **编造竞争性 offer 做杠杆**：触发原因 = 想增加筹码；后果 = 被识破直接出局，行业小、口碑受损。怎么避免：杠杆只用真实的 offer / 薪资 / 技能稀缺性。
  - **为小涨幅把关系谈僵**：触发原因 = 「能多谈就多谈」的执念；后果 = 入职前就和未来雇主关系紧张。怎么避免：权衡边际收益和关系成本，谈判有度。

### W8. 转行求职流程

- **这是谁的工作流**: 求职者的工作流；教练做可迁移技能盘点 + 转行叙事打磨 + 路径设计 + 预期管理。

- **One-liner**: 帮一个人从 A 行业 / 职能进入 B —— 转行求职的核心难点不是「能不能做」，是「招聘方看不出你能做」；解法是「可迁移技能 + 转行叙事 + 一条降低风险的进入路径」。这是 W5（定位）的一个高难度特例。

- **Trigger**: 求职者要换行业 / 换职能（主动转行、行业衰退被迫转、被裁后重新定位、海归 / 回国换赛道等）。

- **Output**: 一份可迁移技能清单（用目标行业语言重新包装）+ 一条解释「为什么转 + 为什么能转」的转行叙事 + 一条进入路径（直接转 / 过渡岗 / 补能力 / 内部转 等）。

- **入门 SOP（minimum viable steps）**:
  1. **可迁移技能盘点**：盘点 A 行业 / 职能里「不绑定具体行业」的能力（项目管理、数据分析、沟通、某类工具、某类问题解决），并**用目标行业 B 的语言重新表述**。**跳过会发生什么**：用 A 行业的术语描述自己 → B 行业招聘方「看不懂 / 觉得不相关」，简历直接刷。
  2. **目标 B 的真实要求调研**：调研目标行业 / 岗位真正要什么（硬门槛、加分项、是否有「必须从这里入」的路径），判断转行的现实难度和缺口。**跳过会发生什么**：对 B 一无所知就转 → 撞上隐形硬门槛（如某些岗位的资质 / 经验硬要求），白费力气。
  3. **转行叙事编织**：把「为什么离开 A + 为什么选 B + 为什么我能胜任 B」串成一条让人信服的连贯故事（不是「我厌倦了 A」这种负面叙事，是「我在 A 积累的 X 正好是 B 需要的」）。**跳过会发生什么**：没有叙事 → 招聘方把你当「逃离者 / 不知道自己要什么的人」，而非「有备而来的转行者」。
  4. **进入路径设计**：设计一条降低风险的路径 —— 直接转（缺口小时）/ 过渡岗（先转职能或先转行业，不一次跨两步）/ 补能力（课程 / 项目 / 证书填硬缺口）/ 内部转岗（在现公司先转）/ networking 进入（转行者尤其需要有人「背书 + 引荐」）。**跳过会发生什么**：硬磕「一步到位直接转」→ 跨度太大、屡投屡拒、信心和时间双重消耗。
  5. **执行（进入 W1/W2/W3 对应弧线）**：按目标 locale 的求职弧线执行，但简历 / 面试都围绕「可迁移性 + 转行叙事」组织。

- **资深路径（差异点 — skip / optimize / add）**:
  - **optimize**: 把「一步到位直接转」优化成「拆成可达的中间步」—— 资深教练知道「同时换行业 + 换职能」跨度最大、最难，通常建议先换一个维度（保留职能换行业，或保留行业换职能），用一个过渡岗做跳板。把「自己投网申硬碰」优化成「转行更要靠 networking」—— 转行者的简历在 ATS / 初筛里天然吃亏（关键词不匹配），有人「背书 + 引荐」能绕过「纸面看不出能做」的问题，networking 对转行者的权重比一般求职更高（evidence: [T03-S017]）。
  - **add**: 资深教练会**额外做「转行的真实成本 + 预期管理」** —— 坦诚告诉求职者：转行通常意味着**短期降薪 / 降级 / 重新证明自己**，时间线比同行跳槽长；不把转行说成「换个赛道就能起飞」（不贩卖确定性，也不 victim-blame —— 转行难是真实约束，不是「你不够努力」）。还会**额外做「可迁移技能的『翻译』示范」** —— 求职者最大的盲区是「我知道我会，但不知道怎么让 B 行业的人看懂」，资深教练会逐条示范怎么把 A 的经历「翻译」成 B 的语言。
  - （本 workflow 资深差异以 optimize + add 为主，**skip 类较少** —— 转行流程的每一步对新手都是必要的，资深的价值在「优化路径」和「补预期管理」，而非「省步骤」。这本身是一个信号：转行是高难度场景，没有「资深抄近道」，只有「资深做得更稳」。）

- **近期变化（近 12 个月）**:
  - **2023-2025 起，由裁员潮 + 行业冷热剧烈分化驱动**：被迫转行（行业衰退 / 被裁后无法回原行业）的求职者增多 —— 教练对「被迫转行」和「主动转行」的处理要分开，被迫转行者的情绪和生计压力更大，预期管理 + 过渡策略权重更高（evidence: [T03-S020], [T03-S021]）。
  - **近 12 个月：转行流程骨架无重大变化（行业稳态）**。「可迁移技能 + 转行叙事 + 路径设计」是长期稳定的转行方法论。
  - **2024-2025 步骤内的小变化**：AI 用于「可迁移技能翻译」「转行叙事草拟」—— 工具层变化，流程稳态。
  - 当前采用率：转行方法论本身稳态；「被迫转行占比上升」是市场周期带来的人群结构变化。

- **典型耗时**:
  - 入门 SOP（求职者侧）：转行求职周期通常显著长于同行跳槽 —— 3-9 个月常见，跨度大 / 需补能力时更长。
  - 资深路径：耗时不一定更短（转行难是外部约束），但「拆成中间步 + networking 进入 + 预期管理」让求职者少走「一步到位硬碰」的弯路，且不被屡拒带崩。

- **关键工具（与 Track 02 关联）**:
  - LinkedIn / 脉脉（场景特化 —— 转行尤其依赖 networking 进入；也用于调研目标行业在职者的真实路径）
  - 牛客 / 一亩三分地 / Glassdoor（场景特化 —— 调研目标行业 / 岗位的真实要求和面经）
  - ChatGPT / Claude（新兴 —— 可迁移技能翻译 / 转行叙事草拟辅助，强时效）

- **关键人物（与 Track 01 关联）**:
  - Richard N. Bolles《What Color Is Your Parachute?》把「可迁移技能盘点」作为转行的钥匙概念（evidence: [T03-S009]）。
  - Bill Burnett & Dave Evans《Designing Your Life》的「奥德赛计划 + 原型」适合转行者「低成本试错」而非「一次定死」（evidence: [T03-S008]）。
  - Mark Granovetter 的弱关系理论解释了「为什么转行者尤其需要 networking」—— 弱关系连接不同社交圈，是进入陌生行业的桥（evidence: [T03-S017]）。

- **常见失败模式**:
  - **用原行业语言描述自己**：触发原因 = 没意识到「招聘方看不懂 ≠ 我不行」；后果 = B 行业招聘方判「不相关」直接刷。怎么避免：逐条把可迁移技能「翻译」成目标行业的语言。
  - **一步到位同时换行业 + 换职能**：触发原因 = 急于彻底转变；后果 = 跨度过大、屡投屡拒。怎么避免：拆成中间步，先换一个维度，用过渡岗做跳板。
  - **负面转行叙事（「我厌倦了 A」）**：触发原因 = 把转行说成「逃离」；后果 = 招聘方把你当「不知道自己要什么的人」。怎么避免：叙事围绕「A 的积累正好是 B 需要的 + 我想清楚了为什么是 B」。
  - **只投网申、不做 networking**：触发原因 = 沿用一般求职习惯；后果 = 转行简历在初筛天然吃亏，纸面看不出能做。怎么避免：转行更要靠 networking，有人背书引荐能绕过「纸面看不出」的硬伤。
  - **对转行成本预期不足**：触发原因 = 「换个赛道就能起飞」的幻想 / 营销话术；后果 = 撞上短期降薪降级的现实后崩盘放弃。怎么避免：教练前置做预期管理 —— 转行通常意味着短期降薪 / 降级 / 时间线更长，这是真实约束。

### W9. 求职教练的接案 / 服务流程

- **这是谁的工作流**: **教练自己的工作流**（不是求职者的）—— 从一个求职者来求助，到完成一段交付。

- **One-liner**: 求职教练接一个客户、把他带到「求职能力提升 + 可控部分做到位」的服务闭环 —— 初诊 → 诊断卡点 → 方案 → 执行陪跑 → 复盘。核心是「诊断对卡点 + 守住能力边界 + 不兜售确定性」。

- **Trigger**: 一个求职者来找教练求助（投了没回音 / 面试老挂 / 不知道方向 / 要谈薪 / 要转行 等）。

- **Output**: 完成一段交付 —— 客户的求职「可控部分」（定位 / 简历 / 面试 / networking / 谈薪）被实质提升，且对「不可控部分」（市场周期 / 结构性约束）有清醒预期。**注意：output 不是「拿到 offer」** —— 教练能提升命中率，不能保证 offer，把 output 定义成「offer」就是兜售确定性。

- **入门 SOP（minimum viable steps）**:
  1. **初诊（intake）**：了解客户的背景、目标、当前求职状态、卡在哪、之前做过什么、预期是什么、约束条件（时间 / 预算 / 签证 / 生计压力）。**跳过会发生什么**：不做初诊直接「上方案」→ 方案是套餐化的、不对人，客户体验差且无效。
  2. **诊断卡点**：判断客户的问题真正卡在哪一环 —— 用「求职漏斗」视角（投递→初筛→面试→offer，哪一层转化率掉）+ 定位 / 简历 / 渠道 / 面试表现 / 谈薪逐项排查。**跳过会发生什么**：不诊断就「客户说要改简历就改简历」→ 改了简历但真正的问题在定位 / 渠道，白做。
  3. **出方案**：基于诊断给一个有优先级的方案 —— 先解决卡点最重的那一环，明确「这次能解决什么 / 不能解决什么」。**跳过会发生什么**：方案没优先级 / 没边界 → 客户不知道先做什么、对结果预期失真。
  4. **执行陪跑**：按方案陪客户做 —— 改简历、mock interview + debrief、定位对话、谈薪演练等（mock + debrief 是教练 ROI 最高的交付物）。**跳过会发生什么**：只给方案不陪跑 → 客户「知道了但做不到」，方案落不了地。
  5. **复盘 + 边界交接**：阶段性复盘进展，校准预期；明确教练能力边界（自己不精通的专业领域 —— 如某类技术岗深度 —— 是「连接资源」而非「越界硬教」）。**跳过会发生什么**：不复盘 → 客户和教练对「进展」「下一步」没共识；不守边界硬教不懂的领域 → 给错误指导，害了客户。

- **资深路径（差异点 — skip / optimize / add）**:
  - **skip**: 资深教练会**跳过对「问题已经很清晰」客户的完整诊断流程** —— 客户明确「我就是要练 PM 面试」且诊断快速确认卡点确在面试表现时，直接进执行陪跑，不为了「流程完整」硬跑一遍定位排查。也会**主动跳过 / 转介自己能力边界外的客户** —— 资深教练知道「不是所有客户我都该接」，自己不擅长的领域（某些专业技术岗、某些 locale）坦诚转介，而非硬接。
  - **optimize**: 把「客户说改什么就改什么」优化成「先诊断再定方案」—— 资深教练不被客户的「自我诊断」带着走（客户常误判自己的卡点，比如以为是简历问题、其实是定位问题）。把「凭经验聊」优化成「用求职漏斗 / 结构化诊断框架」—— 让诊断可复用、可解释，而非每个客户从零凭感觉。
  - **add**: 资深教练会**额外做「预期管理 + 诚实边界」贯穿全程** —— 从初诊就讲清「我能优化可控部分（定位 / 简历 / 面试 / networking / 谈薪），市场周期 / 年龄 / 学历门槛是真实约束，我不能保证 offer」；不做「保 offer」承诺、不做「内推保过」（这是黑产红线）、不靠制造焦虑卖课。还会**额外做「利益关系澄清」** —— 如果自己同时卖内推 / 与机构绑定，向客户讲清这层利益关系（教练理论上站求职者一边，但「教练 + 卖内推 + 抽成」是利益冲突，要透明）（evidence: [T03-S022]，intake warnings #1/#8/#9）。
  - **资深 ≠ 抄近道的硬声明**：本行业资深教练的 skip/optimize/add **全部不指向「兜售确定性」或「教客户作弊」** —— skip 是「不对清晰的客户做无用功 + 不接能力外的客户」，optimize 是「诊断更准、框架更可复用」，add 是「预期管理 + 利益透明 + 守边界」。资深的价值是「把可控部分做到最好 + 帮客户管理预期」，不是「保证结果」。

- **近期变化（近 12 个月）**:
  - **近 12 个月：教练服务流程骨架无重大变化（行业稳态）**。「初诊 → 诊断 → 方案 → 陪跑 → 复盘」是稳定的服务闭环。
  - **2023-2025 起，由就业市场承压驱动**：客户的焦虑水平和「求快 / 求确定性」的诉求上升 —— 这反而让「预期管理 + 诚实边界」这一 add 步骤的权重更高（市场越难，越要顶住「保 offer」诱惑）；也让「制造焦虑卖课」「保 offer 套餐」的行业噪音更盛，资深教练与之划清界限的必要性上升（evidence: [T03-S021], [T03-S020]）。
  - **2024-2025 起**：教练自己也用 AI 辅助交付（帮客户改简历 / 出练习题 / 草拟谈薪话术），交付效率提升 —— 但这是工具层变化，服务流程骨架稳态。
  - 当前采用率：服务流程本身是长期稳态；AI 辅助交付是新增层。

- **典型耗时**:
  - 入门 SOP（教练侧）：一段完整交付从初诊到复盘，按次服务约几小时到十几小时；套餐 / 训练营式可拉到数周。
  - 资深路径：单客户交付时间不一定更短，但「先诊断再方案」避免了「改了没用的东西」的返工，整体交付质量和客户结果更稳。

- **关键工具（与 Track 02 关联）**:
  - 教练用的工具 = 客户工具的「示范 + 教学」版 —— Jobscan（教 ATS）、levels.fyi（教谈薪 anchor）、Pramp / interviewing.io（组织 mock）、Big Interview（行为面训练）。
  - ChatGPT / Claude（新兴 —— 教练辅助交付：改简历 / 出题 / 草拟话术，强时效）
  - 求职过程管理工具（Huntr / Teal / Notion）—— 教练帮客户搭求职看板。

- **关键人物（与 Track 01 关联）**:
  - Andrew LaCivita 是少数把「教练这门生意怎么做」也讲透的人 —— milewalk Academy 的训练营模式本身是「教练执业经营」的一手观察样本（evidence: [T03-S005]）。
  - Alison Green（Ask a Manager）的「对流行求职建议保持怀疑、按场景判断」是教练「诊断 + 不被客户自我诊断带偏」的方法论参照（evidence: [T03-S007]）。
  - **反面样本**：国内校招 / 留学生求职机构（offer 先生 / DBC 职梦 等）的「工业化打包 + 结果承诺」模式 —— 作为「教练服务流程不该怎么做」的反面教材（「保 offer」「付费内推」与「教练只能优化可控部分」的诚实边界直接冲突）（evidence: [T03-S022]）。

- **常见失败模式**:
  - **跳过诊断，客户说改什么就改什么**：触发原因 = 客户的「自我诊断」很笃定 + 教练怕「不给客户想要的」；后果 = 改了简历但真问题在定位 / 渠道，客户花了钱没效果。怎么避免：先诊断再定方案，用求职漏斗排查真正的卡点。
  - **把 output 定义成「拿到 offer」/ 做「保 offer」承诺**：触发原因 = 市场压力下想留住客户 / 行业噪音裹挟；后果 = 兜售确定性，做不到时信任崩塌，且滑向黑产边缘。怎么避免：output 定义成「可控部分实质提升 + 预期清醒」，从初诊就讲清边界。
  - **越界硬教自己不懂的专业领域**：触发原因 = 不想拒客户 / 高估自己；后果 = 给错误的专业指导，害了客户。怎么避免：守能力边界，不精通的领域「连接资源 / 转介」而非硬教。
  - **靠制造焦虑卖课 / 模糊利益关系**：触发原因 = 焦虑营销是行业常见变现套路；后果 = 短期变现、长期口碑崩，且与「教练站求职者一边」的角色冲突。怎么避免：给真实图景不贩卖焦虑；同时卖内推 / 与机构绑定的，向客户透明这层利益关系。
  - **不做预期管理，把市场不可控的失败归给客户**：触发原因 = 回避「我的方法也有上限」；后果 = victim-blaming（「你没拿到 offer 是你不够努力」），既不诚实也伤客户。怎么避免：贯穿全程讲清「可控 vs 不可控」的边界，市场周期 / 结构性约束是真实的。

## Phase 2 提炼提示

### 反复出现 ≥ 3 个 workflow 都包含的步骤模式（候选 playbook 通则）

- **「定位 / 收敛先于一切执行」** —— 出现于 W1（校招）、W2（社招）、W3（海外）、W5（定位本身）、W6（简历，定位完成才启动）、W8（转行）。**几乎每条弧线的第一步都是「先收敛目标」**，且每条弧线的头号失败模式都是「跳过定位直接投 / 直接改简历」。→ 候选 playbook：**「如果求职者上来就问『简历怎么改 / 该投哪』，先回退一步问『目标岗位收敛了吗』** —— 定位不清时改简历 / 投递是高效地做错的事」。
- **「按『簇 / 类型』批量准备，而非逐个从零」** —— 资深路径在 W1（按公司簇准备笔面试）、W4（建一个 STAR 故事库复用）、W6（master 简历 + 按岗位类型做变体）、W3（按面试类型批量准备）反复出现。→ 候选 playbook：**「如果在为每个岗位 / 每场面试从零重做准备 → 抽取可复用资产（故事库 / 简历变体 / 公司簇研究），把『逐个』改成『按类型』」**。
- **「从招聘方 / 岗位的视角和痛点出发，不是从自己出发」** —— W3（Belcak「make the outreach about them」、pain letter）、W4（面试官按 rubric 找证据）、W6（JD 拆解：岗位要什么不是我有什么）、W2（沟通中确认岗位真实性）、W7（理解招聘方议价逻辑）。→ 候选 playbook：**「简历 / 触达 / 面试答案的组织原点是『岗位 / 招聘方在解决什么问题』，不是『我想说什么』」**。
- **「经历要当证据 / 量化，不是流水账」** —— W4（STAR + 可量化 Result）、W5（卖点提炼）、W6（X-Y-Z 量化改写）、W1/W2/W3（行为面准备）。→ 候选 playbook：**「如果经历是『负责 X』式职责罗列 → 套 X-Y-Z / STAR 重构，每段必须有可衡量的 Result」**。
- **「先诊断卡在漏斗哪一层，再行动」** —— W9（教练用求职漏斗诊断卡点）、W1/W2/W3（「投了没回音」回头排查 → 根因常在定位 / 简历）、W5（定位的 trigger 之一是「投了很多没回音」）。→ 候选 playbook：**「如果求职者焦虑『为什么没结果』→ 不情绪化归因，拆求职漏斗每一环转化率，定位是『投递→初筛』掉还是『面试→offer』掉」**。
- **「复盘喂回下一轮」** —— W4（面试三段式的第三段是复盘）、W9（教练服务的第五步是复盘 + 预期校准）。→ 候选 playbook：**「每一场 / 每一阶段做结构化复盘，喂回下一次 —— 只『做』不『复盘』，经验不累积」**。
- **「networking 权重按赛道 / 层级 / locale 判断，不站队」** —— W3（资深教练按岗位层级决定要不要重投 networking）、W8（转行更要靠 networking）、W1（国内校招 networking 权重相对低、网申 + 笔试是主通道）。→ 候选 playbook：**「networking vs 网申不是非此即彼 —— 海外科技 / 咨询 / 高层级 / 转行场景 networking 权重高；国内校招网申 + 笔试是主通道。先判断赛道，再配渠道策略」**（对应 canon 核心概念 22「网申优化 vs networking」、intake warnings #6）。

### 入门 SOP 和资深路径之间最大的差距（候选心智模型）

- **入门 SOP 平均 5-6 步，资深路径几乎不靠「砍步骤」压缩，而是靠「判断哪步该重投 / 哪步该跳过 / 哪步该额外补」**。这一行的资深差异不是「做得快」，是**「分配对注意力」**：
  - **资深特别擅长「判断什么不必做」** —— W1 跳过低性价比海投、W2 跳过广撒网打招呼、W6 跳过 networking 路径下的 ATS 优化、W9 跳过对清晰客户的完整诊断。→ 心智模型候选：**「求职 / 教练的资深，是知道在哪里『不过度投入』—— 不是所有岗位都值得深定制，不是所有场景都该 networking，不是所有客户的自我诊断都要照做」**。
  - **资深的 add 几乎全是「预期管理 + 边界 + 真实性把关」** —— W1 应届生身份管理、W3 市场周期校准、W5 约束条件显性化、W6 优化 vs 造假红线、W7 编造 offer 红线、W8 转行成本预期、W9 诚实边界 + 利益透明。→ 心智模型候选：**「这一行的资深增量，一大半不在『技巧更高』，而在『把不可控说清楚 + 把红线守住 + 把客户预期管住』—— 资深教练卖的是『靠谱』，不是『确定性』」**。
- **W8（转行）几乎没有 skip 类资深差异，全是 optimize + add** —— 这是一个明确信号：**转行是高难度场景，没有「资深抄近道」，只有「资深做得更稳」**。Phase 2 可把「越难 / 越受结构性约束的场景，资深差异越偏向『做得稳』而非『做得省』」作为本行业的一条元规律。

### 近期工作流变化的根本原因（触发事件类型分布）

- **本 track 9 个 workflow 全部标 medium decay，没有纯 high，也没有纯 low 之外的稳态空白** —— 关键结构性发现：**求职工作流的「弧线骨架」是稳态的（定位 → 简历 → 触达 → 面试 → 谈薪 → offer 决策，5-50 年一脉相承），近 12-24 个月在变的是「单个步骤内的做法」，不是整条弧线**。
- **触发变化的事件类型分布**：
  - **AI 化（招聘两端）— 影响 9/9 个 workflow 的「步骤内做法」**：求职者侧（AI 改简历 / AI 模拟面 / AI 公司研究 / AI 谈薪话术）+ 招聘方侧（AI 筛简历 / AI 视频初面 / ATS 语义化升级）。这是最广的驱动力，但**它改的是 step 怎么做，不是 workflow 是什么**。
  - **就业市场周期（2023-2025 裁员潮 + 国内就业承压）— 影响 W1/W2/W3/W7/W8/W9 的「预期管理强度」**：不改流程步骤，改的是「教练对结果的预期管理权重」和「客户的焦虑水平」。
  - **法规变化 — 影响 W7（薪资透明立法使『别先报数字』变形）+ W3/W6（反歧视 / AI 招聘监管影响简历该不该为 AI 优化）**：地区性、持续扩散中。
  - **国内校招政策微调 — 影响 W1（时间窗 / 应届生认定口径逐年微调）**。
- **主要驱动力 → Phase 2 识别的「外部驱动力」**：**AI 化**是头号驱动力（影响最广），但要写清「它重塑的是步骤做法、不是工作流骨架」；**市场周期**是第二驱动力（影响预期管理而非流程）。→ Phase 2.8 诚实边界应写：**「本 skill 工作流节的『骨架』衰减慢（弧线结构稳态），但每个 workflow 里『AI 相关的步骤做法』衰减快，建议每 6-12 个月跑一次 update 专门刷新各 workflow 的『近期变化』字段，骨架部分可保留更久」**。

### 失败模式清单（跨 workflow 汇总 — 供 cli_writer 抽进各 workflow.sh 的失败模式自检）

**A 类 — 跳过定位 / 不诊断（最高频，跨 W1/W2/W5/W6/W8/W9）**：
- 跳过定位直接海投 / 直接改简历 → 高效地做错事（W1, W5, W6）
- 逃离式跳槽，动机不清 → 跳进新的坑（W2）
- 教练跳过诊断、客户说改什么就改什么 → 改了没用的东西（W9）

**B 类 — 红线 / 造假（必须硬划，跨 W1/W2/W6/W7/W9）**：
- 把「美化」和「编造」混为一谈 → 背调 / 试用期暴露（W6）
- 把「付费内推 / 内推保过」当真 → 无效或灰色操作，背调有暴露风险（W1）
- 背调翻车（简历 / 面试夸大了职级 / 在职时间）→ offer 撤回（W2）
- 编造竞争性 offer 做谈薪杠杆 → 被识破出局（W7）
- 教练做「保 offer」承诺 / 靠制造焦虑卖课 → 兜售确定性、滑向黑产（W9）
- AI 实时面试辅助（赛中喂答案）→ 作弊，远程面试反制在收紧（W4，跨 Track 02）

**C 类 — 不懂 locale / 渠道差异（跨 W1/W3/W6）**：
- 海外简历照搬国内格式（带照片 / 年龄）→ 不专业 + 合规顾虑（W3, W6）
- 国内校招照搬海外 networking-first → 错过网申 + 笔试主通道（W1）
- 无差别海投、零 networking（在 networking 权重高的赛道）→ 命中率极低（W3）

**D 类 — 准备方式错 / 不复盘（跨 W1/W4）**：
- 笔试裸考 → 连面试机会都没有（W1）
- 逐字背稿、被追问就崩 + 背诵痕迹减分（W4）
- 行为面讲成流水账 / 职责描述 → 没有可评分的实质（W4）
- 只「面」不「复盘」→ 同样错误反复犯（W4）
- 把面试神秘化（读微表情 / 靠气场）→ 准备方向错（W4）

**E 类 — 预期 / 心态 / victim-blaming（跨 W3/W7/W8/W9）**：
- 把 ghosting 解读成「我一定很差」→ 自我否定（W3）
- 被一场失败带崩 → 影响后续面试（W4）
- 对转行成本预期不足 → 撞上短期降薪降级后崩盘（W8）
- 教练不做预期管理、把市场不可控的失败归给客户 → victim-blaming（W9）

### 入门 vs 资深核心差异（一句话总结，供 Phase 2.4 工作流节 + Phase 2.6 反模式节）

- **入门 = 「能把求职这件事走完」**：知道弧线有哪些步骤，按部就班做。
- **资深 = 「把可控部分做到最好 + 把不可控部分说清楚」**：判断哪步该重投 / 该跳过 / 该额外补（注意力分配），守住优化 vs 造假的红线，贯穿全程做预期管理。**资深的增量不在「抄近道 / 给确定性」，在「分配对注意力 + 守住边界 + 管住预期」** —— 这是 coaching_practitioner 行业 + 「营销噪音 / 包 offer 黑产重」这个特殊语境共同决定的：在一个充斥「保 offer」话术的行业里，**资深的标志恰恰是『不承诺确定性』**。
