## 候选 playbook 规则 (target 8-10)

| # | Pattern（如果 X 则 Y） | 来源 |
|---|------------------------|------|
| P1 | 评估 deal 先问「能不能单独 return the fund」，答案否则其余尽调都次要 | C1；canon power-law → playbook；04 Phase 2 接口 |
| P2 | 冷 inbound / 非 thesis 命中 → 极快 pattern match 处理，first meeting / 深尽调只留给强引荐 + thesis 命中 | 03 Phase 2 接口「先把初筛做掉」；W1/W2/W4/W5 skip |
| P3 | 做单个 deal 决策（尽调 / term sheet / follow-on）→ 先回 portfolio construction 对一遍（ownership / reserve / 占哪个格） | 03 Phase 2 接口「绑回约束框架」；W2/W6/W8 add |
| P4 | 资源是关系型的（deal flow / LP / 帮扶能力）→ 定期主动提前经营，别等需要时才找 | 03 Phase 2 接口「关系是复利资产」；W1/W3/W4/W7 |
| P5 | close / 启动后立刻约定固定信息更新节奏，把「信息失明」当头号风险防 | 03 Phase 2 接口「信息节奏」；W1/W3/W7 |
| P6 | 分配尽调时间 / 投后精力 / reserve 资金 → 都按 power-law 分级，winner 优先，明确 loser 快速降投入 | 03 Phase 2 接口「拒绝平均用力」；W5/W7/W8 |
| P7 | 看到高估值 → 先查 preference 结构（participating / 2x+ 是隐性税），高估值 + 差条款不是好 deal | C7；canon liquidation preference → playbook；W6 |
| P8 | VC 报「估值」→ 先确认 pre 还是 post、option pool 算在哪侧（这一步定创始人实际稀释） | canon pre/post-money → playbook；W6；glossary outsider-tell #1 |
| P9 | 建组合先留 reserve（常见 40-60%）；首轮投满、winner 起飞无钱 follow-on 是新基金第一大结构性错误 | canon reserve → playbook；W3/W8 |
| P10 | 标准化早期 deal → 用市场标准 SAFE / NVCA 模板，把谈判收敛到「估值 + 投资额」；逐条款都想赢是输掉 7-10 年关系 | W6 skip；canon SAFE；03 Phase 2 接口 |
| P11 | 募新基金 → 先找老 LP re-up（老 LP 的钱最便宜），别一开始就广撒网见陌生 LP | W3 skip |
| P12 | 遇任何 VC 问题 → 先问 locale（海外美元 / 国内 RMB / 国内美元），两套生态、两套监管、两套退出 | 02 决策树根节点；C17；全 6 轨 |
| P13 | 遇「该走什么模式 / 该不该集中」→ 先问对方在哪派（thesis / spray / concentrated / solo GP / 国内多元），同问题不同派给相反答案 | C3；§7 谱系；04 分歧 |
| P14 | 寒冬里评判 GP（或自评）→ 看 DPI 不看 TVPI；账面价值不是钱，主动用 secondary 制造流动性 | canon DPI/TVPI → playbook；W3；glossary「DPI is the new IRR」 |
| P15 | 尽调危机期公司 → 不看银行余额，看「增长曲线能否在 runway 用完前转正」（default alive） | canon default alive → playbook；W5 |

**目标 8-10 条 → 选 P1, P2, P3, P6, P7/P8 合并, P9, P10, P12, P13, P14 = 约 10 条。P4/P5 合并进一条「关系与信息节奏」，P11 并入 P4，P15 并入 P1 案例。**
